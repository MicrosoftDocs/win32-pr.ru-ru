# <a name="wpcevent_custom-event"></a>\_Пользовательское событие впцевент

Событие для каждого пользователя, которое поддерживает до трех полей.

События отображаются в **средстве просмотра действий** в **другом** разделе со следующей иерархией:

1.  Издатель

2.  Приложение

3.  Событие


```C++
const EVENT_DESCRIPTOR WPCEVENT_CUSTOM = {0xd, 0x0, 0x10, 0x4, 0x17, 0xd, 0x8000000000000030};
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Издатель* 
</dt> <dd>

Издатель события (например, название компании).

</dd> <dt>

*AppName* 
</dt> <dd>

Имя приложения, создающего событие.

</dd> <dt>

*AppVersion* 
</dt> <dd>

Версия приложения, создающего событие.

</dd> <dt>

*Событие* 
</dt> <dd>

Имя события.

</dd> <dt>

*Значение1* 
</dt> <dd>

Настраиваемое поле 1.

</dd> <dt>

*Value2* 
</dt> <dd>

Настраиваемое поле 2.

</dd> <dt>

*Значение3* 
</dt> <dd>

Настраиваемое поле 3.

</dd> <dt>

*Заблокирован* 
</dt> <dd>

Значение перечисления [**впкфлаг \_**](/windows/win32/api/wpcevent/ne-wpcevent-wpcflag_isblocked) , которое указывает сведения о том, какие события заблокированы и какие элементы управления будут использоваться.

</dd> <dt>

*Причина* 
</dt> <dd>

Пользовательская строка, предоставляющая дополнительные сведения о причине блокировки или запрета блокировки.

</dd> </dl>
