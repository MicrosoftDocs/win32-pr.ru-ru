# <a name="oid-overview"></a>Общие сведения о OID

Расширяемость достигается за счет использования новых [*идентификаторов объектов*](../secgloss/o-gly.md) (OID), новых типов кодирования и новых библиотек DLL.

Идентификаторы объектов CryptoAPI могут принимать любые из следующих форм:

-   Числовая строка, например "1.2.3.500.88"
-   Буквенно-цифровая строка, например *MyFunction*
-   Константа со значением, меньшим или равным 0xFFFF. Эти константы часто связаны с именем с помощью инструкции **\# define** в файле заголовка.

Расширяемые функции принимают OID и аргументы типа кодировки. Эти функции выполняют поиск в системном реестре библиотеки DLL, связанной с аргументами типа OID и кодировки, передаваемыми в функцию. Если библиотека DLL для идентификатора OID найдена, то загружается библиотека DLL и вызывается ее функция. Это позволяет расширять функциональные возможности CryptoAPI по мере необходимости. Использование этой методологии полагается на разработчика новых функций, чтобы написать весь необходимый код для этих функций. Например, для кодирования некоторой новой структуры данных функция в библиотеке DLL должна выполнять весь процесс кодирования.
