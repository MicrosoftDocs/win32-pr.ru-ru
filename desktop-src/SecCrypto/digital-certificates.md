<span data-ttu-id="402d5-104">Проверка подлинности крайне важна для защиты обмена данными.</span><span class="sxs-lookup"><span data-stu-id="402d5-104">Authentication is crucial to secure communications.</span></span> <span data-ttu-id="402d5-105">Пользователи должны иметь возможность доказать свои удостоверения тем, с которыми они взаимодействуют, и должны иметь возможность проверять подлинность других пользователей.</span><span class="sxs-lookup"><span data-stu-id="402d5-105">Users must be able to prove their identity to those with whom they communicate and must be able to verify the identity of others.</span></span> <span data-ttu-id="402d5-106">Проверка подлинности по сети довольно сложна, поскольку пользователи, устанавливающие соединение, физически не встречаются.</span><span class="sxs-lookup"><span data-stu-id="402d5-106">Authentication of identity on a network is complex because the communicating parties do not physically meet as they communicate.</span></span> <span data-ttu-id="402d5-107">Это может позволить перехватывать сообщения и выдавать себя за другое лицо.</span><span class="sxs-lookup"><span data-stu-id="402d5-107">This can allow an unethical person to intercept messages or to impersonate another person or entity.</span></span> <span data-ttu-id="402d5-108">Метод должен быть обработан для поддержания необходимого уровня доверия в процессе связи.</span><span class="sxs-lookup"><span data-stu-id="402d5-108">A method must be worked out to maintain the necessary level of trust within the communication process.</span></span>

<span data-ttu-id="402d5-109">[*Цифровой сертификат*](../secgloss/c-gly.md) — это общие учетные данные, которые предоставляют средства для проверки удостоверения.</span><span class="sxs-lookup"><span data-stu-id="402d5-109">The [*digital certificate*](../secgloss/c-gly.md) is a common credential that provides a means to verify identity.</span></span> <span data-ttu-id="402d5-110">В этом разделе приводятся общие сведения о том, как сертификаты обеспечивают безопасную связь и использование CryptoAPI для использования этих сертификатов и управления ими.</span><span class="sxs-lookup"><span data-stu-id="402d5-110">This section provides an overview of how certificates provide secure communications and how to use CryptoAPI to use and manage those certificates.</span></span>

<span data-ttu-id="402d5-111">Сертификат — это набор данных, определяющий сущность.</span><span class="sxs-lookup"><span data-stu-id="402d5-111">A certificate is a set of data that identifies an entity.</span></span> <span data-ttu-id="402d5-112">Доверенная организация назначает сертификат отдельному объекту или сущности, связывающей открытый ключ с отдельным пользователем.</span><span class="sxs-lookup"><span data-stu-id="402d5-112">A trusted organization assigns a certificate to an individual or an entity that associates a public key with the individual.</span></span> <span data-ttu-id="402d5-113">Лицо или субъект, которому выдан сертификат, называется субъектом этого сертификата.</span><span class="sxs-lookup"><span data-stu-id="402d5-113">The individual or entity to whom a certificate is issued is called the subject of that certificate.</span></span> <span data-ttu-id="402d5-114">Доверенная организация, которая выдает сертификат, является [*центром сертификации*](../secgloss/c-gly.md) (CA) и называется издателем сертификата.</span><span class="sxs-lookup"><span data-stu-id="402d5-114">The trusted organization that issues the certificate is a [*certification authority*](../secgloss/c-gly.md) (CA) and is known as the certificate's issuer.</span></span> <span data-ttu-id="402d5-115">Доверенный центр сертификации выдаст сертификат только после проверки удостоверения субъекта сертификата.</span><span class="sxs-lookup"><span data-stu-id="402d5-115">A trustworthy CA will only issue a certificate after verifying the identity of the certificate's subject.</span></span>

<span data-ttu-id="402d5-116">Сертификаты используют методы шифрования для решения проблемы отсутствия физического контактного лица между этими данными.</span><span class="sxs-lookup"><span data-stu-id="402d5-116">Certificates use cryptographic techniques to address the problem of the lack of physical contact between those communicating.</span></span> <span data-ttu-id="402d5-117">Использование этих методов ограничивает возможность перехвата, изменения или подделки сообщений небезупречным лицом.</span><span class="sxs-lookup"><span data-stu-id="402d5-117">Using these techniques limits the possibility of an unethical person intercepting, altering, or counterfeiting messages.</span></span> <span data-ttu-id="402d5-118">Эти приемы шифрования затрудняют изменение сертификатов.</span><span class="sxs-lookup"><span data-stu-id="402d5-118">These cryptographic techniques make certificates difficult to modify.</span></span> <span data-ttu-id="402d5-119">Таким путем, сущность затрудняет олицетворение другого лица.</span><span class="sxs-lookup"><span data-stu-id="402d5-119">Thus, it is difficult for an entity to impersonate someone else.</span></span>

<span data-ttu-id="402d5-120">Данные в сертификате включают открытый криптографический ключ из [*пары открытого и закрытого ключей*](../secgloss/p-gly.md)субъекта сертификата.</span><span class="sxs-lookup"><span data-stu-id="402d5-120">The data in a certificate includes the public cryptographic key from the certificate subject's [*public/private key pair*](../secgloss/p-gly.md).</span></span> <span data-ttu-id="402d5-121">Сообщение, подписанное закрытым ключом отправителя, может быть получено только получателем сообщения с помощью открытого ключа отправителя.</span><span class="sxs-lookup"><span data-stu-id="402d5-121">A message signed with its sender's private key can only be retrieved by the message's recipient using the sender's public key.</span></span> <span data-ttu-id="402d5-122">Этот ключ можно найти в копии сертификата отправителя.</span><span class="sxs-lookup"><span data-stu-id="402d5-122">This key can be found on a copy of the sender's certificate.</span></span> <span data-ttu-id="402d5-123">Получение подписи с [*открытым ключом*](../secgloss/p-gly.md) из сертификата подтверждает, что подпись была создана с помощью [*закрытого ключа*](../secgloss/p-gly.md)субъекта сертификата.</span><span class="sxs-lookup"><span data-stu-id="402d5-123">Retrieving a signature with a [*public key*](../secgloss/p-gly.md) from a certificate proves that the signature was produced using the certificate subject's [*private key*](../secgloss/p-gly.md).</span></span> <span data-ttu-id="402d5-124">Если отправитель был бдительным и сохранил секрет закрытого ключа, получатель может быть уверен в подлинности отправителя сообщения.</span><span class="sxs-lookup"><span data-stu-id="402d5-124">If the sender has been vigilant and has kept the private key secret, the receiver can be confident in the identity of the message sender.</span></span>

<span data-ttu-id="402d5-125">В сети часто используется доверенное приложение, известное как [*сервер сертификатов*](../secgloss/c-gly.md).</span><span class="sxs-lookup"><span data-stu-id="402d5-125">On a network, there is often a trusted application known as a [*certificate server*](../secgloss/c-gly.md).</span></span> <span data-ttu-id="402d5-126">Центр сертификации, работающий на защищенном компьютере, управляет сервером сертификатов.</span><span class="sxs-lookup"><span data-stu-id="402d5-126">A CA running on a secure computer manages the certificate server.</span></span> <span data-ttu-id="402d5-127">Это приложение имеет доступ к открытому ключу всех его клиентов.</span><span class="sxs-lookup"><span data-stu-id="402d5-127">This application has access to the public key of all its clients.</span></span> <span data-ttu-id="402d5-128">Серверы сертификатов выводят сообщения, известные как сертификаты, каждый из которых содержит открытый ключ одного из пользователей клиента.</span><span class="sxs-lookup"><span data-stu-id="402d5-128">Certificate servers dispense messages known as certificates, each of which contains the public key of one of its client users.</span></span> <span data-ttu-id="402d5-129">Каждый сертификат подписывается с помощью закрытого ключа ЦС.</span><span class="sxs-lookup"><span data-stu-id="402d5-129">Each certificate is signed with the CA's private key.</span></span> <span data-ttu-id="402d5-130">Таким же получателем может быть проверка того, что указанный центр сертификации отправил его.</span><span class="sxs-lookup"><span data-stu-id="402d5-130">Thus the receiver of such a certificate can verify that a specified CA sent it.</span></span>

<span data-ttu-id="402d5-131">Цифровые сертификаты также включают расширения и расширенные свойства, предоставляющие дополнительные сведения о субъекте сертификата, например адрес электронной почты субъекта и действия, которые может выполнять субъект сертификата.</span><span class="sxs-lookup"><span data-stu-id="402d5-131">Digital certificates also include extensions and extended properties that provide additional information about the certificate's subject such as the subject's email address and the activities that the certificate's subject can perform.</span></span>
