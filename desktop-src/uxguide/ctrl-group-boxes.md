# <a name="group-boxes"></a>Поля группы

> [!NOTE]
> это руководство по проектированию было создано для Windows 7 и не обновлялось для более новых версий Windows. Многие рекомендации по-прежнему применяются в принципе, но презентация и примеры не соответствуют нашим [текущим руководствам по проектированию](/windows/uwp/design/).

Поле группы — это прямоугольная рамка с меткой, которая окружает набор связанных элементов управления. Поле группы позволяет отображать связи визуально. Помимо возможности предоставления ключа доступа для группы элементов управления, она не предоставляет никаких функциональных возможностей.

![снимок экрана с окном группы с флажками ](images/ctrl-group-boxes-image1.png)

Типичное поле группы.

> [!Note]  
> Рекомендации, связанные с [макетом](vis-layout.md) , представлены в отдельной статье.

 

## <a name="is-this-the-right-control"></a>Выбор правильного элемента управления

Хотя поля группы являются строгим визуальным средством для указания связей, их чрезмерное использование приводит к недостаточной визуализации и значительно сокращает пространство, доступное на поверхности. Они визуально трудны и должны использоваться экономно, только если нет лучшего решения.

тенденция к проектированию в Windows является более простым и понятным видом, устраняя ненужные строки.

Чтобы решить, является ли поле группы обязательным, учитывайте следующие вопросы.

-   **Есть ли в группе более одного элемента управления?** В противном случае используйте обычную текстовую метку. Редким исключением является использование поля группы с одним элементом управления для обеспечения согласованности с другими полями группы на той же поверхности.

**Неправильный:** ![ снимок экрана с окном группы, содержащим одно текстовое поле ](images/ctrl-group-boxes-image2.png)

В этом примере поле группы содержит только один элемент управления.

-   **Связаны ли элементы управления? Показывает ли связь Добавление ясности?** Если нет, представления элементов управления отдельно за пределами поля группы.
-   **Все элементы управления внутри группы?** Если это так, укажите связь на более крупной поверхности, например в родительском диалоговом окне или странице.

**Неправильный:** ![ снимок экрана с окном группы, содержащим все элементы управления ](images/ctrl-group-boxes-image3.png)

В этом примере все элементы управления (помимо кнопок фиксации) в диалоговом окне находятся в поле Группа.

-   **Можно ли эффективно обмениваться связями, используя только макет?** Если это так, используйте вместо этого [Макет](vis-layout.md) . Связанные элементы управления можно разместить рядом друг с другом и разместить дополнительный интервал между несвязанными элементами управления. Можно также использовать заголовки и отступы для отображения иерархических связей.

![Рисунок из четырех значков, демонстрирующих четыре группы задач ](images/ctrl-group-boxes-image4.png)

В этом примере для отображения отношений управления используется только макет.

-   **Можно ли эффективно обмениваться связями с помощью разделителя?** Если это так, используйте вместо этого разделитель. Разделитель — это горизонтальная линия, которая объединяет элементы управления, расположенные под ним. Разделители предоставляют более простой и понятный внешний вид. Однако, в отличие от полей группы, они работают лучше, если они охватывают всю ширину поверхности.
    -   **Разработчики:** Можно реализовать разделитель с вдавленное прямоугольником с высотой 1.

![Снимок экрана, на котором показаны элементы управления электронной почты, разделенные перекрашенными разделителями прямоугольника.](images/ctrl-group-boxes-image5.png)

В этом примере отмеченные разделители используются для отображения отношений управления.

![снимок экрана элементов управления, разделенных разделителями ](images/ctrl-group-boxes-image6.png)

В этом примере непомеченные разделители используются для отображения отношений элементов управления.

-   **Можно ли эффективно обмениваться связями без текста?** Если это так, рассмотрите возможность использования графических элементов, например [фона](vis-graphic.md) или агрегатов.

## <a name="guidelines"></a>Рекомендации

-   **Не вкладывать поля в группы.** Используйте макет для отображения связей в поле группы.

**Неправильный:** ![ снимок экрана поля группы в поле группы ](images/ctrl-group-boxes-image7.png)

В этом примере вложенные поля группы приводят к ненужным визуальным помехам.

**Исправить:** ![ снимок экрана одного элемента управления в одном поле группы ](images/ctrl-group-boxes-image8.png)

В этом примере та же связь элемента управления отображается с помощью макета.

-   Не помещайте элементы управления в метки поля группы.
    -   **Исключение:** Флажок можно использовать как метку поля группы, если все элементы управления внутри поля включены и отключены флажком.

**Неправильный:** ![ снимок экрана с раскрывающимся списком на метке группы ](images/ctrl-group-boxes-image9.png)

В этом примере раскрывающийся список неправильно помещается в поле группы. В этом примере вместо этого следует использовать [вкладки](https://msdn.microsoft.com/library/windows/desktop/aa511493.aspx) .

-   **Не отключайте поля группы.** Чтобы указать, что группа элементов управления в настоящее время не применяется, отключите все элементы управления в поле группы, но не саму группу. Этот подход является более доступным и может поддерживаться единообразно всеми платформами пользовательского интерфейса.

## <a name="labels"></a>Метки

-   Пометка всех полей группы.
-   Не присваивайте метке ключ доступа. Это не является обязательным и делает другие клавиши доступа более сложными для назначения. Вместо этого назначьте ключи доступа элементам управления в поле группы.
    -   **Исключение:** Если у поверхности много элементов управления, доступные ключи доступа могут быть недостаточно. Если это так, сократите число ключей доступа, назначив их группам, а не элементам управления в полях группы.
-   Используйте [прописные буквы в стиле предложения](glossary.md).
-   Напишите метку с помощью существительного или существительного, а не предложения и не используйте закрывающие знаки препинания, включая двоеточия.
-   Используйте параллельные выражения для меток Box группы в той же поверхности.
-   Оставляйте краткие метки полей группы. Не используйте в качестве метки пояснительный текст. Однако можно иметь пояснительный текст в поле группы.
-   Не повторяйте метку группы в надписях элементов управления в поле. Например, если поле группы помечено как выравнивание, пометка переключателей слева, справа и т. д., без выравнивания по левому краю или по правому краю.
-   Не используйте поля группы в тексте пользовательского интерфейса.

## <a name="documentation"></a>Документация

При ссылке на поля группы:

-   Ознакомьтесь с полями групп только в программистах и других технических документах. В поле Группа используйте два слова в нижнем регистре.
-   В любом другом случае не нужно включать имя поля группы в процедуру, если только диалоговое окно не содержит более одного параметра с тем же именем. В таких случаях используйте в поле с именем группы.
-   По возможности отформатируйте метку, используя полужирный текст. В противном случае заключите метку в кавычки, только если это необходимо для предотвращения путаницы.

Пример. в разделе " **эффекты**" выберите **скрытый**.
