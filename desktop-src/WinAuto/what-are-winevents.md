# <a name="what-are-winevents"></a><span data-ttu-id="6d327-103">Что такое Виневентс?</span><span class="sxs-lookup"><span data-stu-id="6d327-103">What Are WinEvents?</span></span>

<span data-ttu-id="6d327-104">Серверные приложения и операционная система используют Виневентс для уведомления клиентов о возникновении изменений в системе или в пользовательском интерфейсе.</span><span class="sxs-lookup"><span data-stu-id="6d327-104">Server applications and the operating system use WinEvents to notify clients when a change occurs in the system or in the user interface.</span></span>

<span data-ttu-id="6d327-105">Поддержка WinEvent — это функция операционной системы Windows, которая предоставляет следующие возможности:</span><span class="sxs-lookup"><span data-stu-id="6d327-105">WinEvent support is a feature of the Windows operating system that provides:</span></span>

-   <span data-ttu-id="6d327-106">Простой способ регистрации клиентов для получения уведомлений о событиях.</span><span class="sxs-lookup"><span data-stu-id="6d327-106">A simple way for clients to register for event notifications.</span></span>
-   <span data-ttu-id="6d327-107">Механизм внедрения клиентского кода в серверы.</span><span class="sxs-lookup"><span data-stu-id="6d327-107">A mechanism for injecting client code into servers.</span></span>
-   <span data-ttu-id="6d327-108">Маршрутизация событий от серверов к заинтересованным клиентам.</span><span class="sxs-lookup"><span data-stu-id="6d327-108">Routing of events from servers to interested clients.</span></span>
-   <span data-ttu-id="6d327-109">Автоматическое создание событий для большинства элементов управления на основе **HWND**.</span><span class="sxs-lookup"><span data-stu-id="6d327-109">Automatic event generation for most **HWND**-based controls.</span></span>

<span data-ttu-id="6d327-110">Создание событий для элементов управления на основе **HWND** особенно важно для разработчиков серверов.</span><span class="sxs-lookup"><span data-stu-id="6d327-110">Event generation for **HWND**-based controls is especially important for server developers.</span></span> <span data-ttu-id="6d327-111">Время выполнения Microsoft Active Accessibility предоставляет прокси-серверы [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) для всех стандартных элементов пользовательского интерфейса.</span><span class="sxs-lookup"><span data-stu-id="6d327-111">The Microsoft Active Accessibility run-time provides [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) proxies for all standard UI elements.</span></span> <span data-ttu-id="6d327-112">Аналогичным образом система автоматически создает соответствующий Виневентс при каждом создании, удалении, перемещении, изменении размера или выполнении любых других действий с элементом управления на основе **HWND**.</span><span class="sxs-lookup"><span data-stu-id="6d327-112">Similarly, the system automatically generates the appropriate WinEvents whenever it creates, destroys, moves, resizes, or performs any other action on an **HWND**-based control.</span></span>

<span data-ttu-id="6d327-113">Некоторые Виневентс, включая общие события **HWND** , автоматически поддерживаются системой.</span><span class="sxs-lookup"><span data-stu-id="6d327-113">Some WinEvents, including general **HWND** events, are automatically supported by the system.</span></span> <span data-ttu-id="6d327-114">Другие типы Виневентс, такие как изменения состояния или события выбора, характерные для конкретного элемента управления, поддерживаются серверами Microsoft Active Accessibility.</span><span class="sxs-lookup"><span data-stu-id="6d327-114">Other types of WinEvents, such as state changes or selection events specific to a particular control, are supported by Microsoft Active Accessibility servers.</span></span>

<span data-ttu-id="6d327-115">При возникновении события, влияющего на пользовательский интерфейс, серверы могут рассылать уведомления о событиях всем заинтересованным клиентам, вызывая функцию [**нотифивиневент**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) .</span><span class="sxs-lookup"><span data-stu-id="6d327-115">When an event occurs that affects the UI, servers can broadcast an event notification to all interested clients by calling the [**NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) function.</span></span> <span data-ttu-id="6d327-116">Вызов функции включает сведения, определяющие тип произошедшего события, а также элемент пользовательского интерфейса, к которому применяется событие.</span><span class="sxs-lookup"><span data-stu-id="6d327-116">The function call includes information that identifies the type of event that occurred, and the UI element to which the event applies.</span></span> <span data-ttu-id="6d327-117">Клиенты могут использовать эти сведения, чтобы получить объект [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) для элемента пользовательского интерфейса и получить дополнительные сведения.</span><span class="sxs-lookup"><span data-stu-id="6d327-117">Clients can use this information to retrieve an [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) object for the UI element and collect more information.</span></span>

<span data-ttu-id="6d327-118">Например, чтобы уведомить клиентов об изменении имени элемента управления, сервер вызывает [**нотифивиневент**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) и передает [**\_ объект события \_ NAMECHANGE**](event-constants.md) в параметре события.</span><span class="sxs-lookup"><span data-stu-id="6d327-118">For example, to notify clients that a control's name has changed, a server calls [**NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) and passes [**EVENT\_OBJECT\_NAMECHANGE**](event-constants.md) in the event parameter.</span></span> <span data-ttu-id="6d327-119">Система отвечает, определяя, какие клиенты зарегистрированы для получения этого события, и вызывает их зарегистрированную функцию обратного вызова.</span><span class="sxs-lookup"><span data-stu-id="6d327-119">The system responds by determining which clients have registered to receive that particular event and calls their registered callback function.</span></span> <span data-ttu-id="6d327-120">Если ни один клиент не зарегистрировался для события, вызов **нотифивиневент** на сервере сравним с "без операции", и влияние на производительность незначительно.</span><span class="sxs-lookup"><span data-stu-id="6d327-120">If no clients have registered for the event, the server's call to **NotifyWinEvent** is comparable to a "no operation" and the performance impact is negligible.</span></span>

<span data-ttu-id="6d327-121">Серверы вызывают [**нотифивиневент**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) , чтобы объявить о событии системе после возникновения события.</span><span class="sxs-lookup"><span data-stu-id="6d327-121">Servers call [**NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent) to announce the event to the system after the event has occurred.</span></span> <span data-ttu-id="6d327-122">Они никогда не должны уведомлять систему о событии до наступления события.</span><span class="sxs-lookup"><span data-stu-id="6d327-122">They must never notify the system of an event before the event occurs.</span></span>

<span data-ttu-id="6d327-123">Чтобы получать уведомления о событиях, клиенты регистрируют функции обработчика обратного вызова с помощью [**сетвиневенсук**](/windows/desktop/api/Winuser/nf-winuser-setwineventhook).</span><span class="sxs-lookup"><span data-stu-id="6d327-123">To be notified of events, clients register callback hook functions by using [**SetWinEventHook**](/windows/desktop/api/Winuser/nf-winuser-setwineventhook).</span></span> <span data-ttu-id="6d327-124">Клиенты устанавливают одну функцию-ловушку для всех возможных событий или несколько функций-обработчиков для дискретных диапазонов событий.</span><span class="sxs-lookup"><span data-stu-id="6d327-124">Clients set a single hook function for all possible events or multiple hook functions for discrete ranges of events.</span></span> <span data-ttu-id="6d327-125">Дополнительные сведения см. [в разделе Регистрация функции-обработчика](registering-a-hook-function.md).</span><span class="sxs-lookup"><span data-stu-id="6d327-125">For more information, see [Registering a Hook Function](registering-a-hook-function.md).</span></span>

<span data-ttu-id="6d327-126">Когда Microsoft Active Accessibility уведомляется о событии, он вызывает все функции-ловушки, зарегистрированные для этого события, передавая параметры из [**нотифивиневент**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent).</span><span class="sxs-lookup"><span data-stu-id="6d327-126">When Microsoft Active Accessibility is notified of an event, it calls any hook functions that were registered for that event, passing along the parameters from [**NotifyWinEvent**](/windows/desktop/api/Winuser/nf-winuser-notifywinevent).</span></span>
