# <a name="edit-control-msaa-ui-element-reference"></a>Элемент управления Edit (Справочник по элементам пользовательского интерфейса MSAA)

> [!Note]  
> В этом разделе описываются объекты **элемента управления Edit** для целей справочника по элементам ПОЛЬЗОВАТЕЛЬСКОГО интерфейса MSAA. Создание объектов **управления правки** в различных ПЛАТФОРМАХ пользовательского интерфейса не описывается здесь. См. справочную документацию по API для платформы пользовательского интерфейса, которую вы используете.

 

Элементы управления "поле ввода" позволяют пользователю просматривать и редактировать текст. Элементы управления "Правка" создаются с множеством различных стилей, таких как ES " \_ Многострочный". этот стиль создает многострочный элемент управления «поле ввода», например клиентскую область Блокнот и « \_ только для чтения», который создает элемент управления «поле редактирования».

Microsoft Active Accessibility не делает различий между элементами управления "Изменить", созданными с помощью имени класса окон "Правка", и элементами управления Rich Edit, созданными с помощью класса Window с именем "RichEdit" или "RichEdit20A".

## <a name="iaccessible-methods"></a>Методы IAccessible

Элементы управления "поле ввода" поддерживают следующие методы [**IAccessible**](/windows/desktop/api/oleacc/nn-oleacc-iaccessible) :

-   [**акчиттест**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acchittest)
-   [**акклокатион**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation)
-   [**аккнавигате**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accnavigate)
-   [**аккселект**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-accselect)

## <a name="notes"></a>Примечания

-   Microsoft Active Accessibility не поддерживает выбор текста, содержащегося в элементах управления Edit и Rich Edit, поскольку текст представлен в виде строки в свойстве **value** объекта.
-   элемент управления rich edit, предоставляемый Riched20.dll (который используется в текстовых редакторах, таких как WordPad в Windows 98), не отправляет никаких виневентс при изменении позиции курсора во время выделения текста. Когда пользователь нажимает клавишу SHIFT и клавиши со стрелками для выбора текста, объект курсора не активирует [**\_ объект события \_ локатиончанже**](event-constants.md) WinEvent. Если выбор задается программно с помощью расширенных сообщений об изменении, объект курсора не отправляет никаких событий для указания его новой позиции.

    Все приложения, использующие Riched20.dll, представляют эту проблему. Приложения, использующие более ранние версии элемента управления Rich Edit, правильно отправляют события на основе выбранных элементов.

-   Значение **состояния** для элементов управления для редактирования пароля всегда включает в себя [**\_ \_ защищенную систему состояния**](object-state-constants.md)битовой отметки.
