# <a name="list-view-control-msaa-ui-element-reference"></a>Элемент управления List-View (Справочник по элементам пользовательского интерфейса MSAA)

> [!Note]  
> В этом разделе описываются объекты **элементов управления "список** " в целях справочника по элементам ПОЛЬЗОВАТЕЛЬСКОГО интерфейса MSAA. Создание объектов **управления "список-представление** " в различных ПЛАТФОРМАХ пользовательского интерфейса не описывается здесь. См. справочную документацию по API для платформы пользовательского интерфейса, которую вы используете.

 

Элемент управления "представление списка" отображает коллекцию элементов, каждая из которых состоит из значка и метки, и предоставляет несколько способов отображения и упорядочения элементов. Например, дополнительные сведения о каждом элементе отображаются в столбцах справа от значка и метки. Windows Обозреватель использует элемент управления "представление списка" для отображения файлов.

Имя класса Window для элемента управления List-View — это WC \_ LISTVIEW, который определен как "сислиствиев" в коммктрл. h.

## <a name="notes"></a>Примечания

-   Microsoft Active Accessibility не предоставляет текст элемента представления списка в приложениях, использующих функцию обратного вызова для добавления текста к элементу.
-   Элементы представления списка, которые обрезаются, не имеют [**\_ \_ невидимого набора системы состояний**](object-state-constants.md) . Чтобы определить, обрезан ли элемент списка, клиенты могут сравнить координаты элемента, возвращенные [**акклокатион**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) , с областью обрезки окна.
-   Microsoft Active Accessibility не поддерживает использование метода [**аккнавигате**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) для перехода к элементу управления "заголовок" в элементе управления "представление списка", который находится в представлении Details. Однако клиенты используют функцию [**акцессиблеобжектфромпоинт**](/windows/desktop/api/Oleacc/nf-oleacc-accessibleobjectfrompoint) для перехода к элементу управления "заголовок".
-   Для непрямоугольных объектов, таких как элементы представления списка в представлении Details, координаты ограничивающего прямоугольника элемента, извлекаемого [**акклокатион**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acclocation) , завершаются ошибкой при тестировании с помощью [**акчиттест**](/windows/desktop/api/Oleacc/nf-oleacc-iaccessible-acchittest) , так как **акчиттест** определяет границы объекта в зависимости от пикселя.
