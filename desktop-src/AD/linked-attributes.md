# <a name="linked-attributes-ad-ds"></a><span data-ttu-id="0b6e2-105">Связанные атрибуты (AD DS)</span><span class="sxs-lookup"><span data-stu-id="0b6e2-105">Linked Attributes (AD DS)</span></span>

<span data-ttu-id="0b6e2-106">Связанные атрибуты — это пары атрибутов, в которых система вычисляет значения одного атрибута (ссылки назад) на основе значений, заданных для другого атрибута (прямая ссылка) в пределах леса.</span><span class="sxs-lookup"><span data-stu-id="0b6e2-106">Linked attributes are pairs of attributes in which the system calculates the values of one attribute (the back link) based on the values set on the other attribute (the forward link) throughout the forest.</span></span> <span data-ttu-id="0b6e2-107">Значение обратной связи для любого экземпляра объекта состоит из DNs всех объектов, которые имеют DN объекта, заданное в соответствующей ссылке вперед.</span><span class="sxs-lookup"><span data-stu-id="0b6e2-107">A back-link value on any object instance consists of the DNs of all the objects that have the object's DN set in the corresponding forward link.</span></span> <span data-ttu-id="0b6e2-108">Например, "Manager" и "Reports" — это пара связанных атрибутов, где Manager — это прямая ссылка, а отчеты — обратная ссылка.</span><span class="sxs-lookup"><span data-stu-id="0b6e2-108">For example, "Manager" and "Reports" are a pair of linked attributes, where Manager is the forward link and Reports is the back link.</span></span> <span data-ttu-id="0b6e2-109">Теперь предположим, что счет — это менеджер Джо.</span><span class="sxs-lookup"><span data-stu-id="0b6e2-109">Now suppose Bill is Joe's manager.</span></span> <span data-ttu-id="0b6e2-110">Если вы храните различающееся имя объекта пользователя Bill в атрибуте "Manager" объекта пользователя Джо, то DN объекта пользователя Джо будет отображаться в атрибуте "Reports" объекта "счет".</span><span class="sxs-lookup"><span data-stu-id="0b6e2-110">If you store the DN of Bill's user object in the "Manager" attribute of Joe's user object, then the DN of Joe's user object will show up in the "Reports" attribute of Bill's user object.</span></span>

<span data-ttu-id="0b6e2-111">Пара ссылок вперед и назад определяется значениями [**LinkId**](/windows/desktop/ADSchema/a-linkid) двух определений [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) .</span><span class="sxs-lookup"><span data-stu-id="0b6e2-111">A forward link/back link pair is identified by the [**linkID**](/windows/desktop/ADSchema/a-linkid) values of two [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) definitions.</span></span> <span data-ttu-id="0b6e2-112">**LinkId** прямой ссылки — это четное, положительное, ненулевое значение, а **LinkId** связанной ссылки назад — это прямой **LinkId** плюс один.</span><span class="sxs-lookup"><span data-stu-id="0b6e2-112">The **linkID** of the forward link is an even, positive, nonzero value, and the **linkID** of the associated back link is the forward **linkID** plus one.</span></span> <span data-ttu-id="0b6e2-113">Например, **LinkId** для "Manager" — 42, а **LinkId** для "reports" — 43.</span><span class="sxs-lookup"><span data-stu-id="0b6e2-113">For example, the **linkID** for "Manager" is 42 and the **linkID** for "Reports" is 43.</span></span>

-   <span data-ttu-id="0b6e2-121">Значение [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) прямой ссылки должно быть 2.5.5.1, 2.5.5.7 или 2.5.5.14.</span><span class="sxs-lookup"><span data-stu-id="0b6e2-121">The [**attributeSchema**](/windows/desktop/ADSchema/c-attributeschema) value of a forward link must be 2.5.5.1, 2.5.5.7, or 2.5.5.14.</span></span> <span data-ttu-id="0b6e2-122">Эти значения соответствуют синтаксису, содержащему различающееся имя, например, синтаксис [**Object (DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn) .</span><span class="sxs-lookup"><span data-stu-id="0b6e2-122">These values correspond to syntaxes that contain a distinguished name, such as the [**Object(DS-DN)**](/windows/desktop/ADSchema/s-object-ds-dn) syntax.</span></span>
