# <a name="dynamic-objects"></a><span data-ttu-id="8e26c-104">Динамические объекты</span><span class="sxs-lookup"><span data-stu-id="8e26c-104">Dynamic Objects</span></span>

<span data-ttu-id="8e26c-105">В Windows Server 2003 и более поздних версиях Windows службы домен Active Directory обеспечивают поддержку хранения динамических записей в каталоге.</span><span class="sxs-lookup"><span data-stu-id="8e26c-105">In Windows Server 2003 and later versions of Windows, Active Directory Domain Services provide support for storing dynamic entries in the directory.</span></span> <span data-ttu-id="8e26c-106">Динамическая запись — это объект в каталоге, который имеет соответствующее значение срока жизни (TTL).</span><span class="sxs-lookup"><span data-stu-id="8e26c-106">A dynamic entry is an object in the directory which has an associated time-to-live (TTL) value.</span></span> <span data-ttu-id="8e26c-107">TTL для записи задается при создании записи.</span><span class="sxs-lookup"><span data-stu-id="8e26c-107">The TTL for an entry is set when the entry is created.</span></span> <span data-ttu-id="8e26c-108">Срок жизни автоматически уменьшается, а срок действия динамической записи исчезает.</span><span class="sxs-lookup"><span data-stu-id="8e26c-108">The time-to-live is automatically decremented, and when it expires the dynamic entry disappears.</span></span> <span data-ttu-id="8e26c-109">Клиент может привести к тому, что динамическая запись будет оставаться длиннее, чем ее текущий оставшийся срок, путем обновления (изменения) его значения TTL.</span><span class="sxs-lookup"><span data-stu-id="8e26c-109">The client can cause a dynamic entry to remain longer than its current remaining life by refreshing (modifying) its TTL value.</span></span> <span data-ttu-id="8e26c-110">Клиенты, которые хранят динамические данные в каталоге, должны периодически обновлять эти данные, чтобы предотвратить их отображение.</span><span class="sxs-lookup"><span data-stu-id="8e26c-110">Clients that store dynamic data in the directory must periodically refresh that data to prevent it from disappearing.</span></span>

<span data-ttu-id="8e26c-111">Многие приложения и службы, использующие LDAP для хранения и доступа относительно статических и глобально интересных данных с сервера Active Directory, также предпочитают использовать доступ LDAP для удовлетворения своих динамических данных.</span><span class="sxs-lookup"><span data-stu-id="8e26c-111">Many applications and services that use LDAP to store and access relatively static and globally interesting data from an Active Directory server would also prefer to continue using LDAP access for their dynamic data needs.</span></span> <span data-ttu-id="8e26c-112">Более того, для некоторых приложений может быть нежелательно загружается задача сбора мусора объектов в службе каталогов, которая имеет ограниченный срок службы каталога.</span><span class="sxs-lookup"><span data-stu-id="8e26c-112">Moreover, for some applications, it may be desirable to off-load the task of garbage-collecting objects in the DS that have a limited useful life to the directory service.</span></span> <span data-ttu-id="8e26c-113">Разделы каталога приложений (с возможностью управляемого размещения реплик) и "многообъектный доступ к каждому объекту" обеспечивают возможность размещения динамических данных в домен Active Directory Services, позволяя таким образом предоставлять доступ к нему по протоколу LDAP.</span><span class="sxs-lookup"><span data-stu-id="8e26c-113">Application directory partitions (with the ability for controlled placement of replicas) and per-object TTLs together will provide the capability of hosting dynamic data in Active Directory Domain Services, thus allowing LDAP access to it.</span></span>

<span data-ttu-id="8e26c-114">Новый вспомогательный класс объекта с именем **DynamicObject** с OID = 1.3.6.1.4.1.1466.101.119.2 будет определен в базовой схеме AD для использования динамическими записями.</span><span class="sxs-lookup"><span data-stu-id="8e26c-114">A new auxiliary object class called **dynamicObject** with OID = 1.3.6.1.4.1.1466.101.119.2 will be defined in the base AD schema to be used by dynamic entries.</span></span> <span data-ttu-id="8e26c-115">Этот вспомогательный класс содержит атрибут с именем **ентриттл** с OID = 1.3.6.1.4.1.1466.101.119.3 в качестве системного-может содержать атрибут.</span><span class="sxs-lookup"><span data-stu-id="8e26c-115">This auxiliary class contains the attribute called **entryTTL** with OID = 1.3.6.1.4.1.1466.101.119.3 as a system-may-contain attribute.</span></span> <span data-ttu-id="8e26c-116">Значение этого атрибута равно времени в секундах, в течение которого должна существовать соответствующая запись каталога, прежде чем она будет выведена из каталога.</span><span class="sxs-lookup"><span data-stu-id="8e26c-116">The value of this attribute is the "time in seconds" that the corresponding directory entry will continue to exist before disappearing from the directory.</span></span> <span data-ttu-id="8e26c-117">Если клиент не предоставил значение для этого атрибута явно при создании объекта, то DS предоставляет значение по умолчанию, как указано далее.</span><span class="sxs-lookup"><span data-stu-id="8e26c-117">If the client does not supply a value for this attribute explicitly at object creation, then the DS provides a default value as specified later.</span></span>

<span data-ttu-id="8e26c-118">Новая расширенная операция LDAP с OID = 1.3.6.1.4.1.1466.101.119.1 для обновления клиента динамической записи в каталоге будет определена и опубликована в атрибуте **суппортедекстенсион** корневого объекта DSE.</span><span class="sxs-lookup"><span data-stu-id="8e26c-118">A new extended LDAP operation with OID = 1.3.6.1.4.1.1466.101.119.1 for client refresh of a dynamic entry in the directory will be defined and published in the **supportedExtension** attribute of the root DSE object.</span></span>

<span data-ttu-id="8e26c-119">В реальной реализации **ентриттл** является сконструированным атрибутом.</span><span class="sxs-lookup"><span data-stu-id="8e26c-119">In the actual implementation, **entryTTL** is a constructed attribute.</span></span> <span data-ttu-id="8e26c-120">Фактическое время истечения срока действия объекта хранится в виде абсолютного времени, когда объект может быть удален в атрибуте только для системы, именуемый **MS-DS-Entry-Time-to-Live**.</span><span class="sxs-lookup"><span data-stu-id="8e26c-120">The real object expiration time is stored as an absolute time when the object can be destroyed in a system-only attribute called **ms-DS-Entry-Time-To-Live**.</span></span>

<span data-ttu-id="8e26c-121">Все динамические объекты имеют следующие ограничения.</span><span class="sxs-lookup"><span data-stu-id="8e26c-121">All dynamic objects have the following limitations:</span></span>

-   <span data-ttu-id="8e26c-122">Удаленный динамический объект из-за истечения срока жизни TTL не оставляет отметку полного удаления.</span><span class="sxs-lookup"><span data-stu-id="8e26c-122">A deleted dynamic object due to its TTL expiring does not leave a tombstone behind.</span></span>
-   <span data-ttu-id="8e26c-123">Все контроллеры домена, содержащие реплики динамических объектов, должны работать на Windows Server 2003.</span><span class="sxs-lookup"><span data-stu-id="8e26c-123">All DCs holding replicas of dynamic objects must run on Windows Server 2003.</span></span>
-   <span data-ttu-id="8e26c-124">Динамические записи со значениями TTL поддерживаются во всех секциях, кроме раздела конфигурации и раздела схемы.</span><span class="sxs-lookup"><span data-stu-id="8e26c-124">Dynamic entries with TTL values are supported in all partitions except the Configuration partition and Schema partition.</span></span>

-   <span data-ttu-id="8e26c-127">Невозможно изменить статическую запись на динамическую запись и наоборот.</span><span class="sxs-lookup"><span data-stu-id="8e26c-127">There is no way to change a static entry into a dynamic entry and vice-versa.</span></span>
-   <span data-ttu-id="8e26c-128">Нединамическую запись нельзя добавить в динамическую запись.</span><span class="sxs-lookup"><span data-stu-id="8e26c-128">A non-dynamic entry cannot be added subordinate to a dynamic entry.</span></span>
