<span data-ttu-id="5d98e-105">Служба удаленного управления Windows управление оборудованием предназначено для снижения общей стоимости ИТ-администрирования за счет мониторинга и контроля удаленных аппаратных компонентов, особенно перед запуском системы и после сбоя операционной системы.</span><span class="sxs-lookup"><span data-stu-id="5d98e-105">Windows Remote Management hardware management is intended to reduce overall IT administration costs by providing monitoring and control of remote hardware components, especially before the system is started and after an operating system failure.</span></span>

<span data-ttu-id="5d98e-106">Изготовители оборудования (OEM) разработали общую архитектуру для решения этой потребности в управлении оборудованием.</span><span class="sxs-lookup"><span частью этой архитектуры является [*контроллер управления основной платой*](windows-remote-management-glossary.md) (BMC).</span><span class="sxs-lookup"><span data-stu-id="5d98e-107">An important piece of this architecture is the [*baseboard management controller*](windows-remote-management-glossary.md) (BMC).</span></span> <span data-ttu-id="5d98e-108">Контроллер BMC — это специализированное computer.</span></span> <span data-ttu-id="5d98e-109">Контроллер BMC обеспечивает удаленное управление аппаратным обеспечением сервера, получением данных о состоянии и получением уведомлений о критических ошибках и других изменениях состояния оборудования.</span><span class="sxs-lookup"><span data-stu-id="5d98e-109">The BMC provides remote control of server hardware, retrieves status data, and receives notifications about critical errors and other hardware status changes.</span></span> <span data-ttu-id="5d98e-110">Сценарий или приложение, которое наблюдает за удаленным сервером, может получать данные с сервера с помощью [*аппаратного контроллера управления*](windows-remote-management-glossary.md), через удаленную операционную систему или по внешнему [*каналу*](windows-remote-management-glossary.md)непосредственно из BMC.</span><span class="sxs-lookup"><span data-stu-id="5d98e-110">A script or application that is monitoring a remote server can obtain data from the server either [*in-band*](windows-remote-management-glossary.md), through the remote operating system, or [*out-of-band*](windows-remote-management-glossary.md), directly from the BMC.</span></span>

<span data-ttu-id="5d98e-111">В контроллере BMC есть датчики, которые могут обнаружить, например, когда компьютер сервера перегревается или когда напряжение выходит за пределы допустимого диапазона.</span><span class="sxs-lookup"><span data-stu-id="5d98e-111">A BMC has sensors that can detect, for example, when the server computer is overheating or when voltage is out of the acceptable range.</span></span> <span data-ttu-id="5d98e-112">Для определения архитектуры BMC существует несколько стандартов.</span><span class="sxs-lookup"><span data-stu-id="5d98e-112">Several standards exist to define the architecture of BMC.</span></span> <span data-ttu-id="5d98e-113">Стандартный [*интерфейс управления платформой (IPMI)*](windows-remote-management-glossary.md) — это один из таких стандартов, который часто используется.</span><span class="sxs-lookup"><span data-stu-id="5d98e-113">The [*Intelligent Platform Management Interface (IPMI)*](windows-remote-management-glossary.md) is one such standard that is used frequently.</span></span> <span data-ttu-id="5d98e-114">Однако, несмотря на стандарт IPMI, доступ управления к оборудованию сервера является собственным и требует использования средств управления, предоставляемых поставщиками оборудования.</span><span class="sxs-lookup"><span data-stu-id="5d98e-114">However, despite the IPMI standard, management access to server hardware is proprietary and requires use of management tools supplied by OEMs.</span></span> <span data-ttu-id="5d98e-115">Кроме того, удаленный доступ к контроллеру BMC предоставляется с помощью специального сетевого протокола (Remote Management Protocol), который имеет нестандартные механизмы безопасности для проверки подлинности доступа.</span><span class="sxs-lookup"><span data-stu-id="5d98e-115">Also, remote access to a BMC is provided using a specialized wire protocol, Remote Management Control Protocol (RMCP), which has nonstandard security mechanisms for authentication of access.</span></span>

<span data-ttu-id="5d98e-116">Поставщик Microsoft [IPMI](/previous-versions/windows/desktop/ipmiprv/ipmi-provider) и драйвер IPMI позволяют получать данные BMC с удаленных серверных компьютеров с помощью стандартного поставщика WMI с [классами](/previous-versions/windows/desktop/ipmiprv/ipmi-provider)WMI.</span><span class="sxs-lookup"><span data-stu-id="5d98e-116">The Microsoft [IPMI provider](/previous-versions/windows/desktop/ipmiprv/ipmi-provider) and IPMI driver, allow you to obtain BMC data from remote server computers through a standard WMI provider with WMI [classes](/previous-versions/windows/desktop/ipmiprv/ipmi-provider).</span></span> <span data-ttu-id="5d98e-117">Хотя вы можете написать стандартный сценарий WMI, который получает удаленные данные через DCOM, во многих случаях предпочтительным способом получения данных IPMI является использование служебной программы командной строки **WinRM** или [API сценариев WinRM](winrm-scripting-api.md) или [интерфейса API WinRM C++](winrm-c---api.md).</span><span class="sxs-lookup"><span data-stu-id="5d98e-117">While you can write a normal WMI script that obtains remote data through DCOM, in many cases the preferred method of obtaining IPMI data is through the **Winrm** command-line utility or the [WinRM Scripting API](winrm-scripting-api.md) or [WinRM C++ API](winrm-c---api.md).</span></span> <span data-ttu-id="5d98e-118">Служебная программа WinRM и API службы WinRM используют протокол WS-Management и могут получать данные IPMI с локального или удаленного компьютера без использования DCOM.</span><span class="sxs-lookup"><span data-stu-id="5d98e-118">The Winrm utility and the WinRM service APIs rely on the WS-Management protocol and can obtain IPMI data either from local or remote computer without using DCOM.</span></span>

<span data-ttu-id="5d98e-119">В BMC также есть база данных событий, называемая журналом системных событий (SEL), которая записывает события на наблюдаемом компьютере.</span><span class="sxs-lookup"><span data-stu-id="5d98e-119">The BMC also has an event database called the System Event Log (SEL) which records events in the monitored computer.</span></span> <span data-ttu-id="5d98e-120">Нельзя подписываться на то, что эти события доставляются в сценарий, как это можно делать с классами событий WMI.</span><span class="sxs-lookup"><span data-stu-id="5d98e-120">You cannot subscribe to have these events delivered to a script as you can with WMI event classes.</span></span> <span data-ttu-id="5d98e-121">Однако вы можете подписываться на них с помощью программы командной строки Wecutil.exe.</span><span class="sxs-lookup"><span data-stu-id="5d98e-121">However, you can use the Wecutil.exe command-line tool to subscribe to them.</span></span> <span data-ttu-id="5d98e-122">Для получения дополнительных сведений об использовании этого средства в командной строке введите **wecutil/?**.</span><span class="sxs-lookup"><span data-stu-id="5d98e-122">For more information about how to use this tool, at a command prompt type **wecutil /?**.</span></span>
