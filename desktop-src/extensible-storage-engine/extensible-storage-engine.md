<span data-ttu-id="8743d-121">ESE предоставляет механизм хранилища в пользовательском режиме, который управляет данными внутри неструктурированных двоичных файлов, доступных через API-интерфейсы Windows.</span><span class="sxs-lookup"><span data-stu-id="8743d-121">ESE provides a user-mode storage engine that manages data inside of flat, binary files that are accessible through the Windows APIs.</span></span> <span data-ttu-id="8743d-122">Доступ к ESE осуществляется через библиотеку DLL, которая загружается непосредственно в процесс приложения; ни один из методов удаленного доступа не требуется или предоставляется ядром СУБД.</span><span class="sxs-lookup"><span data-stu-id="8743d-122">ESE is accessed through a DLL that is loaded directly into the application's process; no remote access methods are required of or provided by the database engine itself.</span></span> <span data-ttu-id="8743d-123">Хотя ESE не имеет удаленного или межпроцессного доступа, используемые файлы данных можно предоставлять удаленно с помощью протокола SMB через API-интерфейсы Windows, но это не рекомендуется.</span><span class="sxs-lookup"><span data-stu-id="8743d-123">Though ESE has no remote or inter-process access method, the data files it uses can be provided remotely by using server message block (SMB) through the Windows APIs, but this is not recommended.</span></span>
