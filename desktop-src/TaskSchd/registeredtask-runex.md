# <a name="registeredtaskrunex-method"></a>Регистередтаск. Рунекс, метод

Для сценариев запускает зарегистрированную задачу сразу же с помощью указанных флагов и идентификатора сеанса.

## <a name="syntax"></a>Синтаксис


```VB
RegisteredTask.RunEx( _
  ByVal params, _
  ByVal flags, _
  ByVal sessionID, _
  ByRef runningTask _
)
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Параметры* \[ окне\]
</dt> <dd>

Параметры, используемые в качестве значений в действиях задач. Чтобы не указывать значения параметров для действий задач, присвойте этому параметру значение **Nothing**. В противном случае можно указать одно строковое значение или массив строковых значений.

Указанные строковые значения связаны с именами и хранятся в виде пар "имя-значение". Если указать одно строковое значение, то Arg0 будет иметь имя, присвоенное значению. Значение может использоваться в действии задачи, где переменная $ (Arg0) используется в свойствах действия.

Если передать значения, такие как "0", "100" и "250", в виде массива строковых значений, "0" заменит переменные $ (Arg0), "100" заменит переменные $ (arg1), а "250" заменит переменные $ (arg2), используемые в свойствах действия.

Можно указать не более 32 строковых значений.

Дополнительные сведения и список свойств действий, которые могут использовать переменные $ (Arg0), $ (arg1),..., $ (Arg32) в своих значениях, см. в разделе [действия задач](task-actions.md).

</dd> <dt>

*Флаги* \[ окне\]
</dt> <dd>

Константа [ \_ \_ флага запуска задачи](/windows/desktop/api/taskschd/ne-taskschd-task_run_flags) , определяющая, как выполняется задача.

</dd> <dt>

*идентификатор сеанса* \[ окне\]
</dt> <dd>

Сеанс сервера терминалов, в котором требуется запустить задачу.

Если в \_ \_ \_ \_ параметре *flags* не передана константа с идентификатором сеанса, то значение, указанное в этом параметре, игнорируется. Если при \_ выполнении задачи \_ в \_ \_ параметре *flags* передается константа идентификатора сеанса, а значение SessionID меньше или равно 0, то возвращается ошибка недопустимого аргумента.

Если в \_ \_ \_ \_ параметре *flags* передается константа с идентификатором сеанса, а значение SessionID является допустимым идентификатором сеанса больше 0 и если для параметра *user* не указано значение, то служба Планировщик задач попытается запустить задачу в интерактивном режиме в качестве пользователя, выполнившего вход в указанный сеанс.

Если в \_ \_ \_ \_ параметре *flags* передается константа с идентификатором сеанса, а значением SessionID является допустимый идентификатор сеанса больше 0. Если пользователь указан в параметре *User* , то служба Планировщик задач попытается запустить задачу в интерактивном режиме в качестве пользователя, указанного в параметре *User* .

</dd> <dt>

*руннингтаск* \[ заполняет\]
</dt> <dd>

Объект [**руннингтаск**](runningtask.md) , определяющий новый экземпляр задачи.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод не возвращает значение.

## <a name="remarks"></a>Комментарии

Этот метод будет возвращать значение без ошибок, но задача не будет выполняться, если для зарегистрированной задачи свойство [**тасксеттингс. алловдемандстарт**](tasksettings-allowdemandstart.md) имеет значение false.
