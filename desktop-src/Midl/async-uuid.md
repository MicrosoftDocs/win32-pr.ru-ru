# <a name="async_uuid-attribute"></a><span data-ttu-id="80605-104">асинхронный \_ атрибут UUID</span><span class="sxs-lookup"><span data-stu-id="80605-104">async\_uuid attribute</span></span>

    uuid(string-uuid1), 
    async_uuid(string-uuid2)[ [, interface-attribute-list]

## <a name="remarks"></a><span data-ttu-id="80605-121">Комментарии</span><span class="sxs-lookup"><span data-stu-id="80605-121">Remarks</span></span>

<span data-ttu-id="80605-122">Для использования этого атрибута требуется Windows 2000 или более поздней версии Windows.</span><span class="sxs-lookup"><span data-stu-id="80605-122">The use of this attribute requires Windows 2000 or later versions of Windows.</span></span>

<span data-ttu-id="80605-123">При применении атрибута **\[ асинхронного \_ UUID \]** к com-интерфейсу (то есть к интерфейсу, имеющему атрибут [**\[ Object \]**](object.md) ) компилятор MIDL создает асинхронное определение интерфейса в дополнение к традиционной синхронной версии.</span><span class="sxs-lookup"><span data-stu-id="80605-123">When you apply the **\[async\_uuid\]** attribute to a COM interface (that is, an interface that has the [**\[object\]**](object.md) attribute), the MIDL compiler generates an asynchronous definition of the interface, in addition to the traditional, synchronous version.</span></span> <span data-ttu-id="80605-124">Асинхронный интерфейс будет иметь те же имена, что и синхронный интерфейс, но с префиксом Async.</span><span class="sxs-lookup"><span data-stu-id="80605-124">The asynchronous interface will have the same names as the synchronous interface, but with an "Async" prefix.</span></span> <span data-ttu-id="80605-125">Идентификатор интерфейса (IID) будет являться UUID, указанным в качестве параметра для **\[ асинхронного атрибута \_ UUID \]** .</span><span class="sxs-lookup"><span data-stu-id="80605-125">The interface identifier (IID) will be the UUID specified as a parameter to the **\[async\_uuid\]** attribute.</span></span>

<span data-ttu-id="80605-126">Для асинхронного интерфейса MIDL разделяет каждый метод на отдельные методы *Begin* и *Finish* .</span><span class="sxs-lookup"><span data-stu-id="80605-126">For the asynchronous interface, MIDL splits each method into separate *begin* and *finish* methods.</span></span> <span data-ttu-id="80605-127">Метод *Begin* имеет синхронное имя метода с префиксом "Begin \_ " и включает все [**\[ входные \]**](in.md) параметры из синхронного метода.</span><span class="sxs-lookup"><span data-stu-id="80605-127">The *begin* method has the synchronous method name with a "Begin\_" prefix and includes all of the [**\[in\]**](in.md) parameters from the synchronous method.</span></span> <span data-ttu-id="80605-128">Метод *Finish* имеет имя синхронного метода с \_ префиксом Finish и включает все параметры [**\[ out \]**](out-idl.md) из синхронного метода.</span><span class="sxs-lookup"><span data-stu-id="80605-128">The *finish* method has the synchronous method's name with a "Finish\_" prefix and includes all of the [**\[out\]**](out-idl.md) parameters from the synchronous method.</span></span> <span data-ttu-id="80605-129">Если синхронный метод содержит любые **\[ входные параметры \] ,** они будут включены в асинхронные методы *Begin* и *Finish* .</span><span class="sxs-lookup"><span data-stu-id="80605-129">If the synchronous method has any **\[in, out\]** parameters they will be included in both the *begin* and *finish* asynchronous methods.</span></span>

<span data-ttu-id="80605-130">Если асинхронный метод интерфейса имеет атрибут [**\[ Call \_ \]**](call-as.md) , то MIDL создаст объявления для методов *Begin* и *Finish* .</span><span class="sxs-lookup"><span data-stu-id="80605-130">If an asynchronous interface method has the [**\[call\_as\]**](call-as.md) attribute, MIDL will generate declarations for the *begin* and the *finish* methods.</span></span> <span data-ttu-id="80605-131">Необходимо реализовать оба метода.</span><span class="sxs-lookup"><span data-stu-id="80605-131">You must implement both methods.</span></span>

<span data-ttu-id="80605-132">Каждый асинхронный интерфейс является модификатором синхронного интерфейса и, таким образом, не имеет отдельного графа наследования.</span><span class="sxs-lookup"><span data-stu-id="80605-132">Each asynchronous interface is a modifier on a synchronous interface and, as such, does not have a separate inheritance graph.</span></span> <span data-ttu-id="80605-133">Это означает, что нельзя определить синхронный интерфейс из асинхронного интерфейса (отличного от **IUnknown**).</span><span class="sxs-lookup"><span data-stu-id="80605-133">This means that you cannot define a synchronous interface from an asynchronous interface (other than **IUnknown**).</span></span> <span data-ttu-id="80605-134">Кроме того, синхронные интерфейсы наследуются от асинхронных интерфейсов.</span><span class="sxs-lookup"><span data-stu-id="80605-134">Nor can synchronous interfaces inherit from asynchronous interfaces.</span></span> <span data-ttu-id="80605-135">Компилятор MIDL выдаст сообщение об ошибке, если вы попытаетесь выполнить одно из них.</span><span class="sxs-lookup"><span data-stu-id="80605-135">The MIDL compiler will issue an error message if you attempt either one.</span></span>
