# <a name="distributed-routing-table-api"></a>API таблиц распределенной маршрутизации

API таблицы распределенной маршрутизации (DRT) позволяет приложениям публиковать и разрешать цифровые ключи в одноранговой сети.

Приложение, использующее DRT, может выполнить следующие действия:

-   Создание распределенных таблиц маршрутизации для конкретных приложений.
-   Регистрация ключей и связывание этих ключей с IP-адресами и данными приложений.
-   Выполните поиск ключей и получите связанные с ними IP-адреса и данные приложений. Эта функция может использоваться для создания распределенных хэш-таблиц (Дхтс), бессерверных систем разрешения имен и наложения сетей сетки многих топологий.

> [!Note]  
> Администраторы и пользователи приложений с поддержкой DRT должны быть уверены в том, что конечные пользователи своих приложений знают, какие сведения публикуются. Серверы Майкрософт не работают с этой технологией, и информация не отправляется в корпорацию Майкрософт.
