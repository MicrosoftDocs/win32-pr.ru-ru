# <a name="wm_cap_dlg_videocompression-message"></a>\_ \_ Сообщение видеокомпрессион с диалогом WM Cap \_

В сообщении о **\_ \_ \_ видеокомпрессионе WM Cap** появляется диалоговое окно, в котором пользователь может выбрать программу сжатия для использования в процессе записи. Список доступных компрессоров может различаться в зависимости от формата видео, выбранного в диалоговом окне Формат видео драйвера записи. Это сообщение можно отправить явно или с помощью макроса [**капдлгвидеокомпрессион**](/windows/desktop/api/Vfw/nf-vfw-capdlgvideocompression) .


```C++
WM_CAP_DLG_VIDEOCOMPRESSION 
wParam = (WPARAM) 0; 
lParam = 0L; 
```



## <a name="return-value"></a>Возвращаемое значение

Возвращает **значение true** в случае успеха или **false** в противном случае.

## <a name="remarks"></a>Комментарии

Используйте это сообщение с драйверами записи, предоставляющими кадры только в формате RGB для бизнес-аналитики \_ . Это сообщение наиболее полезно в операции записи шага для объединения захвата и сжатия в одной операции. Сжатие кадров с программным компрессором в рамках операции записи в реальном времени, скорее всего, занимает слишком много времени.

Сжатие не влияет на кадры, скопированные в буфер обмена.
