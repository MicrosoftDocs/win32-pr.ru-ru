# <a name="mci_unfreeze-command"></a>Команда "отменить \_ замораживание" MCI

\_Команда РАЗморозка MCI восстанавливает движение в область видеобуфера, замороженную командой [MCI \_ Freeze](mci-freeze.md) . Эта команда распознает цифровые видеоролики, видеомагнитофоны и устройства наложения видео.

Чтобы отправить эту команду, вызовите функцию [**мЦисендкомманд**](/previous-versions//dd757160(v=vs.85)) со следующими параметрами.


```C++
MCIERROR mciSendCommand(
  MCIDEVICEID wDeviceID, 
  MCI_UNFREEZE, 
  DWORD dwFlags, 
  (DWORD) (LPMCI_GENERIC_PARMS) lpUnfreeze
);
```

## <a name="return-value"></a>Возвращаемое значение

Возвращает нуль в случае успеха или ошибку в противном случае.

## <a name="remarks"></a>Remarks

С типом устройства **дигиталвидео** используется следующий дополнительный флаг:

MCI \_ ДГВ \_ Rect

Элемент **RC** структуры, идентифицируемой *лпунфризе* , содержит допустимый отображаемый прямоугольник. Прямоугольник задает область внутри буфера кадров, в пикселях которой должен быть снят бит маски блокировки. Прямоугольные области указываются, как описано [для \_ команды MCI](mci-put.md) . Если этот параметр опущен, прямоугольником по умолчанию выделяется весь буфер кадров. С помощью последовательности команд Freeze и unfreeze с различными прямоугольниками можно описать произвольные шаблоны битов маски блокировки.

Для устройств с цифровыми видео параметр *лпунфризе* указывает на структуру **MCI \_ ДГВ \_ unfreeze \_ пармс** . Дополнительные сведения см. в комментариях к структуре [**MCI \_ ДГВ \_ Rect \_ пармс**](/windows/win32/api/digitalv/ns-digitalv-mci_dgv_rect_parms) .

Для типа устройства **видеомагнитофона** используются следующие дополнительные флаги:

<span id="MCI_VCR_UNFREEZE_INPUT"></span><span id="mci_vcr_unfreeze_input"></span>видеомагнитофон MCI \_ \_ разморозить \_ входные данные
</dt> <dd>

Разморозить входные данные.

</dd> <dt>

<span id="MCI_VCR_UNFREEZE_OUTPUT"></span><span id="mci_vcr_unfreeze_output"></span>\_ \_ незамораживание \_ выходных данных видеомагнитофона MCI
</dt> <dd>

Разморозить выходные данные.

</dd> </dl>

Для типа устройства **оверлея** используется следующий дополнительный флаг:

<dl> <dt>
