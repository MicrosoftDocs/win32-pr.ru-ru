# <a name="about-the-video-compression-manager"></a>О диспетчере сжатия видео

Как правило, устанавливаемые программы сжатия работают с изображениями видео, хранящимися в файлах AVI. В этом обзоре объясняются методики программирования, используемые для доступа к устанавливаемым программам сжатия с помощью ВКМ, и рассматриваются следующие темы:

-   Архитектура ВКМ и видео для архитектуры Windows
-   Сжатие и распаковка данных изображения из приложения
-   Рисование данных из приложения с помощью модулей подготовки отчетов ВКМ
-   Функции и структуры ВКМ

Перед прочтением этого обзора необходимо ознакомиться с графическими службами Microsoft Win32. В частности, точечные рисунки и структуры, связанные с точечными рисунками, такие как [**битмапинфо**](/windows/win32/api/wingdi/ns-wingdi-bitmapinfo) и [**битмапинфохеадер**](/windows/win32/api/wingdi/ns-wingdi-bitmapinfoheader), широко используются ВКМ. Дополнительные сведения о структурах **битмапинфо** и **битмапинфохеадер** см. в разделе [растровые изображения](/previous-versions//ms532349(v=vs.85)).

> [!Note]  
> Диспетчер аудиосжатия (ACM) обеспечивает поддержку на уровне системы для драйверов сжатия аудио и распаковки. Описание служб сжатия звука см. в разделе [Диспетчер](audio-compression-manager.md)аудиосжатия.
