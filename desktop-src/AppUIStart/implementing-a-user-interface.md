### <a name="consistent-ui-is-good-ui"></a><span data-ttu-id="7af98-176">Совместимый пользовательский интерфейс — хороший пользовательский интерфейс</span><span class="sxs-lookup"><span data-stu-id="7af98-176">Consistent UI is Good UI</span></span>

<span data-ttu-id="7af98-180">В редких случаях согласованность не дает пользователю никаких преимуществ и может даже ухудшить взаимодействие с пользователем.</span><span class="sxs-lookup"><span data-stu-id="7af98-180">In rare cases, consistency provides no benefit to the user and can even degrade the user experience.</span></span> <span data-ttu-id="7af98-181">Не выполняйте согласованность пользовательского интерфейса, если такая согласованность снижает возможность выполнения задачи.</span><span class="sxs-lookup"><span data-stu-id="7af98-181">Do not make the UI consistent if that consistency impairs the ability to accomplish a task.</span></span> <span data-ttu-id="7af98-182">Согласованность сама по себе не гарантирует удобство использования.</span><span class="sxs-lookup"><span data-stu-id="7af98-182">Consistency in itself does not guarantee usability.</span></span> <span data-ttu-id="7af98-183">Является ошибкой считать, что согласованность в интерфейсе приводит к хорошему проектированию.</span><span class="sxs-lookup"><span data-stu-id="7af98-183">It is a mistake to think that consistency in the interface will lead to good design.</span></span>

<span data-ttu-id="7af98-184">Например, Пользовательский интерфейс видеоигры обычно очень специфичен для типа игры.</span><span class="sxs-lookup"><span data-stu-id="7af98-184">For example, video game UI is typically very specific to the kind of game.</span></span> <span data-ttu-id="7af98-185">Попытка разработать универсальный пользовательский интерфейс, который хорошо работает для двух специализированных игр, для которых требуется колесо управления, а другой, который лучше всего подходит для джойстика и кнопок, скорее всего, не будет успешным для любой игры.</span><span class="sxs-lookup"><span data-stu-id="7af98-185">Trying to design a generic UI that works well for two specialized games, one that requires a steering wheel and another that works best with a joystick and buttons, is likely not going to be successful for either game.</span></span> <span data-ttu-id="7af98-186">В лучшем случае, скорее всего, будет достигнуто среднее значение, которое не подходит для.</span><span class="sxs-lookup"><span data-stu-id="7af98-186">At best a middle ground is likely to be achieved that is not good for either.</span></span>

<span data-ttu-id="7af98-187">Наличие хороших данных о том, как используются вещи, является ключом к принятии этого решения.</span><span class="sxs-lookup"><span data-stu-id="7af98-187">Having good data about how things are used is the key to making this decision.</span></span> <span data-ttu-id="7af98-188">Будьте в полной мере узнать о преимуществах и недостатках каждого компромисса (скорость и надежность, простота обучения по сравнению с экспертами, Глобальная согласованность и локальная оптимизация) и принятие оптимальных решений относительно всего продукта.</span><span class="sxs-lookup"><span data-stu-id="7af98-188">Be clear about the pros and cons of each tradeoff (speed versus reliability, ease of learning versus expert proficiency, global consistency versus local optimization) and make the best decisions for the feature in relation to the whole product.</span></span>
