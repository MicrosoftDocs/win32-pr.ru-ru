# <a name="inapenforcementclientcallbackgetconnections-method"></a>Метод Инапенфорцементклиенткаллбакк:: соединений

> [!Note]  
> Платформа защиты доступа к сети недоступна начиная с Windows 10

 

Метод обратного вызова **инапенфорцементклиенткаллбакк::** noreturn вызывается напажент и реализуется клиентом принудительного применения для возврата набора соединений.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetConnections(
  [out] Connections **connections
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*подключения* \[ заполняет\]
</dt> <dd>

Указатель на текущий набор обслуживаемых [**соединений**](connections-struct.md).

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Этот метод обратного вызова должен возвращать один из следующих кодов ошибок.



| Код возврата                                                                                                | Описание                                                                                                                                                                                                           |
|------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <dl> <dt>**\_ОК**</dt> </dl>                       | Верните это значение, если операция завершилась удачно.<br/>                                                                                                                                                              |
| <dl> <dt>**\_сервер RPC \_ S \_ недоступен**</dt> </dl> | Возврат этого значения приводит к удалению принудительного применения из списка связанных-SHA и соответствующей записи кэша Напажент для записи на диск. После этого сбой SHA может повторно инициализироваться с помощью Напажент.<br/> |
