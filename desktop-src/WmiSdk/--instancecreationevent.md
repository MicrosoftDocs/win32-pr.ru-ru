# <a name="__instancecreationevent-class"></a>\_\_Класс Инстанцекреатионевент

Класс System **\_ \_ инстанцекреатионевент** сообщает о событии создания экземпляра, которое представляет собой тип [внутреннего события](determining-the-type-of-event-to-receive.md) , создаваемого при добавлении нового экземпляра в пространство имен.

Приведенный ниже синтаксис является упрощенной версией кода MOF и включает все унаследованные свойства. Свойства перечислены в алфавитном порядке, а не в MOF.

## <a name="syntax"></a>Синтаксис

``` syntax
class __InstanceCreationEvent : __InstanceOperationEvent
{
  uint8  SECURITY_DESCRIPTOR[];
  object TargetInstance;
  uint64 TIME_CREATED;
};
```

## <a name="remarks"></a>Комментарии

Класс **\_ \_ инстанцекреатионевент** является производным от [**\_ \_ инстанцеоператионевент**](--instanceoperationevent.md).

**Создание ресурса: \_ \_ инстанцекреатионевент**

Предположим, вы заинтересованы в получении уведомления, если на определенном компьютере запущен Блокнот. При запуске блокнота создается соответствующий процесс. Управление процессами осуществляется с помощью инструментария WMI и представляется \_ классом процесса Win32. Когда запускается блокнот, соответствующий экземпляр \_ класса процесса Win32 станет доступен через инструментарий WMI. Если вы зарегистрировали интерес в этом событии (выполнив соответствующий запрос уведомления о событиях), доступность этого экземпляра приведет к созданию экземпляра класса **\_ \_ инстанцекреатионевент** .

Запросы уведомлений, которые запрашивают уведомление о создании ресурса и используют встроенные события, используют синтаксис, аналогичный приведенному ниже.

`SELECT * FROM __InstanceCreationEvent WITHIN PollingInterval WHERE TargetInstance ISA 'Win32_Process' and TargetInstance.Name = 'notepad.exe'`

Более подробное обсуждение использования **\_ \_ инстанцекреатионевент** в качестве способа мониторинга файловых систем см. в разделе [инструментарий WMI и мониторинг файловой системы](https://www.codeproject.com/Articles/42212/WMI-and-File-System-Monitoring) в кодепрожект.
