# <a name="wmi-object-path-requirements"></a>Требования к пути к объектам WMI

Инструментарий WMI использует пути к объектам в ссылочных свойствах классов ассоциаций для обнаружения связанных объектов, а также для использования путей к объектам в входных или выходных параметрах для нескольких методов. Поскольку пути к объектам обрабатываются как строки для поиска и сравнения, значение пути к объекту при использовании в качестве ссылочного свойства всегда является самой строкой, а не объектом разыменования. При сравнении строк, которые работают с путями объектов, всегда учитывается регистр.

Путь к объекту может использовать следующий синтаксис:

-   Строки, содержащиеся в одинарных кавычках.
-   Косые черты в качестве разделителей.
-   Символы обратной косой черты в качестве разделителей.
-   Шестнадцатеричные константы для целых чисел.
-   Логические константы для классов с ключами, принимающими логические значения.
-   Нотация URL-адреса для представления непечатаемых символов, например %20, для пустого пространства.

Кроме того, строка пути к объекту должна соблюдать следующие ограничения.

-   Предполагается, что локальный сервер с частичным путем к пространству имен. Таким словами, Указание корневого и пространства имен по умолчанию подразумевает корневое и пространство имен по умолчанию на локальном сервере.
-   Нет пробелов ни в одном элементе, ни между элементами.
-   Внедренные кавычки в путях к объектам разрешены, но они должны разделять кавычки escape-символами, как в приложениях C или C++.
-   Только десятичные значения распознаются как числовые части ключей.
