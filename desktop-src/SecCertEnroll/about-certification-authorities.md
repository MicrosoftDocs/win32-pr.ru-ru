# <a name="certification-authorities"></a><span data-ttu-id="0ee21-103">Центры сертификации</span><span class="sxs-lookup"><span data-stu-id="0ee21-103">Certification Authorities</span></span>

<span data-ttu-id="0ee21-104">[*Центр сертификации*](/windows/desktop/SecGloss/c-gly) (ЦС) отвечает за аттестацию идентификации пользователей, компьютеров и организаций.</span><span class="sxs-lookup"><span data-stu-id="0ee21-104">A [*certification authority*](/windows/desktop/SecGloss/c-gly) (CA) is responsible for attesting to the identity of users, computers, and organizations.</span></span> <span data-ttu-id="0ee21-105">Центр сертификации выполняет аутентификацию объектов и подтверждает их подлинность выпуском сертификата с цифровой подписью.</span><span class="sxs-lookup"><span data-stu-id="0ee21-105">The CA authenticates an entity and vouches for that identity by issuing a digitally signed certificate.</span></span> <span data-ttu-id="0ee21-106">Центр сертификации также может управлять сертификатами, осуществлять их отзыв и продление.</span><span class="sxs-lookup"><span data-stu-id="0ee21-106">The CA can also manage, revoke, and renew certificates.</span></span>

<span data-ttu-id="0ee21-107">Центр сертификации может быть общедоступным или частным.</span><span class="sxs-lookup"><span data-stu-id="0ee21-107">A CA can be public or private.</span></span> <span data-ttu-id="0ee21-108">Общедоступный ЦС предоставляет службы сертификации, которые обычно используются для сбора через Интернет.</span><span class="sxs-lookup"><span data-stu-id="0ee21-108">A public CA provides certification services, typically for a fee, to the public over the Internet.</span></span> <span data-ttu-id="0ee21-109">Частный ЦС предоставляет эту службу членам совокупности с разделением, таким как сотрудники Организации или члены другой частной группы.</span><span class="sxs-lookup"><span data-stu-id="0ee21-109">A private CA provides this service to the members of a delimited population such as the employees of a business or members of some other private group.</span></span>

<span data-ttu-id="0ee21-110">Средства, с помощью которых ЦС проверяет подлинность конечного пользователя, изменяются и выходят за рамки данной документации.</span><span class="sxs-lookup"><span data-stu-id="0ee21-110">The means by which a CA authenticates an end user are varied and beyond the scope of this documentation.</span></span> <span data-ttu-id="0ee21-111">Однако очевидно, что методы проверки подлинности различаются по типу поставщика.</span><span class="sxs-lookup"><span data-stu-id="0ee21-111">Clearly, however, methods of authentication vary by type of provider.</span></span> <span data-ttu-id="0ee21-112">Например, частный ЦС может устанавливать удостоверения конечных пользователей, ссылаясь на список групп, например в базу данных сотрудников или Active Directory.</span><span class="sxs-lookup"><span data-stu-id="0ee21-112">For example, a private CA can establish the identity of end users by referring to a group roster such as an employee database or Active Directory.</span></span> <span data-ttu-id="0ee21-113">Методы проверки подлинности, выполняемые общедоступным ЦС, обычно более сложны и частично зависят от уровня гарантии, обещанного сертификатом.</span><span class="sxs-lookup"><span data-stu-id="0ee21-113">The authentication methods performed by a public CA are generally more complex and depend partly on the level of assurance being promised by the certificate.</span></span>

<span data-ttu-id="0ee21-114">По мере роста инфраструктуры открытых ключей (PKI) может стать затруднительным, чтобы один ЦС мог эффективно управлять всеми выдаваемыми сертификатами.</span><span class="sxs-lookup"><span data-stu-id="0ee21-114">As the population of a public key infrastructure (PKI) grows, it can become difficult for a single CA to effectively manage all of the certificates it has issued.</span></span> <span data-ttu-id="0ee21-115">Центр сертификации может компенсировать, добавив авторизацию других центров сертификации в PKI для выдаче сертификатов.</span><span class="sxs-lookup"><span data-stu-id="0ee21-115">The CA can compensate by authorizing other CAs in the PKI to issue certificates.</span></span> <span data-ttu-id="0ee21-116">Первоначальный ЦС называется корнем, а доверенные центры сертификации — подчиненными.</span><span class="sxs-lookup"><span data-stu-id="0ee21-116">The initial CA is called the root, and the CAs it authorizes are called subordinates.</span></span> <span data-ttu-id="0ee21-117">Подчиненные ЦС также могут обозначать собственные дочерние компании в пределах ограничений, заданных корнем.</span><span class="sxs-lookup"><span data-stu-id="0ee21-117">Subordinate CAs can also designate their own subsidiaries within the limits set by the root.</span></span> <span data-ttu-id="0ee21-118">Итоговая структура называется иерархией сертификатов.</span><span class="sxs-lookup"><span data-stu-id="0ee21-118">The resulting structure is called a certificate hierarchy.</span></span> <span цепочкой сертификатов.</span><span class="sxs-lookup"><span data-stu-id="0ee21-120">This is called a certificate chain.</span></span>

<span data-ttu-id="0ee21-121">Термин центр сертификации может ссылаться как на организацию, которая подтверждает подлинность конечного пользователя, так и на сервер, используемый Организацией для выдачи сертификатов и управления ими.</span><span class="sxs-lookup"><span data-stu-id="0ee21-121">The term certification authority can refer to both the organization that vouches for the identity of an end user and the server used by the organization to issue and manage certificates.</span></span> <span data-ttu-id="0ee21-122">Windows Server можно настроить для работы в качестве сервера ЦС, и эта документация обычно относится к серверу при использовании термина ЦС.</span><span class="sxs-lookup"><span data-stu-id="0ee21-122">A Windows server can be configured to act as a CA server, and this documentation usually refers to the server when using the term CA.</span></span>

<span data-ttu-id="0ee21-123">API регистрации сертификатов взаимодействует с центром сертификации главным образом с помощью объекта [**IX509Enrollment**](/windows/desktop/api/CertEnroll/nn-certenroll-ix509enrollment) .</span><span class="sxs-lookup"><span data-stu-id="0ee21-123">The Certificate Enrollment API interacts with a CA mainly by using the [**IX509Enrollment**](/windows/desktop/api/CertEnroll/nn-certenroll-ix509enrollment) object.</span></span> <span data-ttu-id="0ee21-124">Метод [**регистрации**](/windows/desktop/api/CertEnroll/nf-certenroll-ix509enrollment-enroll) этого объекта может автоматически закодировать запрос сертификата, отправить его в центр сертификации и установить выданный сертификат.</span><span class="sxs-lookup"><span data-stu-id="0ee21-124">The [**Enroll**](/windows/desktop/api/CertEnroll/nf-certenroll-ix509enrollment-enroll) method on this object can automatically encode a certificate request, submit it to the CA, and install the issued certificate.</span></span> <span data-ttu-id="0ee21-125">Также можно использовать инициализированный объект **IX509Enrollment** для нестандартной регистрации или для отложенной регистрации.</span><span class="sxs-lookup"><span data-stu-id="0ee21-125">You can also use an initialized **IX509Enrollment** object for out-of-band enrollment or for delayed enrollment.</span></span> <span data-ttu-id="0ee21-126">Кроме того, можно использовать объект [**IX509EnrollmentStatus**](/windows/desktop/api/CertEnroll/nn-certenroll-ix509enrollmentstatus) для отслеживания состояния регистрации.</span><span class="sxs-lookup"><span data-stu-id="0ee21-126">In addition, you can use the [**IX509EnrollmentStatus**](/windows/desktop/api/CertEnroll/nn-certenroll-ix509enrollmentstatus) object to monitor enrollment status.</span></span>
