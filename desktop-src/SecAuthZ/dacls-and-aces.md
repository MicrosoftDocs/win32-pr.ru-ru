# <a name="dacls-and-aces"></a><span data-ttu-id="d614a-103">Списки DACL и ACE</span><span class="sxs-lookup"><span data-stu-id="d614a-103">DACLs and ACEs</span></span>

<span data-ttu-id="d614a-104">Если у объекта Windows нет [*списка управления доступом на уровне пользователей*](/windows/desktop/SecGloss/d-gly) (DACL), система разрешает всем пользователям полный доступ к нему.</span><span class="sxs-lookup"><span data-stu-id="d614a-104">If a Windows object does not have a [*discretionary access control list*](/windows/desktop/SecGloss/d-gly) (DACL), the system allows everyone full access to it.</span></span> <span data-ttu-id="d614a-105">Если объект имеет DACL, система разрешает доступ только к явно разрешенным [*записям контроля доступа*](/windows/desktop/SecGloss/a-gly) (ACE) в списке DACL.</span><span class="sxs-lookup"><span data-stu-id="d614a-105">If an object has a DACL, the system allows only the access that is explicitly allowed by the [*access control entries*](/windows/desktop/SecGloss/a-gly) (ACEs) in the DACL.</span></span> <span data-ttu-id="d614a-106">Если в списке DACL нет записей ACE, система не разрешает доступ никому.</span><span class="sxs-lookup"><span data-stu-id="d614a-106">If there are no ACEs in the DACL, the system does not allow access to anyone.</span></span> <span data-ttu-id="d614a-107">Аналогично, если список DACL имеет ACE, которые разрешают доступ к ограниченному набору пользователей или групп, система неявно запрещает доступ ко всем доверенным лицам, не включенным в Ace.</span><span class="sxs-lookup"><span data-stu-id="d614a-107">Similarly, if a DACL has ACEs that allow access to a limited set of users or groups, the system implicitly denies access to all trustees not included in the ACEs.</span></span>

<span data-ttu-id="d614a-108">В большинстве случаев можно управлять доступом к объекту с помощью ACE, разрешенного доступом; нет необходимости явно запрещать доступ к объекту.</span><span class="sxs-lookup"><span data-stu-id="d614a-108">In most cases, you can control access to an object by using access-allowed ACEs; you do not need to explicitly deny access to an object.</span></span> <span data-ttu-id="d614a-109">Исключением является то, что запись ACE разрешает доступ к группе и вы хотите запретить доступ к члену группы.</span><span class="sxs-lookup"><span data-stu-id="d614a-109">The exception is when an ACE allows access to a group and you want to deny access to a member of the group.</span></span> <span data-ttu-id="d614a-110">Для этого поместите ACE "доступ запрещен" для пользователя в списке DACL перед элементом управления доступом, разрешенным для этой группы.</span><span class="sxs-lookup"><span data-stu-id="d614a-110">To do this, place an access-denied ACE for the user in the DACL ahead of the access-allowed ACE for the group.</span></span> <span data-ttu-id="d614a-111">Обратите внимание, что [порядок записей ACE](order-of-aces-in-a-dacl.md) важен, так как система считывает ACE в последовательности, пока доступ не будет предоставлен или запрещен.</span><span class="sxs-lookup"><span data-stu-id="d614a-111">Note that the [order of the ACEs](order-of-aces-in-a-dacl.md) is important because the system reads the ACEs in sequence until access is granted or denied.</span></span> <span data-ttu-id="d614a-112">Запись ACE пользователя "доступ запрещен" должна быть первой. в противном случае, когда система считывает разрешенный ACE группы доступа, он предоставит доступ ограниченному пользователю.</span><span class="sxs-lookup"><span data-stu-id="d614a-112">The user's access-denied ACE must appear first; otherwise, when the system reads the group's access allowed ACE, it will grant access to the restricted user.</span></span>

<span data-ttu-id="d614a-113">На следующем рисунке показан список DACL, который запрещает доступ одному пользователю и предоставляет доступ к двум группам.</span><span class="sxs-lookup"><span data-stu-id="d614a-113">The following illustration shows a DACL that denies access to one user and grants access to two groups.</span></span> <span data-ttu-id="d614a-114">Члены группы A получают права на чтение, запись и выполнение, выполняя суммирование прав на группирование и разрешение на доступ всем пользователям.</span><span class="sxs-lookup"><span data-stu-id="d614a-114">The members of Group A get Read, Write, and Execute access rights by accumulating the rights allowed to Group A and rights allowed to Everyone.</span></span> <span data-ttu-id="d614a-115">Исключением является Эндрю, которому запрещается доступ к ACE, запрещенному доступом, несмотря на членство в группе «все».</span><span class="sxs-lookup"><span data-stu-id="d614a-115">The exception is Andrew, who is denied access by the access-denied ACE in spite of being a member of the Everyone Group.</span></span>
