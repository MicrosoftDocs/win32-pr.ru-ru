# <a name="operating-system-service-model"></a>Модель службы операционной системы

В модели службы операционной системы приложение, работающее как обычный пользователь, взаимодействует со службой, работающей как **система** , с помощью [удаленного вызова процедур](/windows/desktop/Rpc/rpc-start-page) (RPC).

Стандартное пользовательское приложение помечено в манифесте приложения с **RequestedExecutionLevel** **asInvoker**. Для выполнения операции, требующей прав администратора, стандартное пользовательское приложение выполняет запрос к службе.

Одним из способов использования модели службы операционной системы является Управление приложениями, которые могут повлиять на систему, такие как антивирусные и другие нежелательные программы и шпионские программы. Стандартное пользовательское приложение позволяет вошедшему в систему пользователю управлять некоторыми аспектами службы. Служба отвечает за определение того, какие операции она выполняет для стандартного пользовательского приложения. Например, антивирусная служба может позволить обычным пользователям запускать проверку системы, но она может не позволить обычным пользователям отключить проверку на вирусы в режиме реального времени.

Основным преимуществом использования модели службы операционной системы является то, что запрос на повышение прав не требуется.

Недостаток использования модели службы операционной системы состоит в том, что служба, работающая в системе, использует больше ресурсов, чем задача, а служба не может быть остановлена обычным пользователем. Рассмотрите возможность использования [модели задач с повышенными правами](elevated-task-model.md) , если она достаточна.

Чтобы реализовать модель службы операционной системы, Создайте клиентское приложение обычного пользователя и службу операционной системы. Установка службы в операционной системе во время установки продукта.
