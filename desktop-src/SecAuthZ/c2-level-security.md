# <a name="c2-level-security"></a><span data-ttu-id="5f257-103">Безопасность на уровне C2</span><span class="sxs-lookup"><span data-stu-id="5f257-103">C2-level Security</span></span>

<span data-ttu-id="5f257-104">В следующем списке перечислены некоторые из наиболее важных требований безопасности уровня C2, как определено в американском отделе обороны:</span><span class="sxs-lookup"><span data-stu-id="5f257-104">The following list includes some of the most important requirements of C2-level security, as defined by the U.S. Department of Defense:</span></span>

-   <span data-ttu-id="5f257-105">Управление доступом к ресурсу должно быть возможным путем предоставления или запрета доступа отдельным пользователям или именованным группам пользователей.</span><span class="sxs-lookup"><span data-stu-id="5f257-105">It must be possible to control access to a resource by granting or denying access to individual users or named groups of users.</span></span>
-   <span data-ttu-id="5f257-106">Память должна быть защищена таким образом, чтобы ее содержимое невозможно было прочитать после освобождения [*процесса*](/windows/desktop/SecGloss/p-gly) .</span><span class="sxs-lookup"><span data-stu-id="5f257-106">Memory must be protected so that its contents cannot be read after a [*process*](/windows/desktop/SecGloss/p-gly) frees it.</span></span> <span data-ttu-id="5f257-107">Аналогично, безопасная файловая система, например NTFS, должна защищать удаленные файлы от чтения.</span><span class="sxs-lookup"><span data-stu-id="5f257-107">Similarly, a secure file system, such as NTFS, must protect deleted files from being read.</span></span>
-   <span data-ttu-id="5f257-108">Пользователи должны однозначно идентифицируют себя, например, по паролю при входе в систему.</span><span class="sxs-lookup"><span data-stu-id="5f257-108">Users must identify themselves in a unique manner, such as by password, when they log on.</span></span> <span data-ttu-id="5f257-109">Все действия, которые подлежат аудиту, должны указывать пользователя, выполняющего действие.</span><span class="sxs-lookup"><span data-stu-id="5f257-109">All auditable actions must identify the user performing the action.</span></span>
-   <span data-ttu-id="5f257-110">Системные администраторы должны иметь возможность аудита событий, связанных с безопасностью.</span><span class="sxs-lookup"><span data-stu-id="5f257-110">System administrators must be able to audit security-related events.</span></span> <span data-ttu-id="5f257-111">Однако доступ к данным аудита событий, связанных с безопасностью, должен быть ограничен полномочными администраторами.</span><span class="sxs-lookup"><span data-stu-id="5f257-111">However, access to the security-related events audit data must be limited to authorized administrators.</span></span>
-   <span data-ttu-id="5f257-112">Система должна быть защищена от внешних помех или незаконного изменения, например при изменении работающей системы или системных файлов, хранящихся на диске.</span><span class="sxs-lookup"><span data-stu-id="5f257-112">The system must be protected from external interference or tampering, such as modification of the running system or of system files stored on disk.</span></span>
