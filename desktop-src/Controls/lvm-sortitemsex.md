# <a name="lvm_sortitemsex-message"></a>\_Сообщение LVM сортитемсекс

Использует определяемую приложением функцию сравнения для сортировки элементов элемента управления "представление списка". Индекс каждого элемента изменяется в соответствии с новой последовательностью. Это сообщение можно отправить явно или с помощью макроса [**\_ сортитемсекс ListView**](/windows/desktop/api/Commctrl/nf-commctrl-listview_sortitemsex) .

## <a name="parameters"></a>Параметры

<dl> <dt>

*wParam* 
</dt> <dd>

Определяемое приложением значение, которое передается в функцию сравнения.

</dd> <dt>

*lParam* 
</dt> <dd>

Указатель на определяемую приложением функцию сравнения. Он вызывается во время операции сортировки каждый раз, когда необходимо сравнить относительный порядок двух элементов списка.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает **значение true** , если успешно, или **false** в противном случае.

## <a name="remarks"></a>Комментарии

Функция сравнения имеет следующий вид:

``` syntax
int CALLBACK CompareFunc(LPARAM lParam1, LPARAM lParam2, LPARAM lParamSort);  
```

Это сообщение похоже на [**LVM \_ сортитемс**](lvm-sortitems.md), за исключением типа данных, передаваемых в функцию сравнения. При использовании **LVM \_ сортитемсекс** *lParam1* является текущим индексом первого элемента, а *lParam2* — текущим индексом второго элемента. Вы можете отправить сообщение [**LVM \_ жетитемтекст**](lvm-getitemtext.md) , чтобы получить дополнительные сведения об элементе, если это необходимо.

Функция сравнения должна возвращать отрицательное значение, если первый элемент должен предшествовать второму, положительное значение, если первый элемент должен следовать за вторым, или нуль, если два элемента эквивалентны.

> [!Note]  
> В процессе сортировки содержимое представления списка нестабильно. Если функция обратного вызова отправляет все сообщения в элемент управления "список", не считая [**LVM \_**](lvm-getitem.md) -Item ([**ListView- \_ элемент**](/windows/desktop/api/Commctrl/nf-commctrl-listview_getitem)), результаты могут оказаться непредсказуемыми.
