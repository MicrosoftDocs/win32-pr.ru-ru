# <a name="whats-new-in-synchronization"></a>Новые возможности синхронизации

Windows содержит следующие новые программные элементы для синхронизации.

## <a name="windows-8"></a>Windows 8

### <a name="new-functions"></a>Новые функции

<dl> <dt>

[**делетесинчронизатионбарриер**](/windows/desktop/api/SynchAPI/nf-synchapi-deletesynchronizationbarrier)
</dt> <dd>

Удаляет барьер синхронизации.

</dd> <dt>

[**ентерсинчронизатионбарриер**](/windows/desktop/api/synchapi/nf-synchapi-entersynchronizationbarrier)
</dt> <dd>

Заставляет вызывающий поток ждать в барьере синхронизации до тех пор, пока не будет введено максимальное число потоков.

</dd> <dt>

[**жетоверлаппедресултекс**](/windows/desktop/api/Ioapiset/nf-ioapiset-getoverlappedresultex)
</dt> <dd>

Возвращает результаты операции перекрытия для указанного файла, именованного канала или коммуникационного устройства в течение заданного интервала времени ожидания. Вызывающий поток может выполнить предупреждение.

</dd> <dt>

[**инитиализесинчронизатионбарриер**](/windows/desktop/api/synchapi/nf-synchapi-entersynchronizationbarrier)
</dt> <dd>

Указывает максимальное число потоков и число счетчиков для нового барьера синхронизации.

</dd> <dt>

[**ваитонаддресс**](/windows/desktop/api/SynchAPI/nf-synchapi-waitonaddress)
</dt> <dd>

Ожидает изменения значения по указанному адресу.

</dd> <dt>

[**вакебяддрессалл**](/windows/desktop/api/SynchAPI/nf-synchapi-wakebyaddressall)
</dt> <dd>

Пробуждение всех потоков, ожидающих изменения значения адреса.

</dd> <dt>

[**вакебяддресссингле**](/windows/desktop/api/SynchAPI/nf-synchapi-wakebyaddresssingle)
</dt> <dd>

Пробуждение одного потока, ожидающего изменения значения адреса.

</dd> </dl>
