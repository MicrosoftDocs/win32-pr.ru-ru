# <a name="d3dximage_fileformat-enumeration"></a>\_Перечисление FILEFORMAT D3DXIMAGE

Описывает поддерживаемые форматы файлов изображений. Описания этих форматов см. в разделе Примечания.

## <a name="syntax"></a>Синтаксис


```C++
typedef enum D3DXIMAGE_FILEFORMAT { 
  D3DXIFF_BMP          = 0,
  D3DXIFF_JPG          = 1,
  D3DXIFF_TGA          = 2,
  D3DXIFF_PNG          = 3,
  D3DXIFF_DDS          = 4,
  D3DXIFF_PPM          = 5,
  D3DXIFF_DIB          = 6,
  D3DXIFF_HDR          = 7,
  D3DXIFF_PFM          = 8,
  D3DXIFF_FORCE_DWORD  = 0x7fffffff
} D3DXIMAGE_FILEFORMAT, *LPD3DXIMAGE_FILEFORMAT;
```

## <a name="remarks"></a>Комментарии

Функции, которые начинаются с D3DXLoadxxx, поддерживают все перечисленные форматы. Функции, которые начинаются с D3DXSavexxx, поддерживают все перечисленные форматы, кроме форматов формат (TGA) и Portable пиксмап (стр. ppm).

В следующей таблице перечислены доступные форматы входных и выходных данных.



| Расширение файла | Описание                                                                                                                                                                                                                                                                                                                                        |
|----------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| BMP           | Формат точечного рисунка Windows. Содержит заголовок, описывающий разрешение устройства, на котором создавался прямоугольник пикселей, размеры прямоугольника, размер массива битов, логическую палитру и массив битов, определяющих связь между пикселями в изображении точечного рисунка и записями в логической палитре. |
| DDS           | Формат файла поверхности DirectDraw. Сохраняет текстуры, текстуры тома и карты кубических сред, с уровнями mipmap и без него, а также с сжатием пикселов или без него. См. раздел [DDS](../direct3ddds/dx-graphics-dds.md).                                                                                                                                       |
| DIB           | Формат DIB Windows. Содержит массив битов, Объединенных с структурами, которые задают ширину и высоту растрового изображения, цветовой формат устройства, на котором было создано изображение, и разрешение устройства, которое использовалось для создания этого изображения.                                                                                                              |
| . HDR           | Формат HDR. Кодирует каждый пиксель как РГБЕ 32-разрядный цвет с 8 битами для красного, зеленого и синего, а также общий 8-разрядный показатель степени. Каждый канал по отдельности сжимается с помощью кодирования длины выполнения (RLE).                                                                                                                                       |
| .jpg           | Стандарт JPEG. Указывает сжатие переменных в 24-разрядном RGB-цвете и 8-битных изображениях изображений в формате TIFF с тегом оттенков серого.                                                                                                                                                                                                       |
| . ПФМ           | Формат переносимой схемы float. Необработанный формат изображения с плавающей точкой без сжатия. В заголовке файла указываются ширина, высота, монохромный цвет и порядок слов в машинном образах. Пиксельные данные хранятся как 32-разрядные значения с плавающей запятой, с 3 значениями на пиксель для цвета и одним значением на пиксель для монохромных.                                |
| .png           | Формат PNG. Нечастный формат точечного рисунка, использующий сжатие без потерь.                                                                                                                                                                                                                                                                            |
| стр/мин           | Переносимый формат Пиксмап. Формат двоичного файла или формата ASCII для цветовых изображений, включающий высоту и ширину изображения, а также максимальное значение компонента цвета.                                                                                                                                                                                                 |
| TGA           | Формат графического адаптера Targa или формат. Поддерживает глубину 8, 15, 16, 24 и 32 бит, включая 8-разрядную серую шкалу, и содержит необязательные данные цветовой палитры, данные об источнике изображения (x, y) и размере, а также данные пикселей.                                                                                                                               |



 

Дополнительные сведения о некоторых из этих форматов см. в разделе [Типы точечных рисунков](../gdiplus/-gdiplus-types-of-bitmaps-about.md) .
