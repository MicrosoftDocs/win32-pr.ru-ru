# <a name="group-policies"></a>Групповые политики

> [!NOTE]
> Дополнительные сведения об использовании управления мобильными устройствами (MDM) для установки политик для фоновая интеллектуальная служба передачи (BITS) см. в разделе [CSP политики](/windows/client-management/mdm/policy-configuration-service-provider).

Фоновая интеллектуальная служба передачи (BITS) использует следующие групповые политики для настройки передачи и настройки BITS. Дополнительные сведения о версии BITS, используемой различными версиями Windows, см. в разделе [что нового](what-s-new.md) .

> [!NOTE]  
> Если значение политики не задано, BITS использует значение политики по умолчанию.

**Включение политики BITS**

1.  Откройте групповая политика, введя **gpedit. msc/гпкомпутер: "***ComputerName***"** в окне **выполнения** или в командной строке в окне cmd.
2.  Политики BITS находятся в разделе **Конфигурация компьютера**, **Административные шаблоны**, **сеть**, **Фоновая интеллектуальная служба передачи**.
3.  Щелкните политику правой кнопкой мыши и выберите **изменить**.
4.  Следуйте указаниям по включению и настройке политики.

Групповые политики для BITS находятся в реестре в разделе реестра "программное обеспечение **\_ локального \_ компьютера**" \\  \\ **политики** \\ **Microsoft** \\ **Windows** \\ **BITS**. Обратите внимание, что в реестре перечислены только настроенные политики.

|Политика|Описание|
|-|-|
|жобинактивититимеаут<br/>BITS 1,0|По умолчанию служба BITS сохраняет сведения о задании в течение 90 дней. Если в течение этого периода времени для задания не выполняется никаких действий, служба BITS отменяет задание. Ход перемещения или изменения свойств сбрасывают таймер.<br/> Может потребоваться изменить эту политику, если пользователь не входит в систему или не поддерживает сетевое подключение в течение длительного периода времени.|
|максинтернетбандвидс<br/>BITS 2,0|Ограничивает пропускную способность сети, используемую BITS для фоновой передачи (эта политика не влияет на переносы переднего плана).<br/>Укажите ограничение, которое будет использоваться в течение определенного интервала времени, и предельное значение, которое будет использоваться в другое время. Например, ограничьте использование пропускной способности сети до 10 килобит в секунду (кбит/с) с 8:00 утра. до 5:00 вечера. и использовать все доступные неиспользуемые пропускную способность в остальное время.<br/>**Примечание**. изменение системных часов не влияет на интервал времени, ограничивающий пропускную способность. Например, если текущее время — 2:00 вечера. Кроме того, интервал ограничения пропускной способности начинается в 3:00 часов, а перемещение системных часов вперед на один час не означает, что при этом ограничение пропускной способности будет соблюдено в течение часа. Чтобы отразить изменение системных часов в службе BITS, необходимо перезапустить компьютер или службу BITS.<br/>Укажите предельное значение в килобитах в секунду. Основано ограничение на размер сетевого подключения, а не от размера сетевого адаптера компьютера. BITS использует примерно две килобит, если указано значение меньше двух килобит. Чтобы предотвратить передачу данных BITS, укажите ограничение в ноль. Если задать ограничение в ноль, BITS поместит все фоновые задания в состояние временной ошибки. (Код ошибки имеет значение BG_E_BLOCKED_BY_POLICY.) BITS размещает задания в состоянии очереди по истечении интервала времени.<br/> Если вы отключаете или не настраиваете эту политику, служба BITS использует всю доступную неиспользуемую пропускную способность.<br/> Как правило, эта политика используется для предотвращения передачи BITS от конкурирующих к пропускной способности сети, когда у клиента есть быстрый сетевой адаптер (10 Мбит/с), но он подключен к сети через медленное подключение (56 кбит/с).<br/> Сведения о том, как служба BITS использует пропускную [способность сети, см. в](network-bandwidth.md)этой статье.|
|максдовнлоадтиме<br/>BITS 3,0|Определяет период времени, в течение которого служба BITS может тратить время на активную передачу файлов в задании. По умолчанию — 90 дней.<br/> Чтобы переопределить эту политику для конкретного задания, вызовите метод [**IBackgroundCopyJob4:: сетмаксимумдовнлоадтиме**](/windows/desktop/api/bits3_0/nf-bits3_0-ibackgroundcopyjob4-setmaximumdownloadtime) .|
|MaxJobsPerMachine<br/>BITS 3,0|Ограничивает количество заданий, которые пользователи могут создавать на компьютере. Значение по умолчанию — 300 заданий. Эта политика не применяется к пользователю с правами администратора или учетными записями служб.|
|Maxjobspermachine<br/>BITS 3,0|Ограничивает количество заданий, которые пользователь может создать на компьютере. Значение по умолчанию — 60 заданий на пользователя. Эта политика не применяется к пользователю с правами администратора или учетными записями служб.|
|MaxFilesPerJob<br/>BITS 3,0|Ограничивает число файлов, которые пользователь может добавить в задание. Значение по умолчанию — 200 файлов на задание. Эта политика не применяется к пользователю с правами администратора или учетными записями служб.|
|MaxRangesPerFile<br/>BITS 3,0|Ограничивает количество диапазонов, которые пользователь может указать для файла. Значение по умолчанию — 500 диапазонов. Эта политика не применяется к пользователю с правами администратора или учетными записями служб.|

Этот параметр не действует, если параметры администрирования компьютера для кэша ветвей Windows полностью отключены.|

Служба BITS использует следующие групповые политики для настройки регулирования полосы пропускания.

Расписания обслуживания дополнительно ограничивают пропускную способность сети, используемую для фоновой передачи.<br/> Укажите ограничение, которое будет использоваться для фоновых заданий в расписании обслуживания. Например, если задания с нормальным приоритетом в настоящее время ограничены 256 кбит/с по рабочему расписанию, можно дополнительно ограничить пропускную способность сети заданий обычного приоритета до 0 кбит/с с 8:00 утра. на 10:00. по расписанию обслуживания.<br/>**Примечание**. ограничения пропускной способности, заданные для периода обслуживания, заменяют все ограничения, определенные для работы и других расписаний.|
