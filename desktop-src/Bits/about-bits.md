# <a name="about-bits"></a><span data-ttu-id="afe53-106">Сведения о службе BITS</span><span class="sxs-lookup"><span data-stu-id="afe53-106">About BITS</span></span>

<span data-ttu-id="afe53-107">Используйте фоновая интеллектуальная служба передачи (BITS) для загрузки или передачи файлов на веб-серверы HTTP или на файловые серверы SMB.</span><span class="sxs-lookup"><span data-stu-id="afe53-107">Use Background Intelligent Transfer Service (BITS) to download files from or upload files to HTTP web servers or SMB file servers.</span></span> 

<span data-ttu-id="afe53-108">Служба BITS продолжает передавать файлы после завершения работы приложения, если пользователь, инициировавший перемещение, продолжает входить в систему и поддерживает сетевое подключение.</span><span class="sxs-lookup"><span data-stu-id="afe53-108">BITS continues to transfer files after an application exits as long as the user who initiated the transfer remains logged on and a network connection is maintained.</span></span> <span data-ttu-id="afe53-109">Служба BITS не будет принудительно выполнять подключение к сети.</span><span class="sxs-lookup"><span data-stu-id="afe53-109">BITS will not force a network connection.</span></span> <span data-ttu-id="afe53-110">Служба BITS возобновляет передачу данных после того, как сетевое подключение было потеряно, или после того, как пользователь отключил вход в систему.</span><span class="sxs-lookup"><span data-stu-id="afe53-110">BITS resumes transfers after a network connection that had been lost is reestablished or after a user who had logged off logs back in.</span></span> <span data-ttu-id="afe53-111">Дополнительные сведения см. в разделе [Пользователи и сетевые подключения](users-and-network-connections.md).</span><span class="sxs-lookup"><span data-stu-id="afe53-111">For more information, see [Users and Network Connections](users-and-network-connections.md).</span></span>

<span data-ttu-id="afe53-112">Служба BITS учитывать текущую стоимость сети и перегрузку, чтобы фоновое задание не мешало работе с передним планом пользователя.</span><span class="sxs-lookup"><span data-stu-id="afe53-112">BITS is mindful of the current network cost and congestion so that a background job interferes as little as possible with the user's foreground experience.</span></span> <span data-ttu-id="afe53-113">Служба BITS использует неактивную [пропускную способность сети](network-bandwidth.md) для передачи файлов и увеличит или уменьшит скорость передачи файлов в зависимости от объема доступной пропускной способности сети.</span><span class="sxs-lookup"><span data-stu-id="afe53-113">BITS uses idle [network bandwidth](network-bandwidth.md) to transfer the files and will increase or decrease the rate at which files are transferred based on the amount of idle network bandwidth available.</span></span> <span data-ttu-id="afe53-114">Если сетевое приложение начинает использовать больший объем полосы пропускания, BITS снижает скорость передачи, чтобы защитить интерактивное взаимодействие с пользователем.</span><span class="sxs-lookup"><span data-stu-id="afe53-114">If a network application begins to consume more bandwidth, BITS decreases its transfer rate to preserve the user's interactive experience.</span></span> <span data-ttu-id="afe53-115">Служба BITS использует [политики передачи](how-to-block-a-bits-job-from-downloading-over-an-expensive-connection.md) , заданные приложением, чтобы предотвратить передачу файлов через сетевые подключения с затратами.</span><span class="sxs-lookup"><span data-stu-id="afe53-115">BITS uses app-specified [transfer policies](how-to-block-a-bits-job-from-downloading-over-an-expensive-connection.md) to prevent files from transferring on costed network connections.</span></span>

<span data-ttu-id="afe53-116">Кроме того, служба BITS учитывать энергопотребление.</span><span class="sxs-lookup"><span data-stu-id="afe53-116">BITS is also mindful of power usage.</span></span> <span data-ttu-id="afe53-117">Начиная с Windows 10 Май 2019 обновления, BITS будет передавать файлы, когда компьютер находится в [современном режиме ожидания](/windows-hardware/design/device-experiences/modern-standby) и компьютер подключен к сети.</span><span class="sxs-lookup"><span data-stu-id="afe53-117">Starting with the Windows 10 May 2019 Update, BITS will transfer files when the machine is in [Modern Standby](/windows-hardware/design/device-experiences/modern-standby) mode and the machine is plugged in.</span></span>

<span data-ttu-id="afe53-118">Приложение BITS может использовать различные [уровни приоритета](/windows/desktop/api/Bits/ne-bits-bg_job_priority) BITS, чтобы служба BITS могла выбрать для выполнения задания передачи.</span><span class="sxs-lookup"><span data-stu-id="afe53-118">The BITS application can use the different BITS [priority levels](/windows/desktop/api/Bits/ne-bits-bg_job_priority) to let BITS intelligently pick which transfer jobs to run.</span></span> <span data-ttu-id="afe53-119">Задания с более высоким приоритетом выполняются раньше заданий с более низким приоритетом.</span><span class="sxs-lookup"><span data-stu-id="afe53-119">Higher priority jobs preempt lower priority jobs.</span></span> <span data-ttu-id="afe53-120">Задания с одинаковым приоритетом совместно используют время передачи, благодаря чему большие задания не блокируют маленькие в очереди передачи.</span><span class="sxs-lookup"><span data-stu-id="afe53-120">Jobs at the same priority level share transfer time, which prevents a large job from blocking small jobs in the transfer queue.</span></span> <span data-ttu-id="afe53-121">Задания с более низким приоритетом не получают время передачи, пока задания с более высоким приоритетом не будут завершены или не перейдут в состояние ошибки.</span><span class="sxs-lookup"><span data-stu-id="afe53-121">Lower priority jobs do not receive transfer time until all higher priority jobs are complete or in an error state.</span></span>

<span data-ttu-id="afe53-122">Служба BITS использует Windows BranchCache для однорангового кэширования.</span><span class="sxs-lookup"><span data-stu-id="afe53-122">BITS uses Windows BranchCache for peer caching.</span></span> <span data-ttu-id="afe53-123">Дополнительные сведения см. в [обзоре BranchCache](/previous-versions/windows/it-pro/windows-7/dd755969(v=ws.10)).</span><span class="sxs-lookup"><span data-stu-id="afe53-123">For more information, see the [BranchCache Overview](/previous-versions/windows/it-pro/windows-7/dd755969(v=ws.10)).</span></span>

<span data-ttu-id="afe53-124">Разработчики универсальная платформа Windows (UWP) должны использовать API [Windows. Networking. баккграундтрансфер](/uwp/api/Windows.Networking.BackgroundTransfer) , а не API BITS.</span><span class="sxs-lookup"><span data-stu-id="afe53-124">Universal Windows Platform (UWP) developers should use the [Windows.Networking.BackgroundTransfer](/uwp/api/Windows.Networking.BackgroundTransfer) API and not the BITS API.</span></span>

<span data-ttu-id="afe53-125">Существует три типа [**заданий перемещения**](/windows/desktop/api/Bits/ne-bits-bg_job_type).</span><span class="sxs-lookup"><span data-stu-id="afe53-125">There are three types of [**transfer jobs**](/windows/desktop/api/Bits/ne-bits-bg_job_type).</span></span> <span data-ttu-id="afe53-126">Задание загрузки загружает файлы на клиент, задание отправки отправляет файл на сервер, а задание отправки и ответа отправляет файл на сервер и получает ответный файл из серверного приложения.</span><span class="sxs-lookup"><span data-stu-id="afe53-126">A download job downloads files to the client, an upload job uploads a file to the server, and an upload-reply job uploads a file to the server and receives a reply file from the server application.</span></span>
