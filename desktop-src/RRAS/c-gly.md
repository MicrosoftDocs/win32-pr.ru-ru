# <a name="c-rras"></a>C (RRAS)

<dl> <dt>

<span id="_rras_cyclic_redundancy_check_crc__gly"></span><span id="_RRAS_CYCLIC_REDUNDANCY_CHECK_CRC__GLY"></span>**Циклическая проверка избыточности (CRC)**
</dt> <dd>

CRC — это числовое значение, связанное с блоком данных. Это значение предоставляет сведения, позволяющие приложению определить, были ли изменены данные. CRC аналогичен контрольной сумме, но считается более надежным механизмом проверки ошибок.

</dd> <dt>

<span id="_rras_compression_control_protocol_ccp__gly"></span><span id="_RRAS_COMPRESSION_CONTROL_PROTOCOL_CCP__GLY"></span>**Протокол управления сжатием (CCP)**
</dt> <dd>

Протокол управления сжатием (CCP) позволяет двум компьютерам, которые обмениваются данными через PPP, согласовывать совместимые алгоритмы для отправки и получения сжатых кадров PPP. Два компьютера могут не использовать CCP до фазы сетевого контроля-протокола подключения PPP.

Дополнительные сведения см. в [документе RFC 1962](remote-access-request-for-comments.md). См. также раздел [**PPP \_ CCP \_ info**](/windows/desktop/api/Mprapi/ns-mprapi-ppp_ccp_info) и [**распппккп**](/previous-versions/windows/desktop/legacy/aa377620(v=vs.85)).

</dd> </dl>
