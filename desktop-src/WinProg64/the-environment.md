# <a name="the-environment"></a>Среда

разработчики, работающие с приложениями для 64-разрядных Windows найдут среду разработки практически идентично среде разработки для 32-разрядной Windows. Существующие API-интерфейсы были изменены там, где это необходимо, чтобы они отражали точность платформы, в которой они выполняются. результатом является простота и короткая кривая обучения для разработчика — написание кода для 64-разрядного Windows так же, как написание кода для 32-разрядной Windows.

файлы заголовков Windows поддерживают новые типы данных, позволяющие использовать указатели и связанные с указателями переменные для отражения точности платформы. это означает, что разработчики могут компилировать одну базу исходного кода для собственного запуска на 32-разрядной Windows или 64-разрядной Windows. Эта стратегия сокращает затраты на разработку приложений, использующих 64-разрядное оборудование, например процессоры AMD Opteron или Athlon64 или процессоры Intel Itanium.

Вы получите больше времени на создание приложений 64-bit, если хотите как можно скорее присвоить новые соглашения о типе данных. При изменении кода необходимо одновременно изменить определения данных. протестируйте приложение на 32-разрядной Windows, запустите его с помощью 64-разрядного компилятора (описанного в разделе [средства](the-tools.md)), и приложение будет готово к тестированию при наличии соответствующего 64-разрядного оборудования.
