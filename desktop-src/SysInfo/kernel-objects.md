# <a name="kernel-objects"></a><span data-ttu-id="c4f0b-103">Объекты ядра</span><span class="sxs-lookup"><span data-stu-id="c4f0b-103">Kernel Objects</span></span>

<span data-ttu-id="c4f0b-109">Любой процесс может создать новый обработчик для существующего объекта ядра (даже созданный другим процессом) при условии, что процесс знает имя объекта и имеет доступ к нему в режиме безопасности.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-109">Any process can create a new handle to an existing kernel object (even one created by another process), provided that the process knows the name of the object and has security access to the object.</span></span> <span data-ttu-id="c4f0b-110">Дескрипторы объектов ядра включают права доступа, указывающие действия, которые могут быть предоставлены или запрещены процессу.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-110">Kernel object handles include access rights that indicate the actions that can be granted or denied to a process.</span></span> <span data-ttu-id="c4f0b-111">Приложение задает права доступа при создании объекта или получении существующего обработчика объекта.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-111">An application specifies access rights when it creates an object or obtains an existing object handle.</span></span> <span data-ttu-id="c4f0b-112">Каждый тип объекта ядра поддерживает собственный набор прав доступа.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-112">Each type of kernel object supports its own set of access rights.</span></span> <span data-ttu-id="c4f0b-113">Например, дескрипторы событий могут иметь установленный или ожидающий доступ (или оба), дескрипторы файлов могут иметь доступ на чтение или запись (или и то, и другое) и т. д.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-113">For example, event handles can have set or wait access (or both), file handles can have read class="sxs-lookup"><span data-stu-id="c4f0b-114">For more information, see [Securable Objects](/windows/desktop/SecAuthZ/securable-objects).</span></span>

<span data-ttu-id="c4f0b-115">На следующем рисунке приложение создает объект события.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-115">In the following illustration, an application creates an event object.</span></span> <span data-ttu-id="c4f0b-116">Функция [**CreateEvent**](/windows/desktop/api/synchapi/nf-synchapi-createeventa) создает объект события и возвращает маркер объекта.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-116">The [**CreateEvent**](/windows/desktop/api/synchapi/nf-synchapi-createeventa) function creates the event object and returns an object handle.</span></span>

<span data-ttu-id="c4f0b-118">После создания объекта события приложение может использовать обработчик событий для установки или ожидания события.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-118">After the event object has been created, the application can use the event handle to set or wait on the event.</span></span> <span data-ttu-id="c4f0b-119">Этот маркер остается действительным до тех пор, пока приложение не закроет обработчик или не завершит работу.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-119">The handle remains valid until the application closes the handle or terminates.</span></span>

![приложение, создающее объект события с несколькими дескрипторами](images/cshob-04.png)

<span data-ttu-id="c4f0b-123">Этот метод позволяет приложению иметь дескрипторы с разными правами доступа.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-123">This method enables an application to have handles with different access rights.</span></span> <span data-ttu-id="c4f0b-124">Например, у обработчика 1 может быть задано и ожидание доступа к событию, а у обработчика 2 может быть только ожидание доступа.</span><span class="sxs-lookup"><span data-stu-id="c4f0b-124">For example, Handle 1 might have set and wait access to the event, and Handle 2 might have only wait access.</span></span>
