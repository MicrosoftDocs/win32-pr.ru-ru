# <a name="system-pool-behavior"></a><span data-ttu-id="da60f-103">Поведение системного пула</span><span class="sxs-lookup"><span data-stu-id="da60f-103">System Pool Behavior</span></span>

<span data-ttu-id="da60f-104">В следующем обсуждении показаны действия, выполняемые системным пулом, когда уведомления о событиях отправляются и не ожидаются какие либо биометрические операции.</span><span class="sxs-lookup"><span data-stu-id="da60f-104">The following discussion highlights the actions taken by the system pool when event notices are sent and when no biometric operations are pending.</span></span>

## <a name="event-dispatching"></a><span data-ttu-id="da60f-105">Диспетчеризация событий</span><span class="sxs-lookup"><span data-stu-id="da60f-105">Event Dispatching</span></span>

<span data-ttu-id="da60f-106">Когда биометрическая единица формирует уведомление о событии, системный пул использует каскадный фильтр для отправки уведомления и назначения ему одного из следующих уровней приоритета:</span><span class="sxs-lookup"><span data-stu-id="da60f-106">When a biometric unit generates an event notice, the system pool uses a cascading filter to dispatch the notice and assign it one of the following priority levels:</span></span>

-   <span data-ttu-id="da60f-107">Высокий приоритет назначается явным запросам на сопоставление и регистрации, создаваемым клиентами.</span><span class="sxs-lookup"><span data-stu-id="da60f-107">High priority is assigned to explicit matching and enrollment requests generated by clients.</span></span>
-   <span data-ttu-id="da60f-108">Средний приоритет назначается непредвиденному или непредусмотренному сопоставлению или событиям регистрации.</span><span class="sxs-lookup"><span data-stu-id="da60f-108">Medium priority is assigned to unexpected or unclaimed matching or enrollment events.</span></span>
-   <span data-ttu-id="da60f-109">Низкий приоритет назначается событиям навигации.</span><span class="sxs-lookup"><span data-stu-id="da60f-109">Low priority is assigned to navigation events.</span></span>

<span data-ttu-id="da60f-110">События отслеживания доставляются в следующей последовательности:</span><span class="sxs-lookup"><span data-stu-id="da60f-110">Capture events are delivered in the following sequence:</span></span>

-   <span data-ttu-id="da60f-111">Если текущее окно фокуса ожидает сопоставления или операции регистрации, пример обрабатывается и отправляется клиенту, владеющему текущим окном фокуса.</span><span class="sxs-lookup"><span data-stu-id="da60f-111">If the current focus window is waiting for a matching or an enrollment operation, the sample is processed and sent to the client that owns the current focus window.</span></span>
-   <span data-ttu-id="da60f-112">Если событие отслеживания не произошло из-за текущего окна фокуса, а необработанный обработчик событий зарегистрирован в биометрической службе Windows, событие Capture отправляется этому обработчику.</span><span class="sxs-lookup"><span data-stu-id="da60f-112">If the capture event is unclaimed by the current focus window and an unclaimed event handler has been registered with the Windows Biometric service, the capture event is sent to this handler.</span></span>
-   <span data-ttu-id="da60f-113">Если событие остается недоступным, оно отбрасывается.</span><span class="sxs-lookup"><span data-stu-id="da60f-113">If the event remains unclaimed, it is discarded.</span></span>

<span data-ttu-id="da60f-114">Если событие является событием навигации и обработчик событий навигации зарегистрирован в биометрической службе Windows, событие Capture отправляется в этот обработчик.</span><span class="sxs-lookup"><span data-stu-id="da60f-114">If the event is a navigation event and a navigation event handler has been registered with the Windows Biometric service, the capture event is sent to this handler.</span></span> <span data-ttu-id="da60f-115">Если обработчик событий отсутствует, событие отбрасывается.</span><span class="sxs-lookup"><span data-stu-id="da60f-115">If there is no event handler, the event is discarded.</span></span>

## <a name="idle-mode"></a><span data-ttu-id="da60f-116">Режим простоя</span><span class="sxs-lookup"><span data-stu-id="da60f-116">Idle Mode</span></span>

<span data-ttu-id="da60f-117">Если нет клиентов, ожидающих явного сопоставления или выполнения запросов на регистрацию, системный пул определяет, следует ли автоматически создавать повторяющиеся запросы на захват и отправлять полученное уведомление о событии в обработчик незапрошенных событий или ожидать события навигации и отправлять их в обработчик событий навигации.</span><span class="sxs-lookup"><span data-stu-id="da60f-117">When there are no clients waiting for explicit matching or enrollment requests to complete, the system pool determines whether to automatically generate repeated capture requests and send the resulting event notice to the unclaimed event handler or wait for navigation events and send them to the navigation event handler.</span></span>

<span data-ttu-id="da60f-118">Если в биометрической службе Windows зарегистрирован Неутвержденный обработчик событий, системный пул выполняет следующие действия:</span><span class="sxs-lookup"><span data-stu-id="da60f-118">If an unclaimed event handler has been registered with the Windows Biometric service, the system pool performs the following actions:</span></span>

-   <span data-ttu-id="da60f-119">Режим навигации для датчика отключен.</span><span class="sxs-lookup"><span data-stu-id="da60f-119">The navigation mode for the sensor is disabled.</span></span>
-   <span data-ttu-id="da60f-120">Неутвержденные операции отправляются в обработчик событий независимо от фокуса окна.</span><span class="sxs-lookup"><span data-stu-id="da60f-120">Unclaimed operations are sent to the event handler regardless of window focus.</span></span>
-   <span data-ttu-id="da60f-121">Если для биометрической операции нет необработанных запросов, выполняется автоматическая запись.</span><span class="sxs-lookup"><span data-stu-id="da60f-121">If there are no outstanding requests for a biometric operation, an automatic capture is performed.</span></span>

<span data-ttu-id="da60f-122">Если обработчик навигации зарегистрирован в биометрической службе Windows, системный пул выполняет следующие действия:</span><span class="sxs-lookup"><span data-stu-id="da60f-122">If a navigation handler has been registered with the Windows Biometric service, the system pool does the following:</span></span>

-   <span data-ttu-id="da60f-123">Биометрические единицы в системном пуле помещаются в состояние навигации, если нет ожидающих биометрических операций.</span><span class="sxs-lookup"><span data-stu-id="da60f-123">The biometric units in the system pool are placed into a navigation state if no biometric operations are pending.</span></span>
-   <span data-ttu-id="da60f-124">События навигации отключаются, если клиент отправляет уведомление о сопоставлении или регистрации.</span><span class="sxs-lookup"><span data-stu-id="da60f-124">Navigation events are disabled if a match or enrollment event notice is sent by a client.</span></span>
-   <span data-ttu-id="da60f-125">Если обработчик необработанных событий зарегистрирован, события навигации отключены.</span><span class="sxs-lookup"><span data-stu-id="da60f-125">If an unclaimed events handler has been registered, navigation events are disabled.</span></span>
