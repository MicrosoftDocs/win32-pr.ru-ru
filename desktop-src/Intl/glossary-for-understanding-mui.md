## <a name="globalization"></a><span data-ttu-id="afb9f-108">Глобализация</span><span class="sxs-lookup"><span data-stu-id="afb9f-108">Globalization</span></span>

<span data-ttu-id="afb9f-109">Термин " *глобализация* " указывает на необходимость использования платформы в любом случае.</span><span class="sxs-lookup"><span data-stu-id="afb9f-109">The term *globalization* indicates the need for the platform to work everywhere out of the box.</span></span> <span data-ttu-id="afb9f-110">На практике эта концепция реализована в Windows путем создания единого двоичного файла для операционной системы Windows, начиная с выпуска Windows Vista, посредством поддержки развертывания одного образа в многоязычном контексте и предоставления интерфейсов API NLS для извлечения данных, зависящих от языкового стандарта.</span><span class="sxs-lookup"><span data-stu-id="afb9f-110">In practice, this vision is realized within Windows by having a single worldwide binary for the Windows operating system starting with the Windows Vista release, by supporting single image deployment in a multilingual context and by providing NLS APIs to retrieve locale-specific data.</span></span>

## <a name="localizability"></a><span data-ttu-id="afb9f-111">Пригодность к локализации</span><span class="sxs-lookup"><span data-stu-id="afb9f-111">Localizability</span></span>

<span data-ttu-id="afb9f-112">Термин " *Локализация* " означает, что упрощается локализация продукта, что, в свою очередь, позволяет снизить затраты на локализацию.</span><span class="sxs-lookup"><span data-stu-id="afb9f-112">The term *localizability* means making the product easier to localize, which in turn drives down localization costs.</span></span> <span data-ttu-id="afb9f-113">Это можно сделать с помощью эффективного использования API-интерфейсов ресурсов, контроля экстента и количества ресурсов, добавляемых в систему, и улучшения средств, используемых для работы с этими ресурсами.</span><span class="sxs-lookup"><span data-stu-id="afb9f-113">This can be accomplished by efficient use of resource APIs, controlling the extent and number of resources added to the system, and improving the tools used to work with these resources.</span></span>
