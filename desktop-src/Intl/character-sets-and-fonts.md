# <a name="character-sets-and-fonts"></a>Кодировки и шрифты

Windows позволяет локально определить нестандартные символы в [двухбайтовых наборах символов](double-byte-character-sets.md) (dbcs) и [Unicode](unicode.md). Для DBCS эти нестандартные символы называются определяемыми пользователем символами (EUDC). Юникод обеспечивает аналогичную возможность благодаря своей частной области использования (PUA). Приложения определяют указанный символ, используя соответствующее символьное значение DBCS или Unicode.

Значения символов DBCS, которые могут быть назначены, зависят от заданной кодировки. каждая восточно-азиатская Windows [кодовая страница](code-pages.md) имеет по крайней мере один диапазон зарезервированных значений для использования в качестве еудкс. Диапазоны определяются ключом реестра [еудккодеранже](eudccoderange.md) . Значения Юникода для этой цели всегда берутся из PUA Юникода, значения U + E000 — U + F8FF, U + F0000 — U + FFFF, а U + 100000 — U + 10FFFD.

Чтобы создать символ EUDC или PUA, пользователь выбирает значение символа в пределах указанного диапазона и добавляет [глиф](uniscribe-glossary.md) к шрифту в записи, соответствующей этому символьному значению. Пользователь создает глиф с помощью редактора EUDC или пакета шрифтов, приобретенного у поставщика шрифтов. Любой шрифт DBCS может содержать Еудкс, а любой шрифт Юникода может содержать символы PUA. Шрифт называется "отдельным" шрифтом EUDC/PUA, если он содержит только Еудкс. Шрифт является "интегрированным" шрифтом EUDC/PUA, если он содержит стандартные символы, а также Еудкс.

Системный шрифт EUDC/PUA по умолчанию — это шрифт, который операционная система автоматически связывает с любыми шрифтами DBCS и Юникод, за исключением шрифтов с явно связанными шрифтами EUDC/PUA. Приложения устанавливают шрифт EUDC/PUA системы по умолчанию, устанавливая значение имени Системдефаултеудкфонт в разделе реестра [EUDC](eudc.md) . Аналогичным образом приложения могут связывать отдельные шрифты EUDC/PUA с соответствующими шрифтами, указывая имя шрифта и связанный с ним файл шрифта в ключе EUDC. Операционная система всегда сначала пытается найти EUDC/PUA в текущем выбранном шрифте. Если шрифт не найден, операционная система ищет символ в соответствующем шрифте EUDC/PUA, если он был определен для текущего выбранного шрифта. Если поиск символа по-прежнему не удается, операционная система ищет его в системе по умолчанию шрифт EUDC/PUA системы.

Шрифты TrueType можно устанавливать либо в виде ttf, либо в виде файлов тте. Так как операционная система скрывает тте-файлы, приложения не могут перечислять или иным образом проверять установленные шрифты с помощью функций API GDI. Во многих операционных системах шрифт EUDC/PUA системы по умолчанию и отдельные шрифты EUDC/PUA устанавливаются в виде тте-файлов. Приложения, такие как редакторы EUDC и панель управления, должны использовать записи реестра для добавления, изменения и удаления таких шрифтов.

Использование символов EUDC и PUA не гарантирует надежного сохранения значения на разных компьютерах или наборах символов. Дополнительные сведения об использовании символов EUDC и PUA см. в разделе [символы, определяемые конечным пользователем и частными областями использования](end-user-defined-characters.md) .
