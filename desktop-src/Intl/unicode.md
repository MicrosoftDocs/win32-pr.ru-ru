<span data-ttu-id="505ab-123">Функции, поддерживающие Юникод, описаны в разделе [соглашения для прототипов функций](conventions-for-function-prototypes.md).</span><span class="sxs-lookup"><span <span data-ttu-id="505ab-129">Набор символов Юникода включает множество Объединенных символов, например U + 0308 ("ё"), объединяющий диересис или умляут.</span><span class="sxs-lookup"><span data-stu-id="505ab-129">The Unicode character set includes numerous combining characters, such as U+0308 ("¨"), a combining dieresis or umlaut.</span></span> <span data-ttu-id="505ab-130">Юникод часто представляет один и тот же глиф в форме "", состоящей из "", или "", разделенной "". <span data-ttu-id="505ab-136">В новых приложениях Windows в качестве внутреннего представления данных следует использовать UTF-16.</span><span class="sxs-lookup"><span data-stu-id="505ab-136">New Windows applications should use UTF-16 as their internal data representation.</span></span> <span data-ttu-id="505ab-137">В Windows также предусмотрена широкая поддержка кодовых страниц, и смешанное использование в одном приложении возможно.</span><span class="sxs-lookup"><span data-stu-id="505ab-137">Windows also provides extensive support for code pages, and mixed use in the same application is possible.</span></span> <span data-ttu-id="505ab-138">Даже новые приложения на основе Юникода иногда приходится работать с кодовыми страницами.</span><span class="sxs-lookup"><span data-stu-id="505ab-138">Even new Unicode-based applications sometimes have to work with code pages.</span></span> <span data-ttu-id="505ab-139">Причины этого описаны в разделе [кодовые страницы](code-pages.md).</span><span class="sxs-lookup"><span data-stu-id="505ab-139">Reasons for this are discussed in [Code Pages](code-pages.md).</span></span>
