<span data-ttu-id="b411b-104">Окна состояние, композиция и кандидаты формируют пользовательский интерфейс редактора IME.</span><span class="sxs-lookup"><span data-stu-id="b411b-104">The status, composition, and candidates windows form the user interface for the IME.</span></span> <span data-ttu-id="b411b-105">Окно состояния указывает на то, что редактор IME открыт и предоставляет пользователю средства для установки режимов преобразования.</span><span class="sxs-lookup"><span data-stu-id="b411b-105">The status window indicates that the IME is open and provides the user with the means to set the conversion modes.</span></span> <span data-ttu-id="b411b-106">Окно композиции появляется, когда пользователь вводит текст и, в зависимости от режима преобразования, либо отображает введенный текст, либо отображает преобразованный текст.</span><span class="sxs-lookup"><span data-stu-id="b411b-106">The composition window appears when the user enters text and, depending on the conversion mode, either displays the text as entered or displays converted text.</span></span> <span data-ttu-id="b411b-107">Окно кандидатов отображается в сочетании с окном композиция.</span><span class="sxs-lookup"><span data-stu-id="b411b-107">The candidates window appears in conjunction with the composition window.</span></span> <span data-ttu-id="b411b-108">Он содержит список "кандидатов" characters) for the selected character or characters in the composition window.</span></span> <span data-ttu-id="b411b-109">Пользователь может прокрутить список кандидатов и выбрать нужные символы, а class="sxs-lookup"><span data-stu-id="b411b-110">The user can compose the desired text in this way until the composition string is finalized and the window is closed.</span></span>

<span data-ttu-id="b411b-111">Редактор IME отправляет составные символы в приложение, поддерживающее IME, в виде/ГКС в формате [**WM \_ IME \_ char**](wm-ime-char.md) или [**WM \_ IME \_ композиция**](wm-ime-composition.md) \_ .</span><span class="sxs-lookup"><span data-stu-id="b411b-111">The IME sends the composed characters to the IME-aware application in the form of [**WM\_IME\_CHAR**](wm-ime-char.md) or [**WM\_IME\_COMPOSITION**](wm-ime-composition.md)/GCS\_RESULT messages.</span></span> <span data-ttu-id="b411b-112">Если приложение не обрабатывает эти сообщения, функция [**дефвиндовпрок**](/windows/desktop/api/winuser/nf-winuser-defwindowproca) преобразует их в одно или несколько сообщений [**WM \_ char**](../inputdev/wm-char.md) .</span><span class="sxs-lookup"><span data-stu-id="b411b-112">If the application does not process these messages, the [**DefWindowProc**](/windows/desktop/api/winuser/nf-winuser-defwindowproca) function translates them into one or more [**WM\_CHAR**](../inputdev/wm-char.md) messages.</span></span>

<span data-ttu-id="b411b-113">По умолчанию операционная система автоматически создает окна состояния, композиции и кандидатов и управляет ими для требований к вводу текста.</span><span class="sxs-lookup"><span data-stu-id="b411b-113">By default, the operating system automatically creates and manages status, composition, and candidates windows for text input requirements.</span></span> <span data-ttu-id="b411b-114">Для многих приложений достаточно такой обработки по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="b411b-114">For many applications, this default processing is sufficient.</span></span> <span data-ttu-id="b411b-115">Эти приложения полностью основаны на операционной системе для поддержки IME и говорят, что они не знают о многих задачах, которые операционная система выполняет для управления окнами IME.</span><span class="sxs-lookup"><span data-stu-id="b411b-115">These applications rely entirely on the operating system for IME support and are said to be "IME-unaware" because they are unaware of the many tasks the operating system carries out to manage the IME windows.</span></span>

<span data-ttu-id="b411b-116">С другой стороны, приложение, поддерживающее IME, участвует в создании окон IME и управлении ими.</span><span class="sxs-lookup"><span data-stu-id="b411b-116">An IME-aware application, on the other hand, participates in the creation and management of IME windows.</span></span> <span data-ttu-id="b411b-117">Такие приложения управляют работой, положением и внешним видом окон по умолчанию, отправляя сообщения в эти окна и перехватывая и обрабатывая сообщения из окон.</span><span class="sxs-lookup"><span data-stu-id="b411b-117">Such applications control the operation, position, and appearance of the default windows by sending messages to these windows and by intercepting and processing messages from the windows.</span></span> <span data-ttu-id="b411b-118">В некоторых случаях приложения создают собственные окна IME и обеспечивают полную обработку собственных окон состояния, композиции и кандидатов.</span><span class="sxs-lookup"><span data-stu-id="b411b-118">In some cases, applications create their own IME windows and provide complete processing for their custom status, composition, and candidates windows.</span></span>
