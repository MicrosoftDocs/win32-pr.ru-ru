## <a name="platform-specific-extensions"></a><span data-ttu-id="e8316-337">Расширения для конкретной платформы</span><span class="sxs-lookup"><span data-stu-id="e8316-337">Platform-specific Extensions</span></span>

<span data-ttu-id="e8316-338">Библиотека Директксмас предназначена для упрощения поддержки C++ SIMD-программирования, обеспечивающей отличную поддержку платформ x86, x64 и Windows RT с помощью широко поддерживаемых встроенных инструкций (SSE2 и ARM-NEON).</span><span class="sxs-lookup"><span data-stu-id="e8316-338">The DirectXMath library is intended to simplify C++ SIMD programming providing excellent support for x86, x64, and Windows RT platforms using broadly supported intrinsics instructions (SSE2 and ARM-NEON).</span></span>

<span data-ttu-id="e8316-339">Однако бывают случаи, когда инструкции, относящиеся к конкретной платформе, могут оказаться полезными.</span><span class="sxs-lookup"><span data-stu-id="e8316-339">There are times, however, when platform-specific instructions may prove beneficial.</span></span> <span data-ttu-id="e8316-340">Из-за способа реализации Директксмас во многих случаях довольно просто использовать типы Директксмас непосредственно в стандартных встроенных инструкциях, поддерживаемых компилятором, и использовать Директксмас в качестве резервного пути для платформ, которые не поддерживают расширенную инструкцию.</span><span class="sxs-lookup"><span data-stu-id="e8316-340">Due to the way DirectXMath is implemented, in many cases it is trivial to use DirectXMath types directly in standard compiler-supported intrinsics statements, and to use DirectXMath as the fallback path for platforms that don't support the extended instruction.</span></span>

<span data-ttu-id="e8316-341">Например, ниже приведен упрощенный пример использования инструкции по использованию SSE 4,1 Dot-Product.</span><span class="sxs-lookup"><span data-stu-id="e8316-341">For example, here is a simplified example of leveraging the SSE 4.1 dot-product instruction.</span></span> <span data-ttu-id="e8316-342">Обратите внимание, что необходимо явно защитить путь кода, чтобы избежать создания исключений недопустимых инструкций во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="e8316-342">Note that you must explicitly guard the code-path to avoid generating invalid instruction exceptions at run time.</span></span> <span data-ttu-id="e8316-343">Убедитесь, что пути кода достаточно существенны, чтобы вычислить дополнительную стоимость ветвления, сложность поддержки нескольких путей кода и т. д.</span><span class="sxs-lookup"><span data-stu-id="e8316-343">Ensure the code paths do significant enough work to justify the additional cost of branching, complexity of maintaining multiple code-paths, and so on.</span></span>


```
#include <windows.h>
#include <stdio.h>

#include <DirectXMath.h>

#include <intrin.h>
#include <smmintrin.h>

using namespace DirectX;

bool g_bSSE41 = false;

void DetectCPUFeatures()
{
#ifndef _M_ARM
   // See __cpuid documentation on MSDN for more information

   int CPUInfo[4] = {-1};
   __cpuid( CPUInfo, 0 );

   if ( CPUInfo[0] >= 1 )
   {
       __cpuid(CPUInfo, 1 );
 
       if ( CPUInfo[2] & 0x80000 )
           g_bSSE41 = true;
   }
#endif
}

int main()
{
   if ( !XMVerifyCPUSupport() )
       return -1;

   DetectCPUFeatures();

   ...

   XMVECTORF32 v1 = { 1.f, 2.f, 3.f, 4.f };
   XMVECTORF32 v2 = { 5.f, 6.f, 7.f, 8.f };

   XMVECTOR r2, r3, r4;

   if ( g_bSSE41 )
   {
#ifndef _M_ARM
       r2 = _mm_dp_ps( v1, v2, 0x3f );
       r3 = _mm_dp_ps( v1, v2, 0x7f );
       r4 = _mm_dp_ps( v1, v2, 0xff );
#endif
   }
   else
   {
       r2 = XMVector2Dot( v1, v2 );
       r3 = XMVector3Dot( v1, v2 );
       r4 = XMVector4Dot( v1, v2 );
   }

   ... 

   return 0;
}
```
