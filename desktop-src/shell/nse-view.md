# <a name="displaying-a-self-contained-view-of-a-namespace-extension"></a>Отображение Self-Containedного представления расширения пространства имен

Большинство расширений пространства имен являются подмножеством пространства имен оболочки. при создании точки соединения, как описано в разделе [указание расположения расширения пространства имен](nse-junction.md), Windows Explorer позволяет пользователям переходить к своему расширению пространства имен и из него, как и любые другие папки. однако можно также использовать Windows Explorer, чтобы отобразить только содержимое расширения пространства имен. Этот параметр отображения иногда называется *корневым представлением*. Обычно корневые представления могут быть предпочтительнее обычно для некоторых типов расширений.

при наличии корневого представления создается новый экземпляр обозревателя Windows, который отображает содержимое расширения как отдельное пространство имен. Представление в виде дерева в корневом представлении показывает только те папки, которые являются частью расширения. Пользователи не могут переходить от корневого представления к другим частям пространства имен оболочки.

Расширения реализуются таким же образом для корневых представлений, как и для обычных представлений. единственное различие заключается в способе отображения содержимого расширения с помощью обозревателя Windows Explorer. Также можно иметь обычные и корневые представления одного и того же расширения.

Чтобы открыть представление расширения, необходимо запустить экземпляр Explorer.exe с флагом/root. Существует несколько различных способов запуска корневого представления в зависимости от природы вашего расширения.

-   [Как использовать точку соединения для открытия корневого представления](how-to-use-a-junction-point-to-open-a-rooted-view.md)
-   [Использование файла ярлыка для открытия корневого представления](how-to-use-a-shortcut-file-to-open-a-rooted-view.md)
-   [Отображение корневого представления файла](how-to-display-a-rooted-view-of-a-file.md)
