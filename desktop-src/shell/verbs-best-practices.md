## <a name="best-practices-for-multiple-selection-verbs"></a><span data-ttu-id="f3f7d-139">Рекомендации по использованию нескольких команд выбора</span><span class="sxs-lookup"><span data-stu-id="f3f7d-139">Best Practices for Multiple Selection Verbs</span></span>

<span data-ttu-id="f3f7d-140">Поскольку количество элементов в сценарии с множественным выбором может быть большим, важно учитывать влияние реализации команд на производительность.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-140">Because the number of items in a multi-selection verb scenario can be large, it is important that you consider the performance implications of your verb implementations.</span></span> <span data-ttu-id="f3f7d-141">Например, когда пользователь ищет " \* " над областью, включающей большое количество элементов, а затем нажимает кнопку **выбрать все** и щелкаете правой кнопкой мыши, команда представляет выделенный фрагмент, в котором могут содержаться тысячи элементов.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-141">For example, when a user searches for "\*" over a scope that includes large numbers of items, and then clicks **Select All** and right-clicks, the verb is presented with a selection that can have thousands of items in it.</span></span> <span data-ttu-id="f3f7d-142">В результате глаголы должны учитывать только первый элемент в выделенном фрагменте и общее число элементов.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-142">As a result, verbs should only consider the first item in the selection, and the overall item count.</span></span> <span data-ttu-id="f3f7d-143">Первый элемент определяется как элемент в верхней части представления или элемент, который пользователь впервые щелкнул правой кнопкой мыши.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-143">The first item is defined as either the item at the top of the view, or the item that the user first right-clicked.</span></span>

<span data-ttu-id="f3f7d-144">В Windows 7 и более поздних версиях число элементов, передаваемых в команду, ограничено 16 при запросе контекстного меню.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-144">In Windows 7 and later, the number of items passed to a verb is limited to 16 when a shortcut menu is queried.</span></span> <span data-ttu-id="f3f7d-145">Затем команда создается повторно и повторно инициализируется с полным выделением при вызове этой команды.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-145">The verb is then re-created and re-initialized with the full selection when that verb is invoked.</span></span>

### <a name="heterogenous-selections"></a><span data-ttu-id="f3f7d-150">Разнородных выбора</span><span class="sxs-lookup"><span data-stu-id="f3f7d-150">Heterogenous Selections</span></span>

<span data-ttu-id="f3f7d-151">Оптимистичные глаголы автоматически добавляются в вариант множественного выбора, предполагая, что непроверенные элементы могут обрабатываться командой.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-151">Optimistic verbs are automatically added in the multi-selection case, assuming that uninspected items can be handled by the verb.</span></span> <span data-ttu-id="f3f7d-152">С другой стороны, пессимистические команды не добавляются, если выделенный фрагмент содержит непроверенные элементы и добавляется только в тех случаях, когда количество элементов невелико.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-152">In contrast, pessimistic verbs are not added when the selection contains uninspected items, and are only added in cases where the number of items is small.</span></span>

<span data-ttu-id="f3f7d-153">Команды стиля проигрывателя должны быть оптимистичными и автоматически пропускать необработанные элементы.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-153">Player style verbs should be optimistic, and silently skip the items that are not handled.</span></span> <span data-ttu-id="f3f7d-154">Если сбой работы с элементами может привести к потере или путанице данных, команда должна предупредить пользователей об элементах, которые не могут быть обработаны.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-154">If a failure to act on items can cause data loss or confusion, then the verb should warn users about items that cannot be processed.</span></span> <span data-ttu-id="f3f7d-155">Например, команда Backup должна указывать, что не удалось создать резервную копию некоторых элементов.</span><span class="sxs-lookup"><span data-stu-id="f3f7d-155">For example a "backup" verb should indicate that some items could not be backed up.</span></span>
