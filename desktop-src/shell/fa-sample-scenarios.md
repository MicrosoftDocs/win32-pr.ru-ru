## <a name="designing-a-new-file-association"></a><span data-ttu-id="16b1d-109">Проектирование нового сопоставления файлов</span><span class="sxs-lookup"><span data-stu-id="16b1d-109">Designing a New File Association</span></span>

<span data-ttu-id="16b1d-110">Компания должна выполнить следующие действия.</span><span class="sxs-lookup"><span data-stu-id="16b1d-110">The company should take the following steps.</span></span>

1.  <span data-ttu-id="16b1d-111">**Определите, следует ли обрабатывать новый тип файлов как открытый или частный.**</span><span class="sxs-lookup"><span data-stu-id="16b1d-111">**Decide if the new file type should be treated as public or private.**</span></span> <span data-ttu-id="16b1d-112">Этот новый тип файла является типом носителя.</span><span class="sxs-lookup"><span data-stu-id="16b1d-112">This new file type is a media type.</span></span> <span data-ttu-id="16b1d-113">Так как пользователи обмениваются файлами мультимедиа на различных платформах и могут иметь другие приложения, которым необходимо считывать формат Литвареплайер, наиболее подходящим является [общедоступный](fa-file-types.md) тип файлов.</span><span class="sxs-lookup"><span data-stu-id="16b1d-113">Because users exchange media files across various platforms and there might be other applications that need to read the LitwarePlayer format, a [public](fa-file-types.md) file type is the most appropriate.</span></span>
2.  <span data-ttu-id="16b1d-114">**Определите, был ли уже определен этот тип файла.**</span><span class="sxs-lookup"><span data-stu-id="16b1d-114">**Determine whether this file type is already defined.**</span></span> <span data-ttu-id="16b1d-115">Проверьте базу данных MIME (IANA), а также другие общедоступные базы данных типа файлов в Интернете, чтобы определить, что сравниваемый тип файлов не определен.</span><span class="sxs-lookup"><span data-stu-id="16b1d-115">Check the Internet Assigned Numbers Authority (IANA) MIME database, and other public file type databases on the Internet to determine that no comparable file type has been defined.</span></span> <span data-ttu-id="16b1d-116">Так как это новый формат файла, необходимо определить новый тип файлов.</span><span class="sxs-lookup"><span data-stu-id="16b1d-116">Because this is a new file format, you need to define a new file type.</span></span>
3.  <span data-ttu-id="16b1d-117">**Определите расширение имени файла для нового типа файлов.**</span><span class="sxs-lookup"><span data-stu-id="16b1d-117">**Define a file name extension for the new file type.**</span></span> <span data-ttu-id="16b1d-118">Разработчики выбирают объект `.opa-ltw-audio` , который включает в себя аббревиатуру поставщика, и подсказку о том, что содержит файл.</span><span class="sxs-lookup"><span data-stu-id="16b1d-118">The developers choose the `.opa-ltw-audio`, which incorporates their vendor abbreviation and a hint about the what the file contains.</span></span> <span data-ttu-id="16b1d-119">Исследование определяет, что расширение не используется другими пользователями.</span><span class="sxs-lookup"><span data-stu-id="16b1d-119">Research determines that the extension is not being used by anyone else.</span></span> <span data-ttu-id="16b1d-120">В соответствии с текущими рекомендациями короткое расширение не определено.</span><span class="sxs-lookup"><span data-stu-id="16b1d-120">Following current recommendations, no short extension is defined.</span></span>
4.  <span data-ttu-id="16b1d-121">**Определите тип MIME для типа файла и зарегистрируйте его с помощью IANA.**</span><span class="sxs-lookup"><span data-stu-id="16b1d-121">**Define a MIME type for the file type and register it with the IANA.**</span></span> <span data-ttu-id="16b1d-122">Litware определяет новый тип MIME как *Audio/литвареплайер. 1* и подготавливает приложение типа MIME, следуя рекомендациям, изложенным в документе RFC (запросы на комментарии) 2045, 2046, 2047 и 2048.</span><span class="sxs-lookup"><span data-stu-id="16b1d-122">Litware defines the new MIME type as *audio/LitwarePlayer.1* and prepares a MIME type application, following the guidelines laid out in Request for Comments (RFC) numbers 2045, 2046, 2047, and 2048.</span></span> <span data-ttu-id="16b1d-123">Затем они отправляют приложение в IANA, который добавляет новый тип файлов в базу данных зарегистрированных типов MIME.</span><span class="sxs-lookup"><span data-stu-id="16b1d-123">They then submit the application to the IANA, which adds the new file type to the database of registered MIME types.</span></span>
5.  <span data-ttu-id="16b1d-124">**Определите, существует ли идентификатор ProgID для типа файла.**</span><span class="sxs-lookup"><span data-stu-id="16b1d-124">**Determine whether a ProgID exists for the file type.**</span></span> <span data-ttu-id="16b1d-125">Так как это новый тип файла, для него не существует [ProgID](fa-progids.md) .</span><span class="sxs-lookup"><span data-stu-id="16b1d-125">Because this is a new file type, no [ProgID](fa-progids.md) exists for it.</span></span> <span data-ttu-id="16b1d-126">Litware задает создание нового идентификатора ProgID для Литвареплайер.</span><span class="sxs-lookup"><span data-stu-id="16b1d-126">Litware sets about designing a new ProgID for LitwarePlayer.</span></span> <span data-ttu-id="16b1d-127">Они решают использовать понятное имя "Литвареплайер Audio Player" (которое хранится в виде ресурса в файле LitwarePlayer.exe) и разрабатывает значок по умолчанию для файлов, связанных с Литвареплайер (также хранимых в LitwarePlayer.exe).</span><span class="sxs-lookup"><span data-stu-id="16b1d-127">They decide on the friendly name "LitwarePlayer Audio Player" (which is stored as a resource in the LitwarePlayer.exe file), and they design a default icon to use for files associated with LitwarePlayer (also stored in LitwarePlayer.exe).</span></span> <span data-ttu-id="16b1d-128">Так как Литвареплайер — это новое приложение, это идентификатор ProgID версии 1.</span><span class="sxs-lookup"><span data-stu-id="16b1d-128">Because LitwarePlayer is a new application, this is a version 1 ProgID.</span></span>
6.  <span data-ttu-id="16b1d-129">**Зарегистрируйте идентификатор ProgID.**</span><span class="sxs-lookup"><span data-stu-id="16b1d-129">**Register the ProgID.**</span></span> <span data-ttu-id="16b1d-130">При установке Литвареплайер программа установки создает в реестре следующую запись [ProgID](fa-progids.md) .</span><span class="sxs-lookup"><span data-stu-id="16b1d-130">When LitwarePlayer is installed, the installation program creates the following [ProgID](fa-progids.md) entry in the registry.</span></span>

    ```
    HKEY_CLASSES_ROOT
       Litware.LitwarePlayer.1
          (Default) = LitwarePlayer Audio Player
          FriendlyTypeName = @LitwarePlayer, -120
          CurVer
             (Default) = Litware.LitwarePlayer.1
          DefaultIcon
             (Default) = LitwarePlayer, -142
          shell
             play
                command
                   (Default) = "%ProgramFiles%\LitwarePlayer\LitwarePlayer.exe" "%1"
    ```

    <span data-ttu-id="16b1d-131">В командном ключе %1 передается в качестве пути к файлу для воспроизведения.</span><span class="sxs-lookup"><span data-stu-id="16b1d-131">In the command key, %1 is passed as the path to the file to play.</span></span>

7.  <span data-ttu-id="16b1d-132">**Зарегистрируйте расширение имени файла для типа файла.**</span><span class="sxs-lookup"><span data-stu-id="16b1d-132">**Register the file name extension for the file type.**</span></span> <span data-ttu-id="16b1d-133">При установке Литвареплайер программа установки создает в реестре следующие записи для пользовательского расширения типа файла.</span><span class="sxs-lookup"><span data-stu-id="16b1d-133">When LitwarePlayer is installed, the installation program creates the following entries in the registry for its custom file type extension.</span></span>

    ```
    HKEY_CLASSES_ROOT
       .opa-vwi-audio
          (Default) = Litware.LitwarePlayer.1
          PerceivedType = Audio
          Content Type = audio/LitwarePlayer
    ```

> [!Note]  
> <span data-ttu-id="16b1d-134">При каждом создании или изменении сопоставления файлов уведомите систему о том, что изменение было выполнено путем вызова [**шчанженотифи**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shchangenotify), указав \_ событие ассокчанжед шкне.</span><span class="sxs-lookup"><span data-stu-id="16b1d-134">Whenever a file association is created or changed, notify the system that a change has been made by calling [**SHChangeNotify**](/windows/desktop/api/shlobj_core/nf-shlobj_core-shchangenotify), specifying the SHCNE\_ASSOCCHANGED event.</span></span> <span data-ttu-id="16b1d-135">Если это не сделано, оболочка может не распознать изменения, внесенные до перезагрузки системы.</span><span class="sxs-lookup"><span data-stu-id="16b1d-135">If this is not done, the Shell might not recognize any changes made until the system restarts.</span></span>
