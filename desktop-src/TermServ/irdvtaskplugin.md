# <a name="irdvtaskplugin-interface"></a>Интерфейс Ирдвтаскплугин

Интерфейс **ирдвтаскплугин** реализуется *агентом задачи* обновления виртуальной машины, что позволяет агенту задач управлять обновлениями системы для виртуальной машины. Этот интерфейс используется *агентом триггера*, который реализуется хост-системой.

## <a name="members"></a>Элементы

Интерфейс **ирдвтаскплугин** наследует от интерфейса [**IUnknown**](/windows/desktop/api/unknwn/nn-unknwn-iunknown) . **Ирдвтаскплугин** также имеет следующие типы членов:

-   [Методы](#methods)
-   [Свойства](#properties)

### <a name="methods"></a>Методы

Интерфейс **ирдвтаскплугин** содержит следующие методы.



| Метод                                          | Описание                                                        |
|:------------------------------------------------|:-------------------------------------------------------------------|
| [**Инициализация**](irdvtaskplugin-initialize.md) | Вызывается для инициализации агента задачи.<br/>                    |
| [**StartTask**](irdvtaskplugin-starttask.md)   | Вызывается для запуска задачи обновления на виртуальной машине.<br/> |
| [**Завершение**](irdvtaskplugin-terminate.md)   | Вызывается при завершении работы агента задачи.<br/>          |



 

### <a name="properties"></a>Свойства

Интерфейс **ирдвтаскплугин** имеет следующие свойства.



| Свойство                                                   | Тип доступа          | Описание                                             |
|:-----------------------------------------------------------|:---------------------|:--------------------------------------------------------|
| [**PluginName**](irdvtaskplugin-pluginname.md)<br/> | Только для чтения<br/> | Содержит отображаемое имя агента задач.<br/> |



 

## <a name="remarks"></a>Remarks

Агент задачи выполняется на виртуальной машине, когда для этой виртуальной машины запланировано обновление системы. Агент задач обновляет виртуальную машину при вызове метода [**StartTask**](irdvtaskplugin-starttask.md) .

Чтобы зарегистрировать агент задач, добавьте следующий ключ в реестр виртуальной машины:

**HKey \_ \_** \\ **программное обеспечение** на локальном компьютере. \\  \\  \\  \\  \\  \\ **подключаемые модули** задачи сервера терминалов \\  Microsoft Windows NT CurrentVersion таскажентнаме

В разделе этого раздела реестра добавьте следующие значения:



| Имя                     | Тип                      | Описание                                                                   |
|--------------------------|---------------------------|-------------------------------------------------------------------------------|
| **ЭТОМУ**<br/>     | **REG \_ SZ**<br/>    | Строка, представляющая **идентификатор CLSID** агента задачи.<br/>          |
| **IsEnabled**<br/> | **REG \_ DWORD**<br/> | 0, если агент задач отключен, или 1, если агент задач включен.<br/> |



 

> [!Note]  
> Можно зарегистрировать более одного агента задач, но будет использоваться только один агент задачи. Если включен более одного агента задач, будет использоваться только первый найденный агент.

 

Хотя этот интерфейс поддерживается в операционных системах, указанных в приведенных ниже требованиях, он будет использоваться только в том случае, если виртуальная машина размещена на Windows Server 2012.
