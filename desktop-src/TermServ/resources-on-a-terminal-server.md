<span data-ttu-id="85051-104">В среде службы удаленных рабочих столов несколько пользователей могут одновременно войти в один сервер узла сеансов удаленный рабочий стол (который ранее назывался сервером терминалов).</span><span class="sxs-lookup"><span data-stu-id="85051-104">In a Remote Desktop Services environment, multiple users can log on simultaneously to a single Remote Desktop Session Host (RD Session Host) server (formerly known as a terminal server).</span></span> <span data-ttu-id="85051-105">Следовательно, пользователи совместно используют аппаратные и программные ресурсы сервера, что может создать следующие области состязаний:</span><span class="sxs-lookup"><span data-stu-id="85051-105">Consequently, the users are sharing the hardware and software resources of the server, which can create the following areas of contention:</span></span>

-   <span data-ttu-id="85051-106">Время ЦП.</span><span class="sxs-lookup"><span data-stu-id="85051-106">CPU time.</span></span> <span data-ttu-id="85051-107">Каждый пользователь имеет настольную среду и может запускать любые приложения, доступные для этого рабочего стола.</span><span class="sxs-lookup"><span data-stu-id="85051-107">Each user has a desktop environment and can run whatever applications are available to that desktop.</span></span> <span data-ttu-id="85051-108">Однако все приложения, запущенные всеми пользователями, будут посвящены центральным ресурсам ЦП, доступным на сервере узла сеансов удаленных рабочих столов.</span><span class="sxs-lookup"><span data-stu-id="85051-108">However, all applications run by all users are contending for the central CPU resources available on the RD Session Host server.</span></span> <span data-ttu-id="85051-109">Если один пользователь запускает плохо написанное приложение, интенсивно работающий с ЦП, другие пользователи могут столкнуться с незаметной потерей производительности.</span><span class="sxs-lookup"><span data-stu-id="85051-109">If one user runs a poorly written, CPU-intensive application, other users might experience a visible loss of performance.</span></span>
-   <span data-ttu-id="85051-110">Доступ к диску.</span><span class="sxs-lookup"><span data-stu-id="85051-110">Disk access.</span></span> <span data-ttu-id="85051-111">Пользователи конкурируют за доступ к приложениям и связанным файлам программ.</span><span class="sxs-lookup"><span data-stu-id="85051-111">Users contend for access to applications and related program files.</span></span> <span data-ttu-id="85051-112">Кроме того, пользователи конкурируют за доступ к диску серверной операционной системы, например загрузку библиотек DLL или обмен памятью между файлом подкачки и физической памятью.</span><span class="sxs-lookup"><span data-stu-id="85051-112">In addition, users contend for disk accesses by the server operating system, such as loading DLLs or swapping memory between the paging file and physical memory.</span></span>
-   <span data-ttu-id="85051-113">Память с произвольным доступом (ОЗУ).</span><span class="sxs-lookup"><span data-stu-id="85051-113">Random Access Memory (RAM).</span></span> <span data-ttu-id="85051-114">Каждое приложение, выполняемое каждым пользователем, зависит от ресурсов ОЗУ, доступных на сервере узла сеансов удаленных рабочих столов.</span><span class="sxs-lookup"><span data-stu-id="85051-114">Each application run by every user is contending for the RAM resources available on the RD Session Host server.</span></span> <span data-ttu-id="85051-115">Если один пользователь запускает приложение, интенсивно использующие память, другие пользователи могут столкнуться с потерей производительности.</span><span class="sxs-lookup"><span data-stu-id="85051-115">If one user runs a memory-intensive application, other users might experience a loss of performance.</span></span>
-   <span data-ttu-id="85051-116">Доступ по сети.</span><span class="sxs-lookup"><span data-stu-id="85051-116">Network access.</span></span> <span data-ttu-id="85051-117">Доступ к сети необходим в службы удаленных рабочих столов среде, так как все действия рабочего стола — графические выводимые данные и мыши и клавиатуры — проходят по сетевым связям между клиентским рабочим столом и сервером.</span><span class="sxs-lookup"><span data-stu-id="85051-117">Network access is essential in a Remote Desktop Services environment because all desktop activity—graphical output and mouse/keyboard input—flows over the network links between the client desktop and the server.</span></span> <span data-ttu-id="85051-118">Кроме того, приложения пользователей, работающие на сервере узла сеансов удаленных рабочих столов, конкурируют за доступ к другим сетевым ресурсам.</span><span class="sxs-lookup"><span data-stu-id="85051-118">In addition, the users' applications running on the RD Session Host server contend for access to other network resources.</span></span>
-   <span data-ttu-id="85051-119">Серверное оборудование.</span><span class="sxs-lookup"><span data-stu-id="85051-119">Server hardware.</span></span> <span data-ttu-id="85051-120">Аппаратные компоненты, такие как компакт-диски, дисководы гибких дисков, последовательные порты и параллельные порты, часто основываются на сервере, а не на клиенте.</span><span class="sxs-lookup"><span data-stu-id="85051-120">Hardware components such as CD-ROMs, floppy disk drives, serial ports, and parallel ports are often server based, not client based.</span></span> <span data-ttu-id="85051-121">Совместное использование этих общедоступных компонентов создает новые соображения для пользователей и приложений, обращающихся к этим аппаратным компонентам.</span><span class="sxs-lookup"><span data-stu-id="85051-121">Sharing these traditionally nonshared components creates new considerations for users and for applications that access these hardware components.</span></span> <span data-ttu-id="85051-122">Дополнительные сведения см. в разделе [рекомендации по периферийным устройствам](peripheral-hardware-guidelines.md).</span><span class="sxs-lookup"><span data-stu-id="85051-122">For more information, see [Peripheral Hardware Guidelines](peripheral-hardware-guidelines.md).</span></span>
-   <span data-ttu-id="85051-123">Доступ к глобальным объектам и ресурсам.</span><span class="sxs-lookup"><span data-stu-id="85051-123">Access to global objects and resources.</span></span> <span data-ttu-id="85051-124">В среде службы удаленных рабочих столов пользователи не запускают отдельные копии Windows — некоторые из основных модулей копируются, но остальные модули являются общими для пользователей.</span><span class="sxs-lookup"><span data-stu-id="85051-124">In a Remote Desktop Services environment, users do not run individual copies of Windows—some of the core modules are cloned, but the remaining modules are shared among the users.</span></span> <span data-ttu-id="85051-125">Таким же, пользователи конкурируют за доступ к реестру, файлу подкачки, системным службам и другим глобальным объектам и ресурсам.</span><span class="sxs-lookup"><span data-stu-id="85051-125">Thus, users are competing for access to the registry, the paging file, system services, and other global objects and resources.</span></span>

<span data-ttu-id="85051-126">Многие из предыдущих точек состязаний могут быть устранены путем изменения размера сервера узла сеансов удаленных рабочих столов с достаточным объемом ресурсов ЦП, памяти и дискового пространства для обработки спроса на клиент.</span><span class="sxs-lookup"><span data-stu-id="85051-126">Many of the preceding points of contention can be mitigated by sizing the data-ttu-id="85051-128">Доступность памяти можно развернуть путем установки дополнительного объема физической памяти (это может помочь при увеличении ограничений памяти для выпуска Enterprise, Datacenter или 64-разрядных выпусков Windows Server).</span><span class="sxs-lookup"><span data-stu-id="85051-128">Memory availability can be maximized by installing extra physical memory (the data-stu-id="85051-130">Properly configuring an RD Session Host server is a critical element of perceived application performance.</span></span>

<span data-ttu-id="85051-131">Несмотря на то, что размер оборудования является важной частью создания масштабируемой среды службы удаленных рабочих столов, рекомендации по программному обеспечению не менее важны.</span><span class="sxs-lookup"><span data-stu-id="85051-131">Although hardware sizing is an important part of creating a scalable Remote Desktop Services environment, the software considerations are equally important.</span></span> <span data-ttu-id="85051-132">На самом деле, Точная настройка приложения часто может значительно снизить конкуренцию за ресурсы и улучшить наблюдаемую производительность приложения.</span><span class="sxs-lookup"><span data-stu-id="85051-132">In fact, fine-tuning an application often can do a lot to reduce resource competition and improve perceived application performance.</span></span>
