# <a name="remote-desktop-sessions"></a><span data-ttu-id="9d860-103">Сеансы удаленный рабочий стол</span><span class="sxs-lookup"><span data-stu-id="9d860-103">Remote Desktop Sessions</span></span>

<span data-ttu-id="9d860-104">Когда пользователь входит на компьютер с включенной службы удаленных рабочих столов, для пользователя запускается сеанс.</span><span class="sxs-lookup"><span data-stu-id="9d860-104">When a user logs on to a Remote Desktop Services–enabled computer, a session is started for the user.</span></span> <span data-ttu-id="9d860-105">Каждый сеанс идентифицируется по уникальному ИДЕНТИФИКАТОРу сеанса.</span><span class="sxs-lookup"><span data-stu-id="9d860-105">Each session is identified by a unique session ID.</span></span> <span data-ttu-id="9d860-106">Поскольку каждый вход на клиент подключение к удаленному рабочему столу (RDC) получает отдельный идентификатор сеанса, взаимодействие с пользователем аналогично выполнению одновременного входа на несколько компьютеров. Например, компьютер Office и домашний компьютер.</span><span class="sxs-lookup"><span data-stu-id="9d860-106">Because each logon to a Remote Desktop Connection (RDC) client receives a separate session ID, the user-experience is similar to being logged on to multiple computers at the same time; for example, an office computer and a home computer.</span></span>

<span data-ttu-id="9d860-107">Каждый сеанс удаленного рабочего стола связан с интерактивной рабочей станцией.</span><span class="sxs-lookup"><span data-stu-id="9d860-107">Each remote desktop session is associated with an interactive window station.</span></span> <span data-ttu-id="9d860-108">Единственным поддерживаемым именем Windows Station для интерактивной рабочей станции является «WinSta0»; Поэтому каждый сеанс связывается с собственной рабочей станцией "WinSta0".</span><span class="sxs-lookup"><span data-stu-id="9d860-108">The only supported window station name for an interactive window station is "WinSta0"; therefore each session is associated with its own "WinSta0" window station.</span></span> <span data-ttu-id="9d860-109">Для каждой станции Windows существует три стандартных рабочего стола: Winlogon Desktop, экранная заставка и интерактивный рабочий стол.</span><span class="sxs-lookup"><span data-stu-id="9d860-109">There are three standard desktops for each window station: the Winlogon desktop, the screen saver desktop, and the interactive desktop.</span></span>

<span data-ttu-id="9d860-110">Пользователь, связанный с интерактивной рабочей станцией для сеанса, называется *интерактивным пользователем*.</span><span class="sxs-lookup"><span data-stu-id="9d860-110">The user associated with the interactive window station for a session is known as the *interactive user*.</span></span> <span data-ttu-id="9d860-111">В клиенте подключение к удаленному рабочему столу (RDC) в дополнение к интерактивному пользователю в консоли службы удаленных рабочих столов может быть несколько интерактивных пользователей.</span><span class="sxs-lookup"><span data-stu-id="9d860-111">On a Remote Desktop Connection (RDC) client there can be multiple interactive users in addition to the interactive user on the Remote Desktop Services console.</span></span> <span data-ttu-id="9d860-112">Чтобы получить идентификатор сеанса, который в настоящее время подключен к консоли, используйте функцию [**втсжетактивеконсолесессионид**](/windows/desktop/api/Winbase/nf-winbase-wtsgetactiveconsolesessionid) .</span><span class="sxs-lookup"><span data-stu-id="9d860-112">To retrieve the identifier of the session currently attached to the console, use the [**WTSGetActiveConsoleSessionId**](/windows/desktop/api/Winbase/nf-winbase-wtsgetactiveconsolesessionid) function.</span></span>

<span data-ttu-id="9d860-113">Когда пользователь выходит из системы подключение к удаленному рабочему столу (RDC), сеанс, который клиент находится на сервере узла сеансов удаленный рабочий стол (который ранее назывался сервером терминалов), удаляется, и удаляются и рабочие станции, связанные с этим сеансом.</span><span class="sxs-lookup"><span data-stu-id="9d860-113">When a user logs off from a Remote Desktop Connection (RDC) client, the session that the client has on the Remote Desktop Session Host (RD Session Host) server (formerly known as a terminal server) is deleted and the window stations and desktops associated with that session are removed.</span></span> <span data-ttu-id="9d860-114">Однако, поскольку сеанс консоли службы удаленных рабочих столов никогда не удаляется, станции окон, связанные с сеансом консоли, не удаляются.</span><span class="sxs-lookup"><span data-stu-id="9d860-114">However, because the Remote Desktop Services console session is never deleted, the window stations associated with the console session are not deleted.</span></span> <span data-ttu-id="9d860-115">Это влияет на поведение приложений в среде службы удаленных рабочих столов, когда они настроены для работы в контексте безопасности интерактивного пользователя, также известного как режим активации "Интерактивный пользователь запуска от имени".</span><span class="sxs-lookup"><span data-stu-id="9d860-115">This affects how applications behave in a Remote Desktop Services environment when they are configured to run in the security context of the interactive user, also known as the "RunAs Interactive User" object activation mode.</span></span>

<span data-ttu-id="9d860-116">Дополнительные сведения о запуске локального серверного процесса в указанном сеансе см. в статье [Активация сеанса в сеансе с моникером сеанса](session-to-session-activation-with-a-session-moniker.md) и [Использование моникера сеанса](using-a-session-moniker.md).</span><span class="sxs-lookup"><span data-stu-id="9d860-116">For more information about starting a local server process on a specified session, see [Session-to-Session Activation with a Session Moniker](session-to-session-activation-with-a-session-moniker.md) and [Using a Session Moniker](using-a-session-moniker.md).</span></span> <span data-ttu-id="9d860-117">Дополнительные сведения о контекстах безопасности см. в описании [контекста безопасности клиента](/windows/desktop/SecAuthZ/the-client-security-context).</span><span class="sxs-lookup"><span data-stu-id="9d860-117">For more information about security contexts, see [The Client's Security Context](/windows/desktop/SecAuthZ/the-client-security-context).</span></span>
