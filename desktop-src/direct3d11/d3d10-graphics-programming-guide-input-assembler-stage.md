# <a name="input-assembler-stage"></a><span data-ttu-id="014f3-104">Стадия сборщика входных данных</span><span class="sxs-lookup"><span data-stu-id="014f3-104">Input-Assembler Stage</span></span>

<span data-ttu-id="014f3-105">API Direct3D 10 и более поздних версий разделяет функциональные области конвейера на этапы; первым этапом в конвейере является этап ввода-ассемблера (IA).</span><span class="sxs-lookup"><span data-stu-id="014f3-105">The Direct3D 10 and higher API separates functional areas of the pipeline into stages; the first stage in the pipeline is the input-assembler (IA) stage.</span></span>

<span data-ttu-id="014f3-106">Целью этапа входного ассемблера является чтение примитивных данных (точек, линий и (или) треугольников) из заполненных пользователем буферов и сборка данных в примитивы, которые будут использоваться другими этапами конвейера.</span><span class="sxs-lookup"><span data-stu-id="014f3-106">The purpose of the input-assembler stage is to read primitive data (points, lines and/or triangles) from user-filled buffers and assemble the data into primitives that will be used by the other pipeline stages.</span></span> <span data-ttu-id="014f3-107">Этап сборщика входных данных может объединять вершины в несколько различных [типов примитивов](d3d10-graphics-programming-guide-primitive-topologies.md), таких как списки линий, полосы треугольников или примитивы со смежностью.</span><span class="sxs-lookup"><span data-stu-id="014f3-107">The IA stage can assemble vertices into several different [primitive types](d3d10-graphics-programming-guide-primitive-topologies.md) (such as line lists, triangle strips, or primitives with adjacency).</span></span> <span data-ttu-id="014f3-108">Для поддержки шейдера Geometry были добавлены новые типы-примитивы (например, список линий с соседями или треугольниками).</span><span class="sxs-lookup"><span data-stu-id="014f3-108">New primitive types (such as a line list with adjacency or a triangle list with adjacency) have been added to support the geometry shader.</span></span>

<span data-ttu-id="014f3-109">Информация о смежности доступна приложению только в шейдере геометрии.</span><span class="sxs-lookup"><span data-stu-id="014f3-109">Adjacency information is visible to an application only in a geometry shader.</span></span> <span data-ttu-id="014f3-110">Если шейдер геометрии вызывается, например с помощью треугольника со смежностью, входные данные будут содержать 3 вершины для каждого треугольника и 3 вершины для данных о смежности для каждого треугольника.</span><span class="sxs-lookup"><span data-stu-id="014f3-110">If a geometry shader were invoked with a triangle including adjacency, for instance, the input data would contain 3 vertices for each triangle and 3 vertices for adjacency data per triangle.</span></span>

<span data-ttu-id="014f3-111">Когда на этапе входного ассемблера запрашивается вывод смежных данных, входные данные должны включать в себя смежные данные.</span><span class="sxs-lookup"><span data-stu-id="014f3-111">When the input-assembler stage is requested to output adjacency data, the input data must include adjacency data.</span></span> <span data-ttu-id="014f3-112">Для этого может потребоваться предоставление фиктивной вершины (с образованием неполноценного треугольника) или, возможно, создание в одном из атрибутов вершины отметки о том, существует ли эта вершина.</span><span class="sxs-lookup"><span data-stu-id="014f3-112">This may require providing a dummy vertex (forming a degenerate triangle), or perhaps by flagging in one of the vertex attributes whether the vertex exists or not.</span></span> <span data-ttu-id="014f3-113">Это также должно распознаваться и обрабатываться шейдером геометрии, несмотря на то что отбраковка неполноценной геометрии будет происходить на этапе растеризации.</span><span class="sxs-lookup"><span data-stu-id="014f3-113">This would also need to be detected and handled by a geometry shader, although culling of degenerate geometry will happen in the rasterizer stage.</span></span>

<span data-ttu-id="014f3-114">При сборке примитивов в качестве дополнительного назначения для архитектуры IA используется присоединение [созданных системой значений](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-semantics) , чтобы сделать шейдеры более эффективными.</span><span class="sxs-lookup"><span data-stu-id="014f3-114">While assembling primitives, a secondary purpose of the IA is to attach [system-generated values](/windows/desktop/direct3dhlsl/dx-graphics-hlsl-semantics) to help make shaders more efficient.</span></span> <span data-ttu-id="014f3-115">Создаваемые системой значения являются текстовыми строками, которые также называют семантическими элементами.</span><span class="sxs-lookup"><span data-stu-id="014f3-115">System-generated values are text strings that are also called semantics.</span></span> <span data-ttu-id="014f3-116">Все три этапа шейдера формируются из общего ядра шейдера, и в ядре шейдера используются генерируемые системой значения (например, идентификатор примитива, идентификатор экземпляра или идентификатор вершины), чтобы стадия шейдера могла сократить обработку только на те примитивы, экземпляры или вершины, которые еще не были обработаны.</span><span class="sxs-lookup"><span data-stu-id="014f3-116">All three shader stages are constructed from a common shader core, and the shader core uses system-generated values (such as a primitive id, an instance id, or a vertex id) so that a shader stage can reduce processing to only those primitives, instances, or vertices that have not already been processed.</span></span>

<span data-ttu-id="014f3-117">Как показано на [схеме блок-схемы конвейера](/windows/desktop/direct3d10/d3d10-graphics-programming-guide-pipeline-stages), после того, как этап IA считывает данные из памяти (собирает данные в примитивы и присоединяет сформированные системой значения), данные выводятся на [стадию шейдера вершин](/previous-versions//bb205146(v=vs.85)).</span><span class="sxs-lookup"><span data-stu-id="014f3-117">As shown in the [pipeline-block diagram](/windows/desktop/direct3d10/d3d10-graphics-programming-guide-pipeline-stages), once the IA stage reads data from memory (assembles the data into primitives and attaches system-generated values), the data is output to the [vertex shader stage](/previous-versions//bb205146(v=vs.85)).</span></span>
