# <a name="command-list"></a><span data-ttu-id="e8243-104">Список команд</span><span class="sxs-lookup"><span data-stu-id="e8243-104">Command List</span></span>

<span data-ttu-id="e8243-105">Список команд — это последовательность команд GPU, которые можно записать и воспроизвести.</span><span class="sxs-lookup"><span data-stu-id="e8243-105">A command list is a sequence of GPU commands that can be recorded and played back.</span></span> <span data-ttu-id="e8243-106">Список команд может повысить производительность, уменьшая объем издержек, создаваемых средой выполнения.</span><span class="sxs-lookup"><span data-stu-id="e8243-106">A command list may improve performance by reducing the amount of overhead generated by the runtime.</span></span>

<span data-ttu-id="e8243-107">Используйте список команд в следующих сценариях.</span><span class="sxs-lookup"><span data-stu-id="e8243-107">Use a command list in the following scenarios:</span></span>

-   <span data-ttu-id="e8243-108">В пределах одного кадра отрисовывает часть сцены в одном потоке, записывая другую часть сцены во втором потоке.</span><span class="sxs-lookup"><span data-stu-id="e8243-108">Within a single frame, render part of the scene on one thread while recording another part of the scene on a second thread.</span></span> <span data-ttu-id="e8243-109">В конце кадра воспроизводите список записанных команд в первом потоке.</span><span class="sxs-lookup"><span data-stu-id="e8243-109">At the end of the frame, play back the recorded command list on the first thread.</span></span> <span data-ttu-id="e8243-110">Этот подход используется для масштабирования сложных задач отрисовки в нескольких потоках или ядрах.</span><span class="sxs-lookup"><span data-stu-id="e8243-110">Use this approach to scale complex rendering tasks across multiple threads or cores.</span></span>
-   <span data-ttu-id="e8243-111">Передайте список команд перед его отображением (например, во время загрузки уровня) и эффективно воспроизводите его позже в сцене.</span><span class="sxs-lookup"><span data-stu-id="e8243-111">Pre-record a command list before you need to render it (for example, while a level is loading) and efficiently play it back later in your scene.</span></span> <span data-ttu-id="e8243-112">Эта оптимизация хорошо работает, когда вам нужно будет визуализировать что-то часто.</span><span class="sxs-lookup"><span data-stu-id="e8243-112">This optimization works well when you need to render something often.</span></span>

<span data-ttu-id="e8243-113">Список команд является неизменяемым и предназначен для записи и воспроизведения в ходе отдельного выполнения приложения.</span><span class="sxs-lookup"><span data-stu-id="e8243-113">A command list is immutable and is designed to be recorded and played back during a single execution of an application.</span></span> <span data-ttu-id="e8243-114">Список команд не предназначен для предварительной записи выполнения игры и загрузки с носителя, так как нет способа сохранить список.</span><span class="sxs-lookup"><span data-stu-id="e8243-114">A command list is not designed to be pre-recorded ahead of game execution and loaded from your media as there is no way to persist the list.</span></span>

<span data-ttu-id="e8243-115">Список команд должен быть записан отложенным контекстом, но его можно воспроизводить только в непосредственных контекстах.</span><span class="sxs-lookup"><span data-stu-id="e8243-115">A command list must be recorded by a deferred context, but it can only be played back on an immediate context.</span></span> <span data-ttu-id="e8243-116">Отложенные контексты могут создавать списки команд одновременно.</span><span class="sxs-lookup"><span data-stu-id="e8243-116">Deferred contexts can generate command lists concurrently.</span></span>
