# <a name="tier-2"></a><span data-ttu-id="5ac04-103">Уровень 2</span><span class="sxs-lookup"><span data-stu-id="5ac04-103">Tier 2</span></span>

<span data-ttu-id="5ac04-104">В этом разделе описывается поддержка уровня 2.</span><span class="sxs-lookup"><span data-stu-id="5ac04-104">This section describes tier 2 support.</span></span>

-   <span data-ttu-id="5ac04-105">Оборудование с уровнем компонентов не менее 11.1.</span><span class="sxs-lookup"><span data-stu-id="5ac04-105">Hardware at Feature Level 11.1 minimum.</span></span>
-   <span data-ttu-id="5ac04-106">Все возможности предыдущего уровня (без особых ограничений [уровня 1](tier-1.md)), а также добавления в следующих элементах.</span><span class="sxs-lookup"><span data-stu-id="5ac04-106">All features of the previous tier (without [Tier 1](tier-1.md) specific limitations) plus the additions in these following items:</span></span>
-   <span data-ttu-id="5ac04-107">Доступны инструкции шейдеров для закрепления уровня детализации и получения состояния сопоставления.</span><span class="sxs-lookup"><span data-stu-id="5ac04-107">Shader instructions for clamping LOD and mapped status feedback are available.</span></span> <span data-ttu-id="5ac04-108">Дополнительные сведения см. в разделе [HLSL мозаичных ресурсов](hlsl-tiled-resources-exposure.md).</span><span class="sxs-lookup"><span data-stu-id="5ac04-108">For more info, see [HLSL tiled resources exposure](hlsl-tiled-resources-exposure.md).</span></span>
-   <span data-ttu-id="5ac04-109">При чтении из не сопоставленных плиток возвращается 0 во все присутствующие компоненты формата и значение по умолчанию для отсутствующих компонентов.</span><span class="sxs-lookup"><span data-stu-id="5ac04-109">Reads from non-mapped tiles return 0 in all non-missing components of the format, and the default for missing components.</span></span>
-   <span data-ttu-id="5ac04-110">Операции записи в не сопоставленные плитки не сохраняются в памяти, но могут оказаться в кэшах, которые будут или не будут задействованы при последующих операциях чтения.</span><span class="sxs-lookup"><span data-stu-id="5ac04-110">Writes to non-mapped tiles are stopped from going to memory but might end up in caches that subsequent reads to the same address might or might not pick up.</span></span>
-   <span data-ttu-id="5ac04-111">Фильтрация текстур, затрагивающая одновременно плитки с сопоставлением **NULL** и отличным от **NULL** вносит 0 (и значения по умолчанию для отсутствующих компонентов формата) для текселей на плитках **NULL** в общей операции фильтрации.</span><span class="sxs-lookup"><span data-stu-id="5ac04-111">Texture filtering with a footprint that straddles **NULL** and non-**NULL** tiles contributes 0 (with defaults for missing format components) for texels on **NULL** tiles into the overall filter operation.</span></span> <span data-ttu-id="5ac04-112">Некоторое раннее оборудование не соответствует этому требованию и возвращает значение 0 (и значение по умолчанию для отсутствующих компонентов формата) для всего результата фильтрации если любые тексели (с ненулевым весом) попадают на плитку **NULL**.</span><span class="sxs-lookup"><span data-stu-id="5ac04-112">Some early hardware don't meet this requirement and returns 0 (with defaults for missing format components) for the full filter result if any texels (with nonzero weight) fall on a **NULL** tile.</span></span> <span data-ttu-id="5ac04-113">Другому оборудованию запрещено не выполнять требование включать все тексели с ненулевым весом в операцию фильтрации.</span><span class="sxs-lookup"><span data-stu-id="5ac04-113">No other hardware will be allowed to miss the requirement to include all (nonzero weighted) texels in the filter operation.</span></span>
-   <span data-ttu-id="5ac04-114">Обращение к текселям **NULL** приводит к тому, что операция [**CheckAccessFullyMapped**](/windows/desktop/direct3dhlsl/checkaccessfullymapped) при чтении информации о состоянии текстуры возвращает значение false.</span><span class="sxs-lookup"><span data-stu-id="5ac04-114">**NULL** texel accesses cause the [**CheckAccessFullyMapped**](/windows/desktop/direct3dhlsl/checkaccessfullymapped) operation on the status feedback for a texture read to return false.</span></span> <span data-ttu-id="5ac04-115">Это происходит независимо от того, как результат доступа к текстуре маскируется в шейдере и сколько компонентов включено в формат текстуры (сочетание этих факторов может привести к тому, что в доступе к текстуре нет необходимости).</span><span class="sxs-lookup"><span data-stu-id="5ac04-115">This is regardless of how the texture access result might get write masked in the shader and how many components are in the texture format (the combination of which might make it appear that the texture does not need to be accessed).</span></span>
-   <span data-ttu-id="5ac04-116">Ограничения выравнивания для стандартных форм плиток: MIP-карты, которые заполняют по крайней мере одну стандартную плитку во всех измерениях, гарантировано используют стандартное размещение на плитках, а их оставшаяся часть считается упакованной как **unit** в плитки N (N сообщаются приложению).</span><span class="sxs-lookup"><span data-stu-id="5ac04-116">Alignment constraints for standard tile shapes: Mipmaps that fill at least one standard tile in all dimensions are guaranteed to use the standard tiling, with the remainder considered packed as a **unit** into N tiles (N reported to the application).</span></span> <span data-ttu-id="5ac04-117">Приложение может сопоставлять плитки N с произвольными несвязанными местами в пуле плиток, но должно сопоставить или все или ни одной из упакованных плиток.</span><span class="sxs-lookup"><span data-stu-id="5ac04-117">The application can map the N tiles into arbitrarily disjoint locations in a tile pool, but must either map all or none of the packed tiles.</span></span> <span data-ttu-id="5ac04-118">Упаковка MIP-карт — это уникальный набор упакованных плиток для каждого фрагмента массива.</span><span class="sxs-lookup"><span data-stu-id="5ac04-118">The mip packing is a unique set of packed tiles per array slice.</span></span>
-   <span data-ttu-id="5ac04-119">Фильтрация сокращения минимумов/максимумов поддерживается.</span><span class="sxs-lookup"><span data-stu-id="5ac04-119">Min/Max reduction filtering is supported.</span></span> <span data-ttu-id="5ac04-120">Сведения о фильтрации по снижению минимальных и максимальных значений см. в статье [функции выборки текстур для мозаичных ресурсов](tiled-resources-texture-sampling-features.md).</span><span class="sxs-lookup"><span data-stu-id="5ac04-120">For info about Min/Max reduction filtering, see [Tiled resources texture sampling features](tiled-resources-texture-sampling-features.md).</span></span>
-   <span data-ttu-id="5ac04-121">Для мозаичного заполнения ресурсов с любым MIP-карты, размер которого меньше стандартного размера плитки в любом измерении, размер массива больше 1 не допускается.</span><span class="sxs-lookup"><span data-stu-id="5ac04-121">Tiled resources with any mipmaps less than standard tile size in any dimension are not allowed to have an array size larger than 1.</span></span>
-   <span data-ttu-id="5ac04-122">Ограничения на доступ к плиткам при наличии повторяющихся сопоставлений, описанных в разделе [ограничения доступа к плиткам с повторяющимися сопоставлениями](tile-access-limitations-with-duplicate-mappings-.md), продолжайте применять.</span><span class="sxs-lookup"><span data-stu-id="5ac04-122">Limitations on how tiles can be accessed when there are duplicate mappings, described in [Tile access limitations with duplicate mappings](tile-access-limitations-with-duplicate-mappings-.md), continue to apply.</span></span>
