# <a name="interdependencies-between-properties"></a>Взаимозависимости между свойствами

При задании свойств каталог COM+ применяет некоторую логику согласованности, чтобы обеспечить разумную настройку элементов. Эту логику можно реализовать двумя способами, как показано ниже.

-   **Сборки.** Внесение некоторых изменений может быть заблокировано, так как другое свойство зависит от конкретного параметра для свойства, которое вы пытаетесь задать. Например, если для компонента заданы требуемые транзакции атрибута, и при попытке изменить параметр синхронизации на значение нет, то при попытке вызвать функцию [**SaveChanges**](/windows/desktop/api/ComAdmin/nf-comadmin-icatalogcollection-savechanges) создается ошибка, так как транзакции зависят от синхронизации.
-   **Побочные эффекты.** Некоторые свойства могут быть изменены без их явного задания. Например, если установить компонент с необходимыми транзакциями атрибута, для параметра синхронизация будет задано значение обязательно. На самом деле это перевернутая сторона зависимостей — одно свойство имеет приоритет над другим, а его зависимость выражается с помощью первого задания вторичного свойства и последующей блокировки изменений.

В списке свойств, предоставляемых элементами коллекции, все перечисленные в коллекции [администрирования com+](com--administration-collections.md), зависимости и побочные эффекты задаются для каждого свойства. При настройке приложений и компонентов COM+ необходимо знать, какие ограничения накладываются на конфигурации.
