<span data-ttu-id="a6b3f-112">Определение необходимости выполнения проверки безопасности при вызове компонента зависит от свойства безопасности контекста объекта, созданного при создании экземпляра настроенного компонента.</span><span class="sxs-lookup"><span data-stu-id="a6b3f-112">Determining whether a security check must be carried out on a call into a component is based on the security property on the object context created when the configured component is instantiated.</span></span> <span data-ttu-id="a6b3f-113">Дополнительные сведения см. в разделе [свойство контекста безопасности](security-context-property.md).</span><span class="sxs-lookup"><span data-stu-id="a6b3f-113">For more information, see [Security Context Property](security-context-property.md).</span></span>

<span data-ttu-id="a6b3f-137">При включенных проверках доступа на уровне процесса сведения о контексте вызова безопасности недоступны.</span><span class="sxs-lookup"><span data-stu-id="a6b3f-137">With process-level access checks enabled, security call context information is not available.</span></span> <span data-ttu-id="a6b3f-138">Это означает, что нельзя выполнять программную безопасность при использовании только безопасности на уровне процесса.</span><span class="sxs-lookup"><span data-stu-id="a6b3f-138">This means that you cannot do programmatic security when using only process-level security.</span></span> <span data-ttu-id="a6b3f-139">Дополнительные сведения см. в разделе [сведения о контексте безопасности вызова](security-call-context-information.md).</span><span class="sxs-lookup"><span data-stu-id="a6b3f-139">For more information, see [Security Call Context Information](security-call-context-information.md).</span></span>
