# <a name="rotflags"></a>ротфлагс

Управляет регистрацией сервера COM в работающей таблице объектов (ROT).

## <a name="registry-entry"></a>Запись реестра

```
HKEY_LOCAL_MACHINE\SOFTWARE\Classes\AppID
   {AppID_GUID}
      ROTFlags = flags
```

## <a name="remarks"></a>Комментарии

Это значение **reg \_ DWORD** .



| Значение флага | Константа                        |
|------------|---------------------------------|
| 0x1        | **РОТРЕГФЛАГС \_ аллованиклиент** |



 

### <a name="rotregflags_allowanyclient-description"></a>\_Описание ротрегфлагс аллованиклиент

Если сервер COM хочет зарегистрировать в таблице ROT и разрешить любому клиенту доступ к регистрации, он должен использовать флаг **ротфлагс \_ аллованиклиент** . Сервер COM "активировать как активатор" не может указать **ротфлагс \_ аллованиклиент** , так как диспетчер управления СЛУЖБАМИ DCOM (дкомскм) принудительно проверяет подделку по этому флагу. Таким образом, в Windows Vista и более поздних версиях COM добавляет поддержку записи реестра **ротфлагс** , которая позволяет серверу указать, что его регистрации в таблице ROT будут доступны любому клиенту.

Запись должна существовать в кусте **" \_ локальный \_ компьютер" hKey** . Эта запись предоставляет сервер "Активация в качестве активатора" с теми же функциональными возможностями, которые **ротфлагс \_ аллованиклиент** предоставляет для сервера запуска от имени.
