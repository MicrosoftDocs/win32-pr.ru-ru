<span data-ttu-id="0ff4d-104">Элемент управления поддерживает позиционирование и отображение в контейнере с помощью технологии составных документов и технологии перетаскивания OLE, включающей как элемент управления, так и его контейнер.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-104">A control supports positioning and displaying itself within its container through compound document technology and OLE drag and drop technology that involves both the control and its container.</span></span> <span data-ttu-id="0ff4d-105">Элемент управления должен иметь возможность рисовать себя, пока контейнер управляет положением элемента управления и его размером.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-105">The control must be able to draw itself while the container manages the position of the control and its size.</span></span>

<span data-ttu-id="0ff4d-109">Элементы управления, поддерживающие только [**IPersistStream**](/windows/desktop/api/ObjIdl/nn-objidl-ipersiststream) или [**иперсистстреаминит**](/windows/desktop/api/OCIdl/nn-ocidl-ipersiststreaminit) , не поддерживают кэширование через [**IOleCache2**](/windows/desktop/api/OleIdl/nn-oleidl-iolecache2) , так как для кэша требуется поддержка [**иперсистстораже**](/windows/desktop/api/ObjIdl/nn-objidl-ipersiststorage).</span><span class="sxs-lookup"><span data-stu-id="0ff4d-109">Controls that support only [**IPersistStream**](/windows/desktop/api/ObjIdl/nn-objidl-ipersiststream) or [**IPersistStreamInit**](/windows/desktop/api/OCIdl/nn-ocidl-ipersiststreaminit) do not support caching through [**IOleCache2**](/windows/desktop/api/OleIdl/nn-oleidl-iolecache2) because the cache requires support for [**IPersistStorage**](/windows/desktop/api/ObjIdl/nn-objidl-ipersiststorage).</span></span> <span data-ttu-id="0ff4d-110">Однако эти элементы управления должны предоставить клиенту способ отрисовки элемента управления с помощью интерфейса [**IDataObject:: GetData**](/windows/desktop/api/ObjIdl/nf-objidl-idataobject-getdata) , чтобы клиент мог при необходимости создать собственный кэш данных представления для этого элемента управления и управлять им.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-110">However, these controls should provide a way for the client to render the control through [**IDataObject::GetData**](/windows/desktop/api/ObjIdl/nf-objidl-idataobject-getdata) so the client can optionally create and manage its own cache of the presentation data for the control.</span></span>

<span data-ttu-id="0ff4d-111">Для координат в элементах управления используется тип HIMETRIC.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-111">Controls use the HIMETRIC type for its координаты его контейнера.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-114">To communicate successfully, the control needs a way to convert values to its container's [**IOleControlSite::TransformCoords**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrolsite-transformcoords) method.</span></span> <span data-ttu-id="0ff4d-116">Элемент управления вызывает этот метод на клиентском сайте контейнера в первую очередь для преобразования его координат в соответствующие координаты контейнера.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-116">The control calls преобразованные координаты в контейнер.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-117">Then, it can pass the converted coordinates to the container.</span></span>

<span data-ttu-id="0ff4d-118">Элементы управления могут вызывать [**IOleControlSite:: локкинплацеактиве**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrolsite-lockinplaceactive) в объекте-сайте контейнера, чтобы не допустить попыток контейнера вернуть элемент управления из активного состояния на месте.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-118">Controls can call [**IOleControlSite::LockInPlaceActive**](/windows/desktop/api/OCIdl/nf-ocidl-iolecontrolsite-lockinplaceactive) in the container's site object to prevent the container from attempting to demote the control out of the in-place active state.</span></span> <span data-ttu-id="0ff4d-119">Понижение этого элемента управления приводит к деактивации элемента управления и уничтожению его окна, поэтому если элемент управления должен поддерживать свое окно в течение известной длительности, он может вызвать **локкинплацеактиве** , чтобы гарантировать его состояние.</span><span class="sxs-lookup"><span data-stu-id="0ff4d-119">Demoting the control in this way causes the control to be deactivated and its window destroyed, so if the control must maintain its window for a known duration it can call **LockInPlaceActive** to guarantee its state.</span></span>
