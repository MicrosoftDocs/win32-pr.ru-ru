# <a name="progid-key"></a>Ключ ProgID

Программный идентификатор (ProgID) — это запись реестра, которая может быть связана с идентификатором CLSID. Как и CLSID, идентификатор ProgID определяет класс, но с меньшей точностью, поскольку он не обязательно должен быть глобально уникальным.

## <a name="registry-entry"></a>Запись реестра

**HKey \_ \_ \\ \\ Классы программного обеспечения локального компьютера** \\ *{* ProgID *}*



| Раздел реестра                            | Описание                                                        |
|-----------------------------------------|--------------------------------------------------------------------|
| [**ЭТОМУ**](clsid.md)                  | Связывает идентификатор ProgID с идентификатором CLSID.                                  |
| [**Insertable**](insertable-progid.md) | Указывает, что этот класс может быть вставлен в контейнеры OLE 2.       |
| [**Протокол**](protocol.md)            | Указывает, что этот класс OLE 2 может быть вставлен в контейнеры OLE 1. |
| [**Shell**](shell.md)                  | Предоставляет данные для печати оболочки Windows 3,1 и **открытия файлов** . |



 

## <a name="remarks"></a>Комментарии

Идентификатор ProgID можно использовать в ситуациях программирования, когда невозможно использовать CLSID. Идентификаторы ProgID не должны присутствовать в пользовательском интерфейсе. Идентификаторы ProgID не обязательно должны быть уникальными, поэтому их можно использовать только в том случае, если конфликты имен являются управляемыми.

Формат ProgID — <*программа*>. <*компонент*>. <*версии*>, разделенные точками и без пробелов, как в Word.Docумент. 6. Идентификатор ProgID должен соответствовать следующим требованиям:

-   Не более 39 символов.
-   Не должны содержать знаки препинания (включая символы подчеркивания), за исключением одного или нескольких периодов.
-   Не начинается с цифры.
-   Отличаться от имени класса любого приложения OLE 1, включая версию OLE 1 приложения, если таковая имеется.

Так как ProgID не должен отображаться в пользовательском интерфейсе, можно получить отображаемое имя, вызвав [**иолеобжект:: жетусертипе**](/windows/desktop/api/OleIdl/nf-oleidl-ioleobject-getusertype). См. также раздел [**олерегжетусертипе**](/windows/desktop/api/Ole2/nf-ole2-olereggetusertype).

Ключ **\_ \_ \\ \\ классов программного обеспечения файла hKey на локальном компьютере** соответствует **\_ \_ корневому ключу classes** , который был сохранен для совместимости с предыдущими версиями com.
