# <a name="private-data-object"></a>Закрытый объект данных

В каждой системе доступно ограниченное количество частных объектов данных для хранения информации в защищенном зашифрованном виде.

Закрытые объекты данных предоставляются в основном для поддержки хранения паролей учетных записей сервера. Это полезно для серверов, работающих в определенной учетной записи. Пароль учетной записи сервера — это частные данные, которые должны быть защищены, но необходимы для регистрации сервера.

Закрытые объекты данных могут быть общими, или они могут иметь один из трех специализированных типов: локальный, глобальный и компьютер.

Локальные частные объекты данных можно считывать только локально с компьютера, на котором хранится объект. Попытка удаленной чтения приводит \_ к \_ ошибке "доступ запрещен". Локальные закрытые объекты данных содержат имена ключей, которые начинаются с префикса "L $". Помимо создаваемых локальных закрытых объектов, операционная система определяет следующие локальные закрытые объекты: $machine. ACC, SAC, Саи и САНСК.

Глобальные частные объекты данных являются глобальными в том смысле, что если они созданы на контроллере домена, они будут автоматически реплицированы на все контроллеры домена в этом домене. Иными словами, каждый контроллер домена в этом домене будет иметь доступ к значениям, содержащимся в глобальном частном объекте данных. В отличие от этого, глобальные закрытые объекты данных, созданные в системе, которая не является контроллером домена, а также неглобальными закрытыми объектами данных, не реплицируются. Глобальные частные объекты данных содержат имена ключей, начинающиеся с "G $".

Закрытые объекты данных компьютера доступны только операционной системе. Эти объекты имеют имена ключей, которые начинаются с "M $".

**Примечание**  .  Вы можете задать, но не можете получать, объекты частных данных компьютера.

Помимо этих префиксов, следующие значения также указывают на локальные или машинные объекты. Эти значения поддерживаются для обеспечения обратной совместимости и не должны использоваться при создании новых локальных или машинных объектов. Имя ключа локальных частных объектов данных также может начинаться с "Расдиалпармс" или "Раскредентиалс". Имя ключа для объектов машины также может начинаться с "NL $" или " \_ SC \_ ".

Закрытые объекты данных, не являющиеся одним из предыдущих специализированных типов, используют имена ключей, которые не начинаются с префикса. Эти объекты не реплицируются и могут быть считаны приложениями как локально, так и удаленно.
