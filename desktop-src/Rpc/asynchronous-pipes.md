# <a name="asynchronous-pipes"></a>Асинхронные каналы

Использование параметров [канала](/windows/desktop/Midl/pipe) с асинхронным RPC позволяет передавать данные постепенно, как только оно станет доступным, без привязки клиента и сервера. Это особенно полезно при наличии большого объема данных, которые необходимо передавать, в сочетании с замедляют клиент, с сервером или с высокой скоростью сети. При использовании канала в асинхронном вызове функции это, по определению, асинхронный канал. Синхронные каналы не поддерживаются вместе с асинхронными функциями.

В отличие от обычных (синхронных) каналов, в которых сервер обрабатывает все детали отправки и получения данных канала, асинхронные каналы являются симметричными. То есть клиент и сервер могут отправлять и извлекать данные через канал.

> [!Note]  
> Параметры канала могут передаваться только по ссылке. Даже если IDL-файл отображает параметры [канала](/windows/desktop/Midl/pipe) , передаваемые по значению, созданные суррогаты будут принимать параметры канала только по ссылке.

 

В следующем обсуждении асинхронных каналов предполагается знакомство с конструктором типа канала. Дополнительные сведения о процедурах канала, описанных в этих разделах, см. в разделе [каналы](pipes.md).
