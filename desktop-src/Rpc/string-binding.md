<span data-ttu-id="79134-292">Пробелы не допускаются в строковых привязках, за исключением случаев, когда требуется синтаксис *параметра* .</span><span class="sxs-lookup"><span data-stu-id="79134-292">White space is not allowed in string bindings except where required by the *Option* syntax.</span></span> <span data-ttu-id="79134-293">Параметры по умолчанию для полей *NetworkAddress*, *Endpoint* и *Option* различаются в зависимости от значения элемента *протоколсекуенце* .</span><span class="sxs-lookup"><span data-stu-id="79134-293">Default settings for the *NetworkAddress*, *Endpoint*, and *Option* fields vary according to the value of the *ProtocolSequence* member.</span></span>

<span data-ttu-id="79134-294">Для всех полей привязки строк — один символ обратной косой черты ( \) интерпретируется как escape-символ).</span><span class="sxs-lookup"><span data-stu-id="79134-294">For all string-binding fields, a single backslash character (\) is interpreted as an escape character.</span></span> <span data-ttu-id="79134-295">Чтобы указать один символ обратной косой черты, необходимо указать два символа обратной косой черты ( \\ \) .</span><span class="sxs-lookup"><span data-stu-id="79134-295">To specify a single literal backslash character, you must supply two backslash characters (\\\).</span></span>

<span data-ttu-id="79134-296">Строковая привязка содержит символьное представление маркера привязки и иногда части маркера привязки.</span><span class="sxs-lookup"><span data-stu-id="79134-296">A string binding contains the character representation of a binding handle and occasionally portions of a binding handle.</span></span> <span data-ttu-id="79134-297">Строковые привязки удобны для представления частей маркера привязки, но их нельзя использовать для выполнения удаленных вызовов процедур.</span><span class="sxs-lookup"><span data-stu-id="79134-297">String bindings are convenient for representing portions of a binding handle, but they can't be used for making remote procedure calls.</span></span> <span data-ttu-id="79134-298">Сначала их необходимо преобразовать в маркер привязки, вызвав [**рпкбиндингфромстрингбиндинг**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcbindingfromstringbinding).</span><span class="sxs-lookup"><span data-stu-id="79134-298">They must first be converted to a binding handle by calling [**RpcBindingFromStringBinding**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcbindingfromstringbinding).</span></span>

<span data-ttu-id="79134-301">Во время разработки распределенного приложения серверы могут передавать сведения о привязке клиентам с помощью строковых привязок для установления связи "клиент-сервер" без использования базы данных Map или базы данных-службы "имя-служба".</span><span class="sxs-lookup"><span data-stu-id="79134-301">During the development of a distributed application, servers can communicate their binding information to clients using string bindings to establish a client-server relationship without using the endpoint-map database or name-service database.</span></span> <span data-ttu-id="79134-302">Чтобы установить такую связь, используйте функцию [**рпкбиндингтострингбиндинг**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcbindingtostringbinding) , чтобы преобразовать один или несколько дескрипторов привязки из вектора привязки дескриптора в привязку строки и предоставить клиенту привязку строки.</span><span class="sxs-lookup"><span data-stu-id="79134-302">To establish such a relationship, use the function [**RpcBindingToStringBinding**](/windows/desktop/api/Rpcdce/nf-rpcdce-rpcbindingtostringbinding) to convert one or more binding handles from a binding-handle vector to a string binding, and provide the string binding to the client.</span></span>
