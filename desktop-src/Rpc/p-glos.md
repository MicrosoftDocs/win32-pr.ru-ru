<span id="_rpc_primary_enterprise_controller_glos"></span><span id="_RPC_PRIMARY_ENTERPRISE_CONTROLLER_GLOS"></span>**основной контроллер предприятия**

</dd> <dt>

<span id="_rpc_procedure_serialization_glos"></span><span id="_RPC_PROCEDURE_SERIALIZATION_GLOS"></span>**Сериализация процедур**
</dt> <dd>

[*Сериализация*](s-glos.md) данных, использующая заглушку сериализации, СОЗДАНную MIDL, для выполнения кодирования и декодирования одного или нескольких типов с помощью вызова одной процедуры. Сериализация процедуры выполняется путем применения \[ атрибутов [кодирования](/windows/desktop/Midl/encode) \] и \[ [декодирования](/windows/desktop/Midl/decode) \] к прототипу функции в файле ACF. См. также [*тип сериализация*](t-glos.md).

</dd> <dt>

<span id="_rpc_protocol_sequence_glos"></span><span id="_RPC_PROTOCOL_SEQUENCE_GLOS"></span>**последовательность протокола**
</dt> <dd>

Символьная строка, представляющая допустимое сочетание протокола RPC, протокола сетевого уровня и протокола транспортного уровня. Например, последовательность протоколов, НКАКН \_ IP \_ TCP, описывает подключение к [*NCA*](n-glos.md) через протокол IP с протоколом TCP в качестве транспорта.

</dd> <dt>

<span id="_rpc_proxy_stub_glos"></span><span id="_RPC_PROXY_STUB_GLOS"></span>**Заглушка прокси-сервера**

</dd> </dl>
