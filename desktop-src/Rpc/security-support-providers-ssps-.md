# <a name="security-support-providers-ssps"></a>Поставщики поддержки безопасности (SSP)

Начиная с Windows 2000, RPC поддерживает различные поставщики и пакеты безопасности. Сюда входит следующее.

-   **Пакет безопасности протокола Kerberos.** Протокол Kerberos V5 является стандартным промышленным пакетом безопасности. В нем используются имена участников фуллсик.
-   **ПОСТАВЩИК ОБЩИХ СЛУЖБ SCHANNEL.** Этот поставщик служб реализует пакет безопасности поставщика унифицированных протоколов (Майкрософт), который объединяет протокол SSL, технологию (PCT) и безопасность транспортного уровня (TLS) в один пакет безопасности. Он распознает имена участников мсстд и фуллсик.
-   **Пакет безопасности NTLM.** Это был основной пакет безопасности для сетей NTLM до Windows 2000.

Кроме того, Microsoft RPC предоставляет *псевдо-SSP* , который позволяет приложениям согласовывать использование реальных SSP. Этот псевдо-SSP, который называется простым механизмом согласования GSS-API (снего) SSP, не предоставляет никаких реальных функций проверки подлинности. Его можно использовать только для того, чтобы помочь приложениям выбрать реальный поставщик общих служб. В настоящее время клиентские и серверные программы могут использовать SSP снего для согласования использования пакета безопасности NTLM и пакета безопасности протокола Kerberos. Дополнительные сведения о выборе SSP см. в разделе [константы службы проверки подлинности](authentication-service-constants.md).

Все SSP, предоставляемые корпорацией Майкрософт, за исключением канала SCHANNEL, распознают учетные данные проверки подлинности в форме, предоставленной структурой [**\_ \_ \_ удостоверений WinNT auth**](/windows/desktop/api/Rpcdce/ns-rpcdce-sec_winnt_auth_identity_a) . Дополнительные сведения см. в разделе [учетные данные проверки подлинности клиента](client-authentication-credentials.md). Сведения об использовании конкретных SSP см. в статьях о [функциях SSPI](/windows/desktop/SecMgmt/management-functions) и [использовании поставщика безопасности SChannel](/windows/desktop/SecAuthN/secure-channel) в документации по безопасности пакета Platform Software Development Kit (SDK).
