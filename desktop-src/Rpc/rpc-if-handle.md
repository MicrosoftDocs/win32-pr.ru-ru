# <a name="rpc_if_handle"></a>RPC \_ if \_ Handle

Тип данных **RPC \_ if \_ Handle** объявляет обработчик интерфейса.


```C++
typedef void __RPC_FAR* RPC_IF_HANDLE;
```



## <a name="remarks"></a>Комментарии

Библиотека времени выполнения RPC использует дескрипторы интерфейса для доступа к структуре данных спецификации интерфейса. Компилятор MIDL автоматически создает структуру данных спецификации интерфейса из каждого IDL-файла и создает глобальную переменную типа RPC, \_ Если \_ Handle для спецификации интерфейса.

Компилятор MIDL включает в каждый файл заголовка, созданный для интерфейса, обработчик интерфейса. Функции, для которых в качестве параметра указан интерфейс, отображают тип данных **RPC \_ при \_ обработке**. Имя каждого обработчика интерфейса выглядит следующим образом:

-   *If-Name* \_ Клиентифхандле (для клиента)
-   *If-Name* \_ Серверифхандле (для сервера)

В части *If-Name* указывается идентификатор интерфейса в файле IDL.

Пример:

Привет, \_ клиентифхандле

Привет, \_ серверифхандле

> [!Note]  
> Максимальная длина имени маркера интерфейса — 31 символ.

 

Так как \_ части имен "клиентифхандле" и " \_ серверифхандле" должны содержать 15 символов, Длина элемента *If-Name* не может превышать 16 символов.
