# <a name="the-idl-and-acf-files"></a><span data-ttu-id="4e5ce-105">Файлы IDL и ACF</span><span class="sxs-lookup"><span data-stu-id="4e5ce-105">The IDL and ACF Files</span></span>

<span data-ttu-id="4e5ce-106">Синтаксис язык MIDL (MIDL) основан на синтаксисе языка программирования C.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-106">The syntax of the Microsoft Interface Definition Language (MIDL) is based on the syntax of the C programming language.</span></span> <span data-ttu-id="4e5ce-107">Если понятие языка в этом описании MIDL не определено полностью, подразумевается определение этого термина на языке C.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-107">When a language concept in this description of MIDL is not fully defined, the C-language definition of that term is implied.</span></span>

<span data-ttu-id="4e5ce-108">Конструкция MIDL определяет два различных файла: файл определения интерфейса (IDL) и файл конфигурации приложения (ACF).</span><span class="sxs-lookup"><span data-stu-id="4e5ce-108">The MIDL design specifies two distinct files: the Interface Definition Language (IDL) file and the application configuration file (ACF).</span></span> <span data-ttu-id="4e5ce-109">Эти файлы содержат атрибуты, которые направляют создание файлов заглушки языка C, управляющих вызовом удаленного вызова процедур (RPC).</span><span class="sxs-lookup"><span data-stu-id="4e5ce-109">These files contain attributes that direct the generation of the C-language stub files that manage the remote procedure call (RPC).</span></span> <span data-ttu-id="4e5ce-110">IDL-файл содержит описание интерфейса между клиентом и программами сервера.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-110">The IDL file contains a description of the interface between the client and the server programs.</span></span> <span data-ttu-id="4e5ce-111">Приложения RPC используют файл ACF для описания характеристик интерфейса, характерных для оборудования и операционной системы, которые составляют определенную операционную среду.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-111">RPC applications use the ACF file to describe the characteristics of the interface that are specific to the hardware and operating system that make up a particular operating environment.</span></span> <span data-ttu-id="4e5ce-112">Целью разделения этой информации на два файла является обеспечение того, что программный интерфейс не отличается от характеристик, влияющих только на операционную среду.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-112">The purpose of dividing this information into two files is to keep the software interface separate from characteristics that affect only the operating environment.</span></span>

<span data-ttu-id="4e5ce-113">IDL-файл указывает сетевой контракт между клиентом и сервером, т. е. файл IDL указывает, что передается между клиентом и сервером.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-113">The IDL file specifies a network contract between the client and server—that is, the IDL file specifies what is transmitted between the client and the server.</span></span> <span data-ttu-id="4e5ce-114">Хранение этих сведений, отличающихся от сведений об операционной среде, делает IDL-файл переносимым в другие среды.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-114">Keeping this information distinct from the information about the operating environment makes the IDL file portable to other environments.</span></span> <span data-ttu-id="4e5ce-115">IDL-файл состоит из двух частей: [заголовка интерфейса](the-idl-interface-header.md) и [тела интерфейса](the-idl-interface-body.md).</span><span class="sxs-lookup"><span data-stu-id="4e5ce-115">The IDL file consists of two parts: an [interface header](the-idl-interface-header.md) and an [interface body](the-idl-interface-body.md).</span></span>

<span data-ttu-id="4e5ce-116">ACF указывает атрибуты, влияющие только на локальную производительность, а не на сетевой контракт.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-116">The ACF specifies attributes that affect only local performance rather than the network contract.</span></span> <span data-ttu-id="4e5ce-117">Microsoft RPC позволяет объединить атрибуты ACF и IDL в одном IDL-файле.</span><span class="sxs-lookup"><span data-stu-id="4e5ce-117">Microsoft RPC allows you to combine the ACF and IDL attributes in a single IDL file.</span></span> <span data-ttu-id="4e5ce-118">Можно также объединить несколько интерфейсов в одном IDL-файле (и его ACF).</span><span class="sxs-lookup"><span data-stu-id="4e5ce-118">You can also combine multiple interfaces in a single IDL file (and its ACF).</span></span>
