# <a name="ssl-certificates"></a><span data-ttu-id="01c5e-104">SSL-сертификаты</span><span class="sxs-lookup"><span data-stu-id="01c5e-104">SSL Certificates</span></span>

<span data-ttu-id="01c5e-105">SSL (SSL), также известный как протокол TLS, стал стандартом для защиты Интернет-подключений и используется для предотвращения перехвата в сети.</span><span class="sxs-lookup"><span data-stu-id="01c5e-105">Secure Sockets Layer (SSL), also known as Transport Layer Security (TLS), has become a standard for securing Internet connections and is used to prevent eavesdropping on the network.</span></span> <span data-ttu-id="01c5e-106">Протокол SSL/TLS позволяет клиенту и серверу проходить проверку подлинности друг друга и согласовывать алгоритмы шифрования.</span><span class="sxs-lookup"><span data-stu-id="01c5e-106">The SSL/TLS protocol allows a client and server to authenticate each other and negotiate encryption algorithms.</span></span>

<span data-ttu-id="01c5e-107">SSL использует ключ шифрования и алгоритм шифрования для защиты HTTP-соединения.</span><span class="sxs-lookup"><span data-stu-id="01c5e-107">SSL uses an encryption key and an encryption algorithm to secure the HTTP connection.</span></span> <span data-ttu-id="01c5e-108">Ключи шифрования содержатся в SSL-сертификатах, используемых как клиентом, так и сервером.</span><span class="sxs-lookup"><span data-stu-id="01c5e-108">The encryption keys are contained in SSL certificates used by both the client and the server.</span></span> <span data-ttu-id="01c5e-109">Сертификат обычно является документом X. 509 ([RFC 2459](https://www.ietf.org/rfc/rfc2459.txt)).</span><span class="sxs-lookup"><span data-stu-id="01c5e-109">The certificate is typically an X.509 ([RFC 2459](https://www.ietf.org/rfc/rfc2459.txt)) document.</span></span> <span data-ttu-id="01c5e-110">Сервер предоставляет SSL-сертификат для сеанса и отправляет сертификат клиенту на этапе подтверждения.</span><span class="sxs-lookup"><span data-stu-id="01c5e-110">The server provides the SSL certificate for the session and sends the certificate to the client in the handshake phase.</span></span> <span data-ttu-id="01c5e-111">Клиент отправляет свой сертификат серверу, только если сервер отправляет клиенту запрос на сертификат.</span><span class="sxs-lookup"><span data-stu-id="01c5e-111">The client sends its certificate to the server only if the server sends a request to the client for a certificate.</span></span> <span data-ttu-id="01c5e-112">Таким же клиентом всегда выполняется проверка подлинности сервера, но сервер имеет возможность или не проверять подлинность клиента.</span><span class="sxs-lookup"><span data-stu-id="01c5e-112">Thus the client always authenticates the server but the server has the option whether or not to authenticate the client.</span></span>

<span data-ttu-id="01c5e-113">Сертификаты сервера должны храниться в локальном постоянном хранилище API HTTP-сервера для использования при каждом создании безопасного подключения.</span><span class="sxs-lookup"><span data-stu-id="01c5e-113">Server certificates must be stored in the HTTP Server API's local persistent storage, for use each time a secure connection is created.</span></span> <span data-ttu-id="01c5e-114">Каждая запись хранилища сертификатов также содержит IP-адрес и порт сервера, хэш сертификата (используемый для подписи сообщений) и идентификатор приложения.</span><span class="sxs-lookup"><span data-stu-id="01c5e-114">Each certificate store entry also contains the IP Address and port of the server, the certificate hash (used for signing the messages) and the application ID.</span></span> <span data-ttu-id="01c5e-115">Идентификатор приложения используется для идентификации приложения, владеющего сертификатом.</span><span class="sxs-lookup"><span data-stu-id="01c5e-115">The application ID is used to identify the application that owns the certificate.</span></span>

<span data-ttu-id="01c5e-116">Системные администраторы могут хранить сведения о сертификате сервера SSL с помощью API-интерфейсов конфигурации.</span><span class="sxs-lookup"><span data-stu-id="01c5e-116">System administrators can store SSL server certificate information with the configuration APIs.</span></span> <span data-ttu-id="01c5e-117">Средство администрирования вызывает функцию [**хттпсетсервицеконфигуратион**](/windows/desktop/api/Http/nf-http-httpsetserviceconfiguration) и указывает значение хттпсервицеконфигсслцертинфо для параметра конфигурации службы, чтобы задать сведения для SSL-сертификата.</span><span class="sxs-lookup"><span data-stu-id="01c5e-117">An administrative tool calls the [**HttpSetServiceConfiguration**](/windows/desktop/api/Http/nf-http-httpsetserviceconfiguration) function and specifies HttpServiceConfigSSLCertInfo value for the service configuration parameter to set information for an SSL certificate.</span></span> <span data-ttu-id="01c5e-118">Для каждого IP-адреса и пары портов на компьютере можно настроить только один сертификат сервера.</span><span class="sxs-lookup"><span data-stu-id="01c5e-118">Only one server certificate can be configured for each IP Address and port pair on the machine.</span></span> <span data-ttu-id="01c5e-119">API сервера HTTP также предоставляет функции [**запросов**](/windows/desktop/api/Http/nf-http-httpqueryserviceconfiguration) и [**удаления**](/windows/desktop/api/Http/nf-http-httpdeleteserviceconfiguration) для доступа к существующим сертификатам и их удаления.</span><span class="sxs-lookup"><span data-stu-id="01c5e-119">The HTTP Server API also provides [**query**](/windows/desktop/api/Http/nf-http-httpqueryserviceconfiguration) and [**delete**](/windows/desktop/api/Http/nf-http-httpdeleteserviceconfiguration) functions to access or delete existing certificates.</span></span>
