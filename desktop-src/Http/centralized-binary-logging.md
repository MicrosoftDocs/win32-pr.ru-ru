# <a name="centralized-binary-logging"></a><span data-ttu-id="4016b-103">Централизованное ведение журнала в двоичном формате</span><span class="sxs-lookup"><span data-stu-id="4016b-103">Centralized Binary Logging</span></span>

<span data-ttu-id="4016b-104">Централизованное ведение журнала в двоичном формате — это тип ведения журнала на стороне сервера, который может быть включен в сеансе сервера.</span><span class="sxs-lookup"><span data-stu-id="4016b-104">Centralized binary logging is a type of server side logging that can be enabled on the server session.</span></span> <span data-ttu-id="4016b-105">Централизованная функция ведения журнала в двоичном формате используется в качестве централизованной формы для всех групп URL-адресов, созданных в сеансе сервера, и позволяет всем группам URL-адресов в сеансе сервера передавать двоичные, неформатированные данные журнала в один файл журнала.</span><span class="sxs-lookup"><span data-stu-id="4016b-105">Centralized binary logging functions as a centralized form of logging for all the URL groups created under the server session, and allows all the URL groups under the server session to send binary, unformatted log data to a single log file.</span></span> <span data-ttu-id="4016b-106">Этот тип ведения журнала можно включить только в сеансе сервера. ее нельзя включить в группе URL-адресов.</span><span class="sxs-lookup"><span data-stu-id="4016b-106">This type of logging can be enabled on the server session only; it cannot be enabled on the URL group.</span></span>

## <a name="when-to-use-centralized-binary-logging"></a><span data-ttu-id="4016b-107">Когда следует использовать централизованное ведение журнала в двоичном формате</span><span class="sxs-lookup"><span data-stu-id="4016b-107">When to use Centralized Binary Logging</span></span>

<span data-ttu-id="4016b-108">Когда сеанс сервера содержит несколько групп URL-адресов, процесс создания сотен форматированных файлов журналов для отдельных групп URL-адресов и записи данных журнала на диск может быстро потреблять ценные ресурсы ЦП и памяти, тем самым создавая проблемы с производительностью и масштабируемостью.</span><span class="sxs-lookup"><span data-stu-id="4016b-108">When the server session has numerous URL groups under it, the process of creating hundreds of formatted log files for individual URL groups and writing the log data to a disk can quickly consume valuable CPU and memory resources, thereby creating performance and scalability issues.</span></span> <span data-ttu-id="4016b-109">Централизованное ведение журнала в двоичном формате уменьшает объем системных ресурсов, используемых для ведения журнала, в то же время предоставляя подробные данные журналов для организаций, которым он необходим.</span><span class="sxs-lookup"><span data-stu-id="4016b-109">Centralized binary logging minimizes the amount of system resources that are used for logging, while at the same time providing detailed log data for organizations that require it.</span></span>

<span data-ttu-id="4016b-110">Централизованное ведение журнала в двоичном формате — это свойство сеанса сервера, которое при включении настраивает ведение журнала для всех групп URL-адресов в сеансе сервера.</span><span class="sxs-lookup"><span data-stu-id="4016b-110">Centralized binary logging is a server session property that, when enabled, configures logging for all of the URL groups under the server session.</span></span> <span data-ttu-id="4016b-111">При включении централизованного ведения журнала в двоичном формате данные не могут быть записаны или отформатированы для отдельных групп URL-адресов.</span><span class="sxs-lookup"><span data-stu-id="4016b-111">When centralized binary logging is enabled, data cannot be recorded or formatted for individual URL groups.</span></span> <span data-ttu-id="4016b-112">Централизованный файл журнала двоичного журнала имеет расширение двоичного файла журнала Интернета (. Ибл).</span><span class="sxs-lookup"><span data-stu-id="4016b-112">The centralized binary logging log file has an Internet binary log (.ibl) file name extension.</span></span> <span data-ttu-id="4016b-113">Это расширение имени файла гарантирует, что текстовые программы не пытаются открыть и прочитать файл журнала центрального двоичного журнала.</span><span class="sxs-lookup"><span data-stu-id="4016b-113">This file name extension ensures that text utilities do not try to open and read the central binary logging log file.</span></span>

<span data-ttu-id="4016b-121">Необработанный централизованный файл журнала двоичного журнала состоит из записей фиксированной длины или записей индекса, содержащих идентификаторы строк.</span><span class="sxs-lookup"><span data-stu-id="4016b-121">The raw centralized binary logging log file is made up of fixed-length records or index records that contain string identifiers.</span></span> <span data-ttu-id="4016b-122">Записи индекса появятся, так как для записи максимально возможной информации строковые поля переменной длины заменяются числовыми идентификаторами — индексами, которые сопоставляют строку переменной длины с зарегистрированным идентификатором.</span><span class="sxs-lookup"><span data-stu-id="4016b-122">The index records appear because, in an effort to record as much information as possible, variable-length string fields are replaced by numeric identifiers — indexes — that map the variable-length string to the logged identifier.</span></span>
