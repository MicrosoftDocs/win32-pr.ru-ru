# <a name="istatsconnect-method"></a>метод истатс:: Подключение

метод **Подключение** подключает нпп к сети с помощью указанной сетевой карты и предоставляет сведения о конфигурации для подключения.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT STDMETHODCALLTYPE Connect(
  [in]  HBLOB  hInputBlob,
  [in]  LPVOID StatusCallbackProc,
  [in]  LPVOID UserContext,
  [out] HBLOB  hErrorBlob
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хинпутблоб* \[ окне\]
</dt> <dd>

Обработчик для большого двоичного объекта, указывающий сетевую карту, к которой подключается НПП, и сведения о конфигурации для этого соединения.

</dd> <dt>

*Статускаллбаккпрок* \[ окне\]
</dt> <dd>

Адрес функции обратного вызова пользователя, которая получает обновления состояния, такие как триггеры. Если функция обратного вызова не используется, присвойте этому параметру и параметру *UserContext* **значение NULL**.

</dd> <dt>

*UserContext* \[ окне\]
</dt> <dd>

Значение, передаваемое при вызове функции обратного вызова пользователя. Значением этого параметра обычно является HWND или указатель this. Если функция обратного вызова не указана, присвойте этому параметру и параметру *Статускаллбаккпрок* **значение NULL**.

</dd> <dt>

*херрорблоб* \[ заполняет\]
</dt> <dd>

Обработайте с BLOB-объектом ошибок, который содержит дополнительные сведения об ошибке.

</dd> </dl>

## <a name="remarks"></a>Комментарии

при вызове метода **Подключение** сетевой монитор автоматически вызывает метод **истатс:: Configure** с помощью большого двоичного объекта, предоставленного параметром *хинпутблоб* . обратите внимание, что все коды ошибок, возвращенные вызовом метода **истатс:: Configure** , передаются обратно и возвращаются вызовом **истатс:: Подключение** .

Этот метод должен быть вызван до начала записи кадров. Обратите внимание, что при подключении к сети с помощью этого метода необходимо продолжить использовать интерфейс **истатс** для записи кадров.

Входной большой двоичный объект, заданный параметром *хинпутблоб* , можно получить, вызвав методы **жетнппблобфромуи**, **жетнппблобтабле** и **селектнппблобфромтабле** .

BLOB-объект ошибки, возвращенный параметром *херрорблоб* , содержит записи, которые сетевой монитор не удалось распознать или найти во входном большом двоичном объекте, указанном в *хинпутблоб*. Возвращенный BLOB-объект ошибки содержит сведения об ошибке, которые приложение может использовать для устранения неполадок. Например, если НМЕРР \_ запись BLOB \_ \_ -объекта \_ не \_ существует, запись, которую сетевой монитор не удалось найти, включается в возвращенный большой двоичный объект ошибки.
