# <a name="register-parser-callback-function"></a>Регистрация функции обратного вызова средства синтаксического анализа

Функция **регистрации** экспорта должна быть реализована во всех библиотеках DLL средства синтаксического анализа. Реализация **Register** создает и заполняет [*базу данных свойств*](p.md) для протокола. Сетевой монитор использует базу данных для определения свойств, поддерживаемых протоколом.

## <a name="syntax"></a>Синтаксис


```C++
VOID Register(
  _In_ HPROTOCOL hProtocol
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хпротокол* \[ окне\]
</dt> <dd>

Маркер протокола, который сетевой монитор предоставляет при вызове **Register**. При вызове вспомогательных функций экспорта требуется обработчик *хпротокол* .

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Нет.

## <a name="remarks"></a>Remarks

Сетевой монитор начинает вызывать функцию **Register** сразу после загрузки записи. Сетевой монитор вызывает функцию **Register** для каждого протокола, который он может опознать. Функция [креатепротокол](createprotocol.md) передает указатель на функцию **Register** .

Реализация **Register** включает в себя вызовы следующих функций.

-   Вызов функций [креатепропертидатабасе](createpropertydatabase.md) и [AddProperty](/previous-versions/bb251873(v=msdn.10)) для создания базы данных всех свойств, поддерживаемых протоколом.
-   Вызов функции [креатехандоффтабле](createhandofftable.md) требуется, если протокол использует [*переданный набор*](h.md).

Если DLL-файл средства синтаксического анализа содержит несколько синтаксических анализаторов и средство синтаксического анализа может обнаружить более одного протокола, необходимо реализовать функцию **Register** для каждого протокола.
