# <a name="parserautoinstallinfo-callback-function"></a>Функция обратного вызова Парсераутоинсталлинфо

Функция экспорта **парсераутоинсталлинфо** определяет средство синтаксического анализа или средства синтаксического анализа, которые находятся в библиотеке DLL. **Парсераутоинсталлинфо** должен быть реализован во всех библиотеках DLL средства синтаксического анализа.

## <a name="syntax"></a>Синтаксис


```C++
PPF_PARSERDLLINFO WINAPI ParserAutoInstallInfo(void);
```



## <a name="parameters"></a>Параметры

Эта функция обратного вызова не имеет параметров.

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена успешно, возвращаемое значение является структурой [PF \_ парсердллинфо](pf-parserdllinfo.md) , описывающей ПАРСЕРЫ в библиотеке DLL.

Если функция завершается неудачно, возвращается значение **false**.

## <a name="remarks"></a>Комментарии

При первой загрузке сетевой монитор вызывает метод **парсераутоинсталлинфо** (если он существует) для автоматической установки каждого средства синтаксического анализа, а затем перечислите все библиотеки DLL средства синтаксического анализа в подкаталоге средства синтаксического анализа.

Сведения, возвращаемые в структуре **PF \_ парсердллинфо** , включают следующее:

-   Количество синтаксических анализаторов в библиотеке DLL (обычно один).
-   Имя и краткое описание протокола, обнаруженного каждым синтаксическим анализатором.
-   Необязательный файл справки для каждого протокола.
-   Протоколы, предшествующие каждому протоколу.
-   Протоколы, которые следуют каждому протоколу.

Каждая библиотека DLL анализатора должна содержать одно средство синтаксического анализа. Однако сетевой монитор позволяет создать библиотеку DLL, содержащую более одного средства синтаксического анализа. Например, tcpip.dll является сетевой монитор DLL с несколькими анализаторами.
