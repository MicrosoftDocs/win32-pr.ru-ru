# <a name="irtcconfigure-method"></a>Метод ИРТК:: Configure

Метод [**Configure**](configure.md) отправляет данные конфигурации для отслеживания данных.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT STDMETHODCALLTYPE Configure(
  [in]  HBLOB hConfigurationBlob,
  [out] HBLOB hErrorBlob
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хконфигуратионблоб* \[ окне\]
</dt> <dd>

Маркер для большого двоичного объекта, настроенного вызывающим объектом.

</dd> <dt>

*херрорблоб* \[ заполняет\]
</dt> <dd>

Маркер для большого двоичного объекта ошибок, который содержит дополнительные данные об ошибке.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если метод выполнен успешно, возвращается значение НМЕРР \_ Success.

## <a name="remarks"></a>Remarks

Этот метод необходимо применить для перезапуска НПП, который был запущен, остановлен, но не отключен.

BLOB-объект ошибки, возвращенный *херрорблоб* , содержит записи, которые сетевой монитор не удалось распознать или найти в большом двоичном объекте конфигурации, указанном в *хконфигуратионблоб*. Возвращенный BLOB-объект ошибки содержит данные об ошибке, которые приложение может использовать для устранения неполадок. Например, если \_ запись BLOB-объекта нмерр \_ \_ \_ не \_ существует, запись, сетевой монитор не удается найти, включается в возвращенный большой двоичный объект ошибки.
