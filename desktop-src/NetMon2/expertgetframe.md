# <a name="expertgetframe-function"></a>Функция Експертжетфраме

Функция **експертжетфраме** возвращает запрошенный кадр из загруженной записи.

## <a name="syntax"></a>Синтаксис


```C++
DWORD WINAPI ExpertGetFrame(
  _In_  HEXPERTKEY              hExpertKey,
  _In_  DWORD                   Direction,
  _In_  DWORD                   RequestFlags,
  _In_  DWORD                   RequestedFrameNumber,
  _In_  HFILTER                 hFilter,
  _Out_ LPEXPERTFRAMEDESCRIPTOR pEFrameDescriptor
);
```

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена успешно, возвращается значение НМЕРР \_ Success.

Если функция завершается неудачно, возвращаемое значение указывает причину сбоя. Если возвращаемое значение — НМЕРР \_ \_ , то эксперт должен немедленно очистить и вернуться; пользователь прервал запуск эксперта.

## <a name="remarks"></a>Комментарии

Если задать ФЛАГи \_ присоединения \_ свойств, вызов требует больше ресурсов, чем если бы флаг не был установлен. Если флаг не установлен, указатель указывает на необработанный кадр и на данные о кадре. Если этот флаг установлен, сетевой монитор присоединяет все свойства к кадру путем вызова каждого средства синтаксического анализа, которое заявляет часть кадра. Это может быть очень длительный процесс.

Эксперты не должны устанавливать \_ флаг присоединения флагов, \_ Если для экспертов не требуются свойства, присоединяемые к фрейму анализаторами. Если это возможно, эксперты должны вызвать функцию **експертжетфраме** без флага, а затем извлечь необходимые данные непосредственно из фрейма.

Если эксперт вызывает **експертжетфраме** без флагов \_ присоединить \_ Свойства и требует свойства, связанные с этим кадром (например, событие), эксперт вызывает **експертжетфраме** с теми же параметрами, за исключением следующих:

``` syntax
Direction = EXPERT_GET_SPECIFIED_FRAME;
RequestFlags &= (~EXPERT_DEFER_TO_UI_FILTER) | EXPERT_ATTACH_PROPERTIES;
RequestedFrameNumber= (The actual frame number you want);
hFilter = NULL;
pEFrameDescriptor = (The same one as last time);
```

Использование приведенного выше кода гарантирует, что эксперт получает необходимый кадр без необходимости повторного вызова кода фильтра.

Параметр *хфилтер* можно задать как **лпвоид**. Если он существует, возвращаемый кадр передает этот фильтр. Если у эксперта нет фильтра для передачи функции (если *хфилтер* имеет **значение NULL** ), то возвращаемый кадр не фильтруется.

Функция **експертжетфраме** может вызываться экспертами, реализующими функцию [**запуска**](run.md) или [**настройки**](configure.md) экспорта.
