# <a name="getcapturemactype-function"></a>Функция Жеткаптуремактипе

Функция **жеткаптуремактипе** ВОЗВРАЩАЕТ тип Mac для данной записи.

## <a name="syntax"></a>Синтаксис


```C++
DWORD WINAPI GetCaptureMacType(
  _In_ HCAPTURE hCapture
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хкаптуре* \[ окне\]
</dt> <dd>

Обработчик для записи. Дополнительные сведения о получении маркера записи см. в подразделе "Примечания" этого раздела **жеткаптуремактипе** .

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена успешно, возвращаемое значение является одним из следующих типов MAC.

-   \_неизвестный тип Mac \_
-   \_тип Mac \_ Ethernet
-   \_тип Mac \_ токенринг
-   \_тип Mac \_ FDDI

Если функция завершается неудачно, возвращаемое значение является кодом ошибки.



| Код возврата                                                                                              | Описание                                                 |
|----------------------------------------------------------------------------------------------------------|-------------------------------------------------------------|
| <dl> <dt>**\_неизвестный тип Mac-адреса нмерр \_ \_**</dt> </dl> | Сетевой монитор не удалось найти известный тип MAC.<br/> |



 

## <a name="remarks"></a>Комментарии

Маркер записи может быть получен несколькими способами в зависимости от того, кто выполняет вызов. Для экспертов этот маркер указывается в элементе **хкаптуре** структуры [експертстартупинфо](expertstartupinfo.md) . Для средств синтаксического анализа можно получить маркер записи, вызвав функцию [жетфрамекаптурехандле](getframecapturehandle.md) .

[*Эксперты*](e.md) и [*средства синтаксического анализа*](p.md) могут вызывать функцию **жеткаптуремактипе** .
