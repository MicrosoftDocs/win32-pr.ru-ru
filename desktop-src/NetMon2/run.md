# <a name="run-callback-function"></a>Запустить функцию обратного вызова

Эксперт должен реализовать функцию **Run** . Сетевой монитор вызывает функцию **Run** для запуска эксперта в библиотеке DLL эксперта.

## <a name="syntax"></a>Синтаксис


```C++
BOOL WINAPI Run(
  _In_ HEXPERTKEY         hExpertKey,
  _In_ PEXPERTCONFIG      pConfig,
  _In_ PEXPERTSTARTUPINFO pExpertStartupInfo,
  _In_ DWORD              StartupFlags,
  _In_ HWND               hWndParent
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*хексперткэй* \[ окне\]
</dt> <dd>

Уникальный идентификатор эксперта, который передается обратно в сетевой монитор функции, относящиеся к экспертам.

> [!Note]  
> Идентификатор *хексперткэй* может передавать экспертный ключ, отличный от экспертного ключа, который передает функция [**Configure**](configure.md) .

 

</dd> <dt>

*pConfig* \[ окне\]
</dt> <dd>

Указатель на существующую конфигурацию. Параметр *pConfig* может иметь **значение NULL** , что означает, что эксперт может работать с жестко заданными значениями по умолчанию или со сведениями о запуске, на которые ссылается параметр *пекспертстартупинфо* .

</dd> <dt>

*пекспертстартупинфо* \[ окне\]
</dt> <dd>

Указатель на элемент захвата, имеющий фокус при запуске эксперта.

</dd> <dt>

*Стартупфлагс* \[ окне\]
</dt> <dd>

Признак использования параметра *пекспертстартупинфо* экспертом.

Единственный заданный флаг:



| Значение                                                                                                                                                                                                                                                                                           | Значение                                                                                                                                                                                                                                                                                                                                                                                                                       |
|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="EXPERT_STARTUP_FLAG_USE_STARTUP_DATA_OVER_CONFIG_DATA."></span><span id="expert_startup_flag_use_startup_data_over_config_data."></span><dl> <dt>**\_флаг запуска эксперта \_ \_ использование \_ загрузочных \_ данных \_ для \_ \_ данных конфигурации.**</dt> </dl> | Этот флаг указывает, что эксперт использует параметр *пекспертстартупинфо* и не использует параметр *pConfig* . Как правило, этот флаг можно установить, когда эксперт может начать с щелчка правой кнопкой мыши. Если флаг не установлен, возможны следующие два действия: либо эксперт не начинает с щелчка правой кнопкой мыши, либо эксперт начинает с щелчка правой кнопкой мыши, а затем пользователь настраивает его.<br/> |



 

</dd> <dt>

*хвндпарент* \[ окне\]
</dt> <dd>

Параметр *HWND* родительского элемента (обычно это сетевой монитор пользовательский интерфейс).

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если функция выполнена успешно (то есть при запуске эксперта), возвращается значение **true**.

Если функция завершается неудачно, возвращается значение **false**.

## <a name="remarks"></a>Remarks

При запуске эксперт просматривает кадры в файле записи и либо создает отчет, либо создает события, которые показывают проблемы.
