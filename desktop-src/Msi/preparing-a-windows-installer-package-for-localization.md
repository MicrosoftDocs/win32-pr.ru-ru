# <a name="preparing-a-windows-installer-package-for-localization"></a>подготовка пакета установщик Windows для локализации

локализация пакета установщик Windows на несколько языков может значительно облегчить выполнение следующих действий.

-   Создайте базовую базу данных установки с нейтральной кодовой страницей. См. раздел [Создание базы данных с нейтральной кодовой страницей](creating-a-database-with-a-neutral-code-page.md). Затем можно установить кодовую страницу локализованной базы данных, импортировав таблицу с ненейтральной кодовой страницей в основную базу данных. См. раздел [Настройка кодовой страницы базы данных](setting-the-code-page-of-a-database.md).
-   Упорядочите файлы, которые требуют локализации, в отдельные компоненты и устанавливайте эти файлы в отдельные каталоги. Это гарантирует, что два локализованных пакета никогда не устанавливают файлы с одинаковыми именами в один каталог.

Например, приложение мирового типа, использующее следующие ресурсы, может иметь три компонента.



| Компонент | Ресурс                   |
|-----------|----------------------------|
| РЕАЛЬНОГО     | worldwide.exe              |
| РЕАЛЬНОГО     | записи реестра для всех стран |
| РЕАЛЬНОГО     | международные сочетания         |
| ENG       | engui.dll                  |
| ENG       | readMe.txt                 |
| FRA       | fraui.dll                  |
| FRA       | readMe.txt                 |



 

Файлы, которые необходимо локализовать, можно установить в следующие каталоги:

-   \[ProgramFilesFolder \] \\ World \\worldwide.exe
-   \[\] \\ \\ Английскийengui.dll ProgramFilesFolder \\ World
-   \[\] \\ \\ Английскийreadme.txt ProgramFilesFolder \\ World
-   \[ProgramFilesFolder \] \\ мира на \\ французском языке \\fraui.dll
-   \[ProgramFilesFolder \] \\ мира на \\ французском языке \\readme.txt
