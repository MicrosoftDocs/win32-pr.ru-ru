# <a name="installing-a-com-component-to-a-private-location"></a>Установка COM-компонента в Закрытое расположение

Чтобы заставить приложение COM-клиента всегда использовать одну и ту же копию COM-сервера, создайте пакет установки приложения, чтобы указать отношение [изолированных компонентов](isolated-components.md) между сервером и клиентом COM. При этом устанавливается частная копия компонента COM-Server в расположение, используемое только клиентским приложением. При создании пакета выполните следующие действия.

-   Размещение DLL-сервера COM и клиента exe в отдельных компонентах.
-   Введите запись в [таблицу исолатедкомпонент](isolatedcomponent-table.md) с компонентом COM-Client в \_ столбце Component Shared и в клиентском приложении в \_ столбце приложение компонента. Включите [действие исолатекомпонентс](isolatecomponents-action.md) в таблицы последовательности.
-   Установите бит Мсидбкомпонентаттрибутесшареддллрефкаунт в записи [таблицы компонентов](component-table.md) для общего доступа к компоненту \_ . Установщику требуется этот глобальный параметр refcount в общем расположении для защиты общих файлов и регистрации в случаях, когда используется совместно с другими технологиями установки.
