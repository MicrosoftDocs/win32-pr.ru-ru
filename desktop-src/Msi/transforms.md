# <a name="transforms-property"></a><span data-ttu-id="d917d-103">Свойство TRANSFORMs</span><span class="sxs-lookup"><span data-stu-id="d917d-103">TRANSFORMS property</span></span>

<span data-ttu-id="d917d-104">Свойство **TRANSFORMS** представляет собой список преобразований, применяемых установщиком при установке пакета.</span><span class="sxs-lookup"><span data-stu-id="d917d-104">The **TRANSFORMS** property is a list of the transforms that the installer applies when installing the package.</span></span> <span data-ttu-id="d917d-105">Установщик применяет преобразования в том же порядке, в котором они указаны в свойстве.</span><span class="sxs-lookup"><span data-stu-id="d917d-105">The installer applies the transforms in the same order as they are listed in the property.</span></span> <span data-ttu-id="d917d-106">Преобразования могут быть заданы по имени файла или полному пути.</span><span class="sxs-lookup"><span data-stu-id="d917d-106">Transforms can be specified by their filename or full path.</span></span> <span data-ttu-id="d917d-107">Чтобы указать несколько преобразований, разделяйте каждое имя файла или полный путь точкой с запятой (;).</span><span class="sxs-lookup"><span data-stu-id="d917d-107">To specify multiple transforms, separate each file name or full path with a semicolon (;).</span></span> <span data-ttu-id="d917d-108">Например, чтобы применить три преобразования к пакету, задайте для **преобразований** список имен файлов или список полных путей.</span><span class="sxs-lookup"><span data-stu-id="d917d-108">For example, to apply three transforms to a package, set **TRANSFORMS** to a list of file names or to a list of full paths.</span></span>

``` syntax
TRANSFORMS=transform1.mst;transform2.mst;transform3.mst
TRANSFORMS=\\server\share\path\transform1.mst;\\server2\share2\path2\transform2.mst;\\server3\share3\path3\transform3.mst
```

<span data-ttu-id="d917d-109">Можно указать, что файл преобразования внедряется в хранилище MSI-файла, а не в автономный файл, путем добавления в имя файла двоеточия (:).</span><span class="sxs-lookup"><span data-stu-id="d917d-109">You can indicate that a transform file is embedded in a storage of the .msi file, rather than as a stand-alone file, by prefixing the filename with a colon (:).</span></span> <span data-ttu-id="d917d-110">Например, в следующем примере показано, что в файле. msi внедряются файлы, и, и, transform3. MST, являются автономными.</span><span class="sxs-lookup"><span data-stu-id="d917d-110">For example, the following example indicates that transform1.mst and transform2.mst are embedded inside the .msi file and that transform3.mst is a stand-alone file.</span></span>

``` syntax
TRANSFORMS=:transform1.mst;:transform2.mst;transform3.mst
```

<span data-ttu-id="d917d-111">Установщик требует преобразования, перечисленные в **ПРЕобразованиях** при каждой установке пакета, объявления, установки по требованию или при обслуживании.</span><span class="sxs-lookup"><span data-stu-id="d917d-111">The installer requires the transforms listed in **TRANSFORMS** at every installation, advertisement, installation-on-demand, or maintenance installation of the package.</span></span> <span data-ttu-id="d917d-112">Политика [политики трансформссекуре](transformssecure-policy.md) , свойство **TRANSFORMS** и первый символ строки **преобразований** информирует установщик о том, как управлять устойчивостью источника автономных файлов преобразования.</span><span class="sxs-lookup"><span data-stu-id="d917d-112">The [TransformsSecure policy](transformssecure-policy.md) policy, the **TRANSFORMS** property, and the first character of the **TRANSFORMS** string informs the installer how to handle the source resiliency of stand-alone transform files.</span></span> <span data-ttu-id="d917d-113">Установщик Windows рассматривает параметры [политики трансформсатсаурце](transformsatsource-policy.md) или [**трансформсатсаурце**](transformsatsource.md) так же, как трансформссекуре Policy и [**трансформссекуре**](transformssecure.md).</span><span class="sxs-lookup"><span data-stu-id="d917d-113">Windows Installer treats setting [TransformsAtSource policy](transformsatsource-policy.md) or [**TRANSFORMSATSOURCE**](transformsatsource.md) the same as TransformsSecure policy and [**TRANSFORMSSECURE**](transformssecure.md).</span></span> <span data-ttu-id="d917d-114">Обратите внимание, что преобразования, внедренные в файл MSI, не кэшируются и всегда получаются из пакета.</span><span class="sxs-lookup"><span data-stu-id="d917d-114">Note that transforms embedded in the .msi file are not cached and are always obtained from the package.</span></span>
