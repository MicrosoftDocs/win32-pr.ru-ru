# <a name="companion-files"></a>Сопутствующие файлы

Состояние установки вспомогательного файла зависит не от его собственных сведений о версиях файлов, но и от управления версиями его сопутствующего родителя. См. [правила управления версиями файлов](file-versioning-rules.md). Чтобы указать сопутствующий файл, первичный ключ вспомогательного родителя в [таблице File](file-table.md) должен быть создан в столбце Version записи для сопровождающего.

В следующем примере файл a является сопутствующим родителем, а Филеб — сопутствующий файл.

[Таблица файлов](file-table.md) (частичная)



| Файл  | Версия |
|-------|---------|
| Файл а | 1.0.0.0 |
| филеб | Файл а   |



 

В этом примере состояние установки Филеб зависит от [правил управления версиями файлов](file-versioning-rules.md) и сведений об управлении версиями файла. Если установщик определяет, что версия файла a в пакете должна быть установлена поверх более старой версии файла, которая уже существует на компьютере пользователя, она также установит Филеб из пакета независимо от версии установленных Филеб.

Обратите внимание, что файл, который является путем к ключу для своего компонента, не должен быть сопутствующим файлом. Это приведет к тому, что логика управления версиями файла пути к ключу определяется сопутствующим родительским файлом.
