# <a name="qualified-components"></a>Компоненты с квалификаторами

Квалифицированный компонент — это метод одноуровневого косвенного обращения, аналогичный указателю. Квалифицированные компоненты в основном используются для группирования компонентов с параллельными функциями в категории. Например, если в [таблице Component](component-table.md) есть 30 компонентов, которые относятся к одному шаблону Microsoft Word Fax, локализованному на 30 языков, их можно объединить в категорию компонентов с квалификаторами с помощью [таблицы публишкомпонент](publishcomponent-table.md).

Квалифицированные компоненты добавляются в таблицу Component так же, как обычные компоненты. Каждый компонент должен иметь уникальный идентификатор GUID компонента и идентификатор компонента, указанные в таблице Component. Кроме того, квалифицированные компоненты связаны с GUID категории и квалификатором текстовой строки в таблице Публишкомпонент. На полные компоненты ссылается GUID категории и квалификатор, который просто указывает на обычный компонент в таблице Component.

Например, идентификатор GUID уникального компонента может указывать на разные языковые версии библиотеки ресурсов. В этом случае Группа локализованных библиотек DLL ресурсов состоит из категории, а строки идентификаторов национальных настроек (LCID) обычно используются в качестве квалификаторов. Разработчик может создать пакет установки, использующий эти компоненты для следующих задач:

-   Найдите путь к определенной языковой версии библиотеки ресурсов с помощью [**мсипровидекуалифиедкомпонент**](/windows/desktop/api/Msi/nf-msi-msiprovidequalifiedcomponenta) или [**мсипровидекуалифиедкомпонентекс**](/windows/desktop/api/Msi/nf-msi-msiprovidequalifiedcomponentexa) и установите ресурс.
-   Определите все языковые версии библиотеки DLL ресурсов, которые представлены путем вызова [**мсиенумкомпоненткуалифиерс**](/windows/desktop/api/Msi/nf-msi-msienumcomponentqualifiersa).
-   Подготовьте приложение для поддержки дополнительных языков. В будущем языковой пакет для приложения может использовать квалифицированный компонент для добавления дополнительных языковых версий библиотеки ресурсов.

Дополнительные сведения см. [в разделе Использование компонентов с квалификаторами](using-qualified-components.md).
