# <a name="ice06"></a>ICE06

ICE06 проверяет каждую таблицу, чтобы убедиться, что все столбцы, перечисленные в [ \_ таблице проверки](-validation-table.md) , присутствуют в таблице. Если таблица не существует, все \_ записи проверки для этой таблицы игнорируются.

Целью ICE06 является обнаружение экземпляров, в которых автор пытается использовать новую \_ таблицу проверки, отражающую изменение схемы со старой базой данных, которая не была обновлена. ICE06 также обнаруживает обратную \_ таблицу старой таблицы проверки, используемой в измененной базе данных.

Обратите внимание, что внутренняя проверка, выполняемая [ICE03](ice03.md) , перехватывает экземпляр столбца таблицы, не определенный в \_ таблице проверки, указанной в каталоге Columns. Таким образом, использование ICE03 и ICE06 гарантирует тестирование каждого столбца в базе данных.

## <a name="result"></a>Результат

ICE06 отправляет сообщение об ошибке, если в таблице проверки определен столбец таблицы \_ , не указанный в \_ таблице Columns.

## <a name="example"></a>Пример

В следующем примере ICE06 отправляет сообщение.

Столбец: версия таблицы: ModuleSignature не определен в базе данных.

[ \_ Таблица проверки](-validation-table.md) (частичная)



| Таблица           | Столбец   |
|-----------------|----------|
| ModuleSignature | ModuleID |
| ModuleSignature | Версия  |



 

[ \_ Таблица Columns](-columns-table.md) (частичная)



| Таблица           | number | name     |
|-----------------|--------|----------|
| ModuleSignature | 1      | ModuleID |



 

Столбец Version таблицы ModuleSignature отсутствует в базе данных или указан в \_ таблице Columns.
