# <a name="_transformview-table"></a>\_Таблица переformview

Это временная таблица только для чтения, используемая для просмотра преобразований в режиме представления преобразования. Эта таблица никогда не сохраняется в установщике.

Чтобы вызвать режим представления преобразования, получите маркер и откройте эталонную базу данных. См. раздел [Получение маркера базы данных](obtaining-a-database-handle.md). Вызовите [**мсидатабасеапплитрансформ**](/windows/desktop/api/Msiquery/nf-msiquery-msidatabaseapplytransforma) с мситрансформ \_ Error \_ виевтрансформ. Это приостанавливает преобразование к базе данных и выводит содержимое преобразования в \_ таблицу reformview. Доступ к данным в таблице можно получить с помощью SQL запросов. См. раздел [Работа с запросами](working-with-queries.md).

\_Таблица преобразования не удаляется, когда применяется другое преобразование. Таблица отражает совокупный результат последовательных приложений. Для просмотра преобразований по отдельности необходимо освободить таблицу.

\_Таблица переformview содержит следующие столбцы.



| Столбец  | Type                         | Ключ | Допускает значения NULL |
|---------|------------------------------|-----|----------|
| Таблица   | [Идентификатор](identifier.md) | Да   | Нет        |
| Столбец  | [Text](text.md)             | Да   | Нет        |
| Строка     | [Text](text.md)             | Да   | Да        |
| Данные    | [Text](text.md)             | Нет   | Да        |
| Текущий | [Text](text.md)             | Нет   | Да        |

## <a name="remarks"></a>Комментарии

Объект \_ FormView удерживается в памяти с помощью счетчика блокировок, который можно освободить с помощью следующей команды SQL.

«ALTER TABLE reon \_ Free».

Доступ к данным в таблице можно получить с помощью SQL запросов. В языке SQL есть два основных подразделения: язык описания данных DDL, который используется для определения всех объектов в базе данных SQL, и языка обработки данных DML, который используется для выбора, вставки, обновления и удаления данных в объектах, определенных с помощью DDL.

Операции преобразования языка обработки данных (DML) обозначаются следующим образом. Язык обработки данных DML — это инструкции в SQL, которые управляют, а не определяют данные.



| Операция преобразования | Результат SQL                                    |
|---------------------|-----------------------------------------------|
| Изменение данных         | Таблица рубрик строки Data {Текущее значение} |
| Вставка строки          | Таблица «INSERT» {Row} NULL NULL              |
| Удаление строки          | Таблица «DELETE» {Row} NULL NULL              |



 

Операции преобразования языка описания данных DDL указываются следующим образом. Язык описания данных (DDL) — это инструкции в SQL, которые определяют, а не управляют данными.



| Операция преобразования | Результат SQL                                   |
|---------------------|----------------------------------------------|
| Добавление столбца          | Таблица рубрик NULL {дефн} {номер столбца} |
| Добавить таблицу           | Таблица "CREATE" NULL NULL NULL              |
| Удаление таблицы          | Таблица "DROP" NULL NULL NULL                |
