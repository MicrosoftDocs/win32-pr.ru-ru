# <a name="internal-consistency-evaluators---ices"></a><span data-ttu-id="52dfe-103">Внутренние средства оценки согласованности — ICEs</span><span class="sxs-lookup"><span data-stu-id="52dfe-103">Internal Consistency Evaluators - ICEs</span></span>

<span data-ttu-id="52dfe-104">Внутренние средства оценки согласованности, называемые также стандартом ICEs, являются пользовательскими действиями, написанными на языке VBScript, JScript или в виде библиотеки DLL или EXE.</span><span class="sxs-lookup"><span data-stu-id="52dfe-104">Internal consistency evaluators, also called ICEs, are custom actions written in VBScript, JScript, or as a DLL or EXE.</span></span> <span data-ttu-id="52dfe-105">При выполнении этих настраиваемых действий они проверяют базу данных на наличие записей в записях базы данных, которые являются допустимыми при проверке по отдельности, но могут привести к неправильному поведению в контексте всей базы данных.</span><span class="sxs-lookup"><span data-stu-id="52dfe-105">When these custom actions are executed, they scan the database for entries in database records that are valid when examined individually but that may cause incorrect behavior in the context of the whole database.</span></span> <span data-ttu-id="52dfe-106">Обратите внимание, что это отличается от проверки, выполненной для отдельных записей с помощью [**мсивиевмодифи**](/windows/desktop/api/Msiquery/nf-msiquery-msiviewmodify).</span><span class="sxs-lookup"><span data-stu-id="52dfe-106">Note that this is different than the validation done on individual records using [**MsiViewModify**](/windows/desktop/api/Msiquery/nf-msiquery-msiviewmodify).</span></span>

<span data-ttu-id="52dfe-107">Например, в [таблице Component](component-table.md) может сопоставлено несколько компонентов, допустимых при тестировании по отдельности с помощью [**мсивиевмодифи**](/windows/desktop/api/Msiquery/nf-msiquery-msiviewmodify).</span><span class="sxs-lookup"><span data-stu-id="52dfe-107">For example, the [Component table](component-table.md) may list several components that are all valid when tested individually with [**MsiViewModify**](/windows/desktop/api/Msiquery/nf-msiquery-msiviewmodify).</span></span> <span data-ttu-id="52dfe-108">Однако **мсивиевмодифи** не будет перехватывать ошибку, когда два компонента используют один и тот же [идентификатор GUID](guid.md) в коде компонента.</span><span class="sxs-lookup"><span data-stu-id="52dfe-108">However, **MsiViewModify** would not catch the error when two components use the same [GUID](guid.md) as their component code.</span></span> <span data-ttu-id="52dfe-109">Настраиваемое действие [ICE08](ice08.md) предназначено для проверки того, что таблица Component не содержит ДУБЛИРУЮЩихся идентификаторов GUID кода компонента.</span><span class="sxs-lookup"><span data-stu-id="52dfe-109">The custom action [ICE08](ice08.md) is designed to validate that the Component table does not contain duplicate component code GUIDs.</span></span>

<span data-ttu-id="52dfe-110">Пользовательские действия ICE возвращают четыре типа сообщений:</span><span class="sxs-lookup"><span data-stu-id="52dfe-110">ICE custom actions return four kinds of messages:</span></span>

-   <span data-ttu-id="52dfe-111">[**Ошибки**](merge-errors.md) Сообщения об ошибках сообщают о неправильном поведении базы данных.</span><span class="sxs-lookup"><span data-stu-id="52dfe-111">[**Errors**](merge-errors.md) Error messages report database authoring that cause incorrect behavior.</span></span> <span data-ttu-id="52dfe-112">Например, дублированные идентификаторы GUID компонента приводят к неправильной регистрации компонентов в установщике.</span><span class="sxs-lookup"><span data-stu-id="52dfe-112">For example, duplicate component GUIDs cause the installer to incorrectly register components.</span></span>
-   <span data-ttu-id="52dfe-113">**Предупреждения** Предупреждающие сообщения сообщают о неправильном поведении базы данных в некоторых случаях.</span><span class="sxs-lookup"><span data-stu-id="52dfe-113">**Warnings** Warning messages report database authoring that causes incorrect behavior in certain cases.</span></span> <span data-ttu-id="52dfe-114">Предупреждения также могут сообщать о непредвиденных побочных эффектах создания базы данных.</span><span class="sxs-lookup"><span data-stu-id="52dfe-114">Warnings can also report unexpected side-effects of database authoring.</span></span> <span data-ttu-id="52dfe-115">Например, введите одно и то же имя свойства в двух условиях, отличающихся только регистром букв в имени.</span><span class="sxs-lookup"><span data-stu-id="52dfe-115">For example, entering the same property name in two conditions that differ only by the case of letters in the name.</span></span> <span data-ttu-id="52dfe-116">Так как установщик учитывает регистр, установщик обрабатывает их как различные свойства.</span><span class="sxs-lookup"><span data-stu-id="52dfe-116">Because the installer is case-sensitive, the installer treats these as different properties.</span></span>
-   <span data-ttu-id="52dfe-117">**Сбои** Сообщения об ошибках сообщают о сбое настраиваемого действия ICE.</span><span class="sxs-lookup"><span data-stu-id="52dfe-117">**Failures** Failure messages report the failure of the ICE custom action.</span></span> <span data-ttu-id="52dfe-118">Сбой обычно вызывается базой данных с такими серьезными проблемами, которые невозможно даже запустить ICE.</span><span class="sxs-lookup"><span data-stu-id="52dfe-118">Failure is commonly caused by a database with such severe problems that the ICE cannot even run.</span></span>
-   <span data-ttu-id="52dfe-119">**Информационные** сообщения Информационные сообщения предоставляют сведения из ICE и не указывают на проблему с базой данных.</span><span class="sxs-lookup"><span data-stu-id="52dfe-119">**Informational** Informational messages provide information from the ICE and do not indicate a problem with the database.</span></span> <span data-ttu-id="52dfe-120">Часто это сведения о самом ICE, например краткое описание.</span><span class="sxs-lookup"><span data-stu-id="52dfe-120">Often they are information about the ICE itself, such as a brief description.</span></span> <span data-ttu-id="52dfe-121">Они также могут предоставлять сведения о ходе выполнения при выполнении ICE.</span><span class="sxs-lookup"><span data-stu-id="52dfe-121">They can also provide progress information as the ICE runs.</span></span>
