# <a name="ice19"></a>ICE19

ICE19 проверяет, что объявленные компоненты ссылаются на файл в столбце ключевого пути [таблицы Component](component-table.md) и что объявленное контекстное меню ссылается на каталог в этом столбце.

ICE19 проверяет, что объявленные компоненты или ярлыки имеют ИД компонента. Компоненты в [таблице публишкомпонент](publishcomponent-table.md), которые не объявлены в другой таблице, проверяются только на наличие у них ИД компонента.

## <a name="result"></a>Результат

ICE19 отправляет сообщение об ошибке, если ключевой столбец в таблице Component не ссылается на файл в случае объявленного компонента или каталога в случае объявленного ярлыка. ICE19 отправляет сообщение об ошибке, если ни один из объявленных компонентов или ярлыков не имеет ИД компонента.

## <a name="example"></a>Пример

ICE19 отправляет следующие сообщения об ошибках в приведенном примере:

-   Расширение FLP ссылается на компонент comp1, который не имеет ИД компонента, указанного в [таблице Component](component-table.md).
-   Расширение EXE ссылается на компонент Comp4, который ссылается на каталог в качестве ключевого пути. Ключевой путь имеет значение NULL в таблице Component.
-   Ярлык Shortcut2 ссылается на компонент Comp3, который ссылается на запись реестра в качестве пути к ключу. Значение столбца Attributes в таблице Component равно 4.

[Таблица Component](component-table.md) (частичная)



| Компонент | ComponentId                            | Атрибуты | Путь |
|-----------|----------------------------------------|------------|---------|
| Comp1     | NULL                                   | 0          | Файл1   |
| Comp2     | {00000002-0003-0000-0000-624474736554} | 0          | Файл2   |
| Comp3     | {00000003-0003-0000-0000-624474736554} | 4          | Reg3    |
| Comp4     | {00000004-0003-0000-0000-624474736554} | 0          | NULL    |

> [!Note]  
> Если расширение FLP и exe ссылается на один и тот же компонент, то исполняемый файл или COM-сервер, который их открывает, должен быть одинаковым. Обычно этот EXE-файл является ключевой целью для компонента. Для OFFICE модули doc и XLS не могут ссылаться на один и тот же компонент, так как один и тот же исполняемый файл не открывает оба расширения. Чтобы открыть расширения DOC, необходимо winword.exe, а для открытия модулей XLS необходимо excel.exe.
