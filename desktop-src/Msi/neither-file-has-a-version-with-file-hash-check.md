# <a name="neither-file-has-a-version-with-file-hash-check"></a>Ни один из файлов не имеет версии с проверками хэша файла

Хэширование файлов доступно в установщик Windows. Дополнительные сведения см. в разделе [**мсижетфилехаш**](/windows/desktop/api/Msi/nf-msi-msigetfilehasha) и [Таблица мсифилехаш](msifilehash-table.md). Таблицу Мсифилехаш можно использовать только с файлами без версий.

Если файл ключа устанавливаемого компонента (Copy-A) имеет то же имя, что и файл, уже установленный в целевом расположении (Copy-B), установщик сравнивает номер версии, дату и язык двух файлов.

Если ни один из файлов не имеет номера версии, установщик использует логику, показанную на следующей схеме потока, чтобы определить, следует ли заменить все установленные файлы, относящиеся к компоненту. Поскольку установщик устанавливает только компоненты целиком, если установленный файл ключа заменяется, все файлы компонента заменяются.

Обратите внимание, что на этой диаграмме показаны [правила управления версиями файлов](file-versioning-rules.md)по умолчанию, которые можно переопределить, задав свойство [**REINSTALLMODE**](reinstallmode.md) . Значение свойства **REINSTALLMODE** по умолчанию — "omus".
