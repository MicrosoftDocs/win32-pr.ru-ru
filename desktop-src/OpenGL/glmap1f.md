# <a name="glmap1f-function"></a>Функция glMap1f

Функции [**glMap1d**](glmap1d.md) и **glMap1f** определяют одномерный оценщик.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glMap1f(
         GLenum  target,
         GLfloat u1,
         GLfloat u2,
         GLint   stride,
         GLint   order,
   const GLfloat *points
);
```

## <a name="remarks"></a>Remarks

Оценивающие предоставляют способ использования сопоставлений полинома или рационального полинома для создания вершин, нормалй, координат текстуры и цветов. Значения, созданные средством оценки, отправляются на последующие этапы обработки OpenGL, как если бы они были представлены с помощью команд [**глвертекс**](glvertex-functions.md), [**глнормал**](glnormal-functions.md), [**глтекскурд**](gltexcoord-functions.md)и [**глколор**](glcolor-functions.md) , за исключением того, что созданные значения не обновляют текущие нормальные координаты текстуры или цвета.

Все коэффициенты полинома или рациональной полиномы любой степени (вплоть до максимальной степени, поддерживаемой реализацией OpenGL) можно описать с помощью оценивающих. К ним относятся почти все сплайны, используемые в компьютерных графиках, включая B-сплайны, кривые Безье, Хермите сплайны и т. д.

Оценивающие определяют кривые на основе полиномов Бернстеин. Определить **p** () как

![Формула, показывающая определение p ().](images/map01.png)

где **R**_i_ — это контрольная точка, а () — *я* бернстеинный полином степени *n* (*Order*  = *n* + 1):

![Уравнение, показывающее степень полинома Бернстеина n.](images/map02.png)

Вспомним, что

![Уравнения, отображающие эквивалентность, равное 1.](images/map03.png)

Функция **glMap1** используется для определения базиса и для указания типа создаваемых значений. После определения можно включить и отключить карту, вызвав [**гленабле**](glenable.md) и **глдисабле** с именем Map, одним из девяти предопределенных значений для *целевого объекта* , описанного выше. Функция [glEvalCoord1](glevalcoord-functions.md) оценивает включенные одномерные карты. Когда **glEvalCoord1** представляет значение *u*, функции бернстеин оцениваются с помощью *u*^, где

![Уравнение, показывающее определение ^.](images/map04.png)

Параметры *stride*, *Order* и *points* определяют адресацию массива для доступа к контрольным точкам. Параметр *points* — это расположение первой контрольной точки, которое занимает одно, два, три или четыре смежных расположения памяти в зависимости от того, какая схема определена. Параметр *Order* — это количество контрольных точек в массиве. Параметр *stride* указывает, сколько расположений с плавающей запятой или Double следует переместить на указатель внутренней памяти, чтобы достичь следующей контрольной точки.

Как и в случае со всеми командами OpenGL, которые принимают указатели на данные, это так, как будто содержимое *точек* было скопировано с помощью **glMap1** перед возвратом. Изменения содержимого *точек* не оказывают влияния после вызова **glMap1** .
