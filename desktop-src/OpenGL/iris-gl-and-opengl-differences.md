# <a name="iris-gl-and-opengl-differences"></a>Различия в ГК и OpenGL для IRI

В этом приложении перечислены различия между OpenGL и IRI GL. Указывается термин для каждого отличия, за которым следует описание.



|                                      |                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
|--------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| заключение в накопление                | Операция с буфером накопления OpenGL не определяется, когда значения компонентов превышают 1,0 или удаляются ниже 1,0.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| сглаженные линии                    | Линии OpenGL стипплес со сглаженными линиями. В IRI GL нет.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| дуга                                  | OpenGL поддерживает дуги в своей библиотеке служебной программы.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| списки атрибутов                      | Атрибуты, помещаемые функцией IRI GL пушаттрибутес, отличаются от любых наборов атрибутов, отправленных Глпушаттриб OpenGL. Так как все состояния OpenGL можно считать обратно, можно реализовать любую желаемую семантику push-уведомлений с помощью OpenGL.                                                                                                                                                                                                                                                                                                                                                                                                                           |
| Автоматическое масштабирование текстур            | Интерфейс текстуры OpenGL не поддерживает автоматическое масштабирование изображений до двух измерений. Однако GLU поддерживает масштабирование изображений.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| ббокс                                 | OpenGL не поддерживает условное выполнение списков вывода.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| каллфунк                             | OpenGL не поддерживает обратный вызов из списков вывода. Обратите внимание, что функция IRI GL не поддерживает эту функциональность, если клиент и сервер находятся на разных платформах.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| circle                               | OpenGL поддерживает круги с GLU. В OpenGL как окружности, так и дуги (диски и частичные диски) могут иметь отверстия. Кроме того, можно изменить подразделение примитивов в OpenGL, чьи нормали поверхности доступны для освещения.                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| очистить параметры                        | OpenGL фактически очищает буферы. Она не применяет указанные в данный момент операции с пикселями, такие как смешивание и логикоп, независимо от их режимов. Чтобы отменить использование таких функций, необходимо отобразить многоугольник размера окна.                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| закрытые строки                         | OpenGL отображает все строки с псевдонимами одинарной ширины, так что смежные линии не имеют пикселей. Это означает, что последняя точка независимой линии не рисуется.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| Флаг цвета или обычного                    | Освещение OpenGL явно включено или отключено. Если этот параметр включен, он действует независимо от порядка, в котором указаны цвета и нормали. Вы не можете включить или отключить освещение между командами OpenGL Глбегин и Гленд. Чтобы отключить освещение между Глбегин и Гленд, укажите нулевой внешний, рассеянный и отраженный материал, а затем установите нужный цвет для раскраски материала.                                                                                                                                                                                                                                                            |
| вогнутых участков многоугольники                     | Основной API OpenGL не обрабатывает многоугольники вогнутых участков, но GLU поддерживает декомпозицию несамостоятельных контуров в треугольниках. Эти треугольники могут быть выведены сразу или в виде возврата.                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| текущий вычисленный цвет               | OpenGL не эквивалентен текущему вычисленному цвету. Если вы используете OpenGL в качестве механизма освещения, можно использовать обратную связь для получения цветов, созданных вычислениями освещения.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| Текущее расположение графика            | OpenGL не поддерживает текущую графическую точку. Команды IRI GL, которые зависят от текущей положения графики, такие как относительные линии и многоугольники, не включены в OpenGL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| кривые                               | OpenGL не поддерживает кривые с IRI GL. Рекомендуется использовать кривые НУРБС.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| дефс и привязки                           | OpenGL не имеет концепции объектов «материал», «светлой» или «текстура». только свойства "материал", "светлое" и "Текстура". Однако можно использовать списки вывода для создания собственных объектов.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| депскуе                             | OpenGL не обеспечивает прямую поддержку глубины cueing, но ее поддержка тумана является более общей возможностью, которую можно легко использовать для эмуляции функции IRI GL депскуе.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| Отображение изменения списка                 | Списки дисплеев OpenGL не могут быть едитедонли созданы и уничтожены. Однако можно указать имена списков дисплеев, но можно переопределить отдельные списки для показа в иерархии. Списки дисплеев OpenGL предназначены для кэширования данных, а не для управления базами данных. Они гарантированно хранятся на сервере в средах клиент-сервер, поэтому они не ограничиваются пропускной способностью сети во время выполнения. Списки дисплеев OpenGL можно вызывать между командами Глбегин и Гленд, поэтому иерархия списка отображаемых списков может быть достаточно точной, чтобы ее можно было изменить.                                                                                              |
| проверка ошибок                       | OpenGL проверяет наличие ошибок более тщательно, чем IRI GL. Например, все функции OpenGL, которые не принимаются между Глбегин и Гленд, обнаруживаются как ошибки и не имеют других результатов.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| возвращаемые значения ошибок                  | Если команда OpenGL, возвращающая значение, обнаруживает ошибку, она всегда возвращает ноль. Команды OpenGL, возвращающие данные через переданные указатели, не меняют содержимое массива при обнаружении ошибки.                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| побочные эффекты ошибки                   | Если команда OpenGL приводит к ошибке, ее единственный побочный результат — обновление флага ошибок до соответствующего значения. Другие изменения состояния не вносятся. (Исключением является \_ \_Ошибка памяти, которая является неустранимой.)                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| обратная связь                             | Обратная связь стандартизована в OpenGL, поэтому она не меняется с компьютера на компьютер.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| шрифты и строки                    | OpenGL требует, чтобы глифы символов обрабатываются как отдельные списки вывода. Он предоставляет вызывающую функцию списка вывода, которая принимает список имен списков вывода, каждое из которых представлено как 1, 2 или 4 байта. Функция Глкалллистс добавляет отдельно заданное смещение к каждому имени списка вывода перед вызовом, позволяя спискам имен отображаемых списков обрабатываться как строки. Этот механизм обеспечивает все функции шрифтов IRI GL и значительно больше. Например, можно легко манипулировать символами, составляющими треугольники.                                                                                                                  |
| фронтбуффер                          | IRI GL имеет сложные правила для подготовки к просмотру в интерфейсном буфере в режиме одного буфера. OpenGL обрабатывает отрисовку в интерфейсном буфере простым способом.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| пустые многоугольники                      | Можно использовать емкость набора элементов OpenGL для отрисовки пустых многоугольников. OpenGL не поддерживает другие средства для создания пустых многоугольников.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| Фиксация индекса                       | Там, где это возможно, OpenGL воспринимает индексы цветов и наборов элементов как битовые поля, а не числа. Таким же путем маскирование индексов, а не для фиксации, в поддерживаемый диапазон буфера кадров.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| цвета целых чисел                       | Компоненты цветового целого числа со знаком (красный, зеленый, синий или альфа) сопоставляются линейным образом с плавающими точками, чтобы наиболее отрицательное целое число сопоставлялось с 1,0, а самое положительное целое число сопоставляется 1,0. Это сопоставление происходит при указании цвета, по истечении которого OpenGL заменяет текущий цвет. Компоненты целочисленного цвета без знака сопоставляются линейно с плавающими точками, чтобы 0 сопоставлялся с 0,0, а максимальное целое число сопоставляется с 1,0. Это сопоставление происходит при указании цвета, по истечении которого OpenGL заменяет текущий цвет.                                                                                                                                               |
| целочисленные нормали                      | Целочисленные обычные компоненты сопоставляются точно так же, как и компоненты со знакомым цветом. Наиболее отрицательное целое число сопоставляется с 1,0, а самое положительное целое число сопоставляется с 1,0. фрагменты пикселей. Пиксели, рисуемые Глдравпикселс или Глкопипикселс, всегда растрируются и преобразуются в фрагменты. Полученные фрагменты — это текстурированные, фогжед, глубины в буфере, смешивание и т. д., так же, как будто они были созданы из геометрических точек. Данные фрагментов, не предоставляемые исходными пикселами, дополняются из текущей точки отсчета. Например, пикселы RGBA принимают координаты Z и координат текстуры. Пикселы глубины принимают цвет растровой точки и координаты текстуры. |
| инвариантность                           | OpenGL гарантирует определенную согласованность, которую не поддерживает IRI GL. Например, OpenGL гарантирует, что идентичные последовательности кода, отправленные в ту же систему, отличающиеся только указанной функцией смешивания, будут создавать одинаковые фрагменты пикселей. (Однако фрагменты различаются, если включить и отключить смешение.)                                                                                                                                                                                                                                                                                                                                                 |
| уравнение освещения                    | Уравнение освещения OpenGL немного отличается от формулы ДИАФРАГМы в ГК. OpenGL поддерживает отдельное затухание для каждого источника освещения, а не одно затухание для всех источников освещения, таких как IRI GL. OpenGL регулирует уравнение таким образом, чтобы все внешние, рассеянные и отраженные света были ослаблены. Кроме того, OpenGL позволяет задавать отдельные цвета для внешнего, рассеянного и зеркального интенситиес источников света, а также для внешнего, рассеянного и отраженного отражения материалов. Все цвета источника OpenGL и материала включают альфа. Установка значения "экспонента" для отраженного света не приводит к отсчету отраженного освещения в OpenGL.    |
| мапв                                 | Утилиты OpenGL поддерживают сопоставление координат объекта и окна.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| режим матрицы                          | Где функции IRI GL Орсо, ortho2, перспектива и окна работают с определенной матрицей, все операции матрицы OpenGL работают с текущей матрицей. Все операции матрицы OpenGL, за исключением Гллоадидентити и Гллоадматрикс, умножают текущую матрицу, вместо того чтобы заменять ее (как Орсо, ortho2, перспектива и окно в IRI GL).                                                                                                                                                                                                                                                                                                                       |
| MIP-карты, автоматическое создание        | Интерфейс текстуры OpenGL не поддерживает автоматическое создание образов mipmap. Однако GLU поддерживает автоматическое создание образов mipmap для одномерной и двумерной текстур.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Move, Draw, пмове/пдрав/пклос          | OpenGL поддерживает только графики начального и конечного стиля, так как не ведет текущую графическую точку. Однако для всех команд, связанных с вершиной, по-прежнему принимается спецификация скалярных параметров старых команд перемещения и рисования.                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| режим мпрожектион                     | IRI GL не преобразует геометрию в матрицу моделвиев в режиме матрицы проекции. OpenGL всегда преобразовывает как моделвиев, так и матрицу проекции независимо от режима матрицы.                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| Рисование с несколькими буферами                 | OpenGL преобразуется в каждый цветовой буфер по отдельности, вместо того чтобы вычислять одно новое значение цвета на основе содержимого одного буфера цвета и записывать его во все включенные буферы цветов, как в IRI GL.                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| нурбс                                | OpenGL поддерживает НУРБС с сочетанием основных возможностей (оценивающих) и поддержки GLU. Поддерживаются все возможности НУРБС для IRI GL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| Старый режим многоугольников                     | Для многоугольников с псевдонимами OpenGL всегда используется выборка точек. Режим совместимости с многоугольниками IRI GL, где Пиксели за пределами периметра многоугольника включены в его растрирование, не поддерживается. Если код использует этот режим многоугольника, то, вероятно, для прямоугольников. Старые прямоугольники в режиме многоугольников отображаются на один пиксель больше и выше.                                                                                                                                                                                                                                                                                                                                                |
| Упакованные форматы цвета                 | OpenGL принимает цвета как 8-разрядные компоненты, но эти компоненты обрабатываются как массив байтов, а не как байты, Упакованные в более крупные слова. Поощряя индексирование массивов, а не сдвигов, OpenGL обеспечивает неизменяемое программирование с обратным порядком байтов. Так же, как IRI GL принимает Упакованные цвета для геометрического рендеринга и отрисовки пикселей, OpenGL принимает массивы цветовых компонентов для геометрического рендеринга и отображения пикселей.                                                                                                                                                                                                                                                              |
| patches                              | OpenGL не поддерживает обновления IRI GL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| вритемаск цвета в битах              | OpenGL вритемаскс для компонентов цвета включает или отключает изменения для всего компонента (красного, зеленого, синего или альфа-канала), а не отдельных битов компонентов. Обратите внимание, что для каждого бита вритемаскс поддерживаются как для цветовых индексов, так и для индексов наборов элементов.                                                                                                                                                                                                                                                                                                                                                                                                              |
| Побитовая глубина вритемаск              | OpenGL вритемаскс для компонентов глубины включает или отключает изменения для всего компонента, а не отдельных битов компонента глубины.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| брать                                 | Библиотека служебной программы OpenGL включает поддержку создания матрицы подбора.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| координаты в пикселях                    | В стандартах OpenGL и IRI GL источник системы координат окна находится в левом нижнем углу. OpenGL размещает источник в левом нижнем углу этого пикселя, тогда как IRI GL помещает его в центре нижнего левого пикселя.                                                                                                                                                                                                                                                                                                                                                                                                                              |
| масштаб в пикселях                           | Коэффициенты негативного масштабирования OpenGL отражать текущее расположение графика. IRI GL не определяет операцию отрицательных коэффициентов масштабирования, а вместо этого предоставляет право \_ на \_ пиксмодес отражение справа налево и сверху \_ \_ вниз. Эти режимы отражения отображаются на месте, а не на текущей позиции растрового изображения. OpenGL не определяет режимы отражения.                                                                                                                                                                                                                                                                                                                    |
| пиксмоде                              | Передача пикселей OpenGL работает с отдельными цветовыми компонентами, а не с упакованными группами 4 8-разрядных компонентов, как и в IRI GL. Хотя OpenGL предоставляет значительно больше возможностей, чем IRI GL, он не поддерживает Упакованные конструкции цветов и не позволяет переназначить компоненты цвета (от красного до зеленого, от красного к синему и т. д.) во время операций копирования в пикселях.                                                                                                                                                                                                                                                                                    |
| полф/поли                            | OpenGL не предоставляет прямой поддержки для списков вершин, отличных от списков вывода. Однако такие функции, как полф и поли, можно легко реализовать с помощью API OpenGL.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| вершина заставляющегоа многоугольника             | Плоские сплошные многоугольники ДИАФРАГМы используют цвет последней заданной вершины, а многоугольники OpenGL — цвет первой указанной вершины.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| многоугольник стиппле                      | При использовании IRI GL шаблон стиппле многоугольника отсчитывается относительно экрана. При использовании OpenGL он отсчитывается относительно окна.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| число вершин многоугольника                 | Нет ограничений на количество вершин между Глбегин и Гленд с OpenGL, даже для Глбегин (МНОГОУГОЛЬНИКа). При использовании IRI GL многоугольники ограничены не более чем 255 вершинами.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| реаддисплай                          | Чтение пикселей за пределами границ окна является правильной функцией системы Windows, а не возможностью отрисовки. Используйте функции Windows для замены команды IRI GL реаддисплай.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| относительные перемещение/Рисование/пмове/пдрав/пклос | OpenGL не поддерживает текущую графическую точку, поэтому не поддерживают относительные операции вершин.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| RGBA () логикоп ()                       | OpenGL не поддерживает логические операции с буферами RGBA.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| сбокс ()                               | сбокс — это примитивная прямоугольная рамка ДИАФРАГМы, которая хорошо определена только в случае преобразования без вращения. Она предназначена для визуализации быстрее стандартных прямоугольников. Хотя OpenGL не поддерживает такой примитив, он может быть настроен на отрисовку прямоугольников очень быстро, когда матрицы и другие режимы находятся в состоянии, которое упрощает вычисления.                                                                                                                                                                                                                                                                                                                             |
| скалярные аргументы                     | Все команды OpenGL, которые принимаются между Глбегин и Гленд, имеют точки входа, принимающие скалярные аргументы. Например, glColor4f (красный, зеленый, синий, альфа).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| распашн                              | Функция OpenGL ГлсЦиссор не следит за окном просмотра. Команда в окне просмотра IRI GL автоматически обновляет скрмаск.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| скрбокс ()                             | OpenGL не поддерживает вычисление ограничивающего прямоугольника.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| скрсубдивиде ()                       | OpenGL не поддерживает подразделение экрана.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| режим одиночной матрицы                   | OpenGL всегда поддерживает две матрицы: Моделвиев и проекция. Хотя реализация OpenGL может объединять их в одну матрицу для повышения производительности, она должна всегда представлять собой модель с двумя матрицами.                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| режим подпикселя                        | Вся визуализация OpenGL — это поситионедсубпиксел режим "всегда включено".                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| сваптмеш ()                          | OpenGL не поддерживает возможность сваптмеш. Однако он предлагает два типа сеток треугольников: один, который соответствует поведению по умолчанию для «полосковой» линии ДИАФРАГМы, и другой, соответствующий вызову сваптмеш до третьего и всех последующих вершин при использовании IRI GL.                                                                                                                                                                                                                                                                                                                                                                           |
| аргументы вектора                     | Все команды OpenGL, которые принимаются между Глбегин и Гленд, имеют точки входа, принимающие аргументы вектора. Например, glColor4fv.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| управление окнами                    | OpenGL не включает системные команды Window. Он всегда поддерживается как расширение для окна или операционной системы, которое включает возможности управления устройствами и окнами. Каждое расширение предоставляет специальный системный механизм для создания, уничтожения и манипулирования контекстами визуализации OpenGL. Например, расширение OpenGL для системы Window X (ГЛКС) включает примерно 10 команд для этой цели. Такие команды IRI GL, как гконфиг и DrawMode, не реализованы с помощью OpenGL.                                                                                                                                                                            |
| смещение окна                        | IRI GL возвращает окно просмотра и позиции символов на экране, а не окна, координаты. OpenGL всегда использует координаты окна.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| отрисовка по оси z                          | OpenGL не поддерживает отрисовку цветов в буфере глубины. Он позволяет использовать дополнительные буферы цветов, которые можно реализовать с помощью той же памяти, которая используется для буферов глубины в других конфигурациях окна. Но эти дополнительные буферы цветов не могут совместно использовать память с буфером глубины в любой отдельной конфигурации.                                                                                                                                                                                                                                                                                                                                          |
