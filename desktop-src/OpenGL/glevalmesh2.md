# <a name="glevalmesh2-function"></a>Функция glEvalMesh2

Рассчитывает двухмерную сетку точек или линий.

## <a name="syntax"></a>Синтаксис


```C++
void WINAPI glEvalMesh2(
   GLenum mode,
   GLint  i1,
   GLint  i2,
   GLint  j1,
   GLint  j2
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*mode* 
</dt> <dd>

Значение, указывающее, следует ли вычислять двухмерную сетку точек, линий или многоугольников. Допустимы следующие символьные константы: \_ «точка GL», « \_ линия GL» и « \_ Заливка GL».

</dd> <dt>

*I1* 
</dt> <dd>

Первое целое значение для переменной домена сетки i.

</dd> <dt>

*I2* 
</dt> <dd>

Последнее целочисленное значение для переменной домена сетки i.

</dd> <dt>

*J1* 
</dt> <dd>

Первое целочисленное значение для переменной домена сетки j.

</dd> <dt>

*J2* 
</dt> <dd>

Последнее целочисленное значение для переменной домена сетки j.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Эта функция не возвращает значение.

## <a name="remarks"></a>Комментарии

Используйте [**глмапгрид**](glmapgrid-functions.md) и [**глевалмеш**](glevalmesh-functions.md) в сочетании для эффективного создания и вычисления ряда значений домена сопоставлений с равными пробелами. Функция **глевалмеш** выполняет шаг с целым доменом одномерной или двухмерной сетки, диапазон которого представляет собой домен карт оценки, заданных в [**glMap1**](glmap1.md) и [**glMap2**](glmap2.md). Параметр mode определяет, соединяются ли результирующие вершины как точки, линии или закрашенные многоугольники.

В двухмерном регистре **glEvalMesh2**, Let

? u = (U2 U1)/n

? v = (V2 v1)/m,

где n, U1, U2, m, v1 и v2 являются аргументами самой последней функции [**glMapGrid2**](glmapgrid-functions.md) . Затем, если *mode* имеет значение GL \_ Fill, **glEvalMesh2** эквивалентно следующему:

для (j = J1; j < J2; j + = 1)

{

[**глбегин**](glbegin.md)( \_ четыре \_ ленты GL);

для (i = I1; я <= I2; i + = 1)

{

[**glEvalCoord2**](glevalcoord2d.md)(i? u + U1 (), j? v + v1);

[**glEvalCoord2**](glevalcoord2d.md)(i? u + U1 (), (j + 1)? v + v1);

}

[**гленд**](glend.md)(); }

Если *режим* является \_ строкой GL, вызов **glEvalMesh2** эквивалентен следующему:

для (j = J1; j <= J2; j + = 1)

{

[**глбегин**](glbegin.md)( \_ \_ полоса строк GL);

для (i = I1; я <= I2; i + = 1)

{

[**glEvalCoord2**](glevalcoord2d.md)(i? u + U1, j? v + v1);

}

[**гленд**](glend.md)();

}

для (i = I1; я <= I2; i + = 1)

{

[**глбегин**](glbegin.md)( \_ \_ полоса строк GL);

для (j = J1; j <= J1; j + = 1)

{

[**glEvalCoord2**](glevalcoord2d.md)(i? u + U1, j? v + v1);

}

Гленд ();

}

И, наконец, если *mode* является \_ точкой GL, вызов **glEvalMesh2** эквивалентен следующему:

[**глбегин**](glbegin.md)( \_ пункты GL);

для (j = J1; j <= J2; j + = 1)

{

для (i = I1; я <= I2; i + = 1)

{

[**glEvalCoord2**](glevalcoord2d.md)(i? u + U1, j? v + v1);

}

}

[**гленд**](glend.md)();

Во всех трех случаях единственными абсолютными числовыми требованиями является то, что если i = n, то значение, вычисленное из i? u + U1 — это ровно U2, а если j = m, то значение, вычисленное из j? v + v1 — ровно 2. Следующие функции извлекают сведения, относящиеся к **глевалмеш**:

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументом \_ GL \_ «Карта1» \_ домен Grid

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументом \_ GL \_ MAP2 \_ домен Grid

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументами \_ \_ сегментами сетки GL «Карта1» \_

[**глжет**](glgetbooleanv--glgetdoublev--glgetfloatv--glgetintegerv.md) с аргументами \_ \_ сегментами сетки GL MAP2 \_
