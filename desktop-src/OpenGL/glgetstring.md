# <a name="glgetstring-function"></a>Функция Глжетстринг

Функция **глжетстринг** возвращает строку, описывающую текущее подключение OpenGL.

## <a name="syntax"></a>Синтаксис


```C++
const GLubyte* WINAPI glGetString(
   GLenum name
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*name* 
</dt> <dd>

Одна из следующих символьных констант.

|---------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| <span id="GL_VENDOR"></span><span id="gl_vendor"></span><dl> <dt>**\_поставщик GL**</dt> </dl>             | Возвращает компанию, ответственную за эту реализацию OpenGL. Это имя не меняется с выпуска на выпуск.<br/>                                                  |
| <span id="GL_RENDERER"></span><span id="gl_renderer"></span><dl> <dt>**модуль \_ подготовки отчетов GL**</dt> </dl>       | Возвращает имя модуля подготовки отчетов. Это имя обычно характерно для конкретной конфигурации аппаратной платформы. Он не меняется с выпуска на выпуск.<br/> |
| <span id="GL_VERSION"></span><span id="gl_version"></span><dl> <dt>**\_версия GL**</dt> </dl>          | Возвращает версию или номер выпуска.<br/>                                                                                                                                |

## <a name="remarks"></a>Комментарии

Функция **глжетстринг** возвращает указатель на статическую строку, описывающую некоторые аспекты текущего подключения OpenGL.

Поскольку OpenGL не включает запросы для характеристик производительности реализации, предполагается, что некоторые приложения будут написаны для распознавания известных платформ и изменят использование OpenGL на основе известных характеристик производительности этих платформ. Строки "поставщик GL" \_ и " \_ Подготовка к выпуску GL" вместе уникальным образом определяют платформу и не будут изменяться с выпуска на выпуск. Они должны использоваться алгоритмами распознавания платформ.

Формат и содержимое строки, возвращаемой **глжетстринг** , зависят от реализации, за исключением того, что:

-   Имена расширений не будут содержать пробелы и будут разделены символами пробела в \_ строке РАСШИРЕНИЙ GL.
-   \_Строка версии GL начинается с номера версии. Номер версии использует одну из следующих форм:

    *основной \_ номер*. *дополнительный \_ номер*

    *основной \_ номер*. *дополнительный \_ номер*. *\_ номер выпуска*

-   Сведения, относящиеся к поставщику, могут соответствовать номеру версии. Его формат зависит от реализации, но пробел всегда отделяет номер версии и сведения, относящиеся к поставщику.
-   Все строки завершаются нулем.

Если возникает ошибка, **глжетстринг** возвращает ноль.
