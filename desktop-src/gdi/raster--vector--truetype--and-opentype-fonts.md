# <a name="raster-vector-truetype-and-opentype-fonts"></a><span data-ttu-id="db28c-103">Растровые, векторные, шрифты TrueType и OpenType</span><span class="sxs-lookup"><span data-stu-id="db28c-103">Raster, Vector, TrueType, and OpenType Fonts</span></span>

<span data-ttu-id="db28c-104">Для отображения и печати текста приложения могут использовать четыре различных типа технологий шрифтов:</span><span class="sxs-lookup"><span data-stu-id="db28c-104">Applications can use four different kinds of font technologies to display and print text:</span></span>

-   <span data-ttu-id="db28c-105">Графике</span><span class="sxs-lookup"><span data-stu-id="db28c-105">Raster</span></span>
-   <span data-ttu-id="db28c-106">Вектор</span><span class="sxs-lookup"><span data-stu-id="db28c-106">Vector</span></span>
-   <span data-ttu-id="db28c-107">TrueType</span><span class="sxs-lookup"><span data-stu-id="db28c-107">TrueType</span></span>
-   <span data-ttu-id="db28c-108">Microsoft OpenType</span><span class="sxs-lookup"><span data-stu-id="db28c-108">Microsoft OpenType</span></span>

<span data-ttu-id="db28c-109">Различия между этими шрифтами отражены в том, как *глиф* для каждого символа или символа хранится в соответствующем файле ресурсов Font:</span><span class="sxs-lookup"><span data-stu-id="db28c-109">The differences between these fonts reflect the way that the *glyph* for each character or symbol is stored in the respective font-resource file:</span></span>

-   <span data-ttu-id="db28c-110">В растровых шрифтах глиф — это точечный рисунок, используемый системой для рисования одного символа или символа в шрифте.</span><span class="sxs-lookup"><span data-stu-id="db28c-110">In raster fonts, a glyph is a bitmap that the system uses to draw a single character or symbol in the font.</span></span>
-   <span data-ttu-id="db28c-111">В векторных шрифтах глиф — это коллекция конечных точек линий, которые определяют сегменты линии, используемые системой для рисования символа или символа в шрифте.</span><span class="sxs-lookup"><span data-stu-id="db28c-111">In vector fonts, a glyph is a collection of line endpoints that define the line segments that the system uses to draw a character or symbol in the font.</span></span>
-   <span data-ttu-id="db28c-112">В шрифтах TrueType и OpenType глиф — это коллекция команд line и кривой, а также коллекция подсказок.</span><span class="sxs-lookup"><span data-stu-id="db28c-112">In TrueType and OpenType fonts, a glyph is a collection of line and curve commands as well as a collection of hints.</span></span>

<span data-ttu-id="db28c-113">Система использует команды line и кривых для определения контура точечного рисунка для символа или символа в шрифте TrueType или Microsoft OpenType.</span><span class="sxs-lookup"><span data-stu-id="db28c-113">The system uses the line and curve commands to define the outline of the bitmap for a character or symbol in the TrueType or Microsoft OpenType font.</span></span> <span data-ttu-id="db28c-114">Система использует указания для корректировки длины линий и фигур кривых, используемых для рисования символа или символа.</span><span class="sxs-lookup"><span data-stu-id="db28c-114">The system uses the hints to adjust the length of the lines and shapes of the curves used to draw the character or symbol.</span></span> <span data-ttu-id="db28c-115">Эти указания и соответствующие корректировки основываются на объеме масштабирования, используемом для уменьшения или увеличения размера точечного рисунка.</span><span class="sxs-lookup"><span data-stu-id="db28c-115">These hints and the respective adjustments are based on the amount of scaling used to reduce or increase the size of the bitmap.</span></span> <span data-ttu-id="db28c-116">Шрифт OpenType эквивалентен шрифту TrueType, за исключением того, что шрифт OpenType допускает определения глифов PostScript в дополнение к определениям глифов TrueType.</span><span class="sxs-lookup"><span data-stu-id="db28c-116">An OpenType font is equivalent to a TrueType font except that an OpenType font allows PostScript glyph definitions in addition to TrueType glyph definitions.</span></span>

<span data-ttu-id="db28c-117">Поскольку точечные рисунки для каждого глифа в растровом шрифте предназначены для определенного разрешения устройства, точечные шрифты обычно считаются зависящими от устройства.</span><span class="sxs-lookup"><span data-stu-id="db28c-117">Because the bitmaps for each glyph in a raster font are designed for a specific resolution of device, raster fonts are generally considered to be device dependent.</span></span> <span data-ttu-id="db28c-118">Векторные шрифты, с другой стороны, не зависят от устройства, так как каждый глиф хранится в виде коллекции масштабируемых линий.</span><span class="sxs-lookup"><span data-stu-id="db28c-118">Vector fonts, on the other hand, are not device dependent, because each glyph is stored as a collection of scalable lines.</span></span> <span data-ttu-id="db28c-119">Однако векторные шрифты обычно рисуются медленнее, чем растровые, шрифты TrueType и OpenType.</span><span class="sxs-lookup"><span data-stu-id="db28c-119">However, vector fonts are generally drawn more slowly than raster or TrueType and OpenType fonts.</span></span> <span data-ttu-id="db28c-120">Шрифты TrueType и OpenType обеспечивают как относительно быструю скорость рисования, так и независимость от устройства.</span><span class="sxs-lookup"><span data-stu-id="db28c-120">TrueType and OpenType fonts provide both relatively fast drawing speed and true device independence.</span></span> <span data-ttu-id="db28c-121">С помощью указаний, связанных с глифом, разработчик может масштабировать символы шрифта TrueType или OpenType вверх или вниз, сохраняя их исходную форму.</span><span class="sxs-lookup"><span data-stu-id="db28c-121">By using the hints associated with a glyph, a developer can scale the characters from a TrueType or OpenType font up or down and still maintain their original shape.</span></span>

<span data-ttu-id="db28c-122">Как упоминалось ранее, глифы для шрифта хранятся в файле шрифта-ресурса.</span><span class="sxs-lookup"><span data-stu-id="db28c-122">As previously mentioned, the glyphs for a font are stored in a font-resource file.</span></span> <span data-ttu-id="db28c-123">Файл ресурсов шрифтов фактически является библиотекой DLL, содержащей только данные, но код отсутствует.</span><span class="sxs-lookup"><span data-stu-id="db28c-123">A font-resource file is actually a DLL that contains only data, there is no code.</span></span> <span data-ttu-id="db28c-124">Для растровых и векторных шрифтов эти данные делятся на две части: заголовок, описывающий метрики шрифта и данные глифов.</span><span class="sxs-lookup"><span data-stu-id="db28c-124">For raster and vector fonts, this data is divided into two parts: a header describing the font's metrics and the glyph data.</span></span> <span data-ttu-id="db28c-125">Файл ресурсов шрифта для растрового или векторного шрифта определяется расширением имени файла FON.</span><span class="sxs-lookup"><span data-stu-id="db28c-125">A font-resource file for a raster or vector font is identified by the .fon file name extension.</span></span> <span data-ttu-id="db28c-126">Для шрифтов TrueType и OpenType существует два файла для каждого шрифта: первый файл содержит сравнительно короткий заголовок, а второй содержит фактические данные шрифта.</span><span class="sxs-lookup"><span data-stu-id="db28c-126">For TrueType and OpenType fonts, there are two files for each font: the first file contains a relatively short header and the second contains the actual font data.</span></span> <span data-ttu-id="db28c-127">Первый файл идентифицируется расширением. учетной, а второй — расширением TTF.</span><span class="sxs-lookup"><span data-stu-id="db28c-127">The first file is identified by an .fot extension and the second is identified by a .ttf extension.</span></span>
