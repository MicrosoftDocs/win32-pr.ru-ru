<span data-ttu-id="fbb52-107">Функции, параметры и конструкции параметров используются в документе PrintCapabilities для представления атрибутов устройства, которые вносят вклад в определение конфигурации устройства.</span><span class="sxs-lookup"><span data-stu-id="fbb52-107">Feature/Option and parameter constructs are used in a PrintCapabilities document to represent device attributes that contribute to the definition of the device configuration.</span></span> <span data-ttu-id="fbb52-108">Для примера конструкции "функция-вариант" рассмотрите печатающее устройство, которое может печататься в нескольких решениях.</span><span class="sxs-lookup"><span data-stu-id="fbb52-108">For an example of a Feature/Option construct, consider a printing device that is capable of printing in several resolutions.</span></span> <span data-ttu-id="fbb52-109">Атрибут устройства, определяющий разрешение, может быть представлен как экземпляр компонента, при этом каждое из возможных значений разрешения вывода представлено в виде отдельного экземпляра параметра.</span><span class="sxs-lookup"><span data-stu-id="fbb52-109">The device attribute that defines the resolution can be represented as a Feature instance, with each of the possible output resolution values represented as an individual Option instance.</span></span> <span data-ttu-id="fbb52-110">Один экземпляр параметра может представлять разрешение 300 dpi, другое может представлять разрешение 600 dpi и т. д.</span><span class="sxs-lookup"><span data-stu-id="fbb52-110">One Option instance can represent a resolution of 300 dpi, another can represent a resolution of 600 dpi, and so on.</span></span>

<span data-ttu-id="fbb52-111">Следует отметить, что схема печати требует, чтобы список компонентов, параметров и экземпляров Параметердеф, сообщаемых в каждом документе PrintCapabilities, оставался постоянным независимо от конфигурации.</span><span class="sxs-lookup"><span data-stu-id="fbb52-111">It should be noted that the Print Schema requires that the list of Feature, Option, and ParameterDef instances reported in each PrintCapabilities document must remain constant, independent of the configuration.</span></span> <span data-ttu-id="fbb52-112">Это позволяет однозначным образом выразить конфигурации и зависимости в конфигурациях.</span><span class="sxs-lookup"><span data-stu-id="fbb52-112">This allows configurations and dependencies on configurations to be expressed in an unambiguous way.</span></span> <span data-ttu-id="fbb52-113">Следствием этого требования является то, что у каждой функции и подфункции должен быть четко определенный параметр, независимо от настройки любой другой функции или подфункции.</span><span class="sxs-lookup"><span data-stu-id="fbb52-113">The implication of this requirement is that every Feature and subfeature must have a well-defined setting, regardless of the setting of any other Feature or subfeature.</span></span> <span data-ttu-id="fbb52-114">Таким образом, каждая подфункция должна иметь параметр, эквивалентный отсутствию op (параметр Off, disabled или None), который автоматически выбирается для всех подкомпонентов, когда пользователь выбирает в родительском компоненте параметр без-Op.</span><span class="sxs-lookup"><span data-stu-id="fbb52-114">Therefore, each subfeature must have an Option that is equivalent to a no-op (an Off, Disabled, or None setting), that is automatically selected for all subfeatures when the user selects the no-op Option in the parent Feature.</span></span> <span data-ttu-id="fbb52-115">И наоборот, если для одной из подкомпонентов задан параметр Enabled (включен), родительский компонент и другие связанные с ним подфункции также будут включены.</span><span class="sxs-lookup"><span data-stu-id="fbb52-115">Conversely, when one of the subfeatures is set to an enabled Option, the parent Feature and other associated subfeatures are also enabled.</span></span> <span data-ttu-id="fbb52-116">Тем временем поставщик PrintTicket должен гарантировать (во время проверки PrintTicket), что определяются параметры для всех экземпляров компонента и компонентов, независимо от конфигурации устройства, и что параметры подфункции соответствуют параметрам родительского компонента.</span><span class="sxs-lookup"><span data-stu-id="fbb52-116">In the meantime, the PrintTicket provider must ensure (during PrintTicket validation) that settings for all Feature and subfeature instances are defined, regardless of the device configuration, and that the subfeature settings are consistent with the settings of the parent Feature.</span></span> <span data-ttu-id="fbb52-117">Это гарантирует, что устройству не будет присвоено несоответствие PrintTicket, где некоторые подфункции указывают, что, например, сшивание включена, но другие подфункции указывают, что Ассоциация отключена.</span><span class="sxs-lookup"><span data-stu-id="fbb52-117">This ensures that the device is not given an inconsistent PrintTicket where some subfeatures indicate that, for example, stapling is enabled, but other subfeatures indicate that stapling is disabled.</span></span>

<span data-ttu-id="fbb52-118">Обратите внимание, что авторы PrintCapabilities не обязаны использовать возможности вложенности элементов функций.</span><span class="sxs-lookup"><span data-stu-id="fbb52-118">Note that PrintCapabilities authors are not required to utilize the nesting capabilities of Feature elements.</span></span> <span data-ttu-id="fbb52-119">При желании они могут представлять все экземпляры компонентов в плоской структуре как элементы того же уровня.</span><span class="sxs-lookup"><span data-stu-id="fbb52-119">If they prefer, they can present all Feature instances in a flat structure, as siblings of each other.</span></span> <span data-ttu-id="fbb52-120">Обратите внимание, что вложенная коллекция экземпляров компонентов может быть сведена просто путем перемещения всех подкомпонентов на корневой уровень.</span><span class="sxs-lookup"><span data-stu-id="fbb52-120">Note also that a nested collection of Feature instances can be flattened simply by moving all of the subfeatures out to the root level.</span></span> <span data-ttu-id="fbb52-121">Единственная мера предосторожности, которую следует предпринять, заключается в том, чтобы убедиться, что атрибуты имени этих экземпляров компонентов уникальны.</span><span class="sxs-lookup"><span data-stu-id="fbb52-121">The only precaution that must be taken is to ensure that the name attributes of these Feature instances are unique.</span></span>
