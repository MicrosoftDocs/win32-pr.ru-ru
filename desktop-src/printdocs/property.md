# <a name="property-documents-and-printing"></a>Свойство (документы и печать)

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Элемент Property объявляет устройство, форматирование задания или другое соответствующее свойство, имя которого задается атрибутом Name. Элемент value используется для присвоения значения свойству.

Свойство может быть сложным, возможно, содержать несколько вложенных свойств. Вложенные свойства также представлены элементами свойств.

## <a name="element-tag"></a>Тег элемента

<Property>

| name<br/> | Содержит атрибут Name свойства, которое является либо стандартным свойством, либо закрытым свойством. <br/> |

## <a name="configuration-dependencies"></a>Зависимости конфигурации

Свойство может иметь зависимости конфигурации, кроме случаев, когда оно присутствует в элементе Параметердеф.

## <a name="element-usage"></a>Использование элементов

В дополнение к отображению в элементах Feature и Option, элементы свойств могут отображаться на корневом уровне соответствующих базовых технологий. Схема печати определяет набор элементов свойств, которые можно использовать для описания устройства переносимым способом. Однако если эти свойства недостаточно для ваших потребностей в качестве поставщика PrintCapabilities (обычно потому, что поддерживаемое устройство имеет романские аспекты, не ожидаемые схемой печати), вы можете ввести собственные закрытые элементы свойств. Вы можете усовершенствовать или проанализировать сведения, предоставляемые общедоступным свойством, добавив одно или несколько частных вложенных свойств в качестве содержимого элемента общего свойства.

Элементы свойств определяются с помощью тега XML-элемента <Property> . Каждому свойству присваивается имя с помощью его атрибута Name. Имя должно быть типа QName XML и должно соответствовать соглашению о пространстве имен. Дополнительные сведения см. в разделе [атрибуты XML](xml-attributes.md). Атрибут имени свойства и его расположение в иерархии родительских элементов свойств (если оно является подсвойством) уникально идентифицируют свойство в документе PrintCapabilities или PrintTicket.

Свойство может содержать один или несколько элементов значения или один или несколько дочерних элементов свойств (называемых вложенными свойствами) или их сочетание. Подсвойства полезны, если само свойство состоит из нескольких компонентов. Например, свойство "Консумаблеколор" может иметь компоненты "C", "M" и "Y".

## <a name="example"></a>Пример

``` syntax
<psf:Property name="psk:DisplayName">
  <psf:Value xsi:type="xs:string">6</psf:Value>
</psf:Property>
```
