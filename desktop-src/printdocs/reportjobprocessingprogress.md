# <a name="reportjobprocessingprogress-function"></a>Функция Репортжобпроцессингпрогресс

Сообщает службе диспетчера очереди печати, находится ли задание печати XPS в очереди печати или на этапе отрисовки и какая часть обработки в данный момент выполняется.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT ReportJobProcessingProgress(
  _In_ HANDLE                printerHandle,
  _In_ ULONG                 jobId,
       EPrintXPSJobOperation jobOperation,
       EPrintXPSJobProgress  jobProgress
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*принтерхандле* \[ окне\]
</dt> <dd>

Маркер принтера, для которого функция возвращает сведения. Используйте функцию [**опенпринтер**](openprinter.md) или [**аддпринтер**](addprinter.md) для получения маркера принтера.

</dd> <dt>

Идентификатор *задания* \[ окне\]
</dt> <dd>

Определяет задание печати, для которого извлекаются данные. Чтобы получить идентификатор задания печати, используйте функцию [**AddJob**](addjob.md) или функцию [**стартдок**](/windows/desktop/api/Wingdi/nf-wingdi-startdoca) .

</dd> <dt>

*жобоператион* 
</dt> <dd>

Указывает, находится ли задание на этапе постановки в очередь или на этапе отрисовки.

</dd> <dt>

*жобпрогресс* 
</dt> <dd>

Указывает, какая часть обработки в данный момент выполняется. Это значение относится к событиям в фазе буферизации или подготовки к просмотру в зависимости от значения *жобоператион*.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если операция выполнена успешно, возвращается значение S \_ ОК, в противном случае **HRESULT** будет содержать код ошибки.

Дополнительные сведения о кодах ошибок COM см. в разделе [Обработка ошибок](../com/error-handling-in-com.md).

## <a name="remarks"></a>Комментарии

> [!Note]  
> Это блокирующая или синхронная функция, которая может не возвращать значение немедленно. Скорость возврата этой функцией зависит от факторов времени выполнения, таких как состояние сети, Конфигурация сервера печати и факторы реализации драйвера принтера, которые трудно предсказать при написании приложения. Вызов этой функции из потока, который управляет взаимодействием с пользовательским интерфейсом, может привести к невозможности реагирования приложения.

 

> [!Note]  
> **Репортжобпроцессингпрогресс** будет сообщать о ходе выполнения задания печати XPS только в том случае, если задание печати находится на этапе буферизации или подготовки к просмотру. **Репортжобпроцессингпрогресс** завершится ошибкой, если он вызывается, когда задание печати XPS не находится на этапе буферизации или подготовки к просмотру.
