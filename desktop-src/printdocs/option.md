# <a name="option"></a>Параметр

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Элемент Option содержит все элементы Property и Скоредпроперти, связанные с этим параметром.

## <a name="element-tag"></a>Тег элемента

<Option>

|------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| ограниченного<br/> | Этот атрибут является необязательным для документа PrintCapabilities.<br/> Этот атрибут указывает, доступен ли параметр для выбора или использования. Этому XML-атрибуту можно присвоить одно из следующих значений: None, Принттиккетсеттингс, Админсеттинг или Девицесеттингс. <br/> См. [атрибуты XML](xml-attributes.md)<br/> |
| name<br/>        | Содержит имя параметра (стандартный или закрытый).. XML-атрибут используется таким образом для различения экземпляров параметров. <br/>                                                                                                                                                        |



 

Дополнительные сведения см. в разделе [XML Attributes](xml-attributes.md) .

## <a name="configuration-dependencies"></a>Зависимости конфигурации

Элемент определения параметра не может иметь зависимостей конфигурации.

## <a name="element-usage"></a>Использование элементов

Цель элемента Option состоит в том, чтобы определить одно из состояний, которые может предположить атрибут конфигурации устройства, представленный элементом Feature. Каждое определение элемента параметра содержит один или несколько элементов Скоредпроперти, описывающих внутреннюю или важнейшую характеристику этого параметра. Чтобы упростить перенос и сохранение намерения, схема печати определяет множество общих элементов функций и различные элементы параметров для каждой функции. Поэтому важно использовать элементы параметров печати, определенные схемой, если это возможно, прежде чем создавать собственные определения параметров. понимание процесса определения элементов параметров предоставляет полезные сведения о том, как документ PrintCapabilities и printticket используются в архитектуре печати Microsoft платформа .NET Framework версии 3,0 и Windows Vista.

## <a name="example"></a>Пример

В следующем примере определяется имя для параметра.

``` syntax
<psf:Option name="psk:PrintFront" />
```
