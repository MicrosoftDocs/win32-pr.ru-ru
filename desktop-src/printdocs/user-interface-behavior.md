# <a name="user-interface-behavior"></a>Поведение пользовательского интерфейса

Этот раздел не является актуальным. Самые актуальные сведения см. в [спецификации печати схемы](https://download.microsoft.com/download/D/E/C/DECA6E6B-3E81-48E7-B7EF-6D92A547D03C/print-schema-spec-2-0.zip).

Предположим, что создается клиент PrintCapabilities, который считывает документ PrintCapabilities и выбирает один или несколько параметров из каждого компонента и использует их для создания PrintTicket, указывающего конфигурацию, которая будет использоваться для обработки задания. Для каждой интересующей функции клиент должен проверить каждый вариант и решить, подходит ли этот параметр для поставленной задачи. Для параметра, который не является параметризованным, его можно оценить, обращаясь к значению каждого Скоредпроперти. В случае с параметром непараметризованного размера носителя клиент определяет, соответствуют ли размеры и высота носителя, сообщаемого в каждом параметре, с необходимыми измерениями.

В случае параметризованного параметра клиент должен получить доступ к экземпляру Параметердеф, указанному в экземпляре ParameterRef, который содержится в одном из экземпляров Скоредпроперти. Параметердеф обычно определяет диапазон значений, допустимых для параметра, а также единицы (дюймы, mm и т. д.), представленные значением. Если требуемые размеры попадают в диапазон значений, допустимых для каждого из параметров, клиент имеет дополнительную задачу инициализации параметров (с помощью экземпляра Параметеринит) в PrintTicket. Это особенно важная задача. Например, PrintTicket не должен указывать пользовательский размер носителя без указания размеров носителя, так как итоговый объект PrintTicket будет неоднозначным и неверно определен.

Аналогичный набор обстоятельств должен быть обработан, если клиент является пользовательским интерфейсом. Пользовательский интерфейс обычно отображает значения экземпляров Скоредпроперти, определенных для каждого параметра. Для ясности важно отобразить допустимые диапазон и единицы для параметров в параметризованном параметре. Кроме того, если пользователь выбирает параметр с параметрами, Пользовательский интерфейс должен предлагать пользователю ввести значение, которое будет использоваться для инициализации каждого параметра. Наконец, Пользовательский интерфейс должен составлять PrintTicket, отражающий все выборы пользователя.

Дополнительные сведения о конструкции PrintTicket и спецификации инициализации параметров см. в разделе [создание Device-Specific PrintTicket](creating-a-device-specific-printticket.md). Дополнительные сведения о разыменовании экземпляров ParameterRef и интерпретации экземпляров Параметердеф см. [в разделе Использование параметров](using-parameters.md).
