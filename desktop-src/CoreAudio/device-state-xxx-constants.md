<span data-ttu-id="b2220-125">Методы [**иммдевицеенумератор:: енумаудиоендпоинтс**](/windows/desktop/api/Mmdeviceapi/nf-mmdeviceapi-immdeviceenumerator-enumaudioendpoints), [**иммдевице::**](/windows/desktop/api/Mmdeviceapi/nf-mmdeviceapi-immdevice-getstate)и [**иммнотификатионклиент:: ОНДЕВИЦЕСТАТЕЧАНЖЕД**](/windows/desktop/api/Mmdeviceapi/nf-mmdeviceapi-immnotificationclient-ondevicestatechanged) используют \_ \_ константы состояния устройства XXX.</span><span class="sxs-lookup"><span data-stu-id="b2220-125">The [**IMMDeviceEnumerator::EnumAudioEndpoints**](/windows/desktop/api/Mmdeviceapi/nf-mmdeviceapi-immdeviceenumerator-enumaudioendpoints), [**IMMDevice::GetState**](/windows/desktop/api/Mmdeviceapi/nf-mmdeviceapi-immdevice-getstate), and [**IMMNotificationClient::OnDeviceStateChanged**](/windows/desktop/api/Mmdeviceapi/nf-mmdeviceapi-immnotificationclient-ondevicestatechanged) methods use the DEVICE\_STATE\_XXX constants.</span></span> <span data-ttu-id="b2220-126">Эти методы позволяют клиентам получать сведения об устройствах конечных точек, которые находятся в любом из состояний, представленных \_ \_ константами состояния устройства XXX.</span><span class="sxs-lookup"><span data-stu-id="b2220-126">These methods enable clients to obtain information about endpoint devices that are in any of the states represented by the DEVICE\_STATE\_XXX constants.</span></span>

<span data-ttu-id="b2220-128">В панели управления мультимедиа Windows Mmsys.cpl отображаются устройства конечных точек аудио в системе.</span><span class="sxs-lookup"><span data-stu-id="b2220-128">The Windows multimedia control panel, Mmsys.cpl, displays the audio endpoint devices in the system.</span></span> <span data-ttu-id="b2220-129">Отключение устройства в Mmsys.cpl скрывает устройство от механизмов обнаружения устройств в API-интерфейсах более высокого уровня, но не делает недействительными все объекты потока, экземпляры которых могли быть созданы клиентом до отключения устройства.</span><span class="sxs-lookup"><span data-stu-id="b2220-129">Disabling a device in Mmsys.cpl hides the device from the device-discovery mechanisms in higher-level audio APIs, but it does not invalidate any stream objects that a client might have instantiated before the device was disabled.</span></span> <span data-ttu-id="b2220-130">Например, если поток воспроизводится на устройстве, когда пользователь отключает его в Mmsys.cpl, поток продолжит воспроизводиться непрерывно.</span><span class="sxs-lookup"><span data-stu-id="b2220-130">For example, if a stream is playing on the device when the user disables it in Mmsys.cpl, the stream continues to play uninterrupted.</span></span>
