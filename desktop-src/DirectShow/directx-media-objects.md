# <a name="directx-media-objects"></a>Объекты мультимедиа DirectX

> [!Note]  
> Дмос были заменены [Media Foundation преобразованиями](/windows/desktop/medfound/media-foundation-transforms) (МФТС). интерфейсы DMO по-прежнему поддерживаются. Однако при написании пользовательского кодека или подключаемого модуля обработки звука и видео следует рассмотреть возможность его реализации как MFT.

 

Объекты мультимедиа DirectX (дмос) — это компоненты потоковой передачи данных на основе COM. в некоторых отношениях дмос похожи на фильтры Microsoft DirectShow. как и DirectShow фильтры, дмос принимает входные данные и использует их для создания выходных данных. Однако прикладные программные интерфейсы (API) для дмос гораздо проще, чем соответствующие API для DirectShow. В результате дмос проще в создании, тестировании и использовании. Дмос можно использовать во многих сценариях:

-   приложения на основе DirectShow могут использовать дмос с помощью фильтра DirectShow, который называется фильтром [оболочки DMO](dmo-wrapper-filter.md) . Различие между фильтрами и дмос является прозрачным для приложения. приложение не вызывает напрямую интерфейсы DMO api.
-   Приложения, основанные на Microsoft DirectSound, могут использовать звуковые эффекты дмос. опять же, приложение изолируется от низкоуровневых api DMO интерфейсами api DirectSound более высокого уровня.
-   Приложения могут использовать дмос напрямую.

таким же путем написания DMO создается компонент, который можно использовать в широком спектре приложений. Эта документация содержит следующие разделы:
