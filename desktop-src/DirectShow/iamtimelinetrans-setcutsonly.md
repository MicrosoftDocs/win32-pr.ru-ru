# <a name="iamtimelinetranssetcutsonly-method"></a>Метод Иамтимелинетранс:: Сеткутсонли

> [!Note]  
> \[Не рекомендуется. Этот API может быть удален из будущих выпусков Windows.\]

 

`SetCutsOnly`Метод указывает, визуализируется ли переход в виде вырезания. Если да, то переход выполняется мгновенно в вырезанной точке. Если переход на новую версию занимает много времени, может потребоваться предварительный просмотр его в виде вырезанной области для ускорения просмотра.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT SetCutsOnly(
   BOOL Val
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*Val* 
</dt> <dd>

Указывает, следует ли отображать переход в виде вырезания. Если **значение равно true**, переход отображается как мгновенное вырезание. Если **значение равно false**, переход отрисовывается в течение нормальной длительности.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Если этот метод завершается успешно, возвращается значение **S \_ ОК**. В противном случае возвращается код ошибки **HRESULT** .

## <a name="remarks"></a>Remarks

Отрисовка перехода в виде вырезания не совместима с обменом входными данными. (См. [**иамтимелинетранс:: сетсвапинпутс**](iamtimelinetrans-setswapinputs.md).) При вызове `SetCutsOnly` со значением **true** он временно переопределяет параметр **сетсвапинпутс** . Однако параметр «только быстрые» предназначен для предварительного просмотра, поэтому это ограничение не влияет на выходные данные, поэтому `SetCutsOnly` перед записью файла просто не забывайте вызывать его со значением **false** .
