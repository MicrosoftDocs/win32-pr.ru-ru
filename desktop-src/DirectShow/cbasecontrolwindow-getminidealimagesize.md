# <a name="cbasecontrolwindowgetminidealimagesize-method"></a>Кбасеконтролвиндов. Жетминидеалимажесизе, метод

`GetMinIdealImageSize`Метод получает минимальный оптимальный размер изображения.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT GetMinIdealImageSize(
   long *pWidth,
   long *pHeight
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пвидс* 
</dt> <dd>

Указатель на минимальную идеальную ширину (в пикселях).

</dd> <dt>

*феигхт* 
</dt> <dd>

Указатель на минимальную идеальную высоту (в пикселях).

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращает значение **HRESULT** .

## <a name="remarks"></a>Комментарии

Различные модули подготовки отчетов имеют ограничения производительности на размер отображаемых образов. Несмотря на то, что они должны правильно работать при запросе на отображение изображений, превышающих указанное максимальное значение, модули подготовки отчетов могут выдавать минимальные и максимальные оптимальные размеры через интерфейс [**ивидеовиндов**](/windows/desktop/api/Control/nn-control-ivideowindow) . Этот интерфейс может быть вызван только в том случае, если граф фильтра приостановлен или запущен, так как он не выделяется до тех пор, пока ресурсы не будут выделены и модуль подготовки отчетов не сможет распознать свои ограничения. Если ограничений не существует, модуль подготовки отчетов заполняет параметры *пвидс* и *феигхт* собственными видеоизмерениями и возвращает \_ false. Если существуют ограничения, то записываются ограниченная ширина и высота, а функция – член возвращает \_ ОК.

Измерения применяются к размеру целевого видео, а не к общему размеру окна. Таким образом, при вычислении размера окна, которое необходимо задать, следует учитывать текущий стиль окна (например, заголовок WS \_ и элемент WS \_ ).
