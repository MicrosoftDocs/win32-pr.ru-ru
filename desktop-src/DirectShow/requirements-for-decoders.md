# <a name="requirements-for-decoders"></a>Требования к декодерам

Декодеры, которые доставляют образцы в VMR, должны соблюдать следующие правила.

-   В VMR для каждого видеокадра должен быть доставлен один кадр подизображения. Оба кадра должны иметь одинаковые метки времени.
-   Если подизображение не изменилось, используйте \_ флаг AM гбф \_ нотасинкпоинт в методе [**имемаллокатор::-buffer**](/windows/desktop/api/Strmif/nf-strmif-imemallocator-getbuffer) , чтобы заставить распределитель вернуть буфер, содержащий последний кадр, доставленный в VMR. Просто поставьте новый штамп времени для примера и снова доставьте его в VMR. Если картинка славы пуста, ее все еще следует доставлять. VMR определит пустой фрейм и не смешивает его с видео. Эта проверка выполняется микросхемой VGA и не влияет на производительность воспроизведения.
-   Все примеры, за исключением динамических потоков, должны иметь действительные метки времени начала и окончания. (DVD не является активным потоком.)
-   Метки времени образца носителя должны быть непрерывными
-   Декодер должен идентифицировать себя как VMR-совместимый для использования построителями Graph.
-   Поток субтитров теперь должен содержать внедренные альфа-значения для каждого пикселя. Тип поверхности ARGB4444 идеально подходит для подизображений.
-   Не следует рассчитывать на то, что шаг подизображения совпадает с шириной поверхности. Это не всегда имеет место при использовании VMR.
