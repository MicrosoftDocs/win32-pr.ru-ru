## <a name="remarks"></a><span data-ttu-id="2b075-128">Комментарии</span><span class="sxs-lookup"><span data-stu-id="2b075-128">Remarks</span></span>

<span data-ttu-id="2b075-129">Кадры DV содержат аудио и видео в одном кадре.</span><span class="sxs-lookup"><span data-stu-id="2b075-129">DV frames contain audio and video in the same frame.</span></span> <span data-ttu-id="2b075-130">Фильтр разделителя DV извлекает звуковые данные и доставляет их в виде одного или двух звуковых потоков из ПИН-кодов звука.</span><span class="sxs-lookup"><span data-stu-id="2b075-130">The DV Splitter filter extracts the audio data and delivers it as one or two audio streams, from the audio output pins.</span></span> <span data-ttu-id="2b075-131">Исходный DV-кадр доставляется из ПИН-кода вывода видео в виде видеокадра.</span><span class="sxs-lookup"><span data-stu-id="2b075-131">The original DV frame is delivered from the video output pin, as a video frame.</span></span> <span data-ttu-id="2b075-132">Тип носителя в кадре видеозаписи изменяется с носителя с \_ чередованием на Mac \_ , но в противном случае данные не изменяются.</span><span class="sxs-lookup"><span data-stu-id="2b075-132">The media type on the video frame is changed from MEDIATYPE\_Interleaved to MEDIATYPE\_Video, but otherwise the data is not modified.</span></span> <span data-ttu-id="2b075-133">Тип носителя изменится на сигнал о том, что звуковые данные в кадре следует игнорировать.</span><span class="sxs-lookup"><span data-stu-id="2b075-133">The media type is changed to signal that the audio data into the frame should be ignored.</span></span> <span data-ttu-id="2b075-134">Разделитель DV не задает время носителя в выходных примерах; Если вы создаете нисходящий фильтр, для которого требуется время работы с мультимедиа, то можете наследовать значения времени от числа кадров.</span><span class="sxs-lookup"><span data-stu-id="2b075-134">The DV Splitter does not set a media time on its output samples; if you are writing a downstream filter that requires the media times, then you can derive the times from the frame count.</span></span>

<span data-ttu-id="2b075-135">Только один выходной закрепление за раз предоставляет интерфейсы [**имедиапоситион**](/windows/desktop/api/Control/nn-control-imediaposition) и [**имедиасикинг**](/windows/desktop/api/Strmif/nn-strmif-imediaseeking) .</span><span class="sxs-lookup"><span data-stu-id="2b075-135">Only one output pin at a time exposes the [**IMediaPosition**](/windows/desktop/api/Control/nn-control-imediaposition) and [**IMediaSeeking**](/windows/desktop/api/Strmif/nn-strmif-imediaseeking) interfaces.</span></span>

<span data-ttu-id="2b075-142">Если источником DV является Динамическая камера, обычно нет причин, по которым формат аудио не изменится.</span><span class="sxs-lookup"><span data-stu-id="2b075-142">If the DV source is a live camera, there is normally no reason for the audio format to change.</span></span> <span data-ttu-id="2b075-143">Однако формат может измениться при передаче с ленты Втр, которая содержит несколько разнородных источников.</span><span class="sxs-lookup"><span data-stu-id="2b075-143">However, the format might change if you transmit from a VTR tape that contains several heterogeneous sources.</span></span>
