# <a name="mpeg2_transport_stride-structure"></a>\_ \_ Структура шага передачи MPEG2

`MPEG2_TRANSPORT_STRIDE`Структура описывает формат пакетов транспортного потока MPEG-2 (TS). Эта структура позволяет передавать потоки, в которых 188-байтные транспортные пакеты не являются смежными. Для целей этой документации такие пакеты называются *пакетами STRIDE*.

Пакеты STRIDE идентифицируются по следующему типу носителя:



| Метка | Значение |
|-------------|----------------------------------------|
| Основной тип  | \_Поток MEDIATYPE                      |
| Subtype     | \_ \_ Шаг транспорта медиасубтипе \_ MPEG2 |
| Тип формата | Отформатировать \_ нет                           |



 

Блок формата (**пбформат**) является необязательным. Если блок формата включен, он должен начинаться с структуры STRIDE с использованием **\_ \_ транспорта MPEG2** . Эта структура определяет макет транспортного пакета в пакете STRIDE. Если блок формата имеет **значение NULL**, предполагается, что пакеты используют набор значений по умолчанию. Дополнительные сведения см. в разделе "Примечания".

## <a name="syntax"></a>Синтаксис


```C++
typedef struct _MPEG2_TRANSPORT_STRIDE {
  DWORD dwOffset;
  DWORD dwPacketLength;
  DWORD dwStride;
} MPEG2_TRANSPORT_STRIDE, *PMPEG2_TRANSPORT_STRIDE;
```



## <a name="members"></a>Члены

<dl> <dt>

**двоффсет**
</dt> <dd>

Указывает смещение (в байтах) от начала пакета до первого байта внедренного транспортного пакета. Значение должно находиться в диапазоне от нуля до `(dwStride - dwPacketLength)` включительно.

</dd> <dt>

**двпаккетленгс**
</dt> <dd>

Указывает длину внедренного транспортного пакета в байтах. Для стандартных транспортных пакетов MPEG-2 значение должно быть 188 байт.

</dd> <dt>

**двстриде**
</dt> <dd>

Указывает длину всего пакета STRIDE в байтах. Значение должно быть не меньше `(dwOffset + dwPacketLength)` .

</dd> </dl>

## <a name="remarks"></a>Комментарии

На следующей схеме показаны связи между членами структуры.

![пакет с шагом MPEG-2](images/mpeg2-stride-packet.png)

Входные буферы, содержащие мультиплексированные пакеты STRIDE, имеют некоторые ограничения.

-   Пакеты STRIDE должны упаковываться последовательно в буфер.
-   Ни один байт не может предшествовать первому пакету STRIDE или следовать за последним пакетом STRIDE.
-   Целое число пакетов STRIDE должно помещаться в буфер; то есть длина буфера% Двстриде равна нулю.

Количество пакетов STRIDE на один буфер не ограничено.

Если тип мультимедиа не содержит блок формата (**пбформат** имеет **значение NULL**), используются следующие значения по умолчанию:

-   **двоффсет**: 0
-   **двпаккетленгс**: 188
-   **двстриде**: 188
