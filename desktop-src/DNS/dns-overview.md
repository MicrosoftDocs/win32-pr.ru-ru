# <a name="dns-overview"></a><span data-ttu-id="84f75-103">Обзор DNS</span><span class="sxs-lookup"><span data-stu-id="84f75-103">DNS Overview</span></span>

<span data-ttu-id="84f75-104">DNS — это стандартная служба, используемая для нахождение компьютеров в сети на основе протокола Интернета (IP).</span><span class="sxs-lookup"><span data-stu-id="84f75-104">DNS is an industry-standard service used to locate computers on an Internet Protocol (IP)-based network.</span></span> <span data-ttu-id="84f75-105">IP-сети, такие как сети Интернет и Windows 2000, полагаются на числовые адреса, например 207.46.131.137 на паром информацию по всей сети.</span><span class="sxs-lookup"><span data-stu-id="84f75-105">IP networks, such as the Internet and Windows 2000 networks, rely on number-based addresses, such as 207.46.131.137 to ferry information throughout the network.</span></span> <span data-ttu-id="84f75-106">Сетевые пользователи используют имена на основе символов, такие как `www.microsoft.com` .</span><span class="sxs-lookup"><span data-stu-id="84f75-106">Network users rely on character-based names, such as `www.microsoft.com`.</span></span> <span data-ttu-id="84f75-107">Поэтому необходимо преобразовать символы или понятные пользователю адреса ( `www.microsoft.com` ) в адреса на основе чисел (207.46.131.137), которые может распознать сеть.</span><span class="sxs-lookup"><span data-stu-id="84f75-107">Therefore, it is necessary to translate character or user-friendly addresses (`www.microsoft.com`) into the number-based addresses (207.46.131.137) that the network can recognize.</span></span> <span data-ttu-id="84f75-108">DNS — это служба, которую можно выбрать в Windows 2000 для нахождение ресурсов и их преобразования в соответствующие IP-адреса.</span><span class="sxs-lookup"><span data-stu-id="84f75-108">DNS is the service of choice in Windows 2000 to locate resources and translate them into their corresponding IP addresses.</span></span>

<span data-ttu-id="84f75-109">DNS использует специализированную базу данных записей ресурсов, которые обычно называются записями RR, чтобы реагировать на запросы разрешения имен клиентов.</span><span class="sxs-lookup"><span data-stu-id="84f75-109">DNS uses a specialized database of resource records, commonly referred to simply as RRs, to respond to client name-resolution queries.</span></span> <span data-ttu-id="84f75-110">До DNS разрешение имен в Интернете было достигнуто с помощью [*файла hosts*](h-gly.md), который создается вручную файлами, связанными с именами узлов с IP-адресами.</span><span class="sxs-lookup"><span data-stu-id="84f75-110">Before DNS, name resolution on the Internet was achieved with [*hosts file*](h-gly.md), which are manually created files that associated host names with IP addresses.</span></span>

<span data-ttu-id="84f75-111">При добавлении нового клиента в сеть администратору пришлось вручную обновить файл hosts, а затем скопировать (реплицировать) этот файл на все остальные компьютеры в сети, чтобы новый узел был доступен всем.</span><span class="sxs-lookup"><span data-stu-id="84f75-111">When a new client was added to the network, an administrator had to manually update the hosts file and then copy (replicate) that file to all other computers on the network so that the new host could be reached by all.</span></span> <span data-ttu-id="84f75-112">По мере роста Интернета такая форма разрешения имен была очевидно недостаточной; слишком интенсивное управление и [*масштабирование*](s-gly.md)не выполнялось.</span><span class="sxs-lookup"><span data-stu-id="84f75-112">As the Internet grew, this form of name resolution was clearly insufficient; it was too management intensive, and it did not [*scale*](s-gly.md).</span></span> <span data-ttu-id="84f75-113">Файл hosts стал больше, и так как в нем использовалось [*неструктурированное пространство имен*](f-gly.md) (см. также [пространство имен](name-space.md)), его не удалось секционировать и распределить полностью.</span><span class="sxs-lookup"><span data-stu-id="84f75-113">The hosts file just got bigger, and because it used a [*flat name space*](f-gly.md) (see also [Name Space](name-space.md)), it could not be partitioned and had to be distributed in its entirety.</span></span> <span data-ttu-id="84f75-114">Это решение было DNS.</span><span class="sxs-lookup"><span data-stu-id="84f75-114">The solution was DNS.</span></span>

-   <span data-ttu-id="84f75-115">DNS заменил неструктурированное пространство имен файла hosts на [*Иерархическое пространство имен*](h-gly.md).</span><span class="sxs-lookup"><span data-stu-id="84f75-115">DNS replaced the hosts file's flat name space with a [*hierarchical name space*](h-gly.md).</span></span> <span data-ttu-id="84f75-116">При использовании иерархического пространства имен сведения об именах узлов и IP-адресах могут быть секционированы и распределены. Поэтому масштабируемость достигается.</span><span class="sxs-lookup"><span data-stu-id="84f75-116">With a hierarchical name space, information about host names and IP addresses can be partitioned and distributed; thus, scalability is achieved.</span></span> <span data-ttu-id="84f75-117">Например, в вымышленном домене widgets.products.microsoft.com ответственность за разрешение имен можно секционировать, чтобы различные серверы могли обрабатывать разрешение имен для различных частей пространства имен:</span><span class="sxs-lookup"><span data-stu-id="84f75-117">For example, in the fictional widgets.products.microsoft.com domain, responsibility for name resolution can be partitioned so that various servers can handle name resolution for different parts of the name space:</span></span>
    -   <span data-ttu-id="84f75-118">Один сервер может отвечать за разрешение первой части (microsoft.com), а затем может пересылать запрос разрешения имени следующему DNS-серверу в иерархии.</span><span class="sxs-lookup"><span data-stu-id="84f75-118">One server can be responsible for resolving the first part (microsoft.com), and can then forward the name-resolution request to the next DNS Server in the hierarchy.</span></span>
    -   <span data-ttu-id="84f75-119">Следующий DNS-сервер может отвечать за разрешение следующей части пространства имен (Products).</span><span class="sxs-lookup"><span data-stu-id="84f75-119">The next DNS Server can be responsible for resolving the next part of the name space (products).</span></span>
    -   <span data-ttu-id="84f75-120">Наконец, запрос можно перенаправить на третий DNS-сервер, отвечающий за разрешение последней части имени (widgets).</span><span class="sxs-lookup"><span data-stu-id="84f75-120">Finally, the request can be forwarded to a third DNS server that is responsible for resolving the last part of the name (widgets).</span></span>

<span data-ttu-id="84f75-121">DNS-серверы в каждой части иерархического пространства имен должны поддерживать базу данных записей ресурсов для узлов, но только в их части иерархии.</span><span class="sxs-lookup"><span data-stu-id="84f75-121">DNS Servers in each part of the hierarchical name space need to maintain a database of resource records for hosts, but only in their part of the hierarchy.</span></span> <span data-ttu-id="84f75-122">Таким образом, сервер (или серверы), входящий в состав widgets.products.microsoft.com, сохраняет записи ресурсов только для продуктов, входящих в иерархическое пространство имен, а не для части microsoft.com или части пространства имен.</span><span class="sxs-lookup"><span data-stu-id="84f75-122">Thus, the server (or servers) in the products part of widgets.products.microsoft.com maintain RRs for only the products part of the hierarchical name space — not for the microsoft.com part or the widgets part of the name space.</span></span>
