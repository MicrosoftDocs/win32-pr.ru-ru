<dl> <dt>

<span id="_security_national_institute_of_standards_and_technology_gly"></span><span id="_SECURITY_NATIONAL_INSTITUTE_OF_STANDARDS_AND_TECHNOLOGY_GLY"></span>**Национальный институт стандартов и технологий**
</dt> <dd>

NIST Подразделение США отдела торговли, которое публикует официальные стандарты как для государственных, так и для компьютерных систем с частным сектором. Эти стандарты публикуются как стандарты федеральной обработки информации (FIPS). В 1987 NIST направляется на определение стандартов обеспечения безопасности конфиденциальной, но неклассифицированной информации в компьютерных системах.

</dd> <dt>

<span id="_security_negotiate_gly"></span><span id="_SECURITY_NEGOTIATE_GLY"></span>**Новое**
</dt> <dd>

[*Поставщик поддержки безопасности*](s-gly.md) (SSP), который выступает в качестве уровня приложения между [*интерфейсом поставщика поддержки безопасности*](s-gly.md) (SSPI) и другими SSP. Когда приложение обращается к интерфейсу SSPI для выполнения входа в сеть, оно может указать SSP для обработки запроса. Если в приложении указано Negotiate, то Negotiate анализирует запрос и выбирает лучший поставщик общих служб для выполнения запроса на основе политики безопасности, настроенной пользователем.

</dd> <dt>

<span id="_security_nist_gly"></span><span id="_SECURITY_NIST_GLY"></span>**NIST**
</dt> <dd>

Ознакомьтесь с *национальным институтом стандартов и технологий*.

</dd> <dt>

<span id="_security_nonce_gly"></span><span id="_SECURITY_NONCE_GLY"></span>**требуем**
</dt> <dd>

Случайное значение, используемое для предотвращения атак воспроизведения.

</dd> <dt>

<span id="_security_nonrepudiation_gly"></span><span id="_SECURITY_NONREPUDIATION_GLY"></span>**неподдельность**
</dt> <dd>

Невозможность отказа пользователя от факта выполнения им определенных действий. Например, при каждом удалении пользователем какого-либо файла система может записывать в журнал идентификатор этого пользователя.

</dd> </dl>
