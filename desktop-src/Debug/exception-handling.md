# <a name="exception-handling-error-handling"></a><span data-ttu-id="33700-104">Обработка исключений (обработка ошибок)</span><span class="sxs-lookup"><span data-stu-id="33700-104">Exception Handling (Error Handling)</span></span>

<span data-ttu-id="33700-105">Исключения могут инициироваться оборудованием или программным обеспечением и могут происходить в режиме ядра, а также в коде пользовательского режима.</span><span class="sxs-lookup"><span data-stu-id="33700-105">Exceptions can be initiated by hardware or software, and can occur in kernel-mode as well as user-mode code.</span></span> <span data-ttu-id="33700-106">Структурированная обработка исключений предоставляет единый механизм для обработки исключений режима ядра и пользовательского режима.</span><span class="sxs-lookup"><span data-stu-id="33700-106">Structured exception handling provides a single mechanism for the handling of kernel-mode and user-mode exceptions.</span></span>

<span data-ttu-id="33700-107">Выполнение определенных последовательностей инструкций может привести к исключениям, инициированным оборудованием.</span><span class="sxs-lookup"><span data-stu-id="33700-107">The execution of certain instruction sequences can result in exceptions that are initiated by hardware.</span></span> <span data-ttu-id="33700-108">Например, нарушение прав доступа создается оборудованием, когда процесс пытается выполнить чтение или запись на виртуальный адрес, к которому он не имеет соответствующего доступа.</span><span class="sxs-lookup"><span data-stu-id="33700-108">For example, an access violation is generated by the hardware when a process attempts to read from or write to a virtual address to which it does not have the appropriate access.</span></span>

<span data-ttu-id="33700-109">События, требующие обработки исключений, также могут возникать во время выполнения программной процедуры (например, если указано недопустимое значение параметра).</span><span class="sxs-lookup"><span data-stu-id="33700-109">Events that require exception handling may also occur during execution of a software routine (for example, when an invalid parameter value is specified).</span></span> <span data-ttu-id="33700-110">В этом случае поток может инициировать исключение явным образом путем вызова функции [**RaiseException**](/windows/win32/api/errhandlingapi/nf-errhandlingapi-raiseexception) .</span><span class="sxs-lookup"><span data-stu-id="33700-110">When this happens, a thread can initiate an exception explicitly by calling the [**RaiseException**](/windows/win32/api/errhandlingapi/nf-errhandlingapi-raiseexception) function.</span></span> <span data-ttu-id="33700-111">Эта функция позволяет вызывающему потоку указывать сведения, описывающие исключение.</span><span class="sxs-lookup"><span data-stu-id="33700-111">This function enables the calling thread to specify information that describes the exception.</span></span>

<span data-ttu-id="33700-112">Исключение может быть непрерывным или недопустимым.</span><span class="sxs-lookup"><span data-stu-id="33700-112">An exception can be continuable or noncontinuable.</span></span> <span data-ttu-id="33700-113">Исключение, которое является непродолжаемым, возникает, когда событие не поддерживается в течение оборудования или если продолжение не имеет смысла.</span><span class="sxs-lookup"><span data-stu-id="33700-113">A noncontinuable exception arises when the event is not continuable in the hardware, or if continuation makes no sense.</span></span> <span data-ttu-id="33700-114">Исключение, которое не является непрерывным, не приводит к завершению работы приложения.</span><span class="sxs-lookup"><span data-stu-id="33700-114">A noncontinuable exception does not terminate the application.</span></span> <span data-ttu-id="33700-115">Таким образом, приложение может перехватить исключение и выполнить команду.</span><span class="sxs-lookup"><span data-stu-id="33700-115">Therefore, an application may be able to catch the exception and run.</span></span> <span data-ttu-id="33700-116">Однако непродолжаемое исключение обычно возникает в результате повреждения стека или другой серьезной проблемы, что затрудняет восстановление после исключения.</span><span class="sxs-lookup"><span data-stu-id="33700-116">However, a noncontinuable exception typically arises as a result of a corrupted stack or other serious problem, making it difficult to recover from the exception.</span></span>
