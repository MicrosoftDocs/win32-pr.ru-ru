## <a name="remarks"></a><span data-ttu-id="87805-209">Комментарии</span><span class="sxs-lookup"><span data-stu-id="87805-209">Remarks</span></span>

<span data-ttu-id="87805-210">Функция **AcceptSecurityContext (Negotiate)** является аналогом сервера функции [**InitializeSecurityContext (Negotiate)**](initializesecuritycontext--negotiate.md) .</span><span class="sxs-lookup"><span data-stu-id="87805-210">The **AcceptSecurityContext (Negotiate)** function is the server counterpart to the [**InitializeSecurityContext (Negotiate)**](initializesecuritycontext--negotiate.md) function.</span></span>

<span data-ttu-id="87805-211">Когда сервер получает запрос от клиента, сервер использует параметр *фконтекстрек* , чтобы указать, что он требует для сеанса.</span><span class="sxs-lookup"><span data-stu-id="87805-211">When the server receives a request from a client, the server uses the *fContextReq* parameter to specify what it requires of the session.</span></span> <span data-ttu-id="87805-212">Таким образом, сервер может указать, что клиенты должны иметь возможность использовать конфиденциальную сессию или сеанс с проверкой [*целостности*](../secgloss/i-gly.md), а также могут отклонять клиенты, которые не могут удовлетворить это требование.</span><span class="sxs-lookup"><span data-stu-id="87805-212">In this fashion, a server can specify that clients must be capable of using a confidential or [*integrity*](../secgloss/i-gly.md)-checked session, and it can reject clients that cannot meet that demand.</span></span> <span data-ttu-id="87805-213">Кроме того, сервер может не потребовать ничего, а любой клиент, который может предоставить или требуется, возвращается в параметре *пфконтекстаттр* .</span><span class="sxs-lookup"><span data-stu-id="87805-213">Alternatively, a server can require nothing, and whatever the client can provide or requires is returned in the *pfContextAttr* parameter.</span></span>

<span data-ttu-id="87805-214">Для пакета, поддерживающего многоэтапную проверку подлинности, например взаимную проверку подлинности, вызывающая последовательность выглядит следующим образом:</span><span class="sxs-lookup"><span data-stu-id="87805-214">For a package that supports multiple-leg authentication, such as mutual authentication, the calling sequence is as follows:</span></span>

1.  <span data-ttu-id="87805-215">Клиент передает токен на сервер.</span><span class="sxs-lookup"><span data-stu-id="87805-215">The client transmits a token to the server.</span></span>
2.  <span data-ttu-id="87805-216">В первый раз сервер вызывает **AcceptSecurityContext (Negotiate)** , который создает маркер ответа, который затем отправляется клиенту.</span><span class="sxs-lookup"><span data-stu-id="87805-216">The server calls **AcceptSecurityContext (Negotiate)** the first time, which generates a reply token that is then sent to the client.</span></span>
3.  <span data-ttu-id="87805-217">Клиент получает маркер и передает его в [**InitializeSecurityContext (Negotiate)**](initializesecuritycontext--negotiate.md).</span><span class="sxs-lookup"><span data-stu-id="87805-217">The client receives the token and passes it to [**InitializeSecurityContext (Negotiate)**](initializesecuritycontext--negotiate.md).</span></span> <span data-ttu-id="87805-218">Если **InitializeSecurityContext (Negotiate)** возвращает значение \_ \_ в секунду, выполняется взаимная проверка подлинности и может начаться безопасный сеанс.</span><span class="sxs-lookup"><span data-stu-id="87805-218">If **InitializeSecurityContext (Negotiate)** returns SEC\_E\_OK, mutual authentication has been completed and a secure session can begin.</span></span> <span data-ttu-id="87805-219">Если **InitializeSecurityContext (Negotiate)** возвращает код ошибки, согласование взаимной проверки подлинности завершается.</span><span class="sxs-lookup"><span data-stu-id="87805-219">If **InitializeSecurityContext (Negotiate)** returns an error code, the mutual authentication negotiation ends.</span></span> <span data-ttu-id="87805-220">В противном случае маркер безопасности, возвращенный **InitializeSecurityContext (Negotiate)** , отправляется клиенту, а шаги 2 и 3 повторяются.</span><span class="sxs-lookup"><span data-stu-id="87805-220">Otherwise, the security token returned by **InitializeSecurityContext (Negotiate)** is sent to the client, and steps 2 and 3 are repeated.</span></span>

<span data-ttu-id="87805-221">Параметры *фконтекстрек* и *пфконтекстаттр* представляют собой битовую маску, представляющую различные атрибуты контекста.</span><span class="sxs-lookup"><span data-stu-id="87805-221">The *fContextReq* and *pfContextAttr* parameters are bitmasks that represent various context attributes.</span></span> <span data-ttu-id="87805-222">Описание различных атрибутов см. в разделе [требования к контексту](context-requirements.md).</span><span class="sxs-lookup"><span data-stu-id="87805-222">For a description of the various attributes, see [Context Requirements](context-requirements.md).</span></span>
