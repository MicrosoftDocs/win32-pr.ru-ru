# <a name="initializesecuritycontext-schannel-function"></a><span data-ttu-id="306b2-103">Функция InitializeSecurityContext (Schannel)</span><span class="sxs-lookup"><span data-stu-id="306b2-103">InitializeSecurityContext (Schannel) function</span></span>

<span data-ttu-id="306b2-104">Функция **InitializeSecurityContext (Schannel)** инициирует клиентский, исходящий [*контекст безопасности*](../secgloss/s-gly.md) из маркера учетных данных.</span><span class="sxs-lookup"><span data-stu-id="306b2-104">The **InitializeSecurityContext (Schannel)** function initiates the client side, outbound [*security context*](../secgloss/s-gly.md) from a credential handle.</span></span> <span data-ttu-id="306b2-105">Функция используется для создания [*контекста безопасности*](../secgloss/s-gly.md) между клиентским приложением и удаленным одноранговым узлом.</span><span class="sxs-lookup"><span data-stu-id="306b2-105">The function is used to build a [*security context*](../secgloss/s-gly.md) between the client application and a remote peer.</span></span> <span data-ttu-id="306b2-106">**InitializeSecurityContext (Schannel)** возвращает маркер, который клиент должен передать удаленному одноранговому узлу, который, в свою очередь, отправляется в локальную реализацию безопасности через вызов [**AcceptSecurityContext (Schannel)**](acceptsecuritycontext--schannel.md) .</span><span class="sxs-lookup"><span data-stu-id="306b2-106">**InitializeSecurityContext (Schannel)** returns a token that the client must pass to the remote peer, which the peer in turn submits to the local security implementation through the [**AcceptSecurityContext (Schannel)**](acceptsecuritycontext--schannel.md) call.</span></span> <span data-ttu-id="306b2-107">Создаваемый токен должен считаться непрозрачным для всех вызывающих объектов.</span><span class="sxs-lookup"><span data-stu-id="306b2-107">The token generated should be considered opaque by all callers.</span></span>

<span data-ttu-id="306b2-108">Как правило, функция **InitializeSecurityContext (Schannel)** вызывается в цикле до тех пор, пока не будет установлен достаточный [*контекст безопасности*](../secgloss/s-gly.md) .</span><span class="sxs-lookup"><span data-stu-id="306b2-108">Typically, the **InitializeSecurityContext (Schannel)** function is called in a loop until a sufficient [*security context*](../secgloss/s-gly.md) is established.</span></span>

