# <a name="ibytebuffercopyto-method"></a>Метод Ибитебуффер:: CopyTo

\[Метод **CopyTo** доступен для использования в операционных системах, указанных в разделе требования. Он недоступен для использования в Windows Server 2003 с пакетом обновления 1 (SP1) и более поздней версии, Windows Vista, Windows Server 2008 и последующих версиях операционной системы. Интерфейс [**IStream**](/windows/desktop/api/objidl/nn-objidl-istream) предоставляет аналогичные функциональные возможности.\]

Метод **CopyTo** Копирует указанное число байтов из текущего указателя поиска в объекте в текущий указатель поиска в другом объекте.

## <a name="syntax"></a>Синтаксис


```C++
HRESULT CopyTo(
  [in]  LPBYTEBUFFER *pByteBuffer,
  [in]  LONG         cb,
  [out] LONG         *pcbRead,
  [out] LONG         *pcbWritten
);
```



## <a name="parameters"></a>Параметры

<dl> <dt>

*пбитебуффер* \[ окне\]
</dt> <dd>

Указывает на целевой поток. Поток, на который указывает *пбитебуффер* , может быть новым потоком или клоном исходного потока.

</dd> <dt>

*CB* \[ окне\]
</dt> <dd>

Число байтов, которое необходимо скопировать из исходного потока.

</dd> <dt>

*пкбреад* \[ заполняет\]
</dt> <dd>

Указатель на расположение, в котором этот метод записывает фактическое число байтов, считанных из источника. Можно задать для этого указателя значение **null** , чтобы указать, что вы не заинтересованы в этом значении. В этом случае этот метод не обеспечивает фактическое число считанных байтов.

</dd> <dt>

*пкбвриттен* \[ заполняет\]
</dt> <dd>

Указатель на расположение, в котором этот метод записывает фактическое число байтов, записанных в назначение. Можно задать для этого указателя значение **null** , чтобы указать, что вы не заинтересованы в этом значении. В этом случае этот метод не обеспечивает фактическое число записанных байтов.

</dd> </dl>

## <a name="return-value"></a>Возвращаемое значение

Возвращаемое значение является значением **HRESULT**. Значение S \_ ОК указывает на Успешный вызов.

## <a name="remarks"></a>Комментарии

Этот метод копирует указанные байты из одного потока в другой. Его также можно использовать для копирования потока в себя. Указатель поиска в каждом экземпляре потока корректируется на количество считанных или записанных байтов.
