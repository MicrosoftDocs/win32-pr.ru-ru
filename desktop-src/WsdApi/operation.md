# <a name="operation-element"></a>Operation, элемент

Указывает операцию, для которой создается код.

## <a name="usage"></a>Использование

``` syntax
<operation/>
```

## <a name="attributes"></a>Атрибуты

Атрибуты отсутствуют.

## <a name="child-elements"></a>Дочерние элементы

Нет дочерних элементов.

## <a name="remarks"></a>Remarks

Можно указать любое количество операций. Если операции не указаны, код создается для всех операций во всех соответствующих типах портов. Использование элемента **Operation** ограничит методы, созданные для тех, которые содержатся в операции.

Например, принтер поддерживает следующие операции:

-   **принтжоббипост**
-   **принтжоббиреференце**
-   **CancelJob**
-   **жетжобелементс**
-   **жетактивежобс**
-   **жетжобхистори**
-   **субскрибетопринтерконфигчанже**
-   **унсубскрибетопринтерконфигчанже**

Однако, чтобы включить только методы, связанные с операциями **принтжоббипост** и **жетжобелементс** , скрипт создания кода будет использовать элементы [**идлфунктиондекларатионс**](idlfunctiondeclarations.md) следующим образом:

``` syntax
<idlFunctionDeclarations>
    <operation>PrintJobByPost</operation>
    <operation>GetJobElements></operation>
</idlFunctionDeclarations>
```

При этом создаются объявления функций IDL для всех методов, связанных с двумя операциями (например, **бегинпринтжоббипост**, **ендпринтжоббипост**, **бегинжетжобелементс** и **енджетжобелементс**).
