# <a name="semantics"></a><span data-ttu-id="2c173-155">Семантика</span><span class="sxs-lookup"><span data-stu-id="2c173-155">Semantics</span></span>

<span data-ttu-id="2c173-156">Семантика — это строка, присоединенная к входным или выходным данным шейдера, который передает сведения о предполагаемом использовании параметра.</span><span class="sxs-lookup"><span data-stu-id="2c173-156">A semantic is a string attached to a shader input or output that conveys information about the intended use of a parameter.</span></span> <span data-ttu-id="2c173-157">Семантика необходима для всех переменных, передаваемых между стадиями шейдера.</span><span class="sxs-lookup"><span data-stu-id="2c173-157">Semantics are required on all variables passed between shader stages.</span></span> <span data-ttu-id="2c173-158">Ниже показан синтаксис для добавления семантики в переменную шейдера ([Синтаксис переменной (DirectX HLSL)](dx-graphics-hlsl-variable-syntax.md)).</span><span class="sxs-lookup"><span data-stu-id="2c173-158">The syntax for adding a semantic to a shader variable is shown here ([Variable Syntax (DirectX HLSL)](dx-graphics-hlsl-variable-syntax.md)).</span></span>

<span data-ttu-id="2c173-159">Как правило, данные, передаваемые между стадией конвейера, полностью универсальны и не обрабатываются системой. разрешена произвольная семантика, не имеющая специального смысла.</span><span class="sxs-lookup"><span data-stu-id="2c173-159">In general, data passed between pipeline stages is completely generic and is not uniquely interpreted by the system; arbitrary semantics are allowed which have no special meaning.</span></span> <span data-ttu-id="2c173-160">Параметры (в Direct3D 10 и более поздних версиях), которые содержат эти специальные семантики, называются [семантикой системных значений](#system-value-semantics).</span><span class="sxs-lookup"><span data-stu-id="2c173-160">Parameters (in Direct3D 10 and later) which contain these special semantics are referred to as [System-Value Semantics](#system-value-semantics).</span></span>
