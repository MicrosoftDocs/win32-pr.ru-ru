# <a name="tex---ps"></a>Да-PS

Загружает регистр назначения с использованием данных цвета (RGBA) из текстуры. Текстура должна быть привязана к определенной стадии текстуры (n) с помощью [**сеттекстуре**](/windows/desktop/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-settexture). Выбор текстуры управляется [**сетсамплерстате**](/windows/desktop/api/d3d9/nf-d3d9-idirect3ddevice9-setsamplerstate).

## <a name="syntax"></a>Синтаксис



| Да DST |
|---------|



 

where

-   DST — это регистр назначения.

## <a name="remarks"></a>Remarks



| Версии шейдеров пикселей | 1\_1 | 1\_2 | 1 \_ 3 | 1\_4 | 2 \_ 0 | 2 \_ x | 2 \_ SW | 3 \_ 0 | 3 \_ SW |
|-----------------------|------|------|------|------|------|------|-------|------|-------|
| Да                   | x    | x    | x    |      |      |      |       |      |       |



 

Номер целевой регистрации указывает номер стадии текстуры.

Выборка текстур использует координаты текстуры для поиска или выборки значений цвета в заданных координатах (u, v, w, q), принимая во внимание атрибуты состояния стадии текстуры.

Данные координат текстуры интерполируются на основе данных координат текстурной текстуры и связаны с определенной стадией текстуры. Ассоциация по умолчанию является сопоставлением "один к одному" между номером стадии текстуры и порядком объявления координат текстуры. Это означает, что первый набор координат текстуры, определенный в формате вершины, по умолчанию связан с этапом текстуры 0.

Координаты текстуры могут быть связаны с любым этапом с помощью двух методов. При использовании шейдера вершин с фиксированной функцией или конвейера фиксированной функции флаг состояния стадии текстуры Тсс \_ текскурдиндекс можно использовать в [**сеттекстурестажестате**](/windows/desktop/api/d3d9helper/nf-d3d9helper-idirect3ddevice9-settexturestagestate) для связывания координат с стадией. В противном случае координаты текстуры выводятся с помощью Отна вершин шейдера при использовании программируемого шейдера вершин.
