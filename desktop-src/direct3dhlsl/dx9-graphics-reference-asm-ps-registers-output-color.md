# <a name="output-color-register"></a>Регистр цвета вывода

Выходные регистры цвета шейдера пикселей (oC #) — это регистры только для записи, которые выводят результаты в несколько целевых объектов рендеринга.

Синтаксис



| даваемым # |
|------|



 

Где:



| Имя | Описание       |
|------|-------------------|
| oC0  | Целевой объект прорисовки \# 0 |
| oC1  | Целевой объект прорисовки \# 1 |
| oC2  | Целевой объект прорисовки \# 2 |
| oC3  | Целевой объект прорисовки \# 3 |



 

## <a name="remarks"></a>Примечания

-   Если окн записывается, но отсутствует соответствующий целевой объект прорисовки, то эта запись в окн игнорируется.
-   Состояния рендеринга D3DRS \_ колорвритинабле, D3DRS \_ COLORWRITEENABLE1, D3DRS \_ COLORWRITEENABLE2 и D3DRS \_ COLORWRITEENABLE3 определяют, какие компоненты окн в конечном итоге записываются в целевой объект отрисовки (после смешения, если применимо). Если шейдер записывает некоторые, но не все компоненты, определенные \_ \* состояниями отрисовки D3DRS колорвритинабле для заданного регистра окн, то незаписанные каналы создают неопределенные значения в соответствующем целевом объекте прорисовки. Если ни один из компонентов окн не написан, соответствующий целевой объект рендеринга не должен обновляться (как указано выше), поэтому D3DRS \_ колорвритинабле \* отрисовки не применяются.

### <a name="shader-model-2-restrictions"></a>Ограничения модели шейдеров 2

-   Окн можно записать только с помощью инструкции [MOV-PS](mov---ps.md) .
-   oC0 должен быть всегда записан в шейдер.
-   Отсутствуют исходные свиззле (кроме. ксизв = Default свиззле) или модификатор источника, разрешенный при записи в любой окн.
-   Конечная маска записи (кроме. ксизв = маска по умолчанию) или модификатор инструкции разрешена при записи в любой окн.
-   Если окн записан, то необходимо также записать oC0-окн-1. Например, для записи в oC2 необходимо также выполнить запись в oC0 и oC1.
-   Допускается не более одной записи в любой неoc # на шейдер.
-   Для PS \_ 2 \_ x и PS \_ 3 \_ 0 нельзя выполнять запись в регистры OC # и oD \# в динамическом управлении потоком или затенения (операции записи в OC # внутри статического управления потоком прекрасно).

### <a name="shader-model-3-restrictions"></a>Ограничения модели шейдеров 3

-   Для PS \_ 3 \_ 0 выходные регистры OC # и oD \# могут записываться любое количество раз. Выходные данные шейдера пикселей поступают из содержимого выходных регистров в конце выполнения шейдера. Если запись в выходной регистр не происходит, возможно, из-за управления потоком или если шейдер просто не написал его, соответствующая рендертаржет также не обновляется. Если подмножество каналов в выходном регистре записано, то неопределенные значения будут записаны в оставшиеся каналы.
-   Для PS \_ 3 \_ 0 регистры OC # могут быть записаны с любым вритемаскс.
-   Для PS \_ 2 \_ x и PS \_ 3 \_ 0 нельзя выполнять запись в регистры OC # и oD \# в динамическом управлении потоком или затенения (операции записи в OC # внутри статического управления потоком прекрасно).
-   Вы не можете выполнять вычисления градиента (или операции, которые неявно вызывают такие вычисления градиента, как [текслд-PS \_ 2 \_ 0 и up](texld---ps-2-0.md), [текслдб-PS](texldb---ps.md), [текслдп-PS](texldp---ps.md)) внутри операторов управления потоком, условия ветвления которых различаются для отдельных примитивов (IE: Динамическая Инструкция по управлению потоком). Инструкция затенения не считается динамической системой управления потоком.
