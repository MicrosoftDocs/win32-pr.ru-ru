## <a name="remarks"></a>Remarks

Исходные данные могут поступать из любого типа ресурсов, кроме буферов.

*сркаддресс* предоставляет набор координат текстуры, необходимых для выполнения образца, в виде значений с плавающей запятой, ссылающихся на нормализованное пространство текстуры. Режимы переноса адресов (перенос/зеркальное отображение, зажим, границы и т. д.) применяются к координатам текстуры за пределами \[ 0... 1 \] диапазон, взятый из состояния выборки \# , и применяется после того, как к координатам текстуры применяется любое смещение адреса.

*сркресаурце* — это регистр текстуры (t \# ). Это просто заполнитель для текстуры, включая тип возвращаемых данных для ресурса, для которого выполняется выборка. Все эти сведения объявляются в преамбуле шейдера. Фактический ресурс для выборки привязывается к шейдеру снаружи в области \# (для t \# ).

*срксамплер* — это регистры образцов. Это просто заполнитель для коллекции элементов управления фильтрации, таких как точка и линейная, функции текстурирования и элементы управления, поддерживающие перенос адресов.

Набор сведений, необходимых оборудованию для выполнения выборки, разбит на две ортогональные части. Сначала регистр текстуры предоставляет сведения о типе данных, включая, например, сведения о том, содержит ли текстуру данные SRGB. Он также ссылается на фактическую выборке памяти. Во вторых, регистр образца определяет применяемый режим фильтрации.

### <a name="array-resources"></a>Ресурсы массива

Для массивов Texture1D компонент *сркаддресс* g (POS-свиззле) выбирает, из какого среза массива нужно извлечь. Это всегда обрабатывается как масштабированное значение с плавающей запятой, а не нормализованное пространство для стандартных координат текстуры, и к значению применяется округление до ближайшего значения, за которым следует срез для доступного диапазона Буффераррай.

Для массивов Texture2D компонент *сркаддресс* b (POS-свиззле) выбирает срез массива для выборки, в противном случае используя ту же семантику, описанную для массивов Texture1D.

### <a name="address-offset"></a>Смещение адреса

Необязательный \[ \_ суффикс аоффимми (u, v, w) \] (смещение адреса непосредственно целым числом) указывает, что координаты текстуры для образца должны быть смещены на набор предоставленных шаг текселя значений целых констант. Литеральные значения представляют собой набор из 4-разрядных чисел 2, имеющих целочисленный диапазон \[ — 8, 7 \] . Этот модификатор определен для всех ресурсов, включая Texture1D/2D-массивы и Texture3D, но он не определен для Текстурекубе.

Оборудование может воспользоваться преимуществами немедленного знания о том, что обходной объем пикселей текстуры по общему расположению выполняется с помощью набора примеров инструкций. Это можно передать с помощью \_ аоффимми (u, v, w).

Смещения добавляются к координатам текстуры в шаг текселя пространстве относительно каждого доступного миплевел. Поэтому, несмотря на то, что координаты текстуры предоставляются в виде нормализованных значений float, смещение применяет шаг текселя целое число.

Смещения адресов не применяются вдоль оси массива Texture1D и двумерных массивов.

\_компоненты аоффимми v, w игнорируются для Texture1Ds.

\_компонент аоффимми w не учитывается для Texture2Ds.

Режимы переноса адресов (перенос/зеркальное отображение, зажим, границы и т. д.) из состояния выборки \# применяются после применения смещения адреса к координатам текстуры.

### <a name="return-type-control"></a>Контроль возвращаемого типа

Формат данных, возвращаемый образцом в регистр назначения, определяется форматом ресурса ( \_ Формат DXGI), \* привязанным к параметру *сркресаурце* (t \# ). Например, если указанное значение t \# привязано к ресурсу с форматом \_ \_ A8B8G8R8 \_ UNORM \_ sRGB, то операция выборки преобразует пример пикселей текстуры из гаммы 2,0 в 1,0, примените фильтрацию, и результат записывается в регистр назначения в виде значений с плавающей запятой в диапазоне \[ 0.. 1 \] .

Возвращаемые значения — это 4 вектора (с использованием значений по умолчанию, относящихся к формату для компонентов, которые не представлены в формате). Свиззле On *сркресаурце* определяет, как свиззле результат из 4 компонентов, который возвращается из примера или фильтра текстуры, после чего параметр. Mask на *dest* определяет, какие компоненты в *destе* будут обновлены.

Когда **Пример** считывает 32-разрядное значение с плавающей запятой в 32-разрядный регистр с выборкой точек (без фильтрации), он может или не может сбрасывать ненормальные значения, но в противном случае числа не изменяются. Если неопределенность с выборкой значений Point является проблемой для приложения, используйте инструкцию [LD](ld--sm4---asm-.md) , гарантирующую, что 32-разрядные значения с плавающей запятой считываются без изменений.

### <a name="lod-calculation"></a>Вычисление Лод

Дополнительные сведения о вычислении производных элементов в процессе определения Лод для фильтрации см. в разделе [наследование \_ RTX](deriv-rtx--sm4---asm-.md) и [наследование \_ РТИ](deriv-rty--sm4---asm-.md). В **образце** инструкции неявно вычисляются производные от координат текстуры с помощью того же определения, которое используются в инструкциях **порожденного** шейдера. Это не относится к [образцам \_ l](sample-l--sm4---asm-.md) или [Sample \_ d](sample-d--sm4---asm-.md) . Для этих инструкций Лод или производные приложения предоставляются непосредственно приложением.

Для **примера** инструкции реализации могут использовать одни и те же вычисления Лод во всех 4 пикселях в формате 2x2 (но не в более крупной области) или выполнять вычисления Лод на уровне отдельных пикселей.

### <a name="miscellaneous-details"></a>Прочие сведения

Для buffer & Texture1D, *сркаддресс* . Гба Components (POS-свиззле), игнорируются. Для массивов Texture1D компоненты *сркаддресс* . BA (POS-свиззле) игнорируются. Для Texture2Ds, *сркаддресс* . компонент (POS-свиззле) игнорируется.

При попытке получить из входного слота, к которому не привязано ничего, возвращается значение 0 для всех компонентов.

### <a name="restrictions"></a>Ограничения

-   *сркресаурце* должен быть \# регистром t. *сркресаурце* не может быть константбуффер, который не может быть привязан к \# регистрам t.
-   *срксамплер* должен быть \# регистром s.
-   Относительная адресация в *сркресаурце* или *срксамплер* не разрешена.
-   *сркаддресс* должен быть временным (r \# /X \# ), константбуффер (CB \# ), входным (v \# ) или непосредственным значением (с).
-   *dest* должен быть временным (r \# /x \# ) или выходным \* \# регистром (o).
-   \_аоффимми (u, v, w) не разрешено для Текстурекубес.
