# <a name="dcl_maxoutputvertexcount-sm4---asm"></a>дкл \_ максаутпутвертекскаунт (SM4-ASM)

Объявляет максимальное количество вершин, которые могут выводиться шейдером Geometry.



| \_ *число* максаутпутвертекскаунт дкл |
|-----------------------------------|



 



| Элемент                                                               | Описание                                                             |
|--------------------------------------------------------------------|-------------------------------------------------------------------------|
| <span id="count"></span><span id="COUNT"></span>*расчета*<br/> | \[в \] 32-битовом целом число без знака от 1 до n включительно.<br/> |



 

Шейдер геометрии может выводить не более 1024 32-разрядных значений. Это максимальное значение включает размер входных данных и размер данных, созданных шейдером.

Ниже приведены некоторые ограничения.

-   Если число вершин достигается до завершения выполнения шейдера Geometry, шейдер завершается.
-   Геометрический шейдер может достигать конца своей программы перед выводом всех вершин. Это вполне допустимо.
-   При отладке шейдера Geometry можно указать количество вершин, создаваемых путем подсчета количества формируемых инструкций по выпуску.

Эта инструкция применяется к следующим этапам шейдера:



| Вершинный построитель текстуры | Шейдер геометрии | Построитель текстуры |
|---------------|-----------------|--------------|
|               | x               |              |



 

Эта инструкция включена для облегчения отладки шейдера в сборке; нельзя создать шейдер на языке ассемблера с использованием модели шейдеров 4.

## <a name="example"></a>Пример

Рассмотрим некоторые примеры.

Предполагается, что входные данные состоят из позиций (. ксизв) и Color (. RGB). Каждый компонент принимает один байт; в течение 7 байт максимальное число вершин, которое может создать шейдер, — 1024/(4 + 3) = 146.


```
dcl_maxOutputVertexCount 146
```



Предположим, что шейдер Geometry создает векторы компонентов 32 4. Максимальное число вершин, которое может создать шейдер, — 1024/(32 \* 4) = 8.


```
dcl_maxOutputVertexCount 8
```
