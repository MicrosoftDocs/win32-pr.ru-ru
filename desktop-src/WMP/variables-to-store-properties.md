# <a name="variables-to-store-properties"></a>Переменные для хранения свойств

Во-первых, для хранения времени задержки потребуется переменная. Образец по умолчанию, созданный мастером подключаемых модулей проигрывателя Windows Media, предоставляет переменную с именем m \_ фскалефактор для хранения коэффициента масштабирования, который он использует для обработки. Этому примеру больше не требуется эта переменная, поэтому можно изменить ее имя и тип для хранения значения времени задержки.

1.  Замените каждый экземпляр m \_ фскалефактор в Echo. h и Echo. cpp на m \_ двделайтиме.
2.  Измените тип данных m \_ фскалефактор (теперь m \_ двделайтиме) с double на DWORD в Echo. h.
3.  В конструкторе для Цечо измените значение времени задержки по умолчанию на 1000.
    ```C++
    m_dwDelayTime = 1000;   // Default to a delay time of 1000 ms.
    
    ```

    

Затем объявите две новые переменные члена для хранения процентного сигнала и процентного сигнала, который должен быть смешан в конечном буфере вывода. Термин «задолженность» относится к результату, а термин «сухой» — к исходному сигналу. Добавьте следующие объявления в Echo. h:


```C++
double  m_fWetMix;    // percentage of effect
double  m_fDryMix;    // percentage of dry signal

```



Эти значения хранятся в виде десятичных представлений процентов, поэтому их можно легко использовать в качестве коэффициентов масштабирования. Например, для каждой переменной будет представлено сочетание влияния на 50 процентов и сигнала "50% источника" в виде значения 0,50. Сумма значений для m \_ фветмикс и m \_ фдримикс не должна превышать 1,0 (100 процентов). Со временем эти значения будут доступны как свойства.

Добавьте следующий код в конструктор Цечо, чтобы задать значения по умолчанию 50 процентов.


```C++
m_fWetMix = 0.50;  // default to 50 percent wet
m_fDryMix = 0.50;  // default to 50 percent dry

```
