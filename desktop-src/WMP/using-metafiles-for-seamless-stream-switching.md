# <a name="using-metafiles-for-seamless-stream-switching"></a>Использование метафайлов для простого переключения потоков

Вы можете легко переключаться между потоками с помощью списков воспроизведения метафайлов. Обычно при завершении фрагмента содержимого буферизация выполняется для следующего клипа или потока перед открытием (если это содержимое получено с сервера потокового мультимедиа). Microsoft службы Windows Media позволяет устранять или по крайней мере свести к минимуму минимальное время буферизации, а еще один фрагмент потокового содержимого начать играть практически сразу же. обычным режимом работы проигрыватель Windows Media является открытие следующего потока мультимедиа, на который ссылается список воспроизведения за 20 секунд до конца потока, готового в данный момент. Обычно это обеспечивает простой переход между потоками мультимедиа в зависимости от других факторов, таких как время веб-доступа.

Используйте элемент **event** в списке воспроизведения в сочетании с командами опеневент из кодировщика, чтобы упростить переключение между потоками или файлами. Отправка команды ОПЕНЕВЕНТ 20 секунд или более перед выполнением команды события может привести к уменьшению задержек при переключении потока. затем проигрыватель Windows Media может выполнить предварительную загрузку части предстоящего потокового содержимого в буфер.

используйте Windows Media Encoder, чтобы отправить команду сценария в потоке, используя следующий формат:


```XML
OPENEVENT eventname 

```



Имя события должно быть задано в элементе **event** в списке воспроизведения. когда проигрыватель Windows Media получает команду сценария опеневент из кодировщика, он выполняет поиск элемента **event** в списке воспроизведения и начинает буферизацию клипа или потока, определенного в элементе **EVENT** . проигрыватель Windows Media сохраняет эти сведения до наступления фактического события с тем же именем. при получении именованного события проигрыватель Windows Media переключается на ранее буферизованное содержимое.

> [!Note]  
> Нельзя использовать символы Юникода для команды сценария ОПЕНЕВЕНТ в файле мультимедиа или в элементе **event** в списке воспроизведения.
