# <a name="trackcsv"></a>track.csv

Этот файл содержит запись для каждой записи в каталоге. Каждая запись представляет собой строку, состоящую из полей, разделенных табуляцией, описанных в таблице ниже. Поля должны присутствовать в указанном порядке.

В столбце Format в следующей таблице описывается способ форматирования каждого текстового поля в Юникоде. Он не ссылается на тип данных содержимого. Например, если в столбце Format указано целое число, это означает, что поле содержит строку в Юникоде, представляющую целочисленное значение, а не фактическое целое число.




| Поле | Обязательно | Формат | Описание | 
|-------|----------|--------|-------------|
| TrackID | Да | Неотрицательное целое число. Пример: 32789<br /> | Уникальный идентификатор, предоставленный поставщиком содержимого. Должно быть меньше 2 ^ 32. Совет по повышению производительности. Если идентификаторы, назначенные для дорожек из одного и того же альбома, нумеруются последовательно, сжатие каталога будет более эффективным. Однако последовательная нумерация записей альбомов не требуется.<br /> | 
| тракктитле | Да | Строка в Юникоде. Пример: Raygun отсылает Робберс ' Blues | Имя дорожки. | 
| Duration | Да | Неотрицательное целое число. Пример: 543<br /> | Длительность записи в секундах. | 
| траккнумбер | Да | Неотрицательное целое число. Пример: 7<br /> | Номер записи в альбоме записи. Номера записей начинаются с 1 и увеличиваются между наборами дисков. Номер записи должен быть меньше 256. номер записи, который больше или равен 256, приведет к непредвиденному поведению в проигрыватель Windows Media. | 
| траккприце | Да | Строка в Юникоде. Пример: $0,99. | Отслеживание цены. Необходимо добавить символ валюты. Пустая строка, 0 и — имеют специальное значение. Пустая строка указывает, что цена неизвестна. Ноль в этом поле означает, что запись свободна, а дефис (-) указывает, что эту запись невозможно купить. | 
| канстреам | Да | Логическое. Может иметь значение 0 или 1. Пример: 0<br /> | Указывает, может ли записываться потоковая передача. | 
| кандовнлоад | Да | Логическое. Может иметь значение 0 или 1. Пример: 0<br /> | Указывает, можно ли загрузить запись. | 
| хаспревиевклип | Да | Логическое. Может иметь значение 0 или 1. Пример: 0<br /> | Указывает, имеет ли запись клип предварительного просмотра. | 
| хасвидеоклип | Да | Логическое. Может иметь значение 0 или 1. Пример: 0<br /> | Указывает, содержит ли дорожку видеоклип. | 
| паренталратинг | Да | Перечисление. Может быть N, E или C. Пример: N<br /> | Указывает рейтинг родительских рекомендаций. Значения N, E и C заменяются для обычных, явных и чистых. | 
| линкедалбумид | Да | Неотрицательное целое число. Должен быть ИДЕНТИФИКАТОРом альбома. Пример: 32423 | ИДЕНТИФИКАТОР альбома, который содержит эту запись.<blockquote>[!Note]<br />Каждая запись должна принадлежать к альбому. То есть для каждой записи поле Линкедалбумид должно быть равно одному из идентификаторов альбома в файле album.csv.</blockquote><br /> | 
| LinkedTrackArtist_ArtistIDs | Да | Список целых чисел. Список содержит идентификаторы исполнителей, разделенные точкой с запятой. Пример: 41322; 12321; 82123; | Список идентификаторов, соответствующих участвующим исполнителям. | 
| Composer | Нет | Строка в Юникоде. Пример: Бисовен | Композитор курса. если в жанре записи не установлен набор полей хаскомпосер, значение поля Composer будет игнорироваться. Обычно используется только для «джаз» или «классическая» дорожек. | 
| Популярность | Да | Неотрицательное целое число или float. Пример: 1252,6<br /> | Определяет расположение записи в списке при сортировке по популярности. Меньшее число указывает на более высокую популярность. | 
| StarRating | Нет | Float. Пример: 4,21<br /> | оценка "звезда" будет округляться до ближайшей 1/4 звезды, проигрыватель Windows Media перед отображением в пользовательском интерфейсе проигрыватель Windows Media. |
