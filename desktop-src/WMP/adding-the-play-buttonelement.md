# <a name="adding-the-play-buttonelement"></a>Добавление БУТТОНЕЛЕМЕНТ Play

наконец, можно добавить элементы **буттонелемент** , которые соединяют визуальные кнопки на экране, чтобы проигрыватель Windows Media действия. это ядро обложки, и его можно рассматривать как привязку поверхности обложки к внутреннему механизму проигрыватель Windows Media.

**Буттонелемент** содержатся в **BUTTONGROUP**. В каждом **BUTTONGROUP** всегда должен быть хотя бы один **буттонелемент** .

Вставьте код Play **буттонелемент** после закрывающей угловой скобки **BUTTONGROUP**.


```C++
        <BUTTONELEMENT
            mappingColor = "#00FF00"
            upToolTip = "Play"
            onClick = "JScript: player.URL='https://proseware.com/laure.wma';" />

```



Для определения **буттонелемент** кнопки воспроизведения используются следующие атрибуты:

**маппингколор**

Это значение цвета для региона в файле рисунка сопоставления, созданном ранее. В данном случае это сплошной зеленый цвет. Этот атрибут является обязательным для любого **буттонелемент**. определяя этот цвет, вы указываете проигрыватель Windows Media связать эту цветовую область с XML-кодом этой кнопки.

**Подсказка**

Определяет текст, который будет отображаться, если пользователь наведет указатель мыши на кнопку. Не путайте это с назначением, которое будет отображаться. Всплывающая подсказка — это небольшой заголовок, который занимает некоторое время. Однако изображение с наведением будет мгновенно отображаться в любом выбранном цвете и форме.

**onClick**

Определяет событие, возникающее при нажатии кнопки мыши. значение этого атрибута события называется обработчиком событий и будет либо строкой Microsoft JScript code, либо JScriptной функцией во внешнем текстовом файле, который загружается атрибутом **лоадскрипт** **представления**. в этом случае код JScript вызывает метод **URL-адреса** проигрыватель Windows Media, который загружает и начинает воспроизводить файл с именем «лауре. wma». обратите внимание, что строка заканчивается точкой с запятой внутри кавычек, что является хорошим JScript практике написания кода. Обратите внимание также на то, что в двойных кавычках используется одинарные кавычки для задания имени файла. дополнительные сведения о JScript см. в статье [использование JScript](using-jscript.md) в разделе "сведения о обложках" этого пакета SDK.

Обратите внимание, что отсутствует конечный тег **буттонелемент** . Если элемент не заключает другой элемент, его можно закрыть с помощью косой черты прямо перед закрывающей угловой скобкой. Это говорит о том, что XML завершен с этим элементом. Например, примененная к объекту директива


```C++
<BUTTONELEMENT></BUTTONELEMENT>
```



и


```C++
<BUTTONELEMENT/>
```



передает те же сведения в формате XML.

Возможности обложек обусловлены использованием обработчиков событий. Если пользователь выполняет какое-либо действие с мышью, это событие можно обменять JScript. код может быть одной строкой, что делает проигрыватель Windows Media сделать что-то вроде play или полноценное приложение, написанное на JScript.
