<span data-ttu-id="4e04a-105">Программируемый конвейер Direct3D 10 предназначен для создания графики для игровых приложений реального времени.</span><span class="sxs-lookup"><span data-stu-id="4e04a-105">The Direct3D 10 programmable pipeline is designed for generating graphics for realtime gaming applications.</span></span> <span data-ttu-id="4e04a-106">На следующей диаграмме показан поток данных из ввода для вывода через все программируемые этапы.</span><span class="sxs-lookup"><span data-stu-id="4e04a-106">The following diagram shows the data flow from input to output through each of the programmable stages.</span></span>

<span data-ttu-id="4e04a-108">Все этапы можно настроить с помощью API.</span><span class="sxs-lookup"><span data-stu-id="4e04a-108">All of the stages can be configured using the API.</span></span> <span data-ttu-id="4e04a-109">Этапы, в которых используются общие ядра шейдера (скругленные прямоугольные блоки), являются программируемыми с помощью языка программирования HLSL.</span><span class="sxs-lookup"><span data-stu-id="4e04a-109">Stages featuring common shader cores (the rounded rectangular blocks) are programmable using the HLSL programming language.</span></span> <span data-ttu-id="4e04a-110">Как вы увидите, это делает конвейер чрезвычайно гибким и адаптируемым.</span><span class="sxs-lookup"><span data-stu-id="4e04a-110">As you will see, this makes the pipeline extremely flexible and adaptable.</span></span> <span data-ttu-id="4e04a-111">Назначение каждого из этапов приведено ниже.</span><span class="sxs-lookup"><span data-stu-id="4e04a-111">The purpose of each of the stages is listed below.</span></span>

-   <span data-ttu-id="4e04a-112">[Этап ввода ассемблера](../direct3d11/d3d10-graphics-programming-guide-input-assembler-stage.md) — этап ввода-ассемблера отвечает за предоставление данных (треугольников, линий и точек) конвейеру.</span><span class="sxs-lookup"><span data-stu-id="4e04a-112">[Input-Assembler Stage](../direct3d11/d3d10-graphics-programming-guide-input-assembler-stage.md) - The input-assembler stage is responsible for supplying data (triangles, lines and points) to the pipeline.</span></span>
-   <span data-ttu-id="4e04a-113">[Стадия вершинного шейдера](https://www.bing.com/search?q=Vertex-Shader+Stage). На стадии вершинного шейдера обрабатываются вершины, обычно с выполнением таких операций, как преобразования, скиннинг и освещение.</span><span class="sxs-lookup"><span data-stu-id="4e04a-113">[Vertex-Shader Stage](https://www.bing.com/search?q=Vertex-Shader+Stage) - The vertex-shader stage processes vertices, typically performing operations such as transformations, skinning, and lighting.</span></span> <span data-ttu-id="4e04a-114">Вершинный шейдер всегда принимает одну входную вершину и создает одну выходную вершину.</span><span class="sxs-lookup"><span data-stu-id="4e04a-114">A vertex shader always takes a single input vertex and produces a single output vertex.</span></span>
-   <span data-ttu-id="4e04a-115">[Геометрический элемент шейдера](https://www.bing.com/search?q=Geometry-Shader+Stage) — стадия Geometry обрабатывает все примитивы.</span><span class="sxs-lookup"><span data-stu-id="4e04a-115">[Geometry-Shader Stage](https://www.bing.com/search?q=Geometry-Shader+Stage) - The geometry-shader stage processes entire primitives.</span></span> <span data-ttu-id="4e04a-116">Его входные данные являются полным примитивом (который представляет собой три вершины для треугольника, две вершины для линии или одну вершину для точки).</span><span class="sxs-lookup"><span data-stu-id="4e04a-116">Its input is a full primitive (which is three vertices for a triangle, two vertices for a line, or a single vertex for a point).</span></span> <span data-ttu-id="4e04a-117">Кроме того, каждый примитив может также включать данные вершин для всех присмежных к границе примитивов.</span><span class="sxs-lookup"><span data-stu-id="4e04a-117">In addition, each primitive can also include the vertex data for any edge-adjacent primitives.</span></span> <span data-ttu-id="4e04a-118">Это может включать не более трех вершин для треугольника или дополнительные две вершины для линии.</span><span class="sxs-lookup"><span data-stu-id="4e04a-118">This could include at most an additional three vertices for a triangle or an additional two vertices for a line.</span></span> <span data-ttu-id="4e04a-119">Шейдер геометрии также поддерживает ограниченное усложнение и упрощение геометрии.</span><span class="sxs-lookup"><span data-stu-id="4e04a-119">The Geometry Shader also supports limited geometry amplification and de-amplification.</span></span> <span data-ttu-id="4e04a-120">При наличии входного примитива шейдер геометрии может удалить его или сгенерировать один или несколько новых примитивов.</span><span class="sxs-lookup"><span data-stu-id="4e04a-120">Given an input primitive, the Geometry Shader can discard the primitive, or emit one or more new primitives.</span></span>
-   <span data-ttu-id="4e04a-121">[Стадия потокового вывода](../direct3d11/d3d10-graphics-programming-guide-output-stream-stage.md) — этап потокового вывода предназначен для потоковой передачи примитивных данных из конвейера в память по своему пути к средствам программной прорисовки.</span><span class="sxs-lookup"><span data-stu-id="4e04a-121">[Stream-Output Stage](../direct3d11/d3d10-graphics-programming-guide-output-stream-stage.md) - The stream-output stage is designed for streaming primitive data from the pipeline to memory on its way to the rasterizer.</span></span> <span data-ttu-id="4e04a-122">Поток данных может быть выходным или передаваться в средство программной прорисовки.</span><span class="sxs-lookup"><span data-stu-id="4e04a-122">Data can be streamed out and/or passed into the rasterizer.</span></span> <span data-ttu-id="4e04a-123">Поток данных, выведенный в память, можно снова вернуть в конвейер как входные данные или обратно считать из ЦП.</span><span class="sxs-lookup"><span data-stu-id="4e04a-123">Data streamed out to memory can be recirculated back into the pipeline as input data or read-back from the CPU.</span></span>
-   <span data-ttu-id="4e04a-124">[Этап программы-прорисовки](../direct3d11/d3d10-graphics-programming-guide-rasterizer-stage.md) — средство программной прорисовки, ответственное за примитивы отсечения, подготовка примитивов для шейдера пикселей и определение способа вызова шейдеров пикселей.</span><span class="sxs-lookup"><span data-stu-id="4e04a-124">[Rasterizer Stage](../direct3d11/d3d10-graphics-programming-guide-rasterizer-stage.md) - The rasterizer is responsible for clipping primitives, preparing primitives for the pixel shader and determining how to invoke pixel shaders.</span></span>
-   <span data-ttu-id="4e04a-125">[Стадия построителя текстуры](https://www.bing.com/search?q=Pixel-Shader+Stage). На стадии построителя текстуры принимаются интерполированные данные для примитива и генерируются данные для пикселей, такие как цвет.</span><span class="sxs-lookup"><span data-stu-id="4e04a-125">[Pixel-Shader Stage](https://www.bing.com/search?q=Pixel-Shader+Stage) - The pixel-shader stage receives interpolated data for a primitive and generates per-pixel data such as color.</span></span>
-   <span data-ttu-id="4e04a-126">[Стадия слияния](../direct3d11/d3d10-graphics-programming-guide-output-merger-stage.md) . Стадия слияния отвечает за объединение различных типов выходных данных (значения шейдера пикселей, сведения о глубине и трафарете) с содержимым целевого объекта отрисовки и буферами глубины и шаблона для создания конечного результата конвейера.</span><span class="sxs-lookup"><span data-stu-id="4e04a-126">[Output-Merger Stage](../direct3d11/d3d10-graphics-programming-guide-output-merger-stage.md) - The output-merger stage is responsible for combining various types of output data (pixel shader values, depth and stencil information) with the contents of the render target and depth/stencil buffers to generate the final pipeline result.</span></span>
