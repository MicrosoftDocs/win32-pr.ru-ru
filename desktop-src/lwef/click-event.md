### <a name="remarks"></a>Комментарии

Это событие отправляется только клиенту ввода-активного клиента символа. Когда пользователь щелкает символ или его значок на панели задач без клиента input-Active, сервер отправляет событие активному клиенту. Если символ является видимым ([**видимым**](visible-property.md)  =  ), то в действии пользователя также задается Последний входной-активный клиент в качестве текущего клиента ввода, который отправляет событие [**активатеинпут**](activateinput-event.md) клиенту, а затем отправляет событие **Click** . Если символ является скрытым (**видимым**  =  **false**) и пользователь щелкает значок символа на панели задач с помощью кнопки 1, этот символ также отображается автоматически.

> [!Note]  
> При щелчке символа не отключаются все остальные символьные данные (все символы). Тем не менее, при нажатии клавиши прослушивания происходит сброс выходных данных символа входа и активация события [**рекуесткомплете**](requestcomplete-event.md) с передачей [запроса. состояние](status-property.md) , указывающее на то, что очередь клиента была прервана.
