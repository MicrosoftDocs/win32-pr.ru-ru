### <a name="hdcp"></a>ПОДДЕРЖИВАЮЩ

Для HDCP определены следующие два флага.



| Значение                                         | Описание                                                                                                                                                                    |
|-----------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| \_Тип защиты \_ ОПМ \_ HDCP                   | Используйте этот флаг, если указатель [**иопмвидеуутпут**](/windows/desktop/api/opmapi/nn-opmapi-iopmvideooutput) был создан с семантикой ОПМ.                                                                        |
| \_ \_ совместимый с ОПМ тип защиты \_ Копп \_ \_ HDCP | Используйте этот флаг, если указатель [**иопмвидеуутпут**](/windows/desktop/api/opmapi/nn-opmapi-iopmvideooutput) был создан с семантикой Копп. Этот флаг соответствует \_ флагу Копп ProtectionType \_ HDCP в Копп. |



 

Оба режима (семантика ОПМ и семантика Копп) поддерживают следующие операции для HDCP:

-   Включите или отключите HDCP с помощью команды [ОПМ \_ Set \_ Protection \_ Level](opm-set-protection-level.md) .
-   Запросите, включена ли защита HDCP с помощью [ОПМ \_ Получение \_ виртуальной \_ защиты \_ ](opm-get-virtual-protection-level.md) или [ОПМ \_ Получение \_ фактического состояния \_ \_ уровня защиты](opm-get-actual-protection-level.md) .

Семантика ОПМ также поддерживает следующее:

-   Повторители HDCP. *Повторитель HDCP* — это устройство HDCP, которое может получить содержимое HDCP, а также повторно зашифровать и передать то же содержимое.
-   Отзыв HDCP. Если к выходному видео ОПМ подключено отозванное устройство HDCP, видеопотокы не будут передавать видео. Приложению не нужно анализировать сообщения о возобновлении системы HDCP (Срмс) или определять, было ли отменено выходное устройство. Дополнительные сведения см. в описании команды [**ОПМ \_ Set \_ HDCP \_ СРМ**](opm-set-hdcp-srm.md) .

При использовании семантики Копп интерфейс [**иопмвидеуутпут**](/windows/desktop/api/opmapi/nn-opmapi-iopmvideooutput) не поддерживает повторители HDCP. Кроме того, он не обрабатывает отзыв HDCP. Вместо этого приложение должно проанализировать СРМ, чтобы определить, было ли отозвано устройство HDCP.
