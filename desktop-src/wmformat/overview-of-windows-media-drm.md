# <a name="overview-of-windows-media-drm"></a>общие сведения о Windows Media DRM

Windows Media Digital Rights Management (DRM) — это система защиты содержимого файлов мультимедиа Windows, чтобы неавторизованные пользователи не могли получить к ним доступ. Базовый цикл DRM состоит из трех этапов: Упаковка, лицензирование и чтение.

## <a name="packaging-windows-media-files"></a>упаковка файлов мультимедиа Windows

Windows media DRM предназначен для работы с файлами мультимедиа Windows. Windowsный файл мультимедиа — это файл, который соответствует спецификации ASF и содержит только аудио-и видеоматериалы, которые были сжаты с помощью видеокодеков Windows Media Audio и video.

При упаковке ASF-файла в заголовок добавляется раздел, относящийся к DRM. Заголовок DRM содержит идентификатор ключа, который определяет содержимое в целях лицензирования, и URL-адрес для получения лицензии, который является адресом веб-страницы, которая может выдавать лицензии на чтение защищенного содержимого. Существует гораздо больше информации, которую можно разместить в заголовке DRM, но это необязательно. Заголовок DRM подписывается, чтобы можно было проверить упаковщик.

Содержимое в ASF-файле шифруется во время процесса упаковки. Однако следующие сведения в упакованном файле доступны даже клиентам, у которых нет лицензии.

-   Метаданные, хранящиеся в заголовке ASF.
-   Некоторые метаданные, которые хранятся в заголовке DRM (например, всегда можно получить URL-адрес для приобретения лицензии).

## <a name="licensing-protected-files"></a>Лицензированные защищенные файлы

Для чтения упакованного файла на клиентский компьютер должна быть выдана лицензия. Лицензия — это набор данных, описывающих условия, при которых данные в защищенных файлах могут быть прочитаны. Чаще всего для защищенного файла выдается лицензия в ответ на попытку пользователя выполнить какую-либо операцию над файлом. Однако также возможно, что поставщик лицензий доставляет лицензии клиенту, прежде чем он будет явно запрошен. Дополнительные сведения о лицензиях см. в разделе [лицензии](licenses.md).

## <a name="reading-data-from-protected-files"></a>Чтение данных из защищенных файлов

Когда пользователь пытается выполнить операцию с защищенным файлом (воспроизведение, запись на компакт-диск, копирование на устройство и т. д.), приложение должно проверить наличие лицензий на содержимое на клиентском компьютере. Если на клиентском компьютере существует действительная лицензия, операция может быть продолжена. Если у вас нет лицензии на содержимое или если лицензия на содержимое, установленная на клиентском компьютере, не позволяет выполнить запрошенное действие, необходимо получить лицензию.
