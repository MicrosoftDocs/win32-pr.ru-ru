# <a name="overview-of-the-asf-format"></a>Обзор формата ASF

Расширенный формат системы (ASF) — это расширяемый формат файлов, предназначенный для хранения и воспроизведения синхронизированных цифровых потоков мультимедиа и передачи их по сетям. ASF — это формат контейнера для содержимого на основе видео Windows Media Audio и Windows Media. Расширение WMA или WMV используется для указания файла ASF, содержащего содержимое, закодированное с помощью видеокодеков Windows Media Audio и (или) Windows Media. Пакет SDK Windows Media Format можно использовать для создания и чтения файлов Windows Media, а также файлов ASF, содержащих другие типы сжатых или несжатых данных.

В этом разделе приводится общее описание формата ASF в качестве фоновой информации. Поскольку объекты чтения и записи обрабатывали все задачи анализа и форматирования файлов нижнего уровня, перед использованием этого пакета SDK для создания ASF-файлов не обязательно иметь подробное представление о формате ASF. Полную спецификацию ASF можно найти на [веб-сайте Майкрософт](https://download.microsoft.com/download/7/9/0/790fecaa-f64a-4a5e-a430-0bccdab3f1b4/ASF_Specification.doc).

Основные цели формата ASF:

-   Для обеспечения эффективного воспроизведения с серверов мультимедиа, серверов HTTP и локальных запоминающих устройств.
-   Для поддержки масштабируемых типов мультимедиа, таких как аудио и видео.
-   , Чтобы предоставить единую композицию мультимедиа для широкого диапазона пропускной способности.
-   , Чтобы обеспечить контроль над связями потоков мультимедиа, особенно в сценариях с ограниченной пропускной способностью.
-   Не зависит от конкретной системы композиции мультимедийных данных, операционной системы компьютера или протокола обмена данными.

ASF-файл может содержать несколько независимых или зависимых потоков, в том числе несколько звуковых потоков для многоканального аудио-потока или видеопотоков с несколькими скоростями, которые подходят для передачи с различной пропускной способностью. Потоки могут быть в любом сжатом или несжатом формате. Однако лучшее сжатие достигается с помощью кодеков Microsoft Windows Media Audio и Video 9. В дополнение к стандартным типам потоков аудио и видео, файл ASF может также содержать текстовые потоки, веб-страницы и команды сценариев, а также любой другой произвольный тип данных. ASF поддерживает содержимое мультимедиа в реальном времени и по запросу. Он может использоваться в качестве автомобиля для записи или воспроизведения H. 32X (например, H. 323 и H. 324) или МБОНЕ конференций.

Файл ASF организован в разделы, называемые «объектами». Существует три объекта верхнего уровня: объект заголовка и объект данных (обязательный), а также необязательный объект индекса. Объект Header содержит общие сведения о файле, такие как размер файла, количество потоков, методы исправления ошибок и используемые кодеки. Здесь также хранятся метаданные. Объект заголовка — это единственный объект верхнего уровня, который может содержать другие объекты. Объект данных содержит потоковые данные, упорядоченные по пакетам. Объект простого индекса содержит список связанных пар "индекс-ключ-кадр", позволяющих приложениям эффективно искать файл. Индексом, связанным с каждым ключевым кадром, может быть время презентации, номер кадра видео или метка времени ссылки.

Каждый объект верхнего уровня или более низкого уровня начинается с глобального уникального идентификатора (GUID) и значения размера. Эти числа позволяют модулю чтения файлов анализировать информацию в соответствующих местах в идентифицируемые объекты. Из-за этих идентификаторов GUID объекты более низкого уровня могут быть отправлены в любом порядке и все еще распознаются. Формат ASF предназначен для преодоления неточных приемов данных. Частично загруженный ASF-файл все еще можно прочитать, если он содержит объект заголовка и по крайней мере один объект данных.

Подробные сведения о формате ASF, представленном в спецификации ASF. Спецификацию можно загрузить с [веб-сайта корпорации Майкрософт](https://download.microsoft.com/download/7/9/0/790fecaa-f64a-4a5e-a430-0bccdab3f1b4/ASF_Specification.doc).
