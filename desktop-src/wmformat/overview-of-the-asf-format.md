<span data-ttu-id="00357-107">Расширенный формат системы (ASF) — это расширяемый формат файлов, предназначенный для хранения и воспроизведения синхронизированных цифровых потоков мультимедиа и передачи их по сетям.</span><span class="sxs-lookup"><span data-stu-id="00357-107">The Advanced Systems Format (ASF) is an extensible file format designed primarily for storing and playing synchronized digital media streams and transmitting them over networks.</span></span> <span data-ttu-id="00357-108">ASF — это формат контейнера для содержимого на основе видео Windows Media Audio и Windows Media.</span><span class="sxs-lookup"><span data-stu-id="00357-108">ASF is the container format for Windows Media Audio and Windows Media Video-based content.</span></span> <span data-ttu-id="00357-109">Расширение WMA или WMV используется для указания файла ASF, содержащего содержимое, закодированное с помощью видеокодеков Windows Media Audio и (или) Windows Media.</span><span class="sxs-lookup"><span data-stu-id="00357-109">The extension wma or wmv is used to specify an ASF file that contains content encoded with the Windows Media Audio and/or Windows Media Video codecs.</span></span> <span data-ttu-id="00357-110">Пакет SDK Windows Media Format можно использовать для создания и чтения файлов Windows Media, а также файлов ASF, содержащих другие типы сжатых или несжатых данных.</span><span class="sxs-lookup"><span data-stu-id="00357-110">The Windows Media Format SDK can be used to create and read Windows Media files, as well as ASF files that contain other types of compressed or uncompressed data.</span></span>

<span data-ttu-id="00357-111">В этом разделе приводится общее описание формата ASF в качестве фоновой информации.</span><span class="sxs-lookup"><span data-stu-id="00357-111">This section provides a general description of the ASF format as background information.</span></span> <span data-ttu-id="00357-112">Поскольку объекты чтения и записи обрабатывали все задачи анализа и форматирования файлов нижнего уровня, перед использованием этого пакета SDK для создания ASF-файлов не обязательно иметь подробное представление о формате ASF.</span><span class="sxs-lookup"><span data-stu-id="00357-112">Because the reader and writer objects handle all low-level file parsing and formatting tasks, it is not necessary to have a detailed understanding of ASF before using this SDK to create ASF files.</span></span> <span data-ttu-id="00357-113">Полную спецификацию ASF можно найти на [веб-сайте Майкрософт](https://download.microsoft.com/download/7/9/0/790fecaa-f64a-4a5e-a430-0bccdab3f1b4/ASF_Specification.doc).</span><span class="sxs-lookup"><span data-stu-id="00357-113">The complete ASF specification can be found on the [Microsoft Web site](https://download.microsoft.com/download/7/9/0/790fecaa-f64a-4a5e-a430-0bccdab3f1b4/ASF_Specification.doc).</span></span>

<span data-ttu-id="00357-114">Основные цели формата ASF:</span><span class="sxs-lookup"><span data-stu-id="00357-114">The primary goals of the ASF format are:</span></span>

-   <span data-ttu-id="00357-115">Для обеспечения эффективного воспроизведения с серверов мультимедиа, серверов HTTP и локальных запоминающих устройств.</span><span class="sxs-lookup"><span data-stu-id="00357-115">To support efficient playback from media servers, HTTP servers, and local storage devices.</span></span>
-   <span data-ttu-id="00357-116">Для поддержки масштабируемых типов мультимедиа, таких как аудио и видео.</span><span class="sxs-lookup"><span data-stu-id="00357-116">To support scalable media types such as audio and video.</span></span>
-   <span data-ttu-id="00357-117">, Чтобы предоставить единую композицию мультимедиа для широкого диапазона пропускной способности.</span><span class="sxs-lookup"><span data-stu-id="00357-117">To permit a single multimedia composition to be presented over a wide range of bandwidths.</span></span>
-   <span data-ttu-id="00357-118">, Чтобы обеспечить контроль над связями потоков мультимедиа, особенно в сценариях с ограниченной пропускной способностью.</span><span class="sxs-lookup"><span data-stu-id="00357-118">To allow authoring control over media stream relationships, especially in constrained-bandwidth scenarios.</span></span>
-   <span data-ttu-id="00357-119">Не зависит от конкретной системы композиции мультимедийных данных, операционной системы компьютера или протокола обмена данными.</span><span class="sxs-lookup"><span data-stu-id="00357-119">To be independent of any particular multimedia composition system, computer operating system, or data communications protocol.</span></span>

<span data-ttu-id="00357-120">ASF-файл может содержать несколько независимых или зависимых потоков, в том числе несколько звуковых потоков для многоканального аудио-потока или видеопотоков с несколькими скоростями, которые подходят для передачи с различной пропускной способностью.</span><span class="sxs-lookup"><span data-stu-id="00357-120">An ASF file can contain multiple independent or dependent streams, including multiple audio streams for multichannel audio, or multiple bit rate video streams suitable for transmission over different bandwidths.</span></span> <span data-ttu-id="00357-121">Потоки могут быть в любом сжатом или несжатом формате. Однако лучшее сжатие достигается с помощью кодеков Microsoft Windows Media Audio и Video 9.</span><span class="sxs-lookup"><span data-stu-id="00357-121">The streams can be in any compressed or uncompressed format; however, the best compression is achieved with the Microsoft Windows Media Audio and Video 9 Series codecs.</span></span> <span data-ttu-id="00357-122">В дополнение к стандартным типам потоков аудио и видео, файл ASF может также содержать текстовые потоки, веб-страницы и команды сценариев, а также любой другой произвольный тип данных.</span><span class="sxs-lookup"><span data-stu-id="00357-122">In addition to the standard audio and video media stream types, an ASF file can also contain text streams, Web pages and script commands, and any other arbitrary data type.</span></span> <span data-ttu-id="00357-123">ASF поддерживает содержимое мультимедиа в реальном времени и по запросу.</span><span class="sxs-lookup"><span data-stu-id="00357-123">ASF supports live and on-demand multimedia content.</span></span> <span data-ttu-id="00357-124">Он может использоваться в качестве автомобиля для записи или воспроизведения H. 32X (например, H. 323 и H. 324) или МБОНЕ конференций.</span><span class="sxs-lookup"><span data-stu-id="00357-124">It can be used as a vehicle to record or play back H.32X (for example, H.323 and H.324) or MBONE conferences.</span></span>

<span data-ttu-id="00357-125">Файл ASF организован в разделы, называемые «объектами».</span><span class="sxs-lookup"><span data-stu-id="00357-125">An ASF file is organized into sections called "objects."</span></span> <span data-ttu-id="00357-126">Существует три объекта верхнего уровня: объект заголовка и объект данных (обязательный), а также необязательный объект индекса.</span><span class="sxs-lookup"><span data-stu-id="00357-126">There are three top-level objects, a Header object and a Data object (both required), plus an optional Index object.</span></span> <span data-ttu-id="00357-127">Объект Header содержит общие сведения о файле, такие как размер файла, количество потоков, методы исправления ошибок и используемые кодеки.</span><span class="sxs-lookup"><span data-stu-id="00357-127">The Header object contains general information about the file, such as file size, number of streams, error correction methods, and codecs used.</span></span> <span data-ttu-id="00357-128">Здесь также хранятся метаданные.</span><span class="sxs-lookup"><span data-stu-id="00357-128">Metadata is also stored here.</span></span> <span data-ttu-id="00357-129">Объект заголовка — это единственный объект верхнего уровня, который может содержать другие объекты.</span><span class="sxs-lookup"><span data-stu-id="00357-129">The Header object is the only top level object data-stu-id="00357-130">The Data object contains the stream data, organized in packets.</span></span> <span data-ttu-id="00357-131">Объект простого индекса содержит список связанных пар "индекс-ключ-кадр", позволяющих приложениям эффективно искать файл.</span><span class="sxs-lookup"><span data-stu-id="00357-131">The Simple Index object contains a list of associated index/key-frame <span data-ttu-id="00357-133">Каждый объект верхнего уровня или более низкого уровня начинается с глобального уникального идентификатора (GUID) и значения размера.</span><span class="sxs-lookup"><span data-stu-id="00357-133">Each top-level or lower-level object begins with a globally unique identifier (GUID) and a size value.</span></span> <span data-ttu-id="00357-134">Эти числа позволяют модулю чтения файлов анализировать информацию в соответствующих местах в идентифицируемые объекты.</span><span class="sxs-lookup"><span data-stu-id="00357-134">These numbers allow the file reader to parse the information at appropriate places into identifiable objects.</span></span> <span data-ttu-id="00357-135">Из-за этих идентификаторов GUID объекты более низкого уровня могут быть отправлены в любом порядке и все еще распознаются.</span><span class="sxs-lookup"><span data-stu-id="00357-135">Because of these GUIDs, lower-level objects can be sent in any order and still be recognized.</span></span> <span data-ttu-id="00357-136">Формат ASF предназначен для преодоления неточных приемов данных.</span><span class="sxs-lookup"><span data-stu-id="00357-136">The ASF format is designed to overcome inaccurate data reception.</span></span> <span data-ttu-id="00357-137">Частично загруженный ASF-файл все еще можно прочитать, если он содержит объект заголовка и по крайней мере один объект данных.</span><span class="sxs-lookup"><span data-stu-id="00357-137">A partially downloaded ASF file can still be read, as long as it contains the Header object and at least one Data object.</span></span>
